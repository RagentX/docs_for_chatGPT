Обработка записей о похожих персонах

При наличии похожей персоны в БД открывается форма, содержащая:

·предупреждение "В базе обнаружена похожая информация!";

·сведения о похожих персонах (идентификатор персоны, ФИО, дата рождения, ИНН, название продукта, портфель, номера договора и лицевого счета, серия и номер паспорта, идентификатор долга);

·кнопки выбора варианта обработки записи.

Примечание. Форма с данными о похожей персоне в БД открывается, только если в описании формата отсутствует флаг автоматического поиска похожей персоны.

Предусмотрены следующие варианты обработки записи о похожей персоне:

·Пропустить — отмена вставки записи таблицы в БД;

·Создать новую персону — в этом случае :

·выполняется создание новых строк в таблицах персон, паспортов, долгов, судебной работы, исполнительного производства;

·выполняется попытка корректировки данных в таблицах телефонов, адресов, произвольных атрибутов, поручителей должников, контактных лиц, расчетных лиц;

·Только обновить указанную персону — в этом случае:

·выполняется обновление строк в таблицах персон и паспортов;

·выполняется попытка корректировки данных в таблицах телефонов, адресов, произвольных атрибутов, контактных лиц, расчетных лиц;

·портфель и данные о долге не изменятся;

·Обновить указанную персону и создать для нее новый долг — в этом случае:

·выполняется обновление строк в таблицах персон и паспортов;

·выполняется создание новых строк в таблицах долгов, судебной работы, исполнительного производства;

·выполняется попытка корректировки данных в таблицах телефонов, адресов, произвольных атрибутов, поручителей должников, контактных лиц, расчетных лиц;

·Обновить указанные персону и долг — в этом случае :

·выполняется обновление строк в таблицах персон, паспортов, долгов, судебной работы, исполнительного производства;

·выполняется попытка корректировки данных в таблицах телефонов, адресов, произвольных атрибутов, поручителей должников, контактных лиц, расчетных лиц;

·статус долга не изменится.

Для выбора одного из вариантов установите соответствующий переключатель и нажмите кнопку ОК. Для применения выбранного варианта обработки ко всем последующим записям нажмите кнопку Применить для всех.

Если похожая персона в БД отсутствует, то загрузка данных осуществляется в соответствии с вариантом Создать новую персону в БД.

При выполнении попыток корректировки данных в таблицах телефонов, адресов, произвольных атрибутов, поручителей должников, контактных лиц, расчетных лиц возможны следующие результаты:

·создание новой строки;

·обновление данных в строке;

·отсутствие изменений (аналогичная строка уже имеется в таблице).

По окончании обработки нестандартной записи выводится форма со сведениями об очередной нестандартной записи. После обработки или пропуска последней нестандартной записи процесс загрузки данных в БД завершается.
