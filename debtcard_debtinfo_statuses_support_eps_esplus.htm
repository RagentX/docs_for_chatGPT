Доработка процедуры изменение ответственного

В процедуре debt_set_operator_5x последним шагом должна вызываться процедура eps_wrp$debt_set_operator_after

Клиентская EPS процедура изменение ответственного

ur_eps_wrp$debt_set_operator_after_es_plus

( 

  @debt_id_table  nvarchar(128),

  @debt_id_list   nvarchar(4000),

  @new_value      int,

  @admin_id       int,

  @mode           int,

  @module         int,

  @machine        nvarchar(64),

  @dsc            nvarchar(512),

  @mod_version    nvarchar(16),

  @complete_cnt   int OUTPUT,

  @do_mail_ins    int = 0,

  @do_action_reg  int = 0,

  @reason         int,

  @plan_finish_work_date datetime

)

Алгоритм процедуры:

Для каждого переданного в параметре @debt_id_list долга в рамках отделения необходимо осуществить поиск долгов данного должника (поиск долгов осуществляется по person.inn) и закрепить их за тем же оператором @new_value:

·Отделение, к которому относится долг, определяется через debt_detail.dict_value_2

·Отделение долга не равно 'О' или 'ОБТЭ'. ((dict.name where parent_id = 10002 and dict.code = debt_detail.dict_value_2) not in ('О','ОБТЭ'))

·Долги ищутся только по юридическим лицам (person.typ<>1)

·Долги ищутся независимо от статуса

·У долгов нет признак ЮЛ "Сети" (debt_detail.dict_value_4 <> 5)

·Для каждого найденного долга помимо закрепления необходимо произвести логирование закрепления и произвести уведомление на внутреннюю почту, по аналогии с процедурой debt$set_operator_5x.
