Примечание. Если включен механизм ограничения коммуникации, то из всех SMS, подлежащих рассылке, будут сформированы только разрешенные сообщения. См далее — Учет ограничений коммуникации.

Порядок действий

1.Выберите тип мероприятия — SMS-рассылка (код 1). Откроется форма Мероприятие для данного типа мероприятий.

2.Выберите группу объектов (объект "Долги"), для обработки которых предназначено мероприятие.

3.Настройте параметры мероприятия. Для этого заполните следующие поля:

·Задержка отправки (ч) — введите количество часов задержки;

·Тип телефона — см. словарь № 4 "Тип телефона";

Примечание. Сообщение рассылается по телефонам выбранного типа, относящимся только к долгу, входящему в выбранную группу.

·Статус телефона — выбрите статусы телефонов (см. далее);

·Код качества телефона — выберите один или более значений (см. словарь № 114). Если не выбрано ни одно значение, то будут выгружены телефоны со всеми возможными значениями кода качества;

·Словарный атрибут телефона 1 — см. словарь № 258. Название атрибута настраивается поддержкой;

·Словарный атрибут телефона 2 — см. словарь № 309. Название атрибута настраивается поддержкой;

·Кол-во СМС по долгу — укажите максимальное количество SMS, которые могут быть отправлены по одному долгу в рамках одного мероприятия (если по условиям, выбранным на форме, по долгу будет сформировано более одного сообщения). По умолчанию в поле стоит значение константы Sms.MaxCountOnDebt.Default, если константа имеет значение 0, то поле пусто;

Примечание. Подробнее об ограничениях на количество SMS и отбору наиболее приоритетных — см. далее Логика формирования группы с учетом ограничений и приоритетов.

·Флаг Только активные номера — если он включен, то из всех номеров выбранных типов выбираются только имеющие статус "Активный", а если выключен — рассылка осуществляется по всем номерам выбранных типов;

·Флаг Формировать как заявку — если флаг включен, то SMS-сообщения формируются в статусе 10 "Подготовка заявки", если флаг выключен, SMS-сообщения формируются в статусе 0 "Подготовка к отправке"; по умолчанию флаг выключен;

·Тип сообщения — выберите сценарий из списка. Сценарии задаются в модуле Утилиты а Сценарии сообщений а вкладка SMS-сообщение;

·Акция — выберите акцию;

·Отправитель — поле доступно для редактирования при SmsOriginatorSelect = "Истина"; значение по умолчанию определяется константой SmsDefaultOriginator.

Примечание. Константа InfoDebtSmsForbiddenPeriod влияет на значение даты начала отправки при формировании SMS с помощью данной процедуры. Минимальное и максимальное время отправки корректируются согласно заданному значению константы, независимо от даты отработки мероприятия. Будни  и выходные настраиваются в модуле "Настройка рабочих дней".

А затем нажмите ОК.

4.Настройте График запуска мероприятия.

5.Заполните поля:

·Последовательность — при необходимости введите числовое значение последовательности. В случае одновременного запуска двух и более мероприятий первым будет выполняться задание с наименьшим значением в поле.

·Флаг "Вкл." — если флаг находится в положении Включено, мероприятие запускается, если флаг находится в положении Выключено, мероприятие не запускается. Использование флага дает возможность отменить запуск мероприятия, не удаляя его.

·Флаг "Проверять формирование групп". При включенном флаге мероприятие выполняется только для тех групп, которые были сформированы или переформированы в соответствии со следующими условиями:

·Если значение константы Scheduler.ReformGroupCheck.Mode равно 2, то группа считается переформированной, если дата формирования больше или равна текущей дате.

·Если значение константы Scheduler.ReformGroupCheck.Mode равно 1, то группа считается переформированной, если дата формирования больше текущей даты и времени минус 1 сутки (текущая логика).

·Если значение константы Scheduler.ReformGroupCheck.Mode не равно 1 или 2, то используется логика режима 1.

Внимание! Если мероприятие в модуле Расписание не предполагает использование автогруппы, флаг Проверять формирование группы должен находиться в положении Выключено.

·Комментарий — дополнительная информация.

6.Нажмите OK.

Описание поля "Статус телефона"

При нажатии на поле открывается выпадающий список, который включает:

·Флаг "Все активные" — если включен данный флаг, то выбранными считаются все статусы, которые не перечислены в значении константы Phone.NotActive.Status.List.

·Флаги, соответствующие статусам телефона по словарю № 37.

Примечание. В списке статусов для выбора доступны только те статусы, которые не перечислены в значении константы Phone.NotActive.Status.List. Флаги, соответствующие статусам телефона, которые перечислены в значении константы Phone.NotActive.Status.List, выделены серым. Выбрать эти неактивные статусы нельзя.

По умолчанию включен флаг "Все активные". Если вы выберите один или более статусов в списке, то флаг "Все активные" будет снят. Если при выбранных статусах включить флаг "Все активные", флаги статусов будут сняты.

Если вы снимете пометки со всех флагов в списке Статус телефона, то при нажатии кнопки ОК на форме Параметры запуска будет показано сообщение

Необходимо выбрать хотя бы один статус телефона.

Настройте статусы правильно.

При выполнении мероприятия рассылка осуществляется на телефоны, имеющие статусы, настроенные в поле Статус телефона.

Логика формирования группы с учетом ограничений и приоритетов

Мероприятие для групп должников SMS-рассылка выполняется следующим образом:

1.Для каждого должника из выбранной группы формируется не более, чем Sms.MaxCountOnDebt.Default сообщений. Если константа равна 0, то ограничения не накладываются.
2.Номера телефонов сортируются по типам телефонов в соответствии с приоритетом, заданным в константе Sms.PhoneTyp.Priority. Если константа равна 0, то сортировка по этому параметру не производится.
3.Для каждого типа телефона номера сортируются по подтипам телефонов в соответствии с приоритетом, заданным в константе Sms.PhoneDictValue1.Priority. Если константа равна 0, то сортировка по этому параметру не выполняется.

Таким образом, сортировки влияют на порядок формирования SMS. Сначала формируются сообщения по более приоритетным телефонам должника.

 Примечание для поддержки.

В зависимости от значения константы Sms.JobTextUpd.Use формирование

·Если константа Sms.JobTextUpd.Use выключена, то формирование сообщений происходит также, как и в предыдущих версиях Системы.

·Если константа Sms.JobTextUpd.Use включена, то для формирования текста сообщений используется отдельная процедура.

       Учет ограничений коммуникации

При массовом формировании SMS проверка ограничений коммуникации осуществляется на этапе создания списка номеров.

Осуществляется проверка, все ли SMS могут быть подготовлены с учетом ограничений.

Если все SMS могут быть подготовлены, они подготавливаются.

Если есть SMS, которые не могут быть подготовлены, то в зависимости от значения в роли пользователя доступа, разрешающего превышение лимита, осуществляется следующее:

·Если доступ выключен или отсутствует, то пользователю предлагается выбор:

·Подготовить только разрешенные SMS;

·Не подготавливать SMS вообще.

·Если доступ включен, то пользователю предлагается выбор:

·Подготовить все выбранные SMS;

·Подготовить только разрешенные SMS;

·Не подготавливать SMS вообще.

Примечание. При формировании всех доступных SMS осуществляется проверка возможности формирования каждого SMS.

       Возможность осуществления коммуникации при превышении лимита

Существует ряд доступов, дающих пользователю возможность осуществить коммуникацию сверх лимита.

Если в роли пользователя для данного типа коммуникации такой доступ включен, то при превышении лимита ему выдается соответствующее сообщение:

Формирование сообщения превысит допустимый лимит коммуникации. Вы действительно хотите <название действия для данного типа коммуникации>?

Выберите:

·Yes — если вы хотите совершить коммуникацию сверх лимита;

·No — если вы не хотите совершить коммуникацию сверх лимита.

Доступы на превышение лимитов коммуникации разных типов

·Доступ № 633 COMMUNICATION_SMS_EXCEED_LIMIT — дает право на формирование SMS – сообщений при превышении лимита согласно схеме коммуникации;

·Доступ № 634 COMMUNICATION_EMAIL_EXCEED_LIMIT — дает право на формирование EMAIL – сообщений при превышении лимита согласно схеме коммуникации;

·Доступ № 635 COMMUNICATION_CALL_EXCEED_LIMIT — дает право на совершение исходящего вызова при превышении лимита коммуникации;

·Доступ № 636 COMMUNICATION_VISIT_EXCEED_LIMIT — дает право на подготовку выезда при превышении лимита коммуникации.

Учет статистики по SMS

Статистика по смс должна обновляться (триггером) сразу после вставки сообщения в таблицу debt_sms. В статистике будут учитываться сообщения в следующих статусах: 

·Передано 

·Доставлено

·Заявка 

·В процессе отправки 

·Не известно

Если сообщение будет переведено в статус, отличный от описанных выше, то оно будет исключено из статистики, и наоборот, если сообщение перейдет в указанные статусы, то оно будет учитываться в статистике. 

В качестве даты сообщении для учета в статистике берется фактическая дата отправки (debt_sms.send_date), если эта дата не заполняется, то учитывается дата начала отправки (debt_sms.start_date).

В случае отложенной отправки сообщения, предполагаемые к отправке через некоторое время (в статусе "Готово к отправке") будут учитываться в статистике, соответствующей дню отправки. И такие сообщения будут учитываться при попытке поставить в очередь еще одно сообщение. 

 Обновление статистки по смс, которые изменили статус будет выполняться фоновым заданием.
