Сервис для интеграции с сервисом ФССП "Кворум"

Примечание. Данный функционал включается по согласованию с вендором. Обратитесь в техническую поддержку. Коммерческая константа @Quorum.Use.

Общее описание сервиса

В сервисе поддержана отправка в сервис ФССП "Кворум" следующих запросов:

·Запрос о ходе ИП;

·Запрос на получение номера ИП;

·Запрос на наличие ИП.

В файле конфигурации сервиса поддержана настройка следующих параметров:

·executive_proceedings_info_request_time — время формирования ежедневного запроса на получение информации о ходе ИП;

·executive_proceedings_info_request_group_id — id группы в ПО Контакт с ИП, по которым формируются запросы о ходе ИП;

·executive_proceedings_number_request_time — время формирования ежедневного запроса на получение номера ИП;

·executive_proceedings_number_request_group_id — id группы в ПО Контакт с ИП, по которым требуется определить номер ИП;

·executive_proceedings_bank_request_time — время формирования ежедневного запроса на наличие ИП по долгам;

·executive_proceedings_bank_request_group_id — id группы в ПО Контакт с долгами, по которым требуется определить наличие ИП;

·email_notification — email адреса через запятую, на которые требуется делать рассылку в случае недоступности Адаптера ФССП.

Если при отправке запроса не приходит ответ, делается автоматическая email-рассылка с ящика etl@cont.ru на почтовые адреса, указанные в параметре email_notification файла конфигурации, со следующими параметрами:

·Тема письма: "ЭДО с ФССП: Ошибка обработки запросов на стороне Адаптера ФССП"

·Текст письма:

Дата и время ошибки: %дата и время отправки email%

Причина: не отвечает Адаптер ФССП (Запрос стороны ИП) (если делался запрос о ходе ИП) / не отвечает Адаптер ФССП (Банк ИП) (если делался запрос на получение номера ИП или запрос на наличие ИП у клиента)

Запрос о ходе ИП

Запросы о ходе ИП выполняются в двух режимах:

·с подпиской:

Ежедневно в то время, которое настроено в файле конфигурации сервиса (параметр executive_proceedings_info_request_time) выполняется следующее:

·переформировывается группа с id, указанным в файле конфигурации (параметр executive_proceedings_info_request_group_id);

·дела из переформированной группы добавляются в очередь на формирование запроса в сервис "Кворум" (если оно уже не находится в этой очереди на данный момент).

·без подписки (единичные запросы):

Единичные запросы добавляются вручную через интерфейс:

·кнопка в карточке ИП

·действие на вкладке "Исполнительное производство" РМ Юридический департамент

Через каждые 5 минут из по делам, находящимся в очереди, направляется запрос о ходе ИП в сервис "Кворум".

После выполнения запросов по всем ИП делается информирование по внутренней почте ПО Контакт:

1.Отбираются ИП, для которых выполняются все следующие условия:

·по которым была загружена хотя бы одна новая запись;

·по долгам есть назначенный оператор.

2.Отобранные ИП группируются по операторам, назначенным на связанный с ИП долг.

3.Операторам делается следующая рассылка по внутренней почте ПО Контакт:

·Тема: "Обновления в запросах о ходе ИП %Текущая дата%"

·Текст письма:

Данные о ходе ИП обновлены по следующим делам:

ID Дела: %law_exec.id%, ФИО клиента: %person.f+i+o%, Номер договора: %debt.contract%

ID Дела: %law_exec.id%, ФИО клиента: %person.f+i+o%, Номер договора: %debt.contract%

Запрос на получение номера ИП

Ежедневно в то время, которое настроено в файле конфигурации сервиса (параметр executive_proceedings_number_request_time) выполняется следующее:

1.Переформировывается группа ИП с id, указанным в файле конфигурации (параметр executive_proceedings_number_request_group_id);

2.По каждому ИП, находящемуся в группе, выполняется запрос на получение номера ИП в сервис "Кворум".

Запрос на наличие ИП

Ежедневно в то время, которое настроено в файле конфигурации сервиса (параметр executive_proceedings_bank_request_time) выполняется следующее:

1.Переформировывается группа долгов с id, указанным в файле конфигурации (параметр executive_proceedings_bank_request_group_id);

2.По группе долгов создается список уникальных должников;

3.По каждому должнику выполняется запрос на наличие ИП в сервис "Кворум".
