Управление портфелями обоих направлений

Форма редактирования атрибутов портфеля

Добавление портфеля контрагенту

1.В форме Клиенты и портфели выберите нужную запись о контрагенте. При этом в списке Портфели будут отображаться портфели, уже добавленные контрагенту.

2.Нажмите кнопку Добавить, относящуюся к списку портфелей. Откроется форма редактирования атрибутов портфеля.

3.В форме заполните следующие поля:

·Название — значение поля формируется автоматически, если не установлен флаг Выкл. алгоритма названия портфеля в форме редактирования атрибутов контрагента или если этот флаг не виден. Если флаг установлен, то название можно ввести вручную;

·Направление —  направление обработки портфеля (Входящий / Исходящий);

·Тариф — процент вознаграждения для исходящих портфелей;

·Номер/часть — номер и часть портфеля. Поля обязательны для заполнения;

Примечание. Как правило, номер портфеля определяет контрагент. При помощи второго параметра (Часть) можно сегментировать данный портфель в тех случаях, если один портфель передается в несколько этапов.

·Стадия — стадия взыскания долгов. Пустое значение означает, что стадий несколько. Поле обязательно для заполнения;

Примечание. Если вы создаете портфель для банкротства, выберите для него стадию bankrupt. В такой портфель попадут все долги должника-банкрота.

·Тип долгов — значение выбирается из словаря № 59. Поле обязательно для заполнения и может принимать следующие значения:

·First Collection — "свежие" долги;

·Second Wash, Third Wash и т. д. — долги, которые уже обрабатывались в данной организации или в другой;

·Тип должников — значение выбирается из словаря № 54 (по умолчанию указано физическое лицо). Поле обязательно для заполнения;

·Тип портфеля — значение выбирается из словаря № 58. Поле обязательно для заполнения;Поле обязательно для заполнения и может принимать следующие значения:

·Основной — портфель, содержащий долги, которые первоначально переданы контрагентом в работу;

·Продленный — портфель, содержащий долги, по которым контрагент продлевает работу после завершения работы с основным портфелем;

·Перекинутый — портфель, содержащий долги, которые организация передала в работу другим организациям (аутсорсинг долгов);

·Статус — статус портфеля. Перечень возможных значений атрибута хранится в словаре № 57 и зависит от атрибута Направление;

·Дата подписания — дата подписания портфеля. Поле обязательно для заполнения;

·Дата начала работ;

·Дата оконч. работ — дата окончания работ с портфелем. Если для исходящего портфеля указана дата в этом поле, то по ее наступлению долги будут автоматически возвращены из аутсорсинга (переведены в статус 16 "Возврат из аутсорсинга"), в противном случае — не будут. При очистке этого поля будет выдаваться соответствующее предупреждение;

 Примечание. Если задана константа NotifyPortfolioEndDaysCnt и для портфеля указана дата окончания работы, то операторам по внутренней почте  системы за указанное в константе количество дней до окончания работы по портфелю будет направляться уведомление. Перечень операторов задается константой NotifyPortfolioEndDate.

Кнопка Дополнительные атрибуты портфеля — при создании портфеля кнопка заблокирована и имеет вид , при редактировании портфеля кнопка доступна.

·Комплекс мероприятий — комплекс мероприятий, ассоциированный с портфелем;

·Правило расчета — правило расчета остатков, действующее для портфеля (см. Правила расчета остатков);

·Группа — дополнительный параметр (число), который позволяет группировать портфели при фильтрации;

·Вид деятельности — информационный параметр для аналитики, значение выбирается из словаря № 104;

·Коэффициент успешности — информационный параметр (число) для аналитики;

·Приоритет — информационный параметр (число) для аналитики;

·Тип договора — тип договора, по которому обслуживается портфель. Значение выбирается из словаря № 106 (1 — агентские долги, 2 — цессионные);

·Информация о договоре — дополнительная информация о договоре, используется при создании писем;

·Стоимость покупки — информационный параметр для аналитики. Например, в данном поле можно указать стоимость долгов, которые контрагент продал данной организации (цессия);

·Прогноз взыскания — информационный параметр для аналитики. Например, в данном поле можно указать прогнозную сумму сбора, рассчитанную аналитиками;

·Дата последних платежей;

·Срок автозакрепления — количество дней от даты закрепления долга за оператором, получившим обещание оплаты, по прошествии которых долг автоматически перезакрепляется за оператором, указанным в поле Оператор для автозакрепления;

·Оператор для автозакрепления — оператор, на которого автоматически перезакрепится долг после истечения срока, указанного в поле Срок для автоматического закрепления;

·Комментарий — дополнительная информация о портфеле.

4.Нажмите ОК, чтобы сохранить портфель.

Редактирование портфеля контрагента

1.В форме Клиенты и портфели выберите нужную запись о контрагенте.

2.В списке Портфели выберите портфель.

3.Нажмите относящуюся к списку портфелей кнопку Изменить или выполните двойной щелчок левой клавишей мыши по выбранной записи.

4.В форме редактирования портфеля измените необходимые атрибуты (см. Добавление портфеля контрагенту).

5.При редактировании портфеля появляется возможность настроить Дополнительные атрибуты портфеля.

6.Нажмите кнопку ОК, чтобы сохранить изменения.

Удаление портфеля контрагента

Право на удаление портфеля имеют пользователи, в роли которых доступ № 601 BANK_PORTFOLIO_DELETE имеет значение TRUE.

Портфель может быть удален, только если он имеет статус "Закрыт" или "Загружается". Для того чтобы удалить портфель, выполните следующие действия:

1.В форме Клиенты и портфели выберите нужную запись о контрагенте.

2.В списке Портфели выберите нужный портфель.

3.Нажмите кнопку Удалить, относящуюся к списку портфелей.

4.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить удаление. Портфель будет удален из списка.

Если портфель имеет статус "В работе", будет выдано сообщение о невозможности выполнения операции. Нажмите кнопку ОК, чтобы закрыть его.
