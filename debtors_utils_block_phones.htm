Блокировка телефонов

Show/Hide Hidden Text

Утилита блокировки телефонов осуществляет блокировку телефонов по списку, переданному в Excel-файле. Сначала осуществляется проверка возможности осуществления данной процедуры. Затем осуществляется блокировка в соответствии с записями из файла, прошедшими проверку. Далее для заблокированных номеров осуществляется перенос истории контактов.

Утилита доступна только тем пользователям, в роли которых доступ № 639 DEBT_LIST_BLOCK_PHONE имеет значение TRUE.

Формат файла, содержащего список для блокировки

Для выбора доступен файл только в формате Excel.

Первая строка файла должна содержать "шапку".

Проверка списка телефонов осуществляется, начиная со второй строки.

Список полей файла для загрузки

·ID телефона;

·Номер телефона;

·ID должника;

·ID долга;

·Причина блокировки (указывается код термина по словарю № 313 "Причина блокировки телефона");

·ID телефона для переноса истории.

Обязательность заполнения полей таблицы

В файле для каждой строки обязательно должны быть заполнены столбцы в одной из следующих комбинаций:

A.ID телефона;

B.Номер телефона и ID должника;

C.Номер телефона и ID долга.

Поле Причина блокировки является обязательным для заполнения, если включена константа Phone.Block.Reason.Mandatory.

Порядок действий для вызова утилиты

Для того чтобы осуществить блокировку телефонов:

1.Войдите в модуль Список должников.

2.Нажмите на кнопку Утилиты загрузки. В открывшемся списке выберите пункт Блокировка телефонов.

3.Откроется стандартное окно для выбора файла. Выберите файл, затем нажмите Открыть (Open).

4.Откроется окно предварительного просмотра. Нажмите кнопку Проверить (остальные кнопки в данный момент неактивны). Будет осуществлена Проверка возможности осуществления блокировки.

5.По окончании проверки в таблице будут заполнены поля "Возможность блокировки" и "Причина невозможности блокировки". Вы можете сохранить полученную таблицу в файл, для этого нажмите кнопку Excel.

6.Нажмите кнопку Загрузка в БД. Будет выполнена блокировка.

7.Если заполнено поле ID телефона для переноса истории, будет осуществлен Перенос истории контактов по заблокированному номеру.

8.По окончании процесса будет показано информационное сообщение. Нажмите ОК.

Проверка возможности осуществления блокировки

1.Если формат файла не совпадает с требуемым, то пользователю выводится сообщение:

"Файл не соответствует требуемому формату"

2.Если формат файла верный, то проводится проверка данных в файле:

Для варианта A — заполнено поле "ID телефона":

·Выполняется проверка наличия в таблице phone телефона с phone.id из столбца "ID телефона". Если телефон не найден, то блокировка не производится с причиной:

"Не найден указанный номер телефона"

·Если телефон найден, то определяется его статус блокировки, если телефон уже заблокирован, то блокировка не производится с причиной:

"Телефон уже заблокирован"

·Выполняется проверка наличия термина в словаре "Причина блокировки телефона", указанного в столбце "Причина блокировки". Если код не найден (или он пустой) и включена константа Phone.Block.Reason.Mandatory, то блокировка не производится с причиной:

"Некорректная причина блокировки"

Для варианта В — заполнены поля "Номер телефона" и "ID должника":

·Выполняется поверка существования указанного должника в таблице person. Если должник не найден, то блокировка не производится с причиной:

"Не найден должник с указанным ID"

·Выполняется проверка наличия phone.number2 как в столбце "Номер телефона" (телефон необходимо привести к стандартному формату через dbo.tranform_phone) и phone.parent_id как "ID должника". Если телефон не найден, то блокировка не производится с причиной:

"Не найден указанный номер телефона у должника"

·Если телефон найден, то определяется его статус блокировки, если телефон уже заблокирован, то блокировка не производится с причиной

"Телефон уже заблокирован"

·Выполняется проверка наличия термина в словаре "Причина блокировки телефона", указанного в столбце "Причина блокировки". Если код не найден и включена константа Phone.Block.Reason.Mandatory, то блокировка не производится с причиной:

"Некорректная причина блокировки"

Для варианта С — заполнены поля "Номер телефона" и "ID долга":

·Выполняется поверка существования указанного долга в таблице debt. Если долг не найден, то блокировка не производится с причиной:

"Не найден долг с указанным ID"

·Выполняется проверка наличия phone.number2 как в столбце "Номер телефона" (телефон необходимо привести к стандартному формату через dbo.tranform_phone) и phone.parent_id как debt.parent_id для долга. Если телефон не найден, то блокировка не производится с причиной:

"Не найден указанный номер телефона у должника"

·Если телефон найден, то определяется его статус блокировки, если телефон уже заблокирован, то блокировка не производится с причиной:

"Телефон уже заблокирован"

·Выполняется проверка наличия термина в словаре "Причина блокировки телефона", указанного в столбце "Причина блокировки". Если код не найден и включена константа Phone.Block.Reason.Mandatory, то блокировка не производится с причиной:

"Некорректная причина блокировки"

По окончании проверки в таблице заполняются поля:

·Возможность блокировки — значения "Да" или "Нет";

·Причина невозможности блокировки — для записей, имеющих значение "Нет" в предыдущем поле указывается причина.

Выполнение блокировки

Блокировка выполняется только для записей, имеющих в поле Возможность блокировки значение "Да".

Если в списке были найдены строки, которые не подлежат загрузке, то пользователю выводится сообщение:

"Сформировать файл ошибочных записей?"

В зависимости от выбора пользователя:

·Если пользователь выбирает "Нет", то осуществляется блокировка телефонов, которые прошли проверку.

·Если пользователь выбирает "Да", то осуществляется блокировка телефонов и формируется Excel-файл, содержащий информацию о телефонах, которые не были заблокированы с указанием причины невозможности блокировки.

Перенос истории контактов по заблокированному номеру

Перенос истории контактов осуществляется для номеров, удовлетворяющих условиям:

·Блокировка номера была выполнена

·В поле "ID телефона для переноса истории" указан идентификатор телефона

·Указанный идентификатор телефона для переноса истории найден в системе.

При переносе истории для каждого номера выполняются следующие действия:

1.Обновляется привязка контактов, которые были привязаны к блокируемому номеру, к новому телефонному номеру, который указан в поле "ID телефона для переноса истории".

2.Обновляется привязка SMS, отправленных на блокируемый номер, к новому телефонному номеру.

3.Для нового телефонного номера устанавливаются следующие атрибуты блокируемого телефона:

a.Статус.

b.Запрет на отправку автоматических сообщений.

c.Запрет автоинформирования.
