Особенности загрузки файлов с типом формата "Судебная работа" и "Судебная работа (обновление)"

1.Логика загрузки поля "Наименование суда" (law_court.name):

·По указанному значению в поле "Наименование суда" осуществляется поиск записей в таблице law_court по полю name.

·Если будет найдена запись, то дело закрепляется за этим судом. (law_act.r_court_id = law_court.id). Если записей будет несколько, то закрепляется запись с максимальным id.

·Иначе (запись по указанным данным не найдена), закрепление судебного дела за судом не выполняется (поле (law_act.r_court_id не обновляется).

2.Загрузка остальных полей осуществляется согласно стандартной логике.

Доработка логики загрузки формата "Судебная работа"

·Если в шаблоне есть поле "ссылка на debt" по таблице "Судебная работа", то поля "Фамилия", "Имя", "Отчество" (ФИО) не обязательно должны присутствовать в шаблоне.

·При этом либо ФИО, либо ссылка на debt должны быть заполнены при загрузке файла.

·Если в файле загрузки указана "ссылка на debt":

·Если долг (debt.id) найден по указанному значению, то создаваемое судебное дело прикрепляется к этому долгу. Данные по долгу и должнику в этом случае обновляются.

·Если долг не найден по указанному значению, то при проверке поле должно отмечаться желтым цветом. Запись не должна загружаться.

·Если в файле не указана "ссылка на debt", то проверяется наличие и заполнение полей "ФИО". В этом случае при загрузке по заполненным данным добавляется новый должник и к нему прикрепляется созданное судебное дело.
