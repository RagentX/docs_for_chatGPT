Скоринговые баллы

Show/Hide Hidden Text

Скоринговые баллы — это словарные атрибуты долга, которые настраиваются клиентом. Если стандартных атрибутов долга, заданных в Системе, вам не достаточно, вы можете воспользоваться скоринговыми баллами. Их значения привязываются к долгу. Они отображаются в списочных рабочих местах. Их можно изменять как в списочных РМ вручную, так и автоматически — с помощью расписания (см. описание РМ Расписание). Один из скоринговых баллов можно использовать в качестве стадии долга.

Использование скоринговых баллов регулируется вендором. Обратитесь в техническую поддержку для включения возможности их использования.

Для включения скоринговых баллов включите константы:

·@UseDebtMark1

·@UseDebtMark2

·@UseDebtMark3

·@UseDebtMark4

Затем настройте стринги для них. Также убедитесь, что правильно заполнены словари 101, 102, 244, 245.

Использование скорингового балла в качестве стадии

По умолчанию в Системе включен атрибут Стадия, привязанный к должнику. Таким образом, независимо от количества долгов у должника и их состояния, значение поля Стадия может быть только одно. Если вы будете использовать скориновый балл в качестве стадии, вы сможете назначить свою стадию для каждого долга. В качестве стадии можно использовать первый скоринговый балл.

Использование скорингового балла № 1 в качестве стадии долга включается вендором. Обратитесь в техническую поддержку.

Включите константу @DepartUseMark1. 

Названия для скоринговых баллов

Названия для них создаются клиентом и передаются вендору, затем специалисты технической поддержки настраивают Систему таким образом, чтобы новые названия заменяли названия по умолчанию во всех РМ. При необходимости могут быть настроены разные названия одного и того же балла в разных РМ.

По умолчанию скоринговые баллы имеют названия:

·Балл № 1;

·Балл № 2;

·Балл № 3;

·Балл № 4.

Значения для скоринговых баллов

Значения скоринговых баллов устанавливаются в словарях:

·Балл № 1 — словарь № 101;

·Балл № 2 — словарь № 102;

·Балл № 3 — словарь № 244;

·Балл № 4 — словарь № 245.

Установить названия указанных словарей и настроить их термины может пользователь Системы, наделенный соответствующими полномочиями.

Доступ к долгам в зависимости от значений скоринговых баллов

Доступность пользователю долгов, имеющих определенные значения скоринговых баллов — см. Применение ограничений доступа к долгам для скоринговых баллов в модуле Настройка списка должников.

Настройка доступности скоринговых баллов и их значений для пользователей

Данный раздел описывает настройку того, как пользователь может назначать долгам данные атрибуты.

Скоринговый балл № 1

Если в роли пользователя доступ № 24 DEBT_STAGE имеет значение TRUE, то пользователю доступны значения скорингового балла № 1 из списка, указанные в доступе № 557 DEBT_MARK1_SET_LIST.

Если доступ № 24 DEBT_STAGE в роли пользователя имеет значение FALSE, то изменение скорингового балла № 1 ему не доступно.

Скоринговый балл № 2

Если в роли пользователя доступ № 558 DEBT_MARK2_SET_LIST имеет значение TRUE, то пользователю доступны все значения скорингового балла № 2 из словаря № 102. Если доступ № 558 в роли пользователя имеет значение FALSE, то изменение скорингового балла № 2 ему не доступно.

Скоринговый балл № 3

Если в роли пользователя доступ № 492 DEBT_MARK3_SET имеет значение TRUE, то ему доступны все значения скорингового балла № 3 из словаря № 244.

Если в роли пользователя доступ № 492 DEBT_MARK3_SET имеет значение FALSE, то ему доступны только те значения из словаря № 244, которые перечислены в значении доступа № 559 DEBT_MARK3_SET_LIST, а также значение "пусто". Если же при выключенном доступе № 492 значение доступа № 559 пусто, то пользователю не доступен скоринговый балл № 3.

Скоринговый балл № 4

Если в роли пользователя доступ № 493 DEBT_MARK4_SET имеет значение TRUE, то ему доступны все значения скорингового балла № 4 из словаря № 245.

Если в роли пользователя доступ № 493 DEBT_MARK4_SET имеет значение FALSE, то ему доступны только те значения из словаря № 245, которые перечислены в значении доступа № 560 DEBT_MARK4_SET_LIST, а также значение "пусто". Если же при выключенном доступе № 493 значение доступа № 560 пусто, то пользователю не доступен скоринговый балл № 4.

Соответствие скоринговых баллов

Данный раздел описывает настройку того, как пользователь может назначать долгам данные атрибуты.

Имеется возможность настройки соответствия между значениями скоринговых баллов. Например, если настроены некоторые соответствия между баллом № 1 и баллом № 2, то для каждого значения балла № 1 будет доступен свой набор значений балла № 1. При настройке соответствия есть возможность задать как определенные значения, так и любое значение. когда соответствия настроены, при установке значений для баллов доступны соответствующие значения либо пустые.

Настройка соответствия скоринговых баллов

Для того чтобы настроить функционал соответствия скоринговых баллов, выполните следующие действия:

1.Для включение функционала соответствия настройте значение константы Debt.Marks.UseRelation.List. Укажите через запятую номера баллов, для которых хотите настроить соответствия. Ели в значении константы перечислено от двух до четырех значений, то константа настроена корректно.

2.Настройте для пользователей соответствующие доступы:

·доступ № 657 DICT_MARKS_RELATION_VIEW для просмотра настроенных соответствий;

·доступ № 658 DICT_MARKS_RELATION_EDIT для редактирования соответствий;

·доступ № 659 DEBT_MARK_SET_IGNORE_RELATION для возможности устанавливать долгам не соответствующие значения скоринговых баллов.

3.Настройте соответствия между значениями скоринговых баллов на вкладке Соответствия БАЛЛ № 1 - Балл № 2... в модуле Словари.

После этого при попытке изменении в Системе значений скоринговых баллов для долгов будет осуществляться проверка их соответствия. Если новые значения нарушают соответствия, они не будут применяться. Это верно для пользователей, в роли которых не включен доступ № 659 DEBT_MARK_SET_IGNORE_RELATION.

Проверка скоринговых баллов на соответствие в карточке должника

Если включен функционал соответствия значений скоринговых баллов (в константе Debt.Marks.UseRelation.List перечислено не менее двух значений), то для пользователей, в роли которых отключен или отсутствует доступ DEBT_MARK_SET_IGNORE_RELATION, при попытке обновить значения скоринговых баллов на форме Информация о долге осуществляется проверка соответствия.

Если хотя бы один из баллов, для которых настроены соответствия, заполнен, и его значение есть в таблице соответствий (см. описание вкладки Соответствия БАЛЛ № 1 - Балл № 2... в модуле Словари), то значения остальных баллов могут быть либо соответствующими, либо пустыми.

Примечание. Если Балл № 1 используется в качестве стадии (константа @DepartUseMark1 имеет значение "Истина"), то соответствия для Балл № 1 не проверяются.

Если новые значения скоринговых баллов нарушают соответствия, то после нажатия ОК форма Информация о долге закрывается без обновления скоринговых баллов, а пользователю выводится сообщение:

½По долгу не были обновлены атрибуты из-за настроенных ограничений.

Массовая проверка скоринговых баллов на соответствие в списочных модулях

Если включен функционал соответствия значений скоринговых баллов (в константе Debt.Marks.UseRelation.List перечислено не менее двух значений), то для пользователей, в роли которых отключен или отсутствует доступ DEBT_MARK_SET_IGNORE_RELATION, порядок обновления значений скоринговых баллов через пункт контекстного меню (вызывается правой кнопкой мыши) Установить значение изменен.

Данная доработка применяется в следующих списочных модулях Системы:

·Список должников;

·Постановка задач;

·Обработка должников;

·ГСП.

Для изменения значений скоринговых баллов в этом случае предусмотрено два варианта.

Обновления значений баллов по одному

Обновлять значения баллов можно по одному, используя подпункты Балл № 1, Балл № 2 и т.д. пункта контекстного меню (вызывается правой кнопкой мыши) Установить значение.

При установке значения каждого балла, для которого установлены соответствия (см. описание вкладки Соответствия БАЛЛ № 1 - Балл № 2... в модуле Словари), осуществляется проверка значений остальных баллов. Значения остальных баллов могут быть либо соответствующими, либо пустыми.

Если соответствие нарушено, то для этих долгов значение скорингового балла не обновляется. По окончании процедуры выводится информационное сообщение:

½"По n долгам не были обновлены значения из-за настроенных ограничений",

где n – количество долгов, для которых не было обновлено значение переданного атрибута.

Примечание. Если Балл № 1 используется в качестве стадии (константа @DepartUseMark1 имеет значение "Истина"), то подпункт Балл № 1 отсутствует.

Одновременное обновление всех баллов ("Атрибуты долга")

Обновить одновременно все значения скоринговых баллов можно используя новый подпункт Атрибуты долга пункта контекстного меню (вызывается правой кнопкой мыши) Установить значение.

При этом откроется форма для ввода новых значений.

Введите нужные значения скоринговых баллов и нажмите "ОК". Будет осуществлена проверка соответствий скоринговых баллов. Если хотя бы для одного балла было введено значение, перечисленное в списках соответствия (см. описание Соответствия БАЛЛ № 1 - Балл № 2... в модуле Словари), то значения остальных баллов могут быть либо соответствующими, либо пустыми.

Если соответствие нарушено, то значения скоринговых баллов не обновляются. После закрытия формы выводится информационное сообщение:

½"По n долгам не были обновлены значения из-за настроенных ограничений",

где n – количество долгов, для которых не было обновлено значение переданного атрибута.

Примечание. Если Балл № 1 используется в качестве стадии (константа @DepartUseMark1 имеет значение "Истина"), то соответствия для Балл № 1 не проверяются.
