Сложный фильтр

Обновлено до 41 релиза 21.05.2019.

Списочные модули, в которых доступен сложный фильтр:

·Список должников;

·Постановка задач;

·Группы свободного поиска;

·Платежи;

·Протокол контактов;

·Исходящее информирование;

·Юридический департамент;

·Документооборот.

Примечание. Фильтрация с применением механизма "Сложный фильтр", осуществляется на стороне сервера. Поэтому рекомендуется в начале работы с модулем осуществить фильтрацию с применением сложного фильтра, чтобы отсечь заведомо ненужные записи. А затем к полученной выборке применяйте мультифильтры и остальные возможности упорядочивания записей для поиска нужных.

Условия, заданные сложным фильтром, и условия, заданные мультифильтрами, выполняются вместе и таким образом, как если бы они объединялись логическим оператором И. То же касается и встроенной фильтрации по колонкам (см. Фильтрация данных в списке), ее условия также объединяются с условиями сложного фильтра при помощи логического И.

Примечание. Для каждой вкладки модуля в сложном фильтре можно создать свои запросы, и их текст сохраняется при переходе между вкладками.

Сложный фильтр включается кнопкой , расположенной в правой верхней части рабочей области экрана. Если фильтрация уже была применена, то цвет кнопки изменяется: .

Описание формы "Сложный фильтр"

Форма Сложный фильтр включает следующие вкладки:

·Фильтр, включающая подчиненные вкладки:

·Активный фильтр — предназначена для задания условий фильтрации, применения их или сохранения.

·Сохраненные фильтры — предназначена для отображения сохраненных фильтров, применения, изменения и удаления выбранного сохраненного фильтра.

Примечание. В левой нижней части вкладки Фильтр отображается количество доступных элементов — оно соответствует количеству полей на текущей вкладке текущего модуля, доступных для фильтрации.

·Загрузка из файла — см. Использование данных из файла в качестве критериев фильтрации

Форма "Фильтр"

Применение сложного фильтра

Для создания условий и применения сложного фильтра выполните следующее:

1.Войдите в один из модулей, для которых реализован "Сложный фильтр".

2.Нажмите кнопку Сложный фильтр. Откроется форма Фильтр.

3.На вкладке Фильтр активируйте подчиненную вкладку Активный фильтр.

4.Сформируйте условия фильтрации, как описано в главе Конструктор фильтров и работа с ним. Нажмите кнопку "Обновить SQL текст" для отображения текста SQL-запроса. Текст SQL-запроса не редактируется и отображается только для ознакомления.

5.Нажмите OK. Окно сложного фильтра закроется, фильтр будет сохранен. В форме модуля будет осуществлен поиск данных в соответсвии с условиями сложного фильтра.

6.Если необходимо, задайте также условия в мультифильтрах и нажмите кнопку "Найти". Поиск будет осуществляться среди данных, найденных при помощи сложного фильтра.

Сохранение заданных условий фильтрации

Фильтры сохраняются в привязке к текущему пользователю, а также к в привязке к модулю и вкладке, на которой он был создан. Названия фильтров пользователя для одной вкладки не должны повторяться.

Для того чтобы сохранить фильтр для дальнейшего использования, выполните следующее:

1.В нужном модуле активируйте нужную вкладку.

2.Нажмите кнопку Сложный фильтр. Откроется форма Фильтр.

3.На вкладке Фильтр активируйте подчиненную вкладку Активный фильтр.

4.Сформируйте условия фильтрации, как описано в главе Конструктор фильтров и работа с ним. Нажмите кнопку "Обновить SQL текст" для отображения текста SQL-запроса. Текст SQL-запроса не редактируется и отображается только для ознакомления.

5.Нажмите кнопку Сохранить фильтр.

6.Откроется форма Сохранение сложного фильтра. Заполните следующие поля:

·Название — введите название фильтра, оно должно быть уникальным в пределах текущей вкладки модуля; поле обязательно для заполнения;

·Комментарий — дополнительная информация; поле не обязательно для заполнения.

7.Нажмите кнопку ОК.
