Назначение ответственного оператора по долгам

Операция назначения ответственного оператора по долгам доступна в форме Постановка задач на вкладках Все должники, Новые должники, Проблемные, Поиск информации, Подготовка выездов. Ответственный оператор может быть назначен:

·По всей выборке — для всех долгов из выборки, сформированной с учетом фильтров;

·Только выделенные — для всех долгов, выделенных в текущей выборке с помощью клавиш SHIFT и/или CTRL.

Примечание: если константа WorkTaskAllMode имеет значение "Ложь", то возможность назначить оператора по всей выборке долгов недоступна.

Примечание. По согласованию  с технической поддержкой возможна настройка списка статусов долга, для которых запрещено назначать долгу оператора. Для поддержки: В значении константы WorkTask.SetOperatorList.BlockedStatus перечислены статусы долга, для которых запрещено назначать долгу оператора. Долгам в любых других статусах разрешено назначать оператора.

В каждый момент времени с долгом может быть связан только один ответственный оператор. Во всех модулях и отчетах в атрибуте Оператор для данного долга будет отображаться последний оператор, которому распределили данного должника. Период работы данного оператора с долгом начинается с момента распределения и заканчивается в момент снятия оператора или назначения другого оператора. Все платежи и обещания за данный период являются результатом работы именного этого оператора.

Для того чтобы назначить ответственного оператора, выполните следующие действия:

1.В форме Постановка задач перейдите на одну из вкладок Все должники, Новые должники, Проблемные, Поиск информации, Подготовка выездов.

2.При необходимости сформируйте выборку долгов с помощью фильтров и/или выделите записи с помощью клавиш SHIFT и CTRL.

3.Нажмите кнопку Назначить оператора и в раскрывающемся списке выберите вариант выполнения операции: По всей выборке или Только выделенные.

4.Откроется форма Выбор оператора. В окне отображается информация об отделе, в котором работает каждый оператор, и о количестве закрепленных должников.

Примечание. Если в роли текущего пользователя доступ № 122 WORK_TASK_FULL имеет значение TRUE, то в списке доступных операторов отображаются все незаблокированные пользователей Системы, а также текущий оператор (даже если он заблокирован). Если доступ отсутствует или имеет значение FALSE, то список операторов отображается с учетом Ограничения доступа к данным на основании структуры предприятия.

5.Выберите оператора в списке.

6.Если константа WorkTask.Operator.Change.Reason.Use имеет значение "Истина", то на форме будет также присутствовать поле Причина для указания причины назначения долгов (см. словарь № 326 "Причина назначения/снятия долга"). Выберите причину. Заполнение данного поля не обязательно.

7.Если поддерживается функционал временного назначения ответственных по долгам, то на форме будет присутствовать поле Дата для ввода даты окончания срока закрепления данного оператора за выбранными долгами. Укажите дату. Заполнение данного поля не обязательно.

Ф-л временного закрепления операторов за долгами включается по технической константе WorkTask.SetUserUntilDate.Use.

8.Нажмите кнопку ОК.

9.Будет показано окно подтверждения с ФИО оператора и количеством выбранных долгов. Нажмите кнопку Yes, чтобы подтвердить назначение.

10.После этого выполняется проверка, имеет ли право данный оператор работать с каждым из выбранных для назначения должников. Проверка происходит в соответствии с ограничениями, заданными в модуле Настройка списка должников. Если оператор имеет доступ не ко всем долгам, то будет показано соответствующее предупреждение. Нажмите ОК.

11.По окончании процесса будет выдано информационное сообщение о количестве должников, которых удалось распределить.

Примечание. Если в роли оператора, которому назначают долги, доступ № 321 WORK_TASK_SEND_NOTIFICATION имеет значение TRUE, то ему приходит уведомление о назначении долгов.

Отображение данных в форме "Выбор оператора"

По умолчанию в форме скрыты дополнительные колонки. Добавить в таблицу их можно, включив флаги в списке выводимых полей. Также вы можете изменить порядок следования колонок в таблице. Полученный вид таблицы выбора оператора будет сохранен при следующем входе в модуль. Для каждого из указанных модулей будет сохранен свой вид таблицы. Изменения сохраняются вплоть до сброса пользовательских настроек.

Полный список колонок формы Выбор оператора:

·ФИО — ФИО оператора;

·Роль — роль оператора;

·Отдел — отдел оператора в модуле Структура предприятия;

·Количество в Списке должников — количество долгов, закрепленных за оператором;

·Кол-во судебных дел (предв.) — учитываются только неопределенные судебные дела, находящиеся в статусе 4;

Примечание. Колонка Кол-во судебных дел (предв.) отображается только в случае, когда константа LawActUseIndefiniteAct = "Истина".

·Кол-во приказов — учитываются только приказы, находящиеся в статусах 9, 10, 11;

·Кол-во исков — учитываются только иски, находящиеся в статусах 13, 14, 15, 18;

·Кол-во правопреемств — учитываются только правопреемственные дела, находящиеся в статусах 13, 14, 15;

Примечание. Колонка Кол-во правопреемств отображается только в случае, когда константа LawActUseSuccession = "Истина".

·Кол-во банкротств — учитываются только банкротства, находящиеся в статусах 6, 13, 14, 15;

Примечание. Колонка Кол-во банкротств отображается только в случае, когда константа LawActUseBankrupt = "Истина".

·Кол-во ИП — учитываются только дела исполнительного производства, находящиеся в статусах 3, 4, 5, 6, 10;

·Всего — в колонке отображается суммарное количество активных дел и долгов, закрепленных за оператором. Причем, если функционал неопределенных судебных дел, правопреемства или банкротства отключен (соответствующая константа равна "Ложь"), то дела данного функционала, закрепленные за оператором, не учитываются в суммарном количестве дел;

·Запрет закрепления — колонка отображается для досудебной стадии при значении доступа № 543 DEBT_BAN_SET_FLAG_IGNORE = TRUE.

Учет запрета закрепления

Для досудебной стадии взыскания реализована возможность видеть в списке операторов с включенным признаком запрета закрепления.

В зависимости от значения доступа № 543 DEBT_BAN_SET_FLAG_IGNORE:

·Если доступ имеет значение TRUE, то в списке будут показаны все операторы. Поле Запрет закрепления отображает, для каких пользователей данный признак включен. При попытке осуществить назначение оператора, имеющего запрет на закрепление, пользователю будет показано предупреждение.

·Если доступ имеет значение FALSE, то в списке будут отображаться только те операторы, у которых отсутствует запрет на закрепление. Поле Запрет закрепления на форме отсутствует. Пользователь может осуществить назначение только тех операторов, которые не имеют запрета на закрепление.

   

Доступ DEBT_BAN_SET_FLAG_IGNORE=TRUE
