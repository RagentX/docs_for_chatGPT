Работа с файлом в универсальном формате

В процессе формирования файла в универсальном формате автоматически формируются данные по всем загружаемым полям таблиц персон, паспортов, долгов, портфелей долгов, телефонов, адресов, произвольных атрибутов, поручителей должников. Если для файла MS Excel используется тип формата "Судебная работа", то формируются данные по всем загружаемым полям таблицы судебной работы, а в случае формата "Исполнительное производство" формируются данные по всем загружаемым полям таблицы исполнительного производства. Тип формата для файла MS Excel запоминается в одном из вспомогательных полей файла в универсальном виде. Вспомогательные поля обеспечивают возможность приведения данных из универсального вида к стандартной таблице данных на экране.

Строка файла MS Excel в универсальном формате содержит следующие поля:

·поля персоны, долга, поручителей должника, паспортных данных, данных о портфеле. Полный состав полей определяется на основе данных из таблицы column_spec: строки с is_load = 1, для которых parent_id равен 30 (персона), 8 (долг), 10 (поручитель), 27 (паспорт) или 49 (портфель);

·поля судебной работы (только для исходных файлов с типом формата "Судебная работа"). Полный состав полей определяется на основе данных из таблицы column_spec: строки с is_load = 1, для которых parent_id равен 46 (судебная работа);

·поля исполнительного производства (только для исходных файлов с типом формата "Исполнительное производство"). Полный состав полей определяется на основе данных из таблицы column_spec: строки с is_load = 1, для которых parent_id равен 47 (исполнительное производство);

·поля значений произвольных атрибутов (при наличии произвольных атрибутов в исходном файле);

·поле "Тип формата", определяется типом формата исходного файла;

·поле "Число телефонов" (при наличии данных о телефонах в исходном файле);

·поля данных о телефонах (при наличии данных о телефонах в исходном файле). Для каждого телефона предусмотрены поля "Тип", "Номер" и "Комментарий";

·поле "Число адресов" (при наличии данных об адресах в исходном файле);

·поля данных об адресах (при наличии данных об адресах в исходном файле). Для каждого адреса предусмотрены поля "Тип", "Индекс", "Область", "Район", "Город", "Улица", "Дом", "Корпус", "Квартира", "Код kladr", "Статус адреса в системе DaData" и "Комментарий".

При открытии файла в универсальном формате пустые столбцы и вспомогательные столбцы "Тип формата", "Число телефонов", "Число адресов" автоматически удаляются, файл преобразуется к первоначальному виду.

В процессе открытия файла в универсальном формате пустые столбцы и вспомогательные столбцы "Тип формата", "Число телефонов", "Число адресов" автоматически удаляются, файл преобразуется к первоначальному виду. Поле считается пустым, если ни одна из строк этого поля не содержит информации. Функции преобразования данных выполняются автоматически, без диалога с пользователем.

Проверка и загрузка данных производится согласно разделам Проверка данных, Корректировка данных по результатам проверки, Загрузка данных в базу данных.
