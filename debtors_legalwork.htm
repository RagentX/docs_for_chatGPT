Судебная работа

Show/Hide Hidden Text

Создание судебных дел и исполнительного производства возможно, если Системой поддерживается модуль Юридический департамент.

Меню Судебная работа видимо и доступно в случае, когда значение константы @LegalCollection равно истина. 

Доступы на судебную работу:

·создание неопределенных судебных дел (доступно, если константа LawActUseIndefiniteAct равна "Истина" и доступ № 58 LAW_ACT_CREATE равен TRUE);

·создание судебных приказов по выбранным долгам (доступно, если доступ № 58 LAW_ACT_CREATE равен TRUE);

·создание правопреемства по выбранным долгам (доступно, если доступ № 58 LAW_ACT_CREATE равен TRUE, и константа LawActUseSuccession равна "Истина");

·создание судебных исков по выбранным долгам (доступно, если доступ № 58 LAW_ACT_CREATE равен TRUE);

·создание банкротства по выбранным должникам (доступно, если доступ № 58 LAW_ACT_CREATE равен TRUE);

·создание дел исполнительного производства по выбранным долгам (доступно, если доступ № 167 LAW_EXEC_CREATE равен TRUE и LAW_EXEC_CREATE_FROM_DEBT равен TRUE);

·создание исполнительных надписей по выбранным долгам (доступно, если доступ № 58 LAW_ACT_CREATE равен TRUE и включен функционал исполнительной надписи (включена коммерческая константа @LawAct.ExecInscription.Use)).

Доступы на создание судебных дел и ИП, по которым ответчиком является поручитель должника:

·Доступ № 660 LAW_ACT_GUARANTOR_CREATE дает право создавать дела судебной работы по поручителям должника (то есть, по таким делам поручители являются ответчиками) следующих типов:

·неопределенные судебные дела;

·судебные приказы;

·правопреемственные дела;

·судебные иски;

Примечание. Банкротства по поручителям не создаются.

·Доступ № 661 LAW_EXEC_GUARANTOR_CREATE дает право на создание дел исполнительного производства по поручителям (то есть, по таким делам поручители являются ответчиками).

       Доступы на создание более одного судебного дела по долгу

На работу функционала влияют следующие доступы:

·Доступ № 356 LAW_ACT_MULTIPLE;

·Доступ № 626 LAW_ACT_MULTIPLE_DIFFERENT_TYPE.

Особенности совместной работы доступов:

·Включены оба доступа — разрешается для долга создавать любое количество дел любого типа;

·Включен доступ № 356, выключен доступ № 626 — разрешается для долга создавать не более одного дела каждого типа;

·Выключены оба доступа — разрешается для долга создать только одно судебное дело.

Примечание: если константа LawActGetOperatorFromPrevStage имеет значение TRUE, то при массовом создании судебных исков/приказов карточки судебной работы/исполнительного производства будут закреплены за теми же операторами, что и карточка должника (работает только при выключенной константе DepartChangeClearOperator).

Для того чтобы сформировать дела судебной работы или ИП с долгами по должникам (или также и по поручителям), выполните следующие действия:

1.В форме Список должников активируйте нужную вкладку.

2.Сформируйте выборку, используя при необходимости сложный фильтр и/или мультифильтры.

3.Выберите одну или более записей о долгах. Для выбора нескольких записей используйте клавиши SHIFT и/или CTRL.

4.В контекстном меню (вызывается правой кнопкой мыши) выберите пункт Судебная работа, затем пункт:

·Создать суд. дело (предв.) — в этом случае по выбранным долгам будут созданы неопределенные судебные дела (подробнее см. Карточка судебной работы в судебном производстве);

·Создать судебные приказы — в этом случае по выбранным долгам будут созданы судебные приказы (подробнее см. Карточка судебной работы в приказном производстве);

·Создать правопреемство — в этом случае по выбранным долгам будут созданы дела правопреемства (подробнее см. Карточка судебной работы в правопреемстве);

·Создать судебные иски — в этом случае по выбранным долгам будут созданы судебные иски (подробнее см. Карточка судебной работы в исковом производстве);

·Создать банкротство — в этом случае по выбранным должникам будут созданы дела банкротства (подробнее см. Карточка судебной работы в банкротстве);

Внимание. Банкротство создается по должнику, а не по долгу.

·Исполнительное производство — в этом случае по выбранным долгам будут созданы карточки исполнительного производства (подробнее см. Карточка исполнительного производства);

·Исполнительная надпись — в этом случае по выбранным долгам будут созданы карточки исполнительной надписи (подробнее см. Карточка исполнительной надписи).

5.Если по долгу есть поручители, и пользователь имеет доступы на создание дел по которым поручители являются ответчиками, то для всех типов дел, кроме банкротства, появляется форма подтверждения с вопросом, требуется ли дополнительно создать такое же дело по поручителям. Для создания таких дел (помимо дел по должникам) нажмите ОК.

Для каждого долга из списка осуществляется ряд проверок возможности создания судебного дела или исполнительного производства в зависимости от настроек Системы.

       Проверки при создании судебного дела по долгу

Проверки при создании судебного дела по долгу

·Если доступ № 356 LAW_ACT_MULTIPLE включен:

·Если доступ № 626 LAW_ACT_MULTIPLE_DIFFERENT_TYPE включен:

§Если по данному долгу нет дел, то дело создается.

§Если по данному долгу есть судебные дела, то пользователю выдается сообщение. Если он выбирает создать новое дело, дело создается.

·Если доступ № 626 LAW_ACT_MULTIPLE_DIFFERENT_TYPE выключен:

§Если константа LawAct.Create.CheckActiveStatus.Use включена, то проверяется наличие судебного дела создаваемого типа в активном статусе.

§Если константа LawAct.Create.CheckActiveStatus.Use выключена, то проверяется наличие судебного дела создаваемого типа в любом статусе.

oЕсли такое дело есть (хотя бы одно), то пользователю выдается сообщение, и дело не создается.

oЕсли такого дела нет, то дело создается.

·Если доступ № 356 LAW_ACT_MULTIPLE выключен:

·Если константа LawAct.Create.CheckActiveStatus.Use включена:

§Если по долгу уже существует хотя бы одно судебное дело в активном статусе, то пользователю выдается сообщение, и дело не создается.

§Если по долгу нет ни одного судебного дела в активном статусе, то дело создается.

·Если константа LawAct.Create.CheckActiveStatus.Use выключена:

§Если по долгу нет ни одного судебного дела любого типа в любом статусе, то дело создается.

§Если по долгу есть хотя бы одно судебное дело любого типа в любом статусе, то:

oВыдается сообщение, что повторное дело создано не будет.

oНовое дело не создается, и никакое дело не возобновляется.

Примечание. Список статусов, которые считаются активными для дел различного типа:

·Неопределенные судебные дела (предварительные) — статус <> 4 (см. словарь № 252);

·Приказы — статус <> 9, 10, 11 (см. словарь № 18);

·Иски — статус <> 13, 14,15, 18 (см. словарь № 25);

·Правопреемство — статус <> 13, 14, 15 (см. словарь № 25);

·Банкротство — статус <> 13, 14. 15.

Информация для поддержки

Функционал создания судебного дела используется в модулях:

·Карточка должника — единичное действие;

·Список должников — единичное и массовое действие;

·Группы свободного поиска — единичное и массовое действие.

На работу функционала влияют следующие константы

·Обычная константа LawAct.Create.CheckActiveStatus.Use;

·Техническая константа LawAct.Create.ReturnAndChangeType.Use.

Изначально для обычных клиентов они настроены:

·Константа LawAct.Create.CheckActiveStatus.Use = Истина;

·Константа LawAct.Create.ReturnAndChangeType.Use = Ложь.

Для Почта банка:

·Доступ № 357 LAW_EXEC_MULTIPLE = FALSE;

·Константа LawAct.Create.CheckActiveStatus.Use = Ложь;

·Константа LawAct.Create.ReturnAndChangeType.Use = Истина.

 Внимание! Важно! Пользовательский текст справки написан именно для этих двух случаев — для всех и для Летобанка (Почта банка).

       Проверки при создании исполнительного производства по долгу

Проверка доступа 357 и наличия ранее созданного ИП по данному делу:

·Если доступ № 357 LAW_EXEC_MULTIPLE включен:

oЕсли ИП по данному долгу не существует, то создается новое ИП.

oЕсли ИП по данному долгу существует, то пользователю выдается сообщение. Если пользователь выбирает продолжить формирование, то формируется новое ИП.

·Если доступ № 357 LAW_EXEC_MULTIPLE выключен:

oЕсли константа LawExec.Create.CheckActiveStatus.Use включена, то:

§Если существует ИП в активном статусе для долга, то новое ИП создано не будет.

§Если не существует ИП в активном статусе для долга, то создается новое ИП.

oЕсли константа LawExec.Create.CheckActiveStatus.Use выключена, то:

§Если по долгу нет ни одного ИП в любом статусе, то создается новое ИП.

§Если по долгу есть ИП в любом статусе, то пользователю выводится сообщение, а новое ИП не создается.
