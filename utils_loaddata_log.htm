Формирование лога с данными о работе модуля загрузки данных

Если константа LoadDataDebugMode имеет значение "Истина", автоматически создается текстовый файл load_data <дата выполнения>.log с данными о работе модуля загрузки данных.

Путь для сохранения файла лога зависит зависит от значения константы LogFileInLocalSettingsDir.

Если константа включена, то файл сохранятеся на рабочий компьютер пользователя в директорию:

"C:\Users\%username%\AppData\Roaming\LuxBase\DebtControl".

Если константа выключена, то файл сохраняется в папке, откуда осуществляется запуск модуля.

Каждая текстовая строка файла имеет следующую структуру:

·текущая дата и время (с точностью до секунды);

·название процедуры, инициализировавшей вывод строки;

·информационный текст.

Информационный текст первой строки файла содержит сведения о версии модуля загрузки и о пользователе. Информационный текст последней строки содержит сообщение о возврате в исходное состояние. Информационный текст остальных строк содержит сведения о выполнении следующих функций в модуле загрузки данных:

·выбор формата;

·выбор файла;

·преобразование данных;

·уточнение нестандартных данных при их преобразовании;

·проверка данных в построчном режиме и в режиме массового обновления;

·загрузка данных в построчном режиме и в режиме массового обновления;

·обработка нестандартных записей при загрузке в построчном режиме;

·дозагрузка данных о контактных лицах;

·дозагрузка данных о расчетных лицах;

·дозагрузка телефонов;

·дозагрузка адресов;

·дозагрузка текстовых адресов.

В состав текстовых строк включаются также итоговые строки по результатам выполнения следующих функций:

·преобразование данных;

·проверка данных;

·загрузка данных;

·дозагрузка данных о контактных лицах;

·дозагрузка данных о расчетных лицах;

·дозагрузка телефонов;

·дозагрузка адресов;

·дозагрузка текстовых адресов.

Примечание для поддержки

При загрузке по всем форматам выполняется проверка технической константы LoadData.LogEachPerson.Use. Если константа имеет значение "Истина", то выполняется логирование действий по загрузке каждой записи по данному формату (запись в atcions) с указанием ссылки на должника (r_person_id).
