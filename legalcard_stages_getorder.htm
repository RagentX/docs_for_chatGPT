Получение приказа

Данный функционал доступен только если значение константы LawAct.Order.SendAndReceive.Person.Use равно "Истина".

Получение приказа доступно для дел, находящихся в статусах:

·Приказ отправлен должнику;

·Приказ получен должником;

·Отмена приказа (если дополнительно константа LawAct.OrderCancel.IsFinal имеет значение "Истина");

·Возобновление.

Данная операция может быть выполнена только после внесения всей необходимой информации:

·поле Дата вступления в силу.

·поле Получен Агентством.

Для перевода приказа в статус "Приказ получен" выполните следующие действия:

1.В карточке судебной работы на вкладке Приказное производство в разделе Судебный приказ введите дату получения в поля Получен Агентством и Дата вступления в силу.

2.Нажмите кнопку Приказ получен и в раскрывающемся списке выберите нужный вариант (Банком (для перевода в статус 5 "Приказ получен должником") или Агентством (для перевода в статус 7 "Приказ получен").

3.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить выполнение операции. Если поле Получен Агентством не заполнено, будет выдано информационное сообщение. Нажмите в нем кнопку ОК, после чего произойдет возврат в карточку судебной работы для внесения данных.

4.Выполните прикрепление документа.

5.Далее осуществляется формирование настроенных запланированных действий.

6.После этого приказ будет переведен в статус "Приказ получен должником" (если в п.5 был выбран вариант Банком) или "Приказ получен" (если в п.5 был выбран вариант Агентством).

Примечание. Если константа LawAct.Decision.Components.Compare имеет значение "Истина", то при выполнении действия Получение приказа дополнительно сравниваются текущие значения составляющих на момент подачи в суд со значениями составляющих по решению суда (составляющие для приказа перечислены в константе LawActOrderDebtComponentList), а также значение размера пошлины с утвержденным размером пошлины.

Если хотя бы одна пара значений не совпадает, пользователю отображается окно с сообщением:

Поданные и присужденные суммы не совпадают. Продолжить?

Нажмите ОК чтобы продолжить или Отмена чтобы прервать действие.

Примечание для поддержки

Если был выбран подпункт Банком, то производится дополнительная проверка обязательности заполнения следующих полей карточки приказного производства, согласно значению технической константы LawAct.PositiveResult.Fields.Mandatory:

·Утвержденная пошлина — если в скобке с единицей есть значение 741;

·Сумма по решению суда — если в скобке с единицей есть значение 740;

·Получен Агентством — если в скобке с единицей есть значение 794;

·Дата вступления в силу — если в скобке с единицей есть значение 525.
