Просмотр данных о судебных исках

Данные о судебных исках отображаются в форме Юридический департамент на вкладке Иски.

Вкладка содержит следующие поля:

·ID должника — идентификатор должника, с которым связан иск;

·ID долга — идентификатор долга, с которым связан иск;

·ID дела — уникальный идентификатор (порядковый номер) дела в системе;

·ФИО — фамилия, имя, отчество ответчика по делу (должника или поручителя);

·Тип лица — тип ответчика (должника или поручителя), см. словарь № 54;

·Банк;

·Портфель;

·Ответственный — оператор, ответственный за работу с делом;

·Дело поступило — дата поступления дела;

·Дата ИЛ — дата получения исполнительного листа (из карточки иска);

·Дата ИЛ на обеспеч. — дата получения решения суда, вступившего в силу (из карточки иска);

·Статус — статус судебного дела, см. словарь № 25;

·Оконч. план действий — дата окончания запланированного задания;

·Договор;

·Лицевой счет — номер счета долга;

·ID Клиента — дополнительный идентификатор долга в системе;

·Валюта — валюта долга (см. Список валют);

· Размер требования — сумма долга, выводится в валюте долга, указанной в поле Валюта; выводится сумма значений по выборке;

Примечание. Для Лето банка для исков, находящихся в статусах, перечисленных в константе PochtaBank.DWH.UpdateTotalSum.IskStatusList, поле Размер требования (law_act.total_sum) для долга обновляется по следующей формуле:

½debt.penalty_sum + debt_detail.number_value_1 + debt_detail.number_value_2 + debt_detail.number_value_3 + debt.commission_sum

·Гос. пошлина — сумма госпошлины, выводится в основной валюте Системы; выводится сумма значений по выборке;

·Утв. гос. пошлина — утвержденная сумма пошлины, выводится в основной валюте Системы; выводится сумма значений по выборке;

·Название суда;

·Дата подачи — дата подачи приказа в суд;

·Крайняя дата подачи — крайняя дата подачи приказа в суд (см. также Автоматический расчет крайней даты подачи);

·Дата вынесения решения — дата вынесения решения судом;

·Дата вступления в силу — дата вступления приказа в силу;

·Комментарий;

·Цвет — выделение цветом для удобства работы;

·Название продукта — указывается название продукта, по которому возник долг;

·Тип продукта — см. словарь № 11;

·Остаток — сумма остатка, выводится в валюте долга, указанной в поле Валюта; выводится сумма значений по выборке;

·Дата рождения — дата рождения ответчика (должника или поручителя);

·Серия — серия паспорта ответчика-должника (если ответчик — поручитель, поле остается пустым);

·Номер — номер паспорта ответчика-должника (если ответчик — поручитель, поле остается пустым);

·Филиал — филиал организации, к которому относится долг (см. словарь № 61);

·Получен оригинал кредит. договора — признак получения оригинала кредитного договора;

·Вынесена вся сумма на просроч. — признак вынесения всей суммы долга на просрочку;

·Прикреплены обеспеч. меры.— признак того, что прикреплен документ с описанием обеспечительных мер;

·Регион — регион, в котором находится филиал организации (см. словарь № 39);

·Дата вынесения всей суммы на просрочку;

·Тип иска — входящий или исходящий иск;

·Укомплектованность;

·Балл № 1;

·Дата;

·Примечание — см. словарь № 247;

 Примечание для поддержки. Колонка Примечание  на данной вкладке видна только если в column_spec у колонки с id=884 заполнено поле user_name. Наименование колонки должно быть реализовано по стрингу law_act_list.act_dict_value_1.

·Встречный иск — флаг включен для исков, которые являются встречными;

·Номер дела в суде;

·Тип противопол. стороны — роль участника договора в судебном процессе, см. словарь № 325.

Примечание. Область, на которой отображается информация о датах, способах и результатах контактов с должником, информация о переводе долга на аутсорсинг и возврата из аутсорсинга, доступна для просмотра на форме, если константа DebtListContactHistory имеет значение "Истина". При этом в списке не отображаются контакты тех пользователей, чьи идентификаторы перечислены в значении константы ContactHistory.ContactLog.IgnoreUsersList.
