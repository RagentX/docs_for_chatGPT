Ограничения на дату и сумму обещаний, регистрируемых вручную

Show/Hide Hidden Text

Данные ограничения действуют при регистрации обещания вручную из индивидуальных рабочих мест:

·Карточка должника;

·Карточка судебной работы;

·Карточка исполнительного производства.

Данные ограничения работают для долгов, не включенных в активные акции.

ØОграничения на срок обещания:

ØКарточка должника;

Ограничения на срок обещания

Когда должник обещает осуществить оплату, он указывает дату, не позже которой он это сделает. Эта дата называется обещанная дата.

Карточка должника

Срок обещания — то есть, количество дней между текущей датой и обещанной датой — регулируется константами:

·PromMaxDayHard — для операторов типа Hard;

·PromMaxDaySoft — для оператора любого типа, кроме Hard.

Обещание, нарушающее указанные сроки зарегистрировать невозможно.

Карточки судебной работы и исполнительного производства

Срок обещания регулируется константами:

·30 дней — для операторов типа Hard;

·21 день — для оператора любого типа, кроме Hard.

Обещание, нарушающее указанные сроки зарегистрировать невозможно.

Ограничения на обещанную сумму

Сумма, которую должник обещает оплатить, называется обещанная сумма.

Ограничения на минимальный размер обещанной суммы

Минимальный размер обещанной суммы регулируется константами

·PromMinSumRateHard — для операторов типа Hard;

·PromMinSumRateSoft — для операторов любого типа кроме Hard.

Если обещанная сумма меньше минимальной разрешенной, логика Системы зависит от значения доступа № 515 REG_CONTACT_INSUFFICIENT_PROM_SUM в роли пользователя:

·REG_CONTACT_INSUFFICIENT_PROM_SUM = TRUE — обещание может быть зарегистрировано (нажмите Yes в окне подтверждения), но тогда ему будет присвоен статус "Не верифицировано". Такие обещания не считаются полноценными и не отображаются в списочных РМ. Для того, чтобы такое обещание стало полноценным, его следует подтвердить (верифицировать). См. Подтверждение обещания оплаты по долгу.

·REG_CONTACT_INSUFFICIENT_PROM_SUM = FALSE — обещание зарегистрировать нельзя.

Ограничения на максимальный размер обещанной суммы

Если в роли пользователя доступ № 396 REG_CONTACT_MAX_PROM_SUM_CONTROL имеет значение TRUE, то у таких пользователей работает ограничение на максимальную сумму обещания.

Если обещанная сумма превышает сумму остатка на количество процентов, указанных в константе MaxPromSumPercent, то такое обещание не будет зарегистрировано. При попытке регистрации такого обещания пользователю отображается соответствующее сообщение.
