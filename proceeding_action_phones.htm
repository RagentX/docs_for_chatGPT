Вид формы добавления/редактирования атрибутов контактного телефона

Описание формы добавления/редактирования атрибутов контактного телефона

Форма добавления/редактирования телефонного номера включает следующие элементы:

·Тип — тип телефонного номера, выпадающий список (см. словарь № 4).

Примечание.

·Если форма открывается для должника, то в выпадающий список попадают только те термины из словаря № 4, которые относятся к должнику (код которых находится в интервале от 1 до 100).

·Если форма открывается для поручителя, то в выпадающий список попадают только те термины из словаря № 4, которые относятся к поручителю (код которых находится в интервале от 101 до 200).

·Если форма открывается для залогодателя, то в выпадающий список попадают только те термины из словаря № 4, которые относятся к залогодателю (код которых превышает 200).

·Словарный атрибут телефона 1 — см. словарь № 258. Данное поле заполнено по умолчанию значением, которое определяется константой Phone.DictValue1.Default. (Примечание: отображение данного атрибута и его название настраиваются по согласованию с технической поддержкой.)

·Словарный атрибут телефона 2 — см. словарь № 309. Если словарь № 309 имеет в качестве родительского словарь № 258, то в списке отображаются только термины словаря, являющиеся дочерними по отношению к выбранному термину в поле "Словарный атрибут телефона 1". (Примечание: отображение данного атрибута и его название настраиваются по согласованию с технической поддержкой.)

·Словарный атрибут телефона 3 — см. словарь № 343. (Примечание: отображение данного атрибута и его название настраиваются по согласованию с технической поддержкой.)

Примечание. Названия словарных атрибутов телефона на форме "Телефон" и их отображение или сокрытие настраиваются по согласованию с технической поддержкой.

Для поддержки:

·Словарные атрибуты телефона не отображаются, если для них установлен соответствующий column_spec.is_not_use=1.

·Иначе отображение атрибутов определяется значением column_spec.display_mode:

·NULL - отображаются в карточке и гриде

·1 - отображаются только в карточке

·2 - отображаются только в гриде

Примечание. Право на изменение значения поля Словарный атрибут телефона 2 имеют только пользователи, в роли которых доступ № 628 PHONE_DICT_VALUE_2_SET_LIST имеет значение "ALL" или содержит список терминов из словаря № 309, разрешенных данному пользователю для установки в качестве значения поля.

·Номер — номер телефона. Поле обязательно для заполнения.

Примечание. Если константа PhoneFormatCheck имеет значение "Истина", то при добавлении номера телефона осуществляется проверка формата номера на соответствие 11-ти символьному формату телефона.

·Контактное лицо — фамилия, имя, отчество контактного лица.

·Не отправлять автоматические SMS — если для номера установить этот флаг, то в процессе автоматического формирования SMS-сообщения и формирования SMS в модуле "Исходящее информирование" SMS на него отправляться не будут.

·Запретить голосовое информирование — если установить данный флаг, то номер не будет включен в список телефонных номеров для автоинформатора;

·Комментарий — дополнительная информация.

·Флаг Верифицирован — признак верификации телефона, см. Верификация контактных данных.

·Флаг Активен — признак активности телефона.

Примечание. Флаг "Активен" присутствует на форме только если константа PhoneCardShowCheckState имеет значение "Истина".

Примечание. Если константа PhoneManualCreationDefaultCheckState имеет значение "Истина", то при добавлении нового номера его статус устанавливался в значение "Активный".

·Кнопка OK — сохранить.

Добавление телефона

Добавление телефона доступно только тем пользователям, в роли которых доступ № 13 DEBT_CARD_EXTRA_EDIT имеет значение TRUE.

Для того чтобы добавить контактный телефон в список, выполните следующие действия:

1.Перейдите на вложенную вкладку Телефоны и нажмите кнопку .

2.В открывшейся форме добавления/редактирования атрибутов контактного телефона заполните нужные поля.

3.Нажмите кнопку ОК, чтобы сохранить данные.

Примечание. Если константа Phone.DictValue1.Default не равна нулю и поле Словарный атрибут телефона 1 при добавлении телефона было оставлено пустым, то  данное поле заполняется значением из константы.

Проверка совпадающих номеров

При добавлении нового телефонного номера осуществляется проверка совпадающих номеров согласно значению константы Phone.Add.CheckEqual.Mode. Если константа имеет значение:

·1 — при добавлении телефона никаких дополнительных проверок не осуществляется;

·2 — если добавляемый номер совпадает с одним из существующих телефонных номеров данного должника, то добавления нового телефона не происходит, и пользователю выводится соответствующее сообщение;

·3 — если добавляемый номер совпадает с одним из существующих телефонных номеров данного должника, относящихся к тому же типу, что и добавляемый номер, то добавления нового телефона не происходит, и пользователю выводится соответствующее сообщение.

Редактирование телефона

Редактирование всех атрибутов телефона доступно только тем пользователям, в роли которых доступ № 92 DEBT_CARD_PHONE_ADR_EDIT имеет значение TRUE.

Если доступ № 92 DEBT_CARD_PHONE_ADR_EDIT у пользователя отсутствует или имеет значение FALSE, но при этом доступ № 143 DEBT_CARD_PHONE_ADR_DSC_EDIT имеет значение TRUE, то для изменения доступны только поля:

·Контактное лицо;

·Комментарий.

Если в Системе используются словарные атрибуты телефона, то дополнительно выполняется следующая логика:

Если доступ № 92 DEBT_CARD_PHONE_ADR_EDIT отсутствует или имеет значение FALSE, то при попытке редактирования телефона проверяется значение доступа № 846 PHONE_ADR_ EDIT_EXCEPTION:

·Если данный доступ отсутствует или его значение пусто, то доступ к редактированию словарных атрибутов телефона отсутствует.

·Если данный доступ заполнен, то доступы к редактированию словарных атрибутов телефонов определяются наличием в списке id соответствующих атрибутов согласно таблице column_spec:

·863 — для словарного атрибута телефона 1;

·901 — для словарного атрибута телефона 2;

·2001 — для словарного атрибута телефона 3.

·Если в списке отсутствуют все три вышеперечисленных значения, то доступ к редактированию телефонов у пользователя отсутствует.

Порядок действий

Для того чтобы отредактировать данные о контактном телефоне должника, выполните следующие действия:

1.Перейдите на вложенную вкладку Телефоны, выберите запись и нажмите кнопку .

Примечание: двойной щелчок мыши по выбранной записи приведет к открытию формы регистрации контакта (см. Регистрация контакта).

2.В форме добавления/редактирования атрибутов контактного телефона измените необходимые данные.

3.Нажмите кнопку ОК, чтобы сохранить изменения.

Удаление телефона

Удаление телефона доступно только тем пользователям, в роли которых доступ № 238 DEBT_CARD_PHONE_DEL имеет значение TRUE.

Для запрета удаления телефонов, по которым были зарегистрированы контакты, используется константа BlockPhoneNotificationMode. В зависимости от ее значения:

0 — запрещено удаление телефонов, по которым были зарегистрированы любые контакты

1 — запрещено удаление телефонов, по которы были зарегистрированы успешные контакты (флаг Считать результат успешным включен)

2 — при наличии контактов по удаляемому телефону будет показано соответстующее уведомление.

Для того чтобы удалить контактный телефон, выполните следующие действия:

1.Перейдите на вложенную вкладку Телефоны и выберите нужную запись.

2.Нажмите кнопку .

3.Дальнейшее зависит от значения константы BlockPhoneNotificationMode и наличия контактов по данному телефону:

a)Если по удаляемому телефону не зарегистрировано контактов (или BlockPhoneNotificationMode = 1 и по телефону не зарегистрировано успешных контактов), то будет показано сообщение:

Нажмите Yes, чтобы удалить телефон.

b)Если BlockPhoneNotificationMode = 0 и по телефону были были зарегистрированы контакты, то будет показано сообщение:

Нажмите OK. Телефон удален не будет.

c)Если BlockPhoneNotificationMode = 1 и по телефону были были зарегистрированы успешные контакты, то будет показано сообщение:

Нажмите OK. Телефон удален не будет.

d)Если BlockPhoneNotificationMode = 2 и по телефону были были зарегистрированы контакты, то будет показано сообщение:

Нажмите Yes, чтобы удалить телефон.

Блокирование телефонного номера

Блокирование телефона отличается от операции удаления тем, что заблокированный телефон остается в базе данных и отображается в списке телефонов. Возможность блокирования/разблокирования контактного телефона должника доступна пользователю при наличии у него доступа № 353 DEBT_CARD_PHONE_BLOCK.

Для того чтобы заблокировать контактный телефон, выполните следующие действия:

1.Перейдите на вложенную вкладку Телефоны и выберите нужную запись.

2.Активируйте контекстное меню правой кнопкой мыши и выберите пункт Заблокировать телефон.

3.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить блокирование контактного телефона.

4.Появится форма ввода причины блокировки телефона. Список возможных причин хранится в словаре № 313 "Причина блокировки телефона". Если константа Phone.Block.Reason.Mandatory имеет значение "Истина", то ввод причины обязателен. Выберите причину блокировки телефона (если необходимо).

5.Нажмите ОК.

6.Запись о заблокированном контактном телефоне будет выделена серым шрифтом.

Примечание. Просмотр и разблокирование заблокированных телефонов доступны в карточке должника пользователям, имеющим соответствующие полномочия. Подробнее см. Блокирование/разблокирование контактного телефона.
