Порядок действия для редактирования долга

Для того чтобы отредактировать данные о долге, выполните следующие действия:

1.В карточке должника на вкладке Долги выберите нужную запись в списке долгов и нажмите кнопку Изменить или выполните двойной щелчок левой клавишей мыши по выбранной записи.

2.Откроется форма "Информация о долге". Измените необходимые атрибуты.

3.Нажмите кнопку ОК, чтобы сохранить изменения.

Примечание. Если в роли пользователя отсутствуют или выключены доступы № 12 DEBT_CARD_FIN_EDIT и № 141 DEBT_CARD_FIN_DSC, поля формы не доступны для редактирования. Форма открывается только на просмотр.

Примечание. Для отображения дополнительных данных по долгу может быть настроена вкладка "Дополнительно".

Формы для просмотра и редактирования информации о долге

В Системе используются две основные формы для долгов:

·Долг кредитной организации / коллекторского агентства; (Для банка портфеля долга bank.collect_direct = 1)

·Долг по ЖКХ. (Для банка портфеля долга bank.collect_direct = 2)

Наличие в конкретной конфигурации Системы одной из этих форм или обоих определяется при первой установке Системы. Добавление или удаления форм может быть осуществлено только вендором, обратитесь в техподдержку.

Поля формы "Информация о долге" для долга кредитной организации / коллекторского агентства

·Портфель — портфель, в который входит долг;

Примечание. Если в роли пользователя доступ № 536 DEBT_CHANGE_PORTFOLIO имеет значение TRUE, то при редактировании данных о долге значение портфеля доступно для редактирования.

·Название банка — полное наименование контрагента, которому принадлежит портфель;

·Телефон банка — телефон контрагента (заполняется автоматически);

·Дата передачи — дата подписания портфеля (заполняется автоматически);

·Дата окончания — дата окончания работ портфеля(заполняется автоматически;

·Коэффициент — коэффициент успешности портфеля;

·Балл № 1 —см. словарь № 101, Скоринговые баллы;

·Балл № 2 — см. словарь № 102, Скоринговые баллы;

Примечание. См. также — соответствия скоринговых баллов.

·№ — идентификационный номер долга, присваивается автоматически при сохранении информации о долге (ID долга);

·Название продукта — указывается название продукта, по которому возник долг;

·Тип продукта — см. словарь № 11;

·Лицевой счет — лицевой счет должника из системы/базы данных клиента;

·Договор — номер договора с должником из системы/базы данных клиента;

·ID Клиента — идентификатор должника из системы/базы данных клиента (при наличии);

·Дата выдачи — дата выдачи кредита;

·Город — город, в котором зарегистрирован договор;

·Регион — регион, в котором выдан кредит, выбирается из словаря № 39;

·Филиал — если организация, использующая систему, имеет филиальную структуру, в данном поле указывается филиал, который должен работать с этим долгом. Значение выбирается из словаря № 61;

·Дата выхода на просрочку — дата, с которой долг считается просроченным;

·Валюта — валюта долга (см. Список валют);

·Размер кредита — сумма кредита без учета процентов, комиссий и т.п., выводится в валюте долга, указанной в поле Валюта;

·Проц. ставка — процентная ставка по кредиту;

·Ост. осн. долга — сумма остатка основного долга, выводится в валюте долга, указанной в поле Валюта;

·Причина возникновения долга — значение выбирается из словаря № 13 (история смены отображается при просмотре истории комментариев);

·Остаток — остаток суммы долга на текущую дату, выводится в валюте долга, указанной в поле Валюта;

·Переплата — сумма переплаты по долгу;

·Дата расчета — дата, на которую рассчитано значение в поле Остаток;

·Дата акт. — дата актуальности. Банковский атрибут, который показывает, до какой даты является актуальным остаток долга, рассчитанный на определенную дату;

·Вознаграждение % — поле, которое организация может использовать для хранения информации о сумме своего вознаграждения с данного долга. Возможность просмотра данного поля регулируется доступом AGENCY_RATE_VIEW, редактирование — доступом AGENCY_RATE_EDIT. Значение может вводиться вручную или загружаться с помощью загрузки данных (формат загрузки данных должен содержать поле Комиссия из таблицы долгов);

·Сумма платежей в руб. — сумма подтвержденных платежей по долгу. Поле не редактируется;

·Начальный долг — начальная сумма долга в выбранной валюте на момент передачи его в организацию;

·Мин. сумма платежа — минимальная сумма платежа, банковский атрибут, информационное поле;

·Крайний срок — дата крайнего срока погашения долга по кредиту, информационное поле;

·Штрафные санкции — описание штрафных санкций, информационное поле;

·Составляющие долга:

·Основной долг — сумма основного долга;

·Проср. осн. долг — сумма просроченного основного долга;

·Проценты — сумма процентов по кредиту;

·Проср. проценты — сумма просроченных процентов по кредиту;

·Штрафы — сумма штрафов по кредиту;

·Пени — сумма пени;

·Комиссия — сумма комиссии;

·Проср. комиссия — сумма просроченной комиссии по кредиту;

·Гос. пошлина — сумма государственной пошлины;

·Акция — информация об акциях. Используется для информирования коллекторов о том, что по данному долгу есть возможность предложить должнику какие-либо льготные условия (например, частичное списание долга при условии оплаты до определенной даты). Информационное поле;

·Сумма на счете — остаток на банковском счете должника;

·Аннуитетный платеж — сумма ежемесячного платежа по кредиту, если размер ежемесячного платежа остаётся постоянным на всём периоде кредитования;

·Балл № 3 — см. словарь № 244, Скоринговые баллы;

·Балл № 4 — см. словарь № 245, Скоринговые баллы;

·Комментарий — дополнительная информация (история смены отображается при просмотре истории комментариев);

·Отчет коллектора — комментарий по работе с долгом. См. Отчет коллектора по работе с долгом; (история смены отображается при просмотре истории комментариев);

·Комментарий по реструктуризации — дополнительная информация по реструктуризации долга (история смены отображается при просмотре истории комментариев).

Просмотр показаний приборов учета

По согласованию с технической поддержкой для долгов по ЖКХ доступна настройка дополнительной вкладки "Показания ПУ" (Техническая константа DebtCard.DebtDetailGKH.MeterDisplay.Use, делали для Энергосбыта).

Данные на вкладке доступны только для просмотра.

Вкладка «Показания ПУ» включает следующие элементы:

·Таблицу Приборы учета, отображающую список приборов учета, с полями:

·Услуга — см. словарь № 87, поле по умолчанию скрыто.

·Серийный номер — серийный номер прибора учета.

·Тип ПУ — см. словарь "Тип прибора учета".

·Дата последней поверки.

·Периодичность поверки.

·Дата установки.

·Местоположение.

·Флаг Признак активности ПУ.

·Подчиненную таблицу Показания, отображающую показания выбранного прибора учета, с полями:

·ID — идентификатор записи, поле по умолчанию скрыто.

·№ п.п. регистра.

·Дата получения показания.

·Тип показания — см. словарь "Тип показания".

·Значение показания.

·Дата регистрации — поле по умолчанию скрыто.

·Комментарий — поле по умолчанию скрыто.
