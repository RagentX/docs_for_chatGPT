Отчет по операторам

Отчет "Отчет по операторам" выводит агрегированные данные о портфеле за период в разрезе работы операторов в Excel.

Отчет "Отчет по операторам" доступен только пользователям, в роли которых доступ REP_OPERATOR имеет значение "Истина".

Порядок действий

Для того чтобы сформировать отчет, выполните следующее:

1.Выберите в главном меню пункт Отчеты, затем выберите отчет "Отчет по операторам".

2.Откроется форма "Отчет по операторам". Заполните следующие поля:

·С: — выберите начала периода (по умолчанию в поле указано первое число текущего месяца).

·По: — выберите окончания периода (по умолчанию в поле указано последнее число текущего месяца).

·Панель "Параметры" — указание параметров детализации отчетов:

·Флаг "Детализация" — включение и выключение режима детализации. Если флаг выключен, то будет формироваться не детализированный отчет по всем операторам. Остальные поля панели "Параметры" в этом режиме не доступны. Если флаг включен, то будет формироваться детализированный отчет. В этом слечае следует заполнить остальные поля панели "Параметры".

·поле "Детализация" — выбор оператора или одного из следующих режимов:

·Не учитывать ручн. закреп-ие — не учитывать ручное закрепление.

·Не установлен — отчет будет сформирован по тем долгам, у которых оператор не установлен.

·SOFT.

·Все операторы — отчет будет сформирован по всем операторам.

·Портфели — откроется форма для множественного выбора портфелей. Выбор хотя бы одного портфеля обязателен.

3.Нажмите кнопку "Запустить". Начнется формирование отчета. В процессе будет отображаться индикатор выполнения. Затем откроется стандартное окно операционной системы для указания расположения и названия файла с последующим сохранением.

4.Сохраните полученный файл. По окончании сохранения файла будет показано соответствующее системное сообщение.

Логика формирования отчета

Для каждого оператора рассматриваются все интервалы работы с должниками, которые пересекаются с параметром "период". В частности, за период оператор мог иметь несколько различных интервалов работы с должником (оператора, работающего с должником, переназначали несколько раз). По каждому интервалу работы просматриваются контакты, платежи и обещания, попавшие в период отчета.

Если отчет детализированный, то информация агрегируется для каждого интервала работы. Если отчет не детализированный, то информация агрегируется по каждому оператору.

Примечание: если за отчетный период ни один оператор не имеет интервала работы с должником, то должник не попадает в отчет.

Описание файла отчета

Не детализированный

Не детализированный отчет "Отчет по операторам" представляет собой таблицу Excel со следующими полями:

·ФИО оператора.

·Сумма подтв. платежей.

·Кол-во подтв. платежей.

·Кол-во персон, сделавших подтв. платежи.

·Сумма неподтв. платежей.

·Кол-во неподтв. платежей.

·Кол-во персон, сделавших неподтв. платежи.

·Сумма вып. обещаний.

·Кол-во вып. обещаний.

·Кол-во персон, вып. обещание.

·Сумма част. вып. обещаний.

·Кол-во част. вып. обещаний.

·Кол-во персон, част. вып. обещание.

·Сумма ожидаемых обещаний.

·Кол-во ожидаемых обещаний.

·Кол-во персон, обещание от которых ожидается.

·Сумма не вып. обещаний.

·Кол-во не вып. обещаний.

·Кол-во персон, не вып. обещание.

Детализированный

Сформированный "Отчет по операторам" представляет собой таблицу Excel со следующими полями:

·№ должника.

·№ долга.

·№ договора.

·Номер счета.

·Должник.

·Продукт.

·Банк.

·Портфель.

·Тип продукта.

·Оператор.

·Филиал.

·Размер кредита.

·Начальный долг.

·Остаток к погашению.

·Основной долг.

·Просроченный основной долг.

·Проценты.

·Просроченные проценты.

·Штрафы.

·Пени.

·Комиссия.

·Гос. пошлина.

·Текущее состояние.

·Пред. состояние.

·Дата выхода на просрочку.

·Дата подписания реестра.

·Дата посл. платежа.

·Сумма подтв. платежей.

·Кол-во подтв. платежей.

·Сумма неподтв. платежей.

·Кол-во неподтв. платежей.

·Сумма вып. обещаний.

·Кол-во вып. обещаний.

·Сумма част. вып. обещаний.

·Кол-во част. вып. обещаний.

·Сумма ожидаемых обещаний.

·Кол-во ожидаемых обещаний.

·Сумма не вып. обещаний.

·Кол-во не вып. обещаний.

·Дата посл. звонка.

·Дата посл. выезда.

·Количество всех звонков.

·Количество успешных звонков.

·Количество всех выездов.

·Количество успешных выездов.

Далее в таблице отображается количество контактов по каждому возможному результату контакта, то есть в таблице добавлено столько полей, сколько возможных результатов контактов (узлов дерева контактов) настроено в Системе (см. модуль Настройка дерева контактов).
