Добавление заявления

Примечание. Заявления на возврат госпошлины обрабатываются иначе, но только при соответствующих настройках Системы. См далее Добавление заявления на возврат госпошлины.

Добавление заявления

Для того чтобы добавить заявление, выполните следующие действия:

1.В карточке судебной работы на вкладке Заявления нажмите кнопку .

2.В открывшейся форме заполните следующие поля:

·Дата подачи — дата подачи заявления. Поле обязательно для заполнения;

·Тип заявления — см. словарь № 33 (выводятся значения из словаря 33, для которых уже создан соответствующий шаблон в конструкторе документов). Поле обязательно для заполнения. (Если в поле выбрано одно из значений словаря № 33, коды которых перечислены в константе LawAct.DueSum.Return.RequestCode, то обработка действия меняется. В этом случае — см. далее Добавление заявления на возврат госпошлины).

·Трек-номер — поле отображается, если включена интеграция с почтой России, особенности обработки поля — см. далее);

·Комментарий — поле ввода дополнительной информации.

3.Нажмите кнопку ОК. Заявление будет добавлено в список.

4.Выполните Прикрепление документа к действию. Прикрепленный документ будет доступен на вкладке "Заявления", в области данных Файлы, а также на вкладке Вложения.

5.Далее осуществляется Формирование настроенных запланированных действий.

Добавление заявления на возврат госпошлины

Для того чтобы заявления на возврат госпошлины обрабатывались способом, отличным от обработки обычных заявлений, в значении константы LawAct.DueSum.Return.RequestCode укажите коды по словарю № 33 "Тип заявления", соответствующие заявлениям на возврат госпошлины.

Для того чтобы добавить заявление на возврат госпошлины, выполните следующие действия:

1.В карточке судебной работы на вкладке Заявления нажмите кнопку .

2.В открывшейся форме заполните следующие поля:

·Дата подачи — дата подачи заявления. Поле обязательно для заполнения;

·Тип заявления — выберите тип заявления, код которого по словарю № 33 перечислен в константе LawAct.DueSum.Return.RequestCode. Поле обязательно для заполнения.

3.Форма изменит свой вид, и на ней появятся дополнительные поля. Заполните их:

·Название гос.органа (текстовое поле; при создании по умолчанию заполняется названием суда, выбранным на вкладке Инф.о суде);

·Адрес гос.органа (текстовое поле; при создании по умолчанию заполняется значением в поле Адрес на вкладке Инф.о суде);

·Причина возврата (выпадающий список терминов по словарю № 344 "Причина заявления" с родительским термином "Заявление на возврат госпошлины");

·Реквизиты гос.органа (текстовое поле; при создании по умолчанию выводятся реквизиты суда, выбранного на вкладке Инф.о суде)

При создании нового заявления на возврат госпошлины в поле Реквизиты гос.органа выводятся реквизиты суда, выбранного на вкладке Инф.о суде в следующем виде:

Получатель: <Получатель>, ИНН: <ИНН>, КПП: <КПП>, БИК: <БИК>, ОКАТО: <ОКАТО>, Р/с: <Р/с>, К/с: <К/с>, Банк: <Банк>, КБК: <КБК>

Если на вкладке "Инф.о суде" не выбран суд, то поле при создании заявления заполняется так:

Получатель: , ИНН: , КПП: , БИК: , ОКАТО: , Р/с: , К/с: , Банк: , КБК:

·Сумма ГП к возврату (поле обязательно для заполнения);

·Возвращенная ГП (поле недоступно для редактирования при создании заявления).

4.Нажмите кнопку ОК. Заявление будет добавлено в список.

5.Выполните Прикрепление документа к действию. Прикрепленный документ будет доступен на вкладке "Заявления", в области данных Файлы, а также на вкладке Вложения.

6.Далее осуществляется Формирование настроенных запланированных действий.

Обработка трек-номера при добавлении запроса

Поле Трек-номер отображается на форме, если в Системе включена интеграция с почтой России (Коммерческая константа @PostRussia.Use).

При нажатии кнопки ОК при добавлении запроса, если поле Трек-номер непустое, для него выполняются следующие проверки:

·если в поле Трек-номер введено значение, содержащее буквенные символы (т.е. не только цифры) или введенная строка меньше 14 символов, пользователю выводится сообщение с текстом:

Трек-номер не соответствует внутрироссийскому формату почтового идентификатора. Продолжить?

с вариантами "Да", "Нет".

·при нажатии кнопки "Да" введенное значение сохраняется в новой записи таблицы, хранящей письма и их статусы.

·при нажатии кнопки "Нет" пользователь "возвращается" на форму "Заявление".

·Иначе, т.е. если в поле Трек-номер строка из 14 цифр, то введенное значение сохраняется в новой записи таблицы, хранящей письма и их статусы.

Действие протоколируется. Дополнительно сохраняется номер запроса.
