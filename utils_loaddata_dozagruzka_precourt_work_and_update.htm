Особенности загрузки файлов с типом формата "Досудебная работа" и "Досудебная работа (обновление)"

1.Если в шаблоне загрузки в поле для загрузки указана таблица "Имущество" и присутствуют поля: "Тип имущества" и "Залоговая стоимость", то создаётся новая запись в таблице person_property со связью person_property.r_debt_id = debt.id загружаемого долга.

·Для имущества обязательно должны быть заполнены поля:

·Тип имущества (typ);

·Залоговая стоимость (sum).

·Иначе, (указанные поля не заполнены) запись в таблицу person_property не добавляется.

·Если не указано поле "Статус" (status по словарю № 86), то автоматически проставляется status = 1.

2. Логика загрузки поля "Оценщик" (property_valuer.name для r_valuer_id):

·По указанному значению в поле "Оценщик" осуществляется поиск записей в таблице property_valuer по полю name.

·Если будет найдена запись, то имущество закрепляется за этим оценщиком. (person_property.r_valuer_id = property_valuer.id). Если записей будет несколько, то закрепляется запись с максимальным id

·Иначе (запись по указанным данным не найдена), создается новая запись в таблице property_valuer. Заполняются значения:

·name = %Оценщик (по загружаемому полю)%;

·valuer_address = NULL

·valuer_phone = NULL

·Загрузка остальных полей осуществляется согласно стандартной логике.
