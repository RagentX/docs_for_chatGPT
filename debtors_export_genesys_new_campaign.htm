       Проверка ограничений коммуникации

Для каждого номера осуществляется проверка возможности выгрузки телефона в call-лист согласно настроенной схеме и текущей статистике.

Настройка учета в ограничениях

На каждую форму, которая вызывается при формировании call-листов добавлен чек бокс "Учитывать ограничения". По умолчанию чек-бокс выключен.

·При формировании задания на автоинформирование чекбокс отображается, если константа CommunicationLimit.ContactTypeList содержит значение 10.

·При формировании задания на обзвон, чекбокс отображается, если константа CommunicationLimit.ContactTypeList содержит значение 1.

Если чекбокс включен, то при формировании call-листов ограничения учитываются согласно текущей статистике.  В call-лист попадают только те телефоны, которые привязаны к участнику договора, с которым можно совершить контакт на момент выгрузки.

При использовании шаблонов формирования call-листов также реализована логика учета ограничений.

Проверка наличия схемы коммуникации

1.Проверяется значение константы CommunicationLimit.ContactTypeList:

·Если константа содержит код 1 или 10 (см. словарь № 3), то проверка превышения лимита коммуникации продолжается;

·Если данный код в константе отсутствует, то проверка не осуществляется, и коммуникация разрешается.

2.Проверяется в шаблоне сообщения флаг Учитывать в ограничениях (для звонков и выездов — в модуле Настройка дерева контактов; для SMS и email — в модуле Сценарии сообщений):

·Если флаг включен, то проверка продолжается;

·Если флаг выключен, то проверка завершается, и коммуникация разрешается.

3.Проверяются Персональные ограничения коммуникации, настроенные для данного долга в Карточка должника.

·Если персональные ограничения настроены, то для данного типа контакта проверяется наличие Схемы коммуникации и соответствие даты совершения коммуникации ее периоду. Если ограничения настроены корректно, то осуществляется проверка лимита коммуникации по данной схеме (см. далее).

·Если персональные ограничения не настроены, или настроены некорректно или не период не включает текущую дату, то проверка движется дальше.

4.Проверяются общие ограничения, налагаемые по формуле, указанной в значении константы, соответствующей данному типу коммуникации. При формировании задания на автоинформирование проверяется константа CommunicationLimit.Call.Formula. При формировании задания на обзвон проверяется константа CommunicationLimit.AutoInfo.Formula.

·Если в формуле указан существующий идентификатор Схемы коммуникации и дата совершения коммуникации входит в ее период, то осуществляется проверка лимита коммуникации по данной схеме (см. далее).

·Если общие ограничения не настроены, настроены некорректно или не включают текущую дату, коммуникация разрешается.

Для того чтобы выгрузить данные в новую кампанию или осуществить дозагрузку, выполните следующие действия:

1.Войдите в модуль Список должников.

2.Активируйте нужную вкладку. Для данного модуля это могут быть все вкладки, на которых отбражаются списки должников.

3.C помощью клавиш SHIFT и CTRL выберите должников, чьи номера надо выгрузить для автодайлера.

4.В контекстном меню (вызывается правой кнопкой мыши) выберите пункт Выгрузить–>Автодайлер (GENESYS)–>Новая кампания или Дозагрузка.

5.Откроется форма настройки задания для автодайлера.    

6.На первой странице формы заполните следующие поля:

·Название задания — поле обязательно для заполнения;

·Раздел Параметры задания:

·Дата начала обзвона — дата начала периода, в течение которого будет осуществляться автодозвон или автоинформирование;

·Дата окончания обзвона — дата окончания периода, в течение которого будет осуществляться автодозвон или автоинформирование;

·Часовой период, в который можно звонить — время начала и окончания часового периода обзвона.

·Кнопка Настроить. При нажатии на нее откроется форма, в которой можно настроить период обзвона отдельно для каждого типа телефонов. При нажатии кнопки Установить для всех заданное время будет установлено для всех типов телефонов;

·Переключатель Режим — переключатель режима, который можно установить в одно из следующих значений: "Автоинформирование" (текст сообщения проговаривает робот) или "Кампания обзвона" (робот проговаривает текст сообщения, а затем соединяет абонента с оператором).  

·Кампания — название кампании обзвона, в рамках которой будет выполняться задание;

·Call-list — поле будет заполнено автоматически в зависимости от кампании;

·Сценарии — голосовой сценарий, который будет проигрываться абоненту (поле доступно только для автоинформирования);

·Кол-во попыток дозвона — введите число;

·Кнопка Далее — Нажмите для открытия второй страницы формы.

7.Перейдите ко второй странице формы, нажав Далее.

8.На второй странице формы выполните следующие действия:

·Выберите порядок сортировки для следующих параметров:

·Порядок сортировки (выбирается только одно значение):

·По списку должников;

·По ID долга;

·Статус телефона должника — значения см. словарь № 37, поддерживается множественный выбор;

Примечание. Если в блоке Статус телефона должника не выбрано ни одного значения, то этот параметр не учитывается при формировании выборки.

·Тип телефона — значения см. словарь № 4, поддерживается множественный выбор, обязательно выбрать хотя бы один тип телефона;

Примечание. Если доступ № 320 DEBT_LIST_DIALER_FULL_PHONE_TYPE_LIST в роли пользователя выключен, то отображаются только типы телефонов с кодом < 100, то есть телефоны поручителей, залогодателей не отображаются.

·Словарный атрибут телефона 1 — см. словарь № 258;

·По дате обновления статуса (выбирается только одно значение):

·По убыванию;

·По возрастанию.

Примечание. Название Словарного атрибута телефона 1 и его значения настраиваются отдельно для каждой компании, использующей Систему. Данный атрибут также может отсутствовать в Системе.

Кнопки  и  используются для того, чтобы передвигать выбранную строку вверх или вниз, можно также передвигать всю папку, устанавливая таким образом порядок сортировки;

 Поле Сортировка отражает выбранный на данный момент порядок сортировки;

·Выберите режим:

·Переключатель Режим, можно установить в одно из следующих значений:

·Выгружать должника — будет выгружена одна запись для автодайлера по всем долгам должника;

·Выгружать каждый долг должника — по каждому долгу будет создана своя запись.

·Выгрузить в Excel — если установлен данный флаг, то будет сформирован файл со списком телефонов, если не установлен — не будет (кампания обзвона будет работать без него);

·Учитывать в ограничениях — если флаг включен, то при формировании call-листов ограничения учитываются согласно текущей статистике.

9.Нажмите кнопку Сформировать выборку. Откроется форма с выборкой. Доступно копирование выбранных записей при помощи комбинации клавиш CTRL+V и CTRL+C.

10.Нажмите кнопку Сформировать задание.

       Примечание для поддержки. Функционал ограничений коммуникации. Добавление в статистику

Для автоинформирования

Проверка ограничений коммуникации осуществляется на этапе формирования call-list.

Добавление коммуникации в статистику осуществляется при регистрации контакта с типом "Автоинформирование" для исходов, настроенных специально для регистрации контактов автоинформирования  (настраивается поддержкой).

Для кампании обзвона

Проверка ограничений коммуникации осуществляется на этапе формирования call-list.

Добавление коммуникации в статистику осуществляется при регистрации контакта.
