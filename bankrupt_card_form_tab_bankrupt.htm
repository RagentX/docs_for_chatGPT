Вкладка "Банкротство"

Вкладка Банкротство включает следующие элементы:

·раздел Общая информация, содержащий общие данные о должнике;

·раздел Банкротство, содержащий данные о банкротстве;

·кнопки для выполнения действий с банкротством (см. Этапы работы в банкротстве);

·блок Краткая информация, содержащий краткую информацию;

·раздел Дела судебной работы, содержащий список дел судебной работы должника и кнопки для работы с ним;

·раздел Телефоны, содержащий список телефонов должника и кнопки для работы с ним;

·раздел Адреса, содержащий список адресов должника и кнопки для работы с ним;

·поле Комментарий для ввода дополнительной информации и просмотра истории комментариев по работе с должником;

·поле Указание руководителя — право вводить информацию в поле имеют только те пользователи, для которых доступ LAW_ACT_CARD_INSTRUCT установлен в значение TRUE.

Ссылка: Подробная информация о редактировании полей в карточках судебной работы и ИП.

Раздел "Общая информация"

Раздел Общая информация включает следующие поля:

·Дата поступления;

·Плановая дата подачи;

·Статус;

·Размер требования — law_act.total_sum;значение поля рассчитывается путем суммирования полей, перечисленных в константе LawActBankruptTotalSumFormula. Если константа = 0 (или пусто), то поле остается пустым (редактирование поля разрешается пользователю в соответствии с значением достуа № 710 LAW_ACT_FIELD_EDIT в его роли);

·Примечание — см. словарь № 247;

Примечание. Право на редактирование примечания определяется доступом LAW_ACT_BANKRUPT_SET_DICT_VALUE.

 Примечание для поддержки. Колонка в Юридическом департаменте на вкладках судебных дел и данное поле в карточке судебной работы видны только если в column_spec у колонки с id=884 заполнено поле user_name. Наименование колонки реализовано по стрингу law_act_list.act_dict_value_1.

·Валюта — валюта долга (см. Список валют);

Примечание. Если все долги в одной валюте, то данная валюта устанавливается в этом поле. Если валюта у долгов разная, то переводятся все долги в основную валюту Системы (1) по текущему курсу и банкротству устанавливаем данную валюту.

·Составляющие — список составляющих долга в банкротстве определяется константой LawActBankruptDebtComponentList.

Раздел "Банкротство"

Раздел Банкротство включает следующие поля:

·Способ подачи — способ подачи иска в суд, см. словарь № 16;

·Дата подачи — дата подачи иска в суд;

·Номер дела в суде;

·Дата расторжения договора — дата расторжения кредитного договора с должником;

·Дата вынесения решения — дата вынесения решения судом;

·Дата вступления в силу — дата вступления в силу решения суда;

·Сумма по решению суда — сумма, которую должник должен выплатить по решению суда;

·Утвержденная пошлина — утвержденная сумма пошлины, выводится в основной валюте Системы, поле видно только если константа LawActBankrupt.UseDueSumFields имеет значение "Истина" (редактирование поля разрешается пользователю в соответствии с значением достуа № 710 LAW_ACT_FIELD_EDIT в его роли);

·Дата оплаты пошлины — поле видно только если константа LawActBankrupt.UseDueSumFields имеет значение "Истина";

·Результат по делу;

·Дата получения ИЛ на обесп.;

·Дата мир. соглашения — дата заключения мирового соглашения с должником;

·Сумма мир. соглашения — сумма, которую должен выплатить должник по мировому соглашению;

·Дата срока исполнения недостатков;

·Дата отказа — дата отказа в принятии судебного иска к производству;

·Дата решения по иску — дата вынесения решения по судебному иску;

·Расчетная пошлина —  — поле видно только если константа LawActBankrupt.UseDueSumFields имеет значение "Истина". Если константа LawAct.Prepare.DueSum.Mandatory имеет значение "Истина", то поле обязательно для заполнения. Подробнее см. Госпошлина (редактирование поля разрешается пользователю в соответствии с значением достуа № 710 LAW_ACT_FIELD_EDIT в его роли);

·Флаг Прикрепить обесп. меры;

·Кнопка Прикрепить — Прикрепление файла с описанием обеспечивающих мер.

Блок Краткая информация состоит из одного не редактируемого текстового поля. Информация в нем выводится согласно шаблону, настроенному в модуле Сценарии сообщений на вкладке Специальное, номер которого указан в константе DebtMainInfoFormat в скобке № 5. Настройка информационного блока аналогична настройке информационного блока для карточки должника. Если константа пуста или отсутствует скобка № 5, то блок Краткая информация на вкладке не отображается.

Блок "Дела судебной работы"

Блок Дела судебной работы включает следующие элементы:

·Таблица со списком всех судебных дел данного должника, за исключением дел о банкротстве. Таблица содержит следующие колонки:

·ID долга;

·ID дела — law_act.id;

·Портфель — law_act.r_portfolio_id;

·Ответственный —law_act.r_user_id;

·Статус — law_act.status law_act.act_status;

·Номер договора — debt.contract;

·Лицевой счет — debt.account;

·Размер требования — law_act.total_sum;

·Валюта — debt.currency;

·Название суда — law_court.name по law_act.r_cour_id;

·Адрес суда — law_court.address по law_act.r_cour_id;

·Дата подачи — law_act.start_date;

·Крайняя дата подачи — law_act.deadline;

·Дата вынесения решения — law_act.court_order_date;

·Дата вступления в силу — law_act.court_exec_date;

·Комментарий — law_act.dsc;

·Название продукта;

·Тип продукта;

·Остаток — debt.debt_sum;

·Тип иска — law_act.act_direct;

·Укомплектованность — law_act. full_strength;

·Кнопка Произвольные отчеты — Формирование отчетов по судебной работе;

·Кнопка Печать документов — Печать документов судебной работы.

Раздел "Телефоны"

Раздел Телефоны содержит следующие элементы:

·Список телефонов с полями:

·Тип — тип телефона, см. словарь № 4;

·Номер — номер телефона;

·Контактное лицо;

·Кнопки для работы со списком телефонов.

Действия со списком телефонов — см. Ведение списка контактных телефонов должника.

Окраска телефонов в зависимости от статуса

Строки в списке телефонов выделены цветом фона в зависимости от статуса контактного телефона (см. Статусы телефонов):

·Не звонили ни разу — серый цвет.

·Результата не было — белый цвет.

·Неверный номер — красный цвет.

·Результат — зеленый цвет .

·Последний результат — голубой цвет.

·Автоинформатор — розовый цвет.

Если у пользователя есть доступ 127 DEBT_CARD_ALL_PHONE, то в карточке должника он может видеть заблокированные номера телефонов. Такие записи в списке телефонов выделены серым шрифтом на белом фоне.

·Кнопки для работы со списком адресов.

Окраска адресов в зависимости от статуса

Строки в списке адресов выделены цветом фона в зависимости от статуса адреса. (см. Статусы адресов):

·Неверный адрес — красный цвет.

·Результат — зеленый цвет;

·Последний результат — голубой цвет.

Если у пользователя есть доступ 280 DEBT_CARD_ALL_ADR, то в карточке должника он может видеть заблокированные адреса. Такие записи в списке адресов выделены серым шрифтом на белом фоне.
