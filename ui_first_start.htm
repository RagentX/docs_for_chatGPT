Запуск Системы и обновление модулей

Система Контакт является современным программным продуктом и обновляется в соответствии с изменениями законодательства, переработкой бизнес-процессов пользователей Системы, а также в условиях модернизации технических и программных средств работы с Системой.

Обновления Системы осуществляются путем обновления модулей, составляющих Систему.

Вам может потребоваться обновить модули в следующих случаях:

·выход нового релиза Системы;

·обновление модулей в связи с выполненной заявкой на доработки;

·исправление обнаруженной ошибки или уязвимости.

Когда новые версии модулей тщательно протестированы, специалисты технической поддержки выкладывают обновленные модули в сетевую папку, расположенную на сервере. Обычно сетевая папка имеет название "\\CONTACT-PROD-AP\Contact".

Для того чтобы изменения вступили в силу на локальном компьютере, необходимо осуществить запуск "Контакта" с вашего рабочего места. Для этого выполните двойной клик на ярлык "Contact.exe".

Система сначала автоматически установит обновления, скачав из сетевой папки в локальную папку обновленные модули Системы. При этом будет показано окно с индикацией процесса обновления. После окончания обновления Система запустится и будет доступна для работы.

Решение возможных проблем с обновлением Системы

Если обновление происходит некорректно, выполните следующие проверки:

1.Выйдите из Системы.

2.Убедитесь, что на сервере есть сетевая папка "\\CONTACT-PROD-AP\Contact", и у вас есть доступ к ней. Если сетевая папка отсутствует, или у вас нет доступа к ней, появится соответствующее предупреждение, и обновление не будет осуществлено.

3.Убедитесь, что на вашем локальном компьютере есть локальная папка с модулями Системы.

4.Убедитесь, что в локальной папке Системы есть настроечный файл "config.ini". В нем, в разделе "main", должен быть указан путь к сетевой папке:

[main]

path=\\CONTACT-PROD-AP\Contact

Если указан путь к несуществующей папке или название указано с ошибками, то появится соответствующее предупреждение, и обновление не будет осуществлено.

5.Убедитесь, что никакой другой пользователь не работает в Контакте из вашей локальной папки и не было сообщения, что файл не может обновиться, т.к. занят другим процессом. Иначе следует завершить работу с Системой прежде, чем устанавливать обновления.

Если пользователи работают на терминальном сервере из одной папки, то необходимо принудительно завершить процесс login.exe (или другой из сообщения) на данном сервере, после чего один из пользователей должен запустить Контакт, в результате корректно запустится обновление всех модулей на данном сервере.

6.Если у других пользователей обновление прошло успешно, а у вас — нет, сравните настройки, проверьте корректность файла "config.ini" в вашей локальной папке Контакта.

Особенности обновления на терминальном сервере

На терминальном сервере пользователи как правило работают с Контактом из одной папки. Если один или несколько пользователей не закрыли Контакт, то часть модулей остаются запущенными, в результате чего система не может обновить такой модуль. В результате пользователи могут получать сообщение вида

В этом случае системный администратор принудительно "убивает" все активные сессии на данном терминальном сервере, после чего запускает модуль contact.exe, который автоматически обновит модули.
