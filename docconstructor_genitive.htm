Вывод ФИО в родительном или дательном падежах

В некоторых случаях может понадобиться вывод имен существительных в косвенных падежах. Например, для заполнения полей Кому и От кого в заголовке письма требуется выводить фамилии, имена и отчества в дательном и родительном падежах соответственно.

Для демонстрации последовательности действий, приводящей к выводу ФИО в косвенных падежах, мы создадим в шаблоне письма два поля:

·Кому: — с ФИО должника в дательном падеже;

·От кого: — с ФИО лица, подписывающего документы (указанного в значении константы RepDebtLetterSignFio).

Последовательность шагов

1.Открываем нужный шаблон письма. Переходим на вкладку Данные.

2.Нажмите на кнопку Запрос ADO и располагаем объект Запрос ADO на шаблоне письма.

3.На вкладке Свойства добавленного объекта, в полях Name и UserName задаем имя для добавленного объекта. Для вывода ФИО в дательном падеже для поля Кому мы использовали названия:

·Name — Dative;

·UserName — For Whom.

4.Затем на той же вкладке Свойства в поле SQL нажимаем кнопку .

5.В открывшейся форме Редактор SQL добавляем следующий запрос:

select dbo.[Дательный](:fio1) as fio_for_whom

где:

·:fio1 — параметр в который будем передавать в данном случае ФИО должника;

·fio_for_whom — название столбца, которое будет в дальнейшем использоваться.

6.В форме Редактор SQL нажимаем кнопку Редактировать параметры.

7.В открывшейся форме Параметры будет отображен указанный в запросе параметр :fio1. Задаем для него тип string. Затем нажимаем кнопку .

8.Откроется форма Редактор выражений. Из набора данных frxDBRep1 выбираем поочередно поля f, i, o, добавляя каждое к полученному выражению двойным кликом. В результате в поле Выражение отображается следующий текст:

<frxDBRep1."f"><frxDBRep1."i"><frxDBRep1."o">

Чтобы ФИО должника отображалось правильно, необходимо добавить фрагмент "+'  '+" между элементами списка, чтобы потом при выводе информации между словами были пробелы. Таким образом должно получиться следующее выражение:

<frxDBRep1."f">+' '+<frxDBRep1."i">+' '+<frxDBRep1."o">

9.Нажимаем OK в форме Редактор выражений. Затем нажимаем OK в форме Параметры. Затем нажимаем кнопку OK в форме Редактор SQL.

10.В дереве данных отображается созданный набор данных For Whom.

11.Аналогичным образом настраиваем запрос для ФИО лица, подписывающего документы в родительном падеже (для поля От кого). Здесь:

·Name — Dative;

·UserName — For Whom.

·Текст запроса — "select dbo.[Родительный](:fio2) as fio_Genitive", где:

·:fio2 — параметр в который будем передавать в данном случае ФИО лица, подписывающего документы;

·fio_Genitive — название столбца, которое будет в дальнейшем использоваться;

·Выражение — <frxDBRep1."sign_fio">, где:

·sign_fio — ФИО лица, подписывающего документы, см. Список полей БД.

12.Теперь на вкладке Данные у нас есть два запроса, и в дереве данных присутствуют два набора данных:

·For Whom;

·From_Whom.

13.Нажмите на кнопку для предварительного просмотра. Затем закройте окно предварительного просмотра. После этого в дереве данных в наборах данных For_Whom и From_Whom появятся настроенные поля fio_for_whom и fio_Genitive соответственно.

14.Следующим этапом будет настройка вывода этих данных в шаблоне письма. Для этого переходим в макет письма, создаем объект. В редакторе текста мы написали "Кому: "и установили курсор в конце этой записи.

15.Затем нажали кнопку . Открылся редактор выражений. В нем мы нашли в наборе данных For_Whom поле fio_for_whom и добавили его в текст (осуществили на нем двойной клик, оно появилось в окне Выражение, затем нажали ОК в редакторе выражений).

16.Далее нажимаем ОК в Редакторе текста. В шаблоне письма появляется объект, содержащий текст "Кому: " и ФИО должника в дательном падеже.

17.Аналогичным образом настраиваем объект, содержащий текст "От кого: " и ФИО лица, подписывающего документы, в родительном падеже.

18.В панели инструментов Дерево отчета выбираем первый из созданных нами объектов Text_For_Whom. В панели инструментов Инспектор выбираем вкладку События. В списке событий выбираем OnBeforePrint и осуществляем двойной клик. В основном окне программы будет сгенерирована процедура, не включающая никакого текста.

19.В нее мы вписываем следующий текст:

 Dative.close;

 Dative.Params[0].value := <frxDBRep1."f">+' '+<frxDBRep1."i">+' '+<frxDBRep1."o">;

 Dative.Open;

20.Затем проделываем все то же самое для второго поля Text_From_Whom. Текст процедуры будет иметь вид:

 Genitive.close;

 Genitive.Params[0].value := <frxDBRep1."sign_fio">;

 Genitive.Open;

21.Наши поля настроены. Сохраняем шаблон письма. При выводе его на экран из карточки должника мы получаем строки "Кому : ..." и "От кого: ..." с соответствующими ФИО в соответствующих косвенных падежах.
