Верхняя область формы

·Код — код результата. Для присвоения/очистки кода нажмите зеленую/серую кнопку справа от поля. Код должен быть присвоен всем окончательным результатам в дереве контактов, иначе регистрация контакта для них фиксироваться не будет. Если код не указан, то при сохранении данных будет выдано предупреждающее сообщение;

·Текст — название результата контакта. Поле обязательно для заполнения;

·Ширина, Высота, Форма и Цвет — настройки графического отображения элемента;

Левая область формы, страница 1

·Ввод комментария — значение определяет, требуется ли вводить комментарий при регистрации данного результата. Возможные значения:

·Не показывать — поле для ввода не будет показано;

·Показывать, но не обязательно — поле будет показано, но не является обязательным для заполнения;

·Обязательно для заполнения — поле будет показано и является обязательным для заполнения при регистрации контакта, запрещено регистрировать контакт с данным исходом, если значение не задано;

·Ввод автокомментария — значение определяет, требуется ли вводить автоматический комментарий при регистрации данного результата. При нажатии на поле открывается список всех автокомментариев, настроенных в модуле Сценарии сообщений на вкладке Автоматический комментарий. Проставьте галочки у тех комментариев, которые требуется вводить при регистрации данного исхода;

·Ссылка на файл — значение определяет, требуется ли прикреплять ссылку на сопроводительный файл при регистрации данного результата. Перечень значений такой же, как для поля Ввод комментария. Прикрепление ссылки осуществляется аналогично прикреплению ссылки на файл в модуле Карточка должника. В случае, когда поле не обязательно, и пользователь не прикрепил ссылку, после закрытия формы регистрации контакта пользователю будет предложено выбрать причину неприкрепления ссылки на файл (см. словарь № 88), а затем его руководителю будет отправлено по внутренней почте сообщение с темой "Не прикреплена ссылка на файл" с информацией о должнике, пользователе, зарегистрировавшем контакт, результате контакта и причине неприкрепления ссылки на файл;

·Сохранение файла в базе — значение определяет, требуется ли сохранять прикрепленный сопроводительный файл в базе данных системы. Перечень значений такой же, как для поля Ввод комментария. Прикрепление файла документа осуществляется аналогично прикреплению файла документа в модуле Карточка должника. В случае, когда поле не обязательно, и пользователь не прикрепил документ, после закрытия формы регистрации контакта пользователю будет предложено выбрать причину невозможности прикрепить файл (см. словарь № 88), а затем его руководителю будет отправлено по внутренней почте сообщение с темой "Не прикреплен файл" с информацией о должнике, пользователе, зарегистрировавшем контакт, результате контакта и причине невозможности прикрепить файл;

·Дата повторного звонка — значение определяет, требуется ли указывать дату повторного звонка при регистрации данного результата. Перечень значений такой же, как для поля Ввод комментария;

·Дата необходимого выезда — значение определяет, требуется ли указывать дату выезда при регистрации данного результата. Перечень значений такой же, как для поля Ввод комментария;

·Ввод обещания — значение определяет, требуется ли вводить обещание при регистрации данного результата. Возможные значения:

·Не показывать — поле для ввода не будет показано;

·Показывать — будет показана панель обещания, включающая следующие поля:

oДата — с учетом ограничений на дату обещания;

oСумма — с учетом ограничений на сумму обещания;

oфлаг "Обещание на уровне должника" — поле выводится, если подключен функционал привязки обещания к должнику. Если флаг включен, обещание привязано к должнику и отображается для всех долгов должника. Если флаг выключен, обещание привязано к долгу.

 Примечание. Функционал привязки обещания к должнику включается при помощи константы @DebtUsePromiseOnPerson.

·Показывать "Частичная оплата" — будет показана панель обещания частичной оплаты, включающая следующие поля:

oДата (только будущая);

oСумма;

o% от остатка (вычисляемое поле);

oПричина частичной оплаты (см. словарю № 240 "Причина частичной оплаты");

oфлаг "Обещание на уровне должника" — поле выводится, если подключен функционал привязки обещания к должнику. Если флаг включен, обещание привязано к должнику и отображается для всех долгов должника. Если флаг выключен, обещание привязано к долгу.

·Показывать "Полная оплата" — будет показана панель обещания полной оплаты, включающая следующие поля:

oДата (только будущая);

oСумма (вычисляемое поле);

o% от остатка (100%);

oфлаг "Обещание на уровне должника" — поле выводится, если подключен функционал привязки обещания к должнику. Если флаг включен, обещание привязано к должнику и отображается для всех долгов должника. Если флаг выключен, обещание привязано к долгу.

·Показывать "Частичная оплата по акции" — будет показана панель обещания частичной оплаты, а также панель информации, относящейся к акции;

·Показывать "Полная оплата по акции" — будет показана панель обещания полной оплаты, а также панель информации, относящейся к акции;

·Ввод неподтвержденного платежа — значение определяет, требуется ли вводить данные о неподтвержденном платеже и дате учета информации при регистрации данного результата. Возможные значения:

·Не показывать — поля для ввода данных о сумме и дате платежа не будут показаны;

·Показывать без даты учета информации — поле для ввода данных о сумме и дате платежа будут показаны, а поле даты учета информации нет;

·Показывать с датой учета информации — будут показаны поля для ввода данных о сумме и дате платежа, а также поле даты учета информации;

·Показывать "Частичная оплата" — будет показана панель платежа с частичной оплатой, включающая следующие поля:

oДата (только прошлая);

oСумма;

o% от остатка (вычисляемое поле);

oПричина частичной оплаты (см. словарю № 240 "Причина частичной оплаты";

·Показывать "Полная оплата" — будет показана панель платежа с полной оплатой, включающая следующие поля:

oДата (только прошлая);

oСумма (вычисляемое поле);

o% от остатка (100%);

·Показывать "Частичная оплата по акции"" — будет показана панель платежа частичной оплаты, а также панель информации, относящейся к акции;

·Показывать "Полная оплата по акции" — будет показана панель платежа полной оплаты, а также панель информации, относящейся к акции;

·Создание графика платежей — значение определяет, требуется ли создавать график платежей при регистрации данного результата. Перечень значений: такой же, как для поля Ввод комментария;

·Не показывать — поле для ввода не будет показано;

·Показывать, но не обязательно — поле для создания графика платежей будет показано, но не является обязательным для заполнения;

·Обязательно для заполнения — поле для создания графика платежей будет показано и является обязательным для заполнения при регистрации контакта, запрещено регистрировать контакт с данным исходом, если значение не задано;

· Удалять из автообзвона — значение определяет, будут ли удаляться телефоны должника из кампаний автодозвона при регистрации контакта с данным результатом. Возможные значения:

·Не удалять — телефоны удалены не будут;

·Все телефоны должника — при регистрации контакта с данным результатом из всех кампаний автодозвона будут удалены все телефоны данного должника, кроме той группы, которая обрабатывается непосредственно в данный момент;

·Только текущий телефон — при регистрации контакта с данным результатом из всех кампаний автодозвона будет удален только тот телефон, для которого регистрируется контакт;

·Удалять из автоинформатора. Возможные значения:

·Не удалять;

·Все телефоны должника — из всех кампаний автоинформатора при регистрации данного результата будут удалены все телефоны данного должника, кроме той группы, которую автодайлер обрабатывает непосредственно в данный момент;

·Только текущий телефон — из всех кампаний автоинформатора при регистрации данного результата будет удален телефон, для которого регистрируется результат;

·два числовых атрибута контакта (на примере выше это атрибуты Километраж (начало) и Километраж (конец)) — перечень значений для этих атрибутов такой же, как для поля Ввод комментария. При регистрации контакта с данным результатом можно будет указать значения этих атрибутов. Названия атрибутов для отображения в форма редактирования параметров результата и в форме регистрации контакта настраиваются в таблице strings;

·Причина звонка должника — значение определяет, требуется ли указывать причину звонка должника (возможные причины см. в словаре № 118) при регистрации данного результата. Перечень значений такой же, как для поля Ввод комментария;

·Пользовательский статус — статус, который будет автоматически присвоен долгу после регистрации данного результата (см. словарь № 6; пользовательскими в нем являются статусы с кодом >= 101, их пользователи могут добавлять сами);

·Причина смены пользовательского статуса — значение определяет, требуется ли указывать причину смены пользовательского статуса (возможные причины см. в словаре № 113) при регистрации данного результата. Перечень значений такой же, как для поля Ввод комментария;

·Результат верификации — считать ли номер телефона верифицированным в случае данного исхода контакта . Возможные значения:

·Не определено — значение верификации не определено;

·Успех — считать номер верифицированным;

·Неуспех — считать номер неверифицированным;

·Анкеты для заполнения — выберите анкеты, которые будут предлагаться для заполнения. Список анкет — словарь № 157 (см. Верификация данных через анкету).

·Балл № 1 — см. словарь № 101, Скоринговые баллы, в результате контакта с данным исходом долгу присваивается настроенное значение данного параметра;

Примечание. Поля Балл № 1 — Балл № 4 видны, только если для них настроены соответствующие коммерческие константы:

·для mark1 — @UseDebtMark1 = TRUE  или @DepartUseMark1 = TRUE;

·для mark2 — @UseDebtMark2 = TRUE;

·для mark3 — @UseDebtMark3 = TRUE;

·для mark4 —  @UseDebtMark4 = TRUE;

·Балл № 2 — см. словарь № 102, Скоринговые баллы, в результате контакта с данным исходом долгу присваивается настроенное значение данного параметра;

·Балл № 3 — см. словарь № 244, Скоринговые баллы, в результате контакта с данным исходом долгу присваивается настроенное значение данного параметра;

·Балл № 4 — см. словарь № 245, Скоринговые баллы, в результате контакта с данным исходом долгу присваивается настроенное значение данного параметра;

·Произвольные атрибуты — см. Настройка произвольных атрибутов для исхода контакта.

·Ввод показаний — ввод показаний ПУ. Данное поле включается по согласованию с технической поддержкой(Техническая константа RegContact.MeterDisplay.Use). Перечень значений такой же, как для поля Ввод комментария.

·Ввод подтверждения обещания — ввод подтверждения намерения или подтверждения оплаты. Данное поле включается по согласованию с технической поддержкой(Техническая константа ConfirmIntent.Use). Возможные значения:

·Не вводить намерения — поле ввода не будет отображено;

·Ввод подтверждения намерения — будет отображаться панель для ввода намерения, обязательная для заполнения;

·Ввод подтверждения оплаты — будет отображаться панель для ввода подтверждения оплаты, обязательная для заполнения;

·Ввод подтверждения намерения. Не обязательно — будет отображаться панель для ввода намерения, не обязательная для заполнения;

·Ввод подтверждения оплаты. Не обязательно — будет отображаться панель для ввода подтверждения оплаты, не обязательная для заполнения.

Правая область формы

·Балл — данный параметр предназначен для автоматического перерасчета параметра "Приоритет" в модуле "Постановка задач". При регистрации контакта с данным результатом имеющийся приоритет увеличивается на заданное для результата количество баллов. Если параметр Балл не задан, то увеличения не происходит. Долг, для которого приоритет не был задан (пуст), также останется с пустым приоритетом;

·МАХ + 1 — установленный флаг позволяет автоматически сделать приоритет долга максимальным среди всех имеющихся долгов у оператора. При этом вычисляется максимальное значение по всем долгам оператора и прибавляется единица, т. е. долг попадет в конец списка;

·Контакт не состоялся — если установлен данный флаг, то долг будет переведен в статус 13 "Контакт не установлен";

·Неверный номер/адрес — если установлен данный флаг и зафиксирован соответствующий результат, то номеру телефона или адресу присваивается статус неверный номер/адрес. См. также константу MaxResultWrongStatus;

·Ввод ФИО контактного лица — если установлен данный флаг, то при регистрации контакта будет предложено ввести ФИО контактного лица (для юридических лиц);

·Ввод нового номера должника — если установлен данный флаг, при регистрации контакта будет показано окно для ввода телефона, как в карточке должника при добавлении контактного телефона;

·Ввод причины возникновения задолженности — если установлен данный флаг, то оператор не сможет зарегистрировать контакт без определения (ввода) причины возникновения задолженности; при этом в форме регистрации контакта выше сценария разговора будет отображаться напоминание "Обязательно уточните причину возникновения задолженности", написанное красным цветом;

·Признак отказа от оплаты — если установлен данный флаг, то при регистрации данного результата долг будет переведен в статус 3 "Отказ";

·Ввод информации о переводе звонка — если установлен данный флаг, то при регистрации результата пользователь должен ввести информацию о переводе звонка. Рекомендуется применять только для входящих звонков;

·Считать результат успешным — данный флаг позволяет разделить контакты на успешные и неуспешные в зависимости от требований Заказчика (успешные контакты отдельно учитываются в отчетах и модулях). При регистрации успешного контакта долг будет переведен из статусов 2 "Не обработан" и 13 "Контакт не установлен" в статус 4 "Контакт установлен", а неуспешного — только из статуса 2 "Не обработан" в 13 "Контакт не установлен";

·Предложить акцию — будет показана панель информации, относящейся к акции, включающая следующие поля:

oАкция (выбор из списка);

oСумма к оплате (вычисляемое поле);

oСумма в руб (вычисляемое поле);

oДата окончания (вычисляемое поле);

·Перевести выезд в статус выполнения — если флаг установлен, то при регистрации результата по долгу, для которого выезд находится в статусе "Необходим" или "Подготовлен", итоговый статус выезда будет определяться флагом Считать результат успешным. Новый статус выезда будет либо "Выполнен успешно" (если результат считается успешным) либо "Выполнен не успешно" (если результат не считается успешным);

·Перераспределение на оператора — если установлен данный флаг, то при регистрации данного результата долг будет автоматически перезакреплен на другого оператора;

Пример. Перераспределение выполняется следующим образом.

При добавлении пользователя в модуле Пользователи для него можно установить флаг Авт. перезакреп. Допустим, для оператора А он установлен. Имеется должник, который закреплен за оператором А, и оператор Б регистрирует для такого должника результата, для которого установлен флаг Перераспределение на оператора. Долг автоматически будет перезакрепится на оператора Б.

Если бы флаг Авт. перезакреп. для оператора А не был включен, то перезакрепление бы не произошло.

·Не отображать в списке контактов — при регистрации контакта с результатом, у которого установлен этот флаг, он не будет отображаться в карточке должника на вкладке Контакты, если константа HideUnvisibleContactResult имеет значение "Истина";

·Изменение телефона/адреса — если флаг включен, то на форме ввода  данных о результате контакта будет отображена панель для изменения данных телефона или адреса, по которому регистрируется контакт;

·Учитывать в ограничениях — если флаг включен, то зарегистрированный контакт с данным исходом учитывается в атрибуте Количество контактов, учитываемых в ограничении.

Примечание. Данное поле видно в следующих случаях:

·Тип контакта Звонок, и константа CommunicationLimit.ContactTypeList содержит значение 1.

·Тип контакта Выезд, и константа CommunicationLimit.ContactTypeList содержит значение 2.

·Подтип (для отчетов) — подтип результата контакта, значение выбирается из словаря № 111;

·Код результата автодайлера — при регистрации контакта с результатом, для которого указан код результата автодайлера, этот код будет передаваться в систему IP-телефонии;

·Код качества телефона — значение атрибута определяет, какой код качества будет присвоен при регистрации контакта с данным результатом (см. словарь № 114). Код качества показывает, возможно ли установить контакт с должником по данному телефону, и может принимать следующие значения:

·Контакт клиента;

·Клиента знают, имеют связь;

·Клиента знают, связи не имеют;

·Клиента не знают;

·Неизвестно;

·Кол-во дней до следующего звонка — значение атрибута определяет максимальное количество дней до даты повторного звонка должнику, которую можно указать при регистрации контакта. Если значение не указано, то максимальное количество дней до следующего звонка определяется значением константы RecallMaxDay. Если значение задано, и константа также задано, то максимальное количество дней определяется меньшим из этих двух значений. Ввод значения имеет смысл, если установлен флаг Дата повторного звонка;

·Формула для автосценария — введите номер формулы, имеющей тип "Сценарий разговора" и дающей в качестве результата строку с перечислением через запятую номеров сценариев разговоров, описанных в модуле Сценарии сообщений для должников. Формула может иметь разные значения в зависимости от различных параметров, например возраста должника, пола, количества дней просрочки и т.д. В процессе регистрации контакта при выборе данного исхода отобразится конкатенация перечисленных сценариев. Формула может меняться при навигации по дереву. Если значение для узла не задано, то по умолчанию в качестве ID сценария берется значение, указанное в константе FormulaAutoMessage;

 Примечание. Конкатенация нескольких сценариев доступна только при значении константы _RegContactNewModeTalkPattern равном True. Если константа выключена, то можно использовать только одно сообщение.

·Не показывать при ручной регистрации контакта —  если установлен данный флаг, то при ручной регистрации контакта  данный результат в дереве контактов отображаться не будет.
