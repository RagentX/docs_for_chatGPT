Массовое получение приказа

Пункт контекстного меню (вызывается правой кнопкой мыши) Приказ получен на вкладке Приказы включает следующие подпункты:

·Должником;

·Агентством.

Условия на отображение пункта "Приказ получен" и его подпунктов

Примечание. Внимание! Первоначально доступ к данному функционалу включается по согласованию с технической поддержкой! Технические константы LawAct.Order.Receive.Bank.Use и LawAct.Order.Receive.Аgent.Use.

Пункт доступен пользователям, в роли которых LAW_ACT_ORDER_FLOW_FULL имеет значение TRUE.

Наличие или отсутствие данного действия в контекстном меню настраивается технической поддержкой.

Если константа LawAct.Order.SendAndReceive.Person.Use имеет значение "Ложь", пункт Приказ получен не отображается в контекстном меню.

Если техническая константа LawMandatoryDocAttachActivityList включает значение 6, то подпункт Банком не отображается в контекстном меню (см. полное Описание LawMandatoryDocAttachActivityList).

Если техническая константа LawMandatoryDocAttachActivityList включает значение 7, то подпункт Агентом не отображается в контекстном меню.

Если оба подпункта Банком и Агентом не отображаются в контекстном меню, то пункт Приказ получен также не отображается в контекстном меню.

Выполнение действия "Приказ получен" -> "Должником"

Действие может быть выполнено для дел, которые находятся в статусах:

·Приказ отправлен должнику;

·Возобновление;

·Отмена приказа.

Данная операция может быть выполнена только после внесения всей необходимой информации:

·поле "Дата вступления в силу";

·поле "Получен агентством".

Для того, чтобы отметить получение приказа банком, выполните следующие действия:

1.В модуле "Юридический департамент" активируйте вкладку "Приказы".

2.Создайте выборку при помощи фильтров.

3.Выберите нужные записи.

4.Активируйте контекстное меню правой кнопкой мыши и выберите пункт "Приказ получен", а затем, в открывшемся списке выберите "Банком".

5.Откроется форма подтверждения. Нажмите "Yes".

6.Если все выбранные дела имеют либо не разрешенный статус, либо незаполненную обязательную информацию, то пользователю отображается предупреждение с текстом:

Для всех приказов не заполнена необходимая информация. Доступных для обработки приказов нет. Операция прервана.

и действие не выполняется.

7.Если не все из выбранных дел имеют и разрешенный статус и заполненную обязательную информацию, то пользователю отображается предупреждение с текстом:

Для %количество приказов, для которых перевод невозможен% не заполнена необходимая информация. Продолжить, исключив ошибочные записи?

с возможностью отменить или подтвердить действие. Нажмите "Yes", чтобы продолжить, или "No", чтобы отменить действие.

8.Если все выбранные приказы имеют разрешенный статус, и для них заполнена необходимая информация, то все выбранные приказы переводятся в статус 5 "Приказ получен должником". Если в п.7 было выбрано "Yes", то выполняется перевод только для разрешенных дел.

9.После завершения действий пользователю отображается информационное окно с количеством дел переведенных и непереведенных в статус дел:

Приказов переведено в статус "Приказ получен: %количество переведенных приказов%. В процессе перевода возникли ошибки: %количество непереведенных приказов%."

Примечание для поддержки

Если был выбран подпункт Банком, то производится дополнительная проверка обязательности заполнения следующих полей карточки приказного производства, согласно значению технической константы LawAct.PositiveResult.Fields.Mandatory:

·Утвержденная пошлина — если в скобке с единицей есть значение 741;

·Сумма по решению суда — если в скобке с единицей есть значение 740;

·Получен Агентством — если в скобке с единицей есть значение 794;

·Дата вступления в силу — если в скобке с единицей есть значение 525.
