Вкладка "Связанные объекты"

Вкладка "Связанные объекты" предназначена для установления, редактирования и удаления связи между записями об имуществе.

Вкладка содержит следующие элементы:

·Таблица со списком объектов, связанных с данным имуществом, с полями:

·ID связи;

·ID объекта;

·Тип объекта;

·Адрес;

·Статус;

·Подстатус;

·Цена экспонирования;

·Цена реализации;

·Тип связи (см. словарь № 358 "Тип связи")

·С;

·По;

·Флаг "Удаление" — данный флаг включен у записей, которые могут быть удалены (только у записей о связях текущего объекта с другими объектами, данный флаг имеет значение только при режиме отображения связанных объектов "много – ко – многим").

Примечание. Поле "По" может быть пустым.

·Кнопки:

· Карточка имущества — открывается карточка имущества для выбранного в списке объекта (в новом окне);

· Добавить — Добавление связи (доступна пользователям, в роли которых доступ № 791 PROPERTY_CARD_RELATED_OBJECT_ADD имеет значение "TRUE");

· Редактировать — Редактирование связи (доступна пользователям, в роли которых доступ № 792 PROPERTY_CARD_RELATED_OBJECT_EDIT имеет значение "TRUE");

· Удалить — Удаление связи (доступна пользователям, в роли которых доступ № 793 PROPERTY_CARD_RELATED_OBJECT_DELETE имеет значение "TRUE").

·Флаг Отображать архивные связи — если флаг выключен, то на вкладке отображаются только те связанные, объекты, у которых заполнено значение "По", и оно меньше текущей даты (т.е., не отображаются связи, которые уже не актуальны). Если флаг включен, то отображаются все связи, без всяких условий на дату.

Примечание. В списке связанных объектов отображаются только те объекты, связь которых с данным имуществом актуальна (то есть текущая дата меньше даты окончания действия связи).

Отображения связанных объектов "много – ко – многим"

Доступен режим отображения связанных объектов "много – ко – многим". Для того чтобы включить этот режим вместо стандартного режима "один – ко – многим", обратитесь в техническую поддержку.

В режиме отображения связанных объектов "один – ко – многим" в списке отображаются только связи текущего объекта с другими объектами.

В режиме отображения связанных объектов "много – ко – многим" в списке отображаются не только связи текущего объекта с другими объектами, а также связи других объектов с текущим и связи между объектами, которые связаны с текущим объектом, но не связаны друг с другом напрямую.

Флаг "Удаление" включен только у записей, которые описывают связи текущего объекта с другими объектами.

Если техническая константа PropertyCard.ObjectLinkDisplay.Mode имеет значение 1, то остается старая реализация, то есть на вкладке "Связанные объекты" отображается список объектов, связанных с данным имуществом.

Если техническая константа PropertyCard.ObjectLinkDisplay.Mode имеет значение 2 для типов связи, указанных в значении константы PropertyCard.ObjectLinkDisplay.LinkType.List (коды словаря № 358 "Тип связи"), то отображаются связи по таблице person_property_link, для которых:

·Связи, у которых ID текущего объекта находится в поле "Ссылка на имущество 1".

·Связи, у которых ID текущего объекта находится в поле "Ссылка на имущество 2".

·Связи между объектами, связанными по "Ссылка на имущество 1" (например, для "Ссылка на имущество 1" = 123. Он связан с "Ссылка на имущество 2" = 124, а также "Ссылка на имущество 2" = 125 и "Ссылка на имущество 2" = 126. В карточке объекта 124 будут выводиться связи не только с объектом 123, а ещё и с объектом 125 и 126. Аналогично для объекта 125 будут выводиться связи с 123, 124 и 126 и т.д.).

·Связи между объектами, связанными по "Ссылка на имущество 2" (например, для "Ссылка на имущество 2" = 123. Он связан с "Ссылка на имущество 1" = 124, а также с "Ссылка на имущество 1" = 125 и "Ссылка на имущество 1" = 126. В карточке объекта 124 будут выводиться связи не только с объектом 123, а ещё и с объектом 125 и 126. Аналогично для объекта 125 будут выводиться связи с 123, 124 и 126 и т.д.).

·Также выводятся строки, у которых "Ссылка на имущество 1" = "Ссылка на имущество 2". Например, открыта карточка имущества ID = 123. Выводятся связи, у которых "Ссылка на имущество 1" = 123 (например, это "Ссылка на имущество 2" = 124), а также строки из таблицы, у которых "Ссылка на имущество 1" = 124. При этом, если объект 124 связан с объектом 125, а объект 125 связан с объектом 126, то в карточке объекта 123 так же будет выводиться связь с объектом 126.

 Примечание. Если некоторые условия выводят дубли, то необходимо выводить только одну запись, в приоритете записи из таблицы person_property_link, у которых ID текущего объекта находится в поле "Ссылка на имущество 1".

Флаг "Удаление" включен для строки, если она существует в таблице person_property_link (т.е. в таблице person_property_link есть запись, у которой в поле «Ссылка на имущество 1» указан идентификатор имущества, карточка которого открыта, а в поле «Ссылка на имущество 2» указано значение поля «ID объекта» выбранной строки в гриде). Для строк, у которых данное условие не выполняется, флаг выключен.
