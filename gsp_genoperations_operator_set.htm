Назначить оператора

Show/Hide Hidden Text

В меню кнопки Работа с оператором в пункте Назначить оператора доступны следующие подпункты:

·ОДВ — Оператор Досудебного Взыскания (оператор, ответственный за долг на досудебной стадии взыскания. Назначается и освобождается также в модуле Постановка задач);

·ОСВ — Оператор Судебного Взыскания (оператор, отвечающий за дело на судебной стадии. Назначается в модуле Юридический департамент путем группового изменения атрибута — Ответственный, на судебных вкладках);

·ОИП —  Оператор Исполнительного Производства (оператор, отвечающий за дело на стадии исполнительного производства. Назначается в модуле Юридический департамент путем группового изменения атрибута — Ответственный, на вкладках, относящихся к исполнительному производству);

Подпункт Назначить оператора виден, если  активен один из подпунктов:

·ОДВ — видно, если на вкладке есть поле ID долга, есть доступ WORK_TASK_SET и в РМ Группы должников было предоставлено право Работа с оператором. Назначить оператора ОДВ;

·ОСВ — видно, если на вкладке есть поле ID СВ (судебного взыскания), есть доступ LAW_ACT_CHANGE_OPERATOR и в РМ Группы должников было предоставлено право Работа с оператором. Назначить оператора ОСВ;

·ОИП — видно, если на вкладке есть поле ID ИП (исполнительного производства), есть доступ LAW_EXEC_CHANGE_OPERATOR и в РМ Группы должников было предоставлено право Работа с оператором. Назначить оператора ОИП.

Назначение оператора досудебного взыскания

Примечание. По согласованию  с технической поддержкой возможна настройка списка статусов долга, для которых запрещено назначать долгу оператора. Для поддержки: В значении константы WorkTask.SetOperatorList.BlockedStatus перечислены статусы долга, для которых запрещено назначать долгу оператора. Долгам в любых других статусах разрешено назначать оператора.

Для того чтобы назначить оператора судебного взыскания, выполните следующие действия:

1.В модуле Группы свободного поиска, активируйте вкладку, на которой доступен данный функционал.

2.Создайте выборку долгов с помощью фильтров и/или выделите записи с помощью клавиш SHIFT и CTRL.

3.Нажмите кнопку Работа с оператором.

4.В открывшемся меню включенный флаг Только выделенные означает выполнение операции только для выделенных записей, выключенный — для всей выборки.

5.Выберите пункт Назначить оператора, далее выберите подпункт ОСВ.

6.В открывшемся диалоговом окне подтвердите выполнение операции, нажав кнопку Yes.

7.Откроется форма выбора оператора. Форма имеет разный вид в зависимости от наличия запрета закрепления в роли текущего пользователя и других системных настроек. Выберите оператора.

8.Если константа WorkTask.Operator.Change.Reason.Use имеет значение "Истина", то на форме будет также присутствовать поле Причина для указания причины назначения долгов (см. словарь № 326 "Причина назначения/снятия долга"). Выберите причину. Заполнение данного поля не обязательно.

9.Если поддерживается функционал временного назначения ответственных по долгам, то на форме будет присутствовать поле Дата для ввода даты окончания срока закрепления данного оператора за выбранными долгами. Укажите дату. Заполнение данного поля не обязательно.

Ф-л временного закрепления операторов за долгами включается по технической константе WorkTask.SetUserUntilDate.Use.

10.Нажмите ОК.

11.Появится информационное окно с указанием количества обработанных долгов. Нажмите ОК.

Назначение оператора судебного взыскания

Если для выбранных долгов не созданы судебные дела, будет показано сообщение: "Среди выбранных строк нет подходящих для обработки." Никаких действий произведено не будет.

3.Нажмите кнопку Работа с оператором.

4.В открывшемся меню включенный флаг Только выделенные означает выполнение операции только для выделенных записей, выключенный — для всей выборки.

5.Выберите пункт Назначить оператора, далее выберите подпункт ОДВ.

6.В открывшейся форме выберите оператора и нажмите ОК.

7.В открывшемся диалоговом окне подтвердите выполнение операции, нажав кнопку Yes.

Отображение данных в форме "Выбор оператора"

По умолчанию в форме скрыты дополнительные колонки. Добавить в таблицу их можно, включив флаги в списке выводимых полей. Также вы можете изменить порядок следования колонок в таблице. Полученный вид таблицы выбора оператора будет сохранен при следующем входе в модуль. Для каждого из указанных модулей будет сохранен свой вид таблицы. Изменения сохраняются вплоть до сброса пользовательских настроек.

Полный список колонок формы Выбор оператора:

·ФИО — ФИО оператора;

·Роль — роль оператора;

·Отдел — отдел оператора в модуле Структура предприятия;

·Количество в Списке должников — количество долгов, закрепленных за оператором;

·Кол-во судебных дел (предв.) — учитываются только неопределенные судебные дела, находящиеся в статусе 4;

Примечание. Колонка Кол-во судебных дел (предв.) отображается только в случае, когда константа LawActUseIndefiniteAct = "Истина".

·Кол-во приказов — учитываются только приказы, находящиеся в статусах 9, 10, 11;

·Кол-во исков — учитываются только иски, находящиеся в статусах 13, 14, 15, 18;

·Кол-во правопреемств — учитываются только правопреемственные дела, находящиеся в статусах 13, 14, 15;

Примечание. Колонка Кол-во правопреемств отображается только в случае, когда константа LawActUseSuccession = "Истина".

·Кол-во банкротств — учитываются только банкротства, находящиеся в статусах 6, 13, 14, 15;

Примечание. Колонка Кол-во банкротств отображается только в случае, когда константа LawActUseBankrupt = "Истина".

·Кол-во ИП — учитываются только дела исполнительного производства, находящиеся в статусах 3, 4, 5, 6, 10;

·Всего — в колонке отображается суммарное количество активных дел и долгов, закрепленных за оператором. Причем, если функционал неопределенных судебных дел, правопреемства или банкротства отключен (соответствующая константа равна "Ложь"), то дела данного функционала, закрепленные за оператором, не учитываются в суммарном количестве дел;

·Запрет закрепления — колонка отображается для досудебной стадии при значении доступа № 543 DEBT_BAN_SET_FLAG_IGNORE = TRUE.

Учет запрета закрепления

Для досудебной стадии взыскания реализована возможность видеть в списке операторов с включенным признаком запрета закрепления.

В зависимости от значения доступа № 543 DEBT_BAN_SET_FLAG_IGNORE:

·Если доступ имеет значение TRUE, то в списке будут показаны все операторы. Поле Запрет закрепления отображает, для каких пользователей данный признак включен. При попытке осуществить назначение оператора, имеющего запрет на закрепление, пользователю будет показано предупреждение.

·Если доступ имеет значение FALSE, то в списке будут отображаться только те операторы, у которых отсутствует запрет на закрепление. Поле Запрет закрепления на форме отсутствует. Пользователь может осуществить назначение только тех операторов, которые не имеют запрета на закрепление.

   

Доступ DEBT_BAN_SET_FLAG_IGNORE=TRUE
