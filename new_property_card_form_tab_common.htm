Поля вкладки Общая карточки имущества перечислены в словаре № 363 "Параметры имущества для отображения в модуле Карточка имущества".

Поддержана возможность открывать ссылки в полях модуля (например, поле "Ссылка на альбом"). Коды полей, из которых могут открываться ссылки, перечислены в технической константе  PropertyCard.PropertyParamForLink.List. Если в таком поле хранится ссылка на файл, то при двойном клике на поле открывается проводник и место, на которое указывает ссылка (функционал аналогично открытию ссылок на вкладке "Вложения" в РМ "Карточка должника").

Редактирование данных

Для пользователей, в роли которых доступ № 805 PROPERTY_CARD_STATUS_SUBSTATUS_EDIT имеет значение "Истина", доступно редактирование следующих полей:

·Статус — термины словаря № 86 "Статус имущества";

Примечание. По согласованию с технической поддержкой вы можете ограничить список терминов, которые можно ввести в данное поле при редактировании. Для этого сообщите технической поддержке список терминов, которые разрешено ввести в данное поле при редактировании. Тогда при нажатии на поле «Статус» в списке доступных для выбора терминов отображаются только разрешенные термины, а также текущий статус имущества, если он не входит в этот список.Информация для разработчиков: Список разрешенных статусов имущества настраивается в технической константе Property.Status.List.

·Подстатус — термины словаря № 364 "Подстатус имущества", являющиеся дочерними по отношению к выбранному статусу имущества.

Область данных "Текущие процессы"

Примечание. Наличие или отсутствие на форме области данных "Текущие процессы" настраивается по согласованию с технической поддержкой.

Область данных "Текущие процессы" отображается, если константа PropertyCard.ProcessesAndAgentsFields.Mode равна 1 или "all".

В области данных "Текущие процессы" отображаются задачи по следующей логике:

1.Задачи группируются по типу задачи.

2.Из каждой группы выбирается задача с максимальным значением ID (debt_task.id).

3.Задачи, с максимальным ID, полученные для каждой группы необходимо вывести в следующем формате:

<ПРОЦЕСС (debt_task.typ, термин из соответствующего словаря)> - <Подтип (debt_task.subtyp, термин из соответствующего словаря)> - <Статус(debt_task.status, термин из соответствующего словаря)>

Область данных "Агенты"

Примечание. Наличие или отсутствие на форме области данных "Агенты" настраивается по согласованию с технической поддержкой.

Область данных "Агенты" отображается, если константа PropertyCard.ProcessesAndAgentsFields.Mode равна 2 или "all".

В область данных "Агенты" выводится информация по имуществу в следующем формате:

·Куратор: ФИО пользователя, ответственного за имущество (при наличии соответствующих полномочий доступно Назначение ответственного куратора имущества);

·Агент по реализации: <значение для параметра имущества "Ответственный за процесс "Реализация"">;

·Агент по сопровождению: <значение для параметра имущества "Ответственный за процесс "Сопровождение"">;

·Агент по монетизации: <значение для параметра имущества "Ответственный за процесс "Монетизация"">;

·Агент по инвентаризации: <значение для параметра имущества "Ответственный за процесс "Инвентаризация"">;

·Агент по выселению: <значение для параметра имущества "Ответственный за процесс "Выселение"">;

·Агент по выбытию с баланса: <значение для параметра имущества "Ответственный за процесс "Выбытие с баланса"">;

·Агент по оспариванию выбытия: <значение для параметра имущества "Ответственный за процесс "Оспаривание выбытия"">;

·Агент по списанию: <значение для параметра имущества "Ответственный за процесс "Списание"">;

·Агент по оценке: <значение для параметра имущества "Ответственный за процесс "Оценка"">.

Подчиненные вкладки на вкладке "Общая"

На вкладке "Общая" в модуле "Карточка имущества" ниже полей с общими данными по имуществу расположены вложенные вкладки:

·Телефоны;

·Адреса;

·Email.

На них реализован стандартный функционал для работы с контактными данными должника.
