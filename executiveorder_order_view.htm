Вкладка "Общая информация"

Общая информация о должнике, его телефонах и адресах, информация о суде и работе с делом в исполнительном производстве должника отображается в карточке исполнительного производства на вкладке Общая информация. Вкладка содержит следующие элементы:

·раздел Информация о долге, содержащий данные о договоре и долге;

·раздел Исполнительное производство, содержащий данные о работе по данному делу и кнопки для переходов по этапам жизненного цикла исполнительного производства;

·раздел Информация об ИД, содержащий данные об исполнительном документе по делу;

·раздел Телефоны, содержащий список телефонов ответчика по делу (должника или поручителя) и кнопки для работы с ним;

·раздел Адреса, содержащий список адресов ответчика по делу (должника или поручителя) и кнопки для работы с ним;

·поле Комментарий для ввода дополнительной информации  и кнопку для просмотра истории комментариев по работе с должником;

·поле Указание руководителя — право вводить информацию в поле имеют только те пользователи, для которых доступ LAW_EXEC_CARD_INSTRUCT установлен в значение TRUE.

Ссылка: Подробная информация о редактировании полей в карточках судебной работы и ИП.

Раздел "Информация о долге"

Раздел Информация о долге содержит следующие поля:

·ФИО должника — данное поле отображается только для тех дел, в которых ответчиком является поручитель или связанное лицо;

·Общая сумма — общая сумма долга по решению суда. Поле доступно для редактирования;

·Валюта — валюта долга (см. Список валют);

·        Остаток — остаток суммы к оплате, выводится в валюте долга, указанной в поле Валюта. Поле доступно для редактирования;

·Остаток по ИД;

поле Остаток по ИД выводится на вкладку, если в константе LawExecCard.DateFields.Use есть значение"5".

 Примечание. Для Лето банка значение в поле Остаток по ИД (law_exec.rest_sum) в исполнительном производстве рассчитывается по следующей формуле:

·Если ИП создано по приказу, то формула имеет вид:

½debt_detail.number_value_1 + debt_detail.number_value_2 + law_act.due_sum

Если заполнено поле Дата получения листа на руки (law_exec.receipt_act_dt), то из итоговой суммы вычитаются все подтвержденные платежи по долгу, которые были совершены позже этой даты.

·Если ИП создано по иску, то формула имеет вид:

½debt_detail.number_value_1 + debt_detail.number_value_2 + debt_detail.number_value_10 + debt_detail.number_value_11 + debt_detail.number_value_12 + law_act.due_sum

Если заполнено поле Дата получения листа на руки (law_exec.receipt_act_dt), то из итоговой суммы вычитаются все подтвержденные  платежи по долгу, которые были совершены позже этой даты.

 Примечание. Поле Остаток по ИД доступно для редактирования только при наличии у пользователя доступа LAW_EXEC_CARD_DEBT_EDIT. 

·Утвержденная пошлина — утвержденная сумма пошлины, выводится в основной валюте Системы (при переводе судебного дела на ИП значение автоматически переносится из поля Сумма пошлины карточки судебной работы). Поле доступно для редактирования;

·Дата договора;

·Номер договора;

·Тип продукта — см. словарь № 11;

·Примечание — см. словарь № 248;

Примечание. Атрибут Примечание доступен для редактирования, если у пользователя есть доступ LAW_EXEC_SET_DICT_VALUE. В этом случае при редактировании поля наряду со словарными значениями из словаря № 248 можно выбрать и значение "пусто". Для поддержки: см. Функционал "Подстатус ИП" на месте поля "Примечание".

·признак Наличие залогового имущества.

·Составляющие — список составляющих долга в исполнительном производстве; поле доступно только для долгов по ЖКХ.

·Переключатель "По решению суда/Остаток", показывающий, какие составляющие долга указаны в предыдущем поле; поле доступно только для долгов по ЖКХ.

Примечание. Составляющие доступны только для дел, созданных по долгам по ЖКХ и только в случае использования функционала ведения судебной работы по периодам задолженности. Составляющие загружаются в дело исполнительного производства из соответствующего дела судебной работы. Составляющие в карточке исполнительного производства доступны для редактирования.

Раздел "Исполнительное производство"

Раздел Исполнительное производство содержит следующие поля:

·Статус — статус дела в исполнительном производстве, см. словарь № 77 (см. Этапы работы в исполнительном производстве);

·Дата создания ИП — дата создания дела в исполнительном производстве;

·Плановая дата подачи в ФСПП — плановая дата подачи дела в ФССП, рассчитывается при помощи константы LawExecDaysForCompletePlanDate следующим образом:

1.Если в значении константы встречается запись, заключенная в круглые скобки и имеющая вид (%bank_id%:%days%), в которой левая часть (%bank_id%) обозначает идентификатор банка, которому принадлежит портфель долга, то дата подачи в ФССП устанавливается как сумма даты формирования ИП и количества дней, указанных в правой части записи (%days%).

2.Если указание количества дней для данного банка в константе отсутствует, то Система осуществляет поиск такой записи, заключенной в круглые скобки, левая часть которой равна нулю. Если такая запись найдена, то дата подачи в ФССП устанавливается как сумма даты формирования ИП и количества дней, указанных в правой части этой записи (%days%).

3.Если п. 1 и п. 2 не выполнены, дата подачи в ФССП автоматически не заполняется.

Поле Плановая дата подачи в ФСПП выводится на вкладку, если в константе LawExecCard.DateFields.Use есть значение "4".

·Дата подачи — фактическая дата подачи дела в ФССП (может быть настроено ограничение: не более определенного количества дней до текущей даты и не более определенного количества дней после текущей даты; для настройки или отмены такого ограничения обратитесь в техническую поддержку техническая константа LawExec.DateFields.Periods);

Примечание. При каждом изменении Дата подачи сравнивается с Датой возбуждения (если поле заполнено). Если введенное значение больше даты возбуждения, то Дата подачи не обновляется, а появляется сообщение:

Дата возбуждения не может быть меньше даты подачи.

·№ исп. производства — номер дела в исполнительном производстве;

·Способ подачи — см. словарь № 16;

·Дней в работе — количество дней в работе, в зависимости от значения константы LawExecWorkDaysMode:

·Если LawExecWorkDaysMode = 1, то расчет должен быть следующим: от текущей даты отнимается дата сдачи в ФССП;

·Если LawExecWorkDaysMode = 2, то от текущей даты отнимается значение даты формирования ИП;

·Если LawExecWorkDaysMode = 3, то от текущей даты отнимается значение даты возбуждения ИП;

·Если LawExecWorkDaysMode = 4, то от текущей даты отнимается значение даты получения исполнительного листа;

·При любом другом значении константы, поле "Дней в работе" остается пустым.

·Дата возбуждения — дата возбуждения исполнительного производства, см. Возбуждение исполнительного производства/отказ в возбуждении исполнительного производства  (может быть настроено ограничение: не более определенного количества дней до текущей даты и не более определенного количества дней после текущей даты; для настройки или отмены такого ограничения обратитесь в техническую поддержку техническая константа LawExec.DateFields.Periods);

Примечание. При каждом изменении Дата возбуждения сравнивается со следующими датами:

· Датой подачи (если поле заполнено). Если введенное значение меньше даты подачи, то Дата возбуждения не обновляется, а появляется сообщение:

Дата возбуждения не может быть меньше даты подачи.

· Датой окончания (если дата окончания заполнена, например, при регистрации окончания ИП или погашения задолженности). Если введенное значение больше даты окончания, то Дата возбуждения не обновляется, а появляется сообщение:

Дата окончания не может быть меньше даты возбуждения.

·Дата ограничения выезда — дата, до которой ограничен выезд должника заграницу в связи с возбуждением дела в исполнительном производстве;

Поле Дата ограничения выезда выводится на вкладку, если в константе LawExecCard.DateFields.Use есть значение "2".

 Примечание. Порядок ввода значений в поле Дата ограничения выезда регулируется технической константой LawExecCard.RestrictionLeaveDt.Change :

·Если константа имеет значение "Истина", то изменения значения поля разрешены.

·Если константа имеет значение "Ложь", то после первоначального ввода значения в поле и сохранения изменений в карточке, последующие изменения значения данного поля не разрешаются.

·Дата получения ИП на взыскание — дата получения исполнительного производства на взыскание долга;

Поле Дата получения ИП на взыскание выводится на вкладку, если в константе LawExecCard.DateFields.Use есть значение "3".

·Дата получения ИП на арест —  дата получения исполнительного производства на арест имущества должника;

Поле Дата получения ИП на арест выводится на вкладку, если в константе LawExecCard.DateFields.Use есть значение "1".

·Причина отказа в возбуждении — причина отказа в возбуждении исполнительного производства (см. Возбуждение исполнительного производства/отказ в возбуждении исполнительного производства);

·Дата отказа в возбуждении — дата отказа в возбуждении исполнительного производства (см. Возбуждение исполнительного производства/отказ в возбуждении исполнительного производства);

·Причина приостановления — см. словарь № 266;

·Дата приостановления — дата последнего приостановления ИП (см. Приостановление ИП);

·Плановая дата окончания ИП — плановая дата окончания дела в исполнительном производстве;

·Дата постановления об оконч. ИП — дата постановления об окончании исполнительного производства;

·Сумма мир. соглашения — сумма, которую должен выплатить должник по мировому соглашению (см. Заключение мирового соглашения);

·Дата мир. соглашения — дата заключения мирового соглашения с должником (см. Заключение мирового соглашения);

·Дата обжалования.

Раздел "Информация об ИД"

Раздел Информация об ИП содержит следующие поля:

·№ исп. документа — поле доступно для редактирования;

·Дата вынесения — дата вынесения решения судом. Поле доступно для редактирования;

·Название — название суда. Поле доступно для редактирования;

·Тип исп. документа — тип исполнительного документа (см. словарь № 124);

·Дата вступления в силу — дата вступления решения Суда в силу;

·Дата получения листа на руки — дата получения исполнительного листа на руки должником;

·Доп. информация — при нажатии открывается окно, содержащее сценарий специального сообщения, настроенного в модуле Сценарии сообщений (вкладка Специальное) и имеющего идентификационный номер, указанный в значении константы DebtMainInfoFormat после символов "(4:". Настройка информационного блока аналогична настройке информационного блока для карточки должника.

Примечание. Если в значении константы DebtMainInfoFormat после символов "(4:" указано число, не являющееся идентификационным номером специального сценария, то поле Доп. информация отсутствует.

Раздел "Телефоны"

В разделе Телефоны отображаются телефоны ответчика по данному делу:

·Если ответчиком является должник, то отображаются телефоны должника. Если если включена константа CardAllPhoneShow, то отображается также  информация по телефонам поручителей и залогодателей.

·Если ответчиком является поручитель, то в списке отображаются только телефоны поручителя.

Раздел Телефоны содержит следующие элементы:

·Список телефонов с полями:

·Тип — тип телефона, см. словарь № 4;

·Номер — номер телефона;

·Контактное лицо;

·Кнопки для работы со списком телефонов.

Действия со списком телефонов — см. Ведение списка контактных телефонов должника.

Окраска телефонов в зависимости от статуса

Строки в списке телефонов выделены цветом фона в зависимости от статуса контактного телефона (см. Статусы телефонов):

·Не звонили ни разу — серый цвет.

·Результата не было — белый цвет.

·Неверный номер — красный цвет.

·Результат — зеленый цвет .

·Последний результат — голубой цвет.

·Автоинформатор — розовый цвет.

Если у пользователя есть доступ 127 DEBT_CARD_ALL_PHONE, то в карточке должника он может видеть заблокированные номера телефонов. Такие записи в списке телефонов выделены серым шрифтом на белом фоне.

Раздел "Адреса"

В разделе Адреса отображаются адреса ответчика по данному делу.

·Если ответчиком является должник, то отображаются адреса должника. Если включена константа CardAllAddressShow, отображается также информация по адресам поручителей и залогодателей.

·Если ответчиком является поручитель, то в списке отображаются только адреса поручителя.

Раздел Адреса содержит следующие элементы:

·Список адресов с полями:

·Тип — тип адреса, см. словарь № 5;

·Адрес — адрес должника;

·Комментарий — поле с дополнительной информацией о должнике и кнопка для просмотра истории комментариев;

·Кнопки для работы со списком адресов.

Действия со списком адресов — см. Ведение списка адресов должника.

См. также:

·Отметка адреса подачи;

·Просмотр адреса на карте;

·Печать документов исполнительного производства;

·Печать документа на выезд к приставу.

Окраска адресов в зависимости от статуса

Строки в списке адресов выделены цветом фона в зависимости от статуса адреса. (см. Статусы адресов):

·Не выезжали ни разу — серый цвет;

·Результата не было — белый цвет;

·Неверный адрес — красный цвет.

·Результат — зеленый цвет;

·Последний результат — голубой цвет.

Если у пользователя есть доступ 280 DEBT_CARD_ALL_ADR, то в карточке должника он может видеть заблокированные адреса. Такие записи в списке адресов выделены серым шрифтом на белом фоне.
