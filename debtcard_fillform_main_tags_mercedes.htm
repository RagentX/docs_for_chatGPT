Для клиента Мерседес

Осуществлена доработка процедуры ur_eps_wrp$single_sms_upd_mers.

Поддержана обработка следующих новых клиентских тегов (по задаче 27665):

Тег

Описание

%utag.all_stages%

Выводятся все уникальные стадии (debt.makr1) всех активных договоров должника через запятую. Порядок следования стадий в порядке возрастания кодов терминов по словарю 101.

Например, если у должника 2 договора в стадии "SOFT-COLLECTION", а еще 1 договор в стадии "OUTSOURCING.HARD", то вывести необходимо "SOFT-COLLECTION, OUTSOURCING.HARD"

%utag.cnt_cars_all%

"Кол-во машин на клиента" на вкладке "Call for Fleet Clients" (см. поле cnt_cars_all в представлении uv_gsp_call_mercedes)

%utag.sum_for_all_debt%

Сумма debt.debt_sum по всем активным долгам должника.

Выводится в рублях по курсу на текущий момент.

%utag.leasing_total_sum%

Сумма полей debt.debt_sum+debt.peni_sum+debt.commission_sum

%utag.leasing_cars_info%

Выводится количество автомобилей "Машина (1C)", сгруппированных по полю "Наименование объекта" в карточке машины:

Считаются только машины, связанные с данным договором.

Перечислить их через запятую по следующему шаблону: %кол-во% %наименование объекта%

Например, "4 Actros 1841 LS ; 1 C 180 ; 3 Axor 1843 LS"

%utag.CA_info%

Только если долг находится в КА (debt.r_portfolio_out_id не пустое, и статус указанного портфеля = Передан), то требуется выводить Коллекторское агентство и дату передачи (по портфелю) в следующем виде:

КА: bank.full_name. Дата передачи: portfolio.sign_date

%utag.prom_status%

Статус последнего обещания по debt_promise_last по словарю 258 (учесть, что имеются и обещания по должнику. Поиск нужного обещания можно посмотреть в SINGLE_SMS_TEXT_UPD_X).

Логика определения статуса обещания: http://confluence.luxbase.int:8090/pages/viewpage.action?pageId=59408391

%utag.last_call_dt%

Дата последнего контакта с типом "Звонок" или "Входящий" по договору 

%utag.last_call_result%

Результат последнего контакта с типом "Звонок" или "Входящий" по договору

Описание

Логика формирования

%utag.sum_for_all_debt%

"Итого просроченная задолженность" по всем активным договорам. Выводится в рублях по курсу на текущий момент.

Сумма просроченной задолженности (debt.debt_sum) для всех активных долгов должника (debt.status <> 6,7,8) 

%utag.all_debt_peni_sum%

"Итого за день (Штрафы)" по всем активным договорам. Выводится в рублях по курсу на текущий момент.

Сумма штрафов (debt.peni_sum) для всех активных долгов должника (debt.status <> 6,7,8)

%utag.stock%

Кодовое слово

debt.stock

%utag.other_debt_info%

Если у должника больше 1 активного договора, то по каждому другому договору требуется вывести следующий текст:

Другой долг: Сумма - %debt.debt_sum% %debt.cur_name%, Штрафы в день - %debt.peni_sum% %debt.cur_name%, Договор - %debt.contract%, Лицевой счет - %debt.account%, Предмет залога: %utag.bank_car_mark_model%

 

%debt.debt_sum%, %debt.peni_sum%, %debt.contract%, %debt.account%, %debt.cur_name% - подставляются значения аналогично данным тегам в SINGLE_SMS_TEXT_UPD_X

%utag.bank_car_mark_model% - Марка и модель машины. Значения параметров имущества с типом 96 + 97. Если по долгу несколько машин, то требуется каждую вывести через запятую. Аналогичный тег поддержан в ur_eps_wrp$single_sms_upd_mers

Если у должника больше одного активного договора (debt.status <> 6,7,8), то последовательно для каждого договора, кроме текущего, требуется вывести информацию, представленную в соседней колонке. Например, если у должника 3 активных долга, только по 2 из них (исключая текущий) необходимо вывести информацию в данному теге.
