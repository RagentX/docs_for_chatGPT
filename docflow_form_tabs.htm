Отображение данных на вкладках модуля

Документы подразделяются по направлениям на:

·Входящие;

·Исходящие.

Документы имеют тип — см. словарь № 332 "Тип документа".

Документы имеют статус — см. словарь № 329 "Статус документа".

Документ является активным, если его статус не перечислен в значении константы Docflow.ArchiveStatusList.

Документ является архивным, если его статус указан в значении константы Docflow.ArchiveStatusList.

Входящий документ считается относящимся к пользователю, если данный пользователь является регистратором документа, его получателем, или его исполнителем.

Исходящий документ считается относящимся к пользователю, если данный пользователь является регистратором документа или его составителем.

Вкладка "Входящие"

На вкладке Входящие отображаются входящие документы, имеющие активный статус.

Если в роли пользователя доступ № 673 DOCFLOW_ALL_DOCUMENTS_VIEW имеет значение TRUE, то в списке отображаются все входящие документы, иначе в списке отображаются только документы текущего пользователя и его подчиненных.

Записи о документах могут быть окрашены — см. Изменение цвета для просроченных документов.

Список колонок на вкладке "Входящие"

Вкладка включает следующие колонки:

·Номер документа — номер входящего документа (docflow.document_incoming_number).

·Тип документа — см. словарь № 332 "Тип документа" (docflow.document_type).

·Форма — см. словарь № 331 "Форма документа" (docflow.document_form).

·Получатель — через запятую выводятся все получатели документа (по таблице docflow_receiver), например:

Пример: Метов И.А., Москалев Н.В.;

·Исполнитель — через запятую выводятся все исполнители документа (по таблице docflow_operator), например:

Пример: Рыжиков А.В., Якимов А.Б.

·Дата регистрации — указывается дата (без времени); значение поля устанавливается автоматически при создании документа и не может быть изменено (docflow.reg_dt).

·Регистратор — значение поля устанавливается автоматически при создании документа и не может быть изменено (docflow.r_reg_user_id).

·Отправитель — отправитель входящего документа (docflow.r_sender_id) (отправителем входящего документа может быть, как физ./юр.лицо, так и суд/ФССП), например:

Пример: Нагатинский районный суд города Москвы.

·Исходящий номер (docflow.document_outgoing_number).

·Дата исходящего —  дата, проставленная на документе отправителем (docflow.document_dt).

·Дата оригинала —  дата поступления оригинала документа (docflow.original_dt).

·Приоритет — см. словарь № 334 "Приоритет отработки документа" (docflow.priority).

·Содержание —  описание документа (содержание) (docflow.dsc). Пример:

Повестка в суд 16.08 по Иванову И.И.

·Комментарий —  комментарий (docflow.comment). Пример:

В документе не хватает второго листа.

·Адрес отправителя — адрес отправителя; отображается только при включенной константе DocflowCard.SenderAddress.Use. При этом для отображения выбирается адрес с типом, указанным в значении константы DocflowCard.SenderAddress.Typ. Если таких адресов несколько, то выбирается тот, у которого id наименьший.

·Комментарий ОД:

·Для судов, ФССП — соответствующий комментарий по суду или ФССП (law_court.dsc).

·Для физических, юридических лиц и ИП выводится соответствующий комментарий (person.dsc).

·Статус — статус документа (см. словарь № 329 "Статус документа") (docflow.status).

·Наименование документа — данное поле может быть скрыто по согласованию с технической поддержкой. Значение выводится из поля docflow.dict_value_1. Если у данного column_spec стоит признак is_not_use = 1, то требуется скрывать поле с формы.

·Наличие связанного документа — флаг; если есть связанный документ, то флаг включен.

·Наличие связанного объекта — флаг; если есть хотя бы один связанный объект, то флаг включен.

·Комментарий исполнителя — комментарий исполнителя (docflow.operator_comment).

·Статус — статус документа (см. словарь № 329 "Статус документа") (docflow.status).
