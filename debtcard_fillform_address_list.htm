Форма редактирования атрибутов адреса

Описание формы редактирования атрибутов адреса

Форма редактирования атрибутов адреса содержит следующие элементы:

·Тип — тип адреса. Поле обязательно для заполнения.

·Индекс.

·Словарный атрибут адреса 1 — см. словарь № 372. Данное поле заполнено по умолчанию значением, которое определяется константой Address.DictValue1.Default. (Примечание: отображение данного атрибута и его название настраиваются по согласованию с технической поддержкой.)

·Словарный атрибут адреса 2 — см. словарь № 373. Если словарь № 373 имеет в качестве родительского словарь № 372, то в списке отображаются только термины словаря, являющиеся дочерними по отношению к выбранному термину в поле "Словарный атрибут адреса 1". (Примечание: отображение данного атрибута и его название настраиваются по согласованию с технической поддержкой.)

·Словарный атрибут адреса 3 — см. словарь № 374. (Примечание: отображение данного атрибута и его название настраиваются по согласованию с технической поддержкой.)

Примечание. Названия словарных атрибутов адреса на форме "Адрес" и их отображение или сокрытие настраиваются по согласованию с технической поддержкой.

Для поддержки:

·Словарные атрибуты телефона не отображаются, если для них установлен соответствующий column_spec.is_not_use=1

·Иначе отображение атрибутов определяется значением column_spec.display_mode:

·NULL - отображаются в карточке и гриде

·1 - отображаются только в карточке

·2 - отображаются только в гриде

·Использовать KLADR — если данный флаг установлен, то значения полей Область, Район, Населенный пункт будут выбираться из списков, сформированных на основании классификатора адресов России, если не установлен — будет использоваться ручной ввод данных.

·Область — в зависимости от флага Использовать KLADR значение выбирается в списке или вводится вручную.

·Район — в зависимости от флага Использовать KLADR значение выбирается в списке или вводится вручную.

·Населенный пункт — в зависимости от флага Использовать KLADR значение выбирается в списке или вводится вручную. Данное поле обязательно для заполнения.

·Использовать справочник улиц — если данный флаг установлен, то значения поля Улица будет выбираться из списка улиц, сформированных на справочника; Данный флаг может быть скрыт на форме, для этого установите константе DebtCard.AddrCard.StreetKLADR.Use значение "Истина". При этом введенные названия улиц не будут проверяться справочником улиц.

Примечание: справочник улиц, если он не скрыт, доступен не для всех населенных пунктов.

·Улица — в зависимости от флага Использовать справочник улиц значение выбирается в списке или вводится вручную.

·Дом.

·Корпус.

·Квартира.

·Комментарий — дополнительная информация.

·Должник проживает по данному адресу — флаг устанавливается вручную и является признаком проживания должника по указанному адресу.

·Признак возвращенного письма — флаг устанавливается вручную или автоматически при регистрации возвращенного письма в РМ "Исходящее информирование". Если установлен данный флаг, то в процессе автоматического формирования писем в системе письма на него формироваться не будут.

Копирование адреса или строки списка адресов

·Для копирования выбранного адреса в буфер обмена выберите в контекстном меню (вызывается правой кнопкой мыши) пункт Копировать адрес;

·Для копирования всей выбранной строки в буфер обмена выберите в контекстном меню (вызывается правой кнопкой мыши) пункт Копировать строку;

Добавление адреса

Добавление адреса доступно только тем пользователям, в роли которых доступ № 13 DEBT_CARD_EXTRA_EDIT имеет значение TRUE.

Для того чтобы добавить адрес в список адресов должника, выполните следующие действия:

1.В форме со списком адресов нажмите кнопку "Добавить".

2.Откроется форма редактирования атрибутов адреса.

3.В форме заполните все необходимые поля.

4.Нажмите кнопку ОК, чтобы сохранить данные.

Проверка совпадающих адресов

При добавлении нового адреса осуществляется проверка совпадающих адресов согласно значению константы Address.Add.CheckEqual.Mode. Если константа имеет значение:

·1 — при добавлении адреса никаких дополнительных проверок не осуществляется;

·2 — если добавляемый адрес совпадает с одним из существующих адресов данного должника, то добавления не происходит, и пользователю выводится соответствующее сообщение;

·3 — если добавляемый адрес совпадает с одним из существующих адресов данного должника, относящихся к тому же типу, что и добавляемый адрес, то добавления нового телефона не происходит, и пользователю выводится соответствующее сообщение.

Проверяется совпадение по каждому полю адреса. Незаполненные поля считаются совпадающими. При сравнении незаполненного и заполненного поля, эти поля также считаются совпадающими. Не совпадающими считаются поля, которые оба заполнены и имеют различные значения.

       Логика проверки совпадения адресов

Проверка осуществляется без учета регистра. Если значение хотя бы одно поля нового адреса не совпадает со значением этого поля в существующем адресе, то адреса считаются несовпадающими.

Сверка адреса проводится для следующих полей: 

·Страна (country);

·Область (oblast);

·Район (district)

·Нас. пункт (city);

·Улица (street);

·Дом (house);

·Корпус (building);

·Квартира (flat).

Поля считаются совпадающими, если:

·Значения в поле совпадают;

·Оба значения пусты;

·Одно из значений пусто, а другое не пусто.

Редактирование адреса

Редактирование всех атрибутов адреса доступно только тем пользователям, в роли которых доступ № 92 DEBT_CARD_PHONE_ADR_EDIT имеет значение TRUE.

Если доступ № 92 DEBT_CARD_PHONE_ADR_EDIT у пользователя отсутствует или имеет значение FALSE, но при этом доступ № 143 DEBT_CARD_PHONE_ADR_DSC_EDIT имеет значение TRUE, то для изменения доступно только поле:

·Комментарий.

Если в Системе используются словарные атрибуты телефона, то дополнительно выполняется следующая логика:

Если доступ № 92 DEBT_CARD_PHONE_ADR_EDIT отсутствует или имеет значение FALSE, то при попытке редактирования адреса проверяется значение доступа № 846 PHONE_ADR_ EDIT_EXCEPTION:

·Если данный доступ отсутствует или его значение пусто, то доступ к редактированию словарных атрибутов адреса отсутствует.

·Если данный доступ заполнен, то доступы к редактированию словарных атрибутов адресов определяются наличием в списке id соответствующих атрибутов согласно таблице column_spec:

·2071 — для словарного атрибута адреса 1;

·2072 — для словарного атрибута адреса 2;

·2073 — для словарного атрибута адреса 3.

·Если в списке отсутствуют все три вышеперечисленных значения, то доступ к редактированию адресов у пользователя отсутствует.

Для того чтобы отредактировать атрибуты адреса, выполните следующие действия:

1.Выберите нужный адрес в списке адресов и нажмите кнопку .

2.В форме редактирования атрибутов адреса измените необходимые данные.

3.Нажмите кнопку ОК, чтобы сохранить изменения.
