Дозагрузка адресов для персон

В режиме массовой дозагрузки адресов для персон файл MS Excel обрабатывается по специальному алгоритму. Строки файла должны иметь следующую структуру:

1.Идентификатор персоны в таблице персон.

2.Тип адреса (код термина словаря № 5).

3.Почтовый индекс.

4.Область.

5.Район.

6.Город.

7.Улица.

8.Дом.

9.Корпус.

10.Квартира.

11.Статус адреса в системе DaData (код/название термина из словаря № 72).

12.Комментарий.

13.Код KLADR.

14.Полный код KLADR.

15.Код ОКАТО.

16.Идентификатор поручителя (необязательный реквизит).

17.Идентификатор залогодателя (необязательный реквизит).

18.Координата X (широта, необязательный реквизит).

19.Координата Y (долгота, необязательный реквизит).

Проверка и преобразование данных

В процессе открытия файла автоматически выполняется преобразование данных. Осуществляется проверка следующих данных:

·идентификатора персоны на непустое целочисленное значение, имеющееся в таблице персон;

·типа адреса на наличие в словаре типов адресов;

·почтового индекса, области, района, города, улицы, дома, корпуса, квартиры, комментария на предельный размер текста;

·контроля корректности кода KLADR;

·идентификатора поручителя (при его наличии) на целочисленное значение, имеющееся в таблице поручителей;

·идентификатора залогодателя (при его наличии) на целочисленное значение, имеющееся в таблице залогодателей.

Загрузка данных по каждой записи сводится к попытке корректировки данных в таблице адресов.

Поддержка константы LoadData.Address.LoadWithTransform

Если значение константы LoadData.Address.LoadWithTransform имеет значение "Ложь", то составляющие адреса загружаются без преобразования, в частности:

·При открытии Excel файла через кнопку "Открыть" отключается автоматическое преобразование всех составляющих адреса;

·Логика проверки данных через кнопку "Проверить" остается прежней, так как она не преобразует составляющие адреса;

·Логика добавления данных через кнопку "Сохранить в БД" остается прежней, исключив любое преобразования составляющих адреса (т.е. данные для адреса добавляются в таблицу address в том виде, в котором они указаны в гриде). Полный адрес (address.full_adr) собирается также, без дополнительных преобразований составляющих, с использованием функции address$get_full_adr для MSSQL и с использованием представления address_full_kladr_v для ORACLE.

Если значение константы LoadData.Address.LoadWithTransform имеет значение "Истина", то составляющие адреса загружаются с использованием текущей логики (т.е. с преобразованиями).

Чистка адресов (Поддержка константы LoadDataUseDataq)

Если у пользователя включена  константа LoadDataUseDataq, то на последнем этапе работы процедуры по дозагрузке адресов, на экране пользователя появляется информационное окно "Чистка адресов через DaData".

·Если пользователь выбирает режим Производить для всех адресов, то после нажатия кнопки ОК создается нужное количество заявок на очистку адресов через DaData.  Заявки на очистку адреса создаются путем добавления записей в таблицу dataq_request, со следующим набором полей:
