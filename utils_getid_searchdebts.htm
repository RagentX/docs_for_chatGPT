Поиск долгов

Для того чтобы выполнить поиск долгов, в форме Утилита определения ID долга нажмите кнопку Обработать. Во время поиска долгов кнопки Обработать, Выгрузить файлы Excel, Выгрузить файлы Csv, Отменить, а также возможность редактирования ячеек таблицы становятся недоступны. В процессе поиска долгов в нижней строке формы будет отображаться тип операции (поиск долгов), текущие данные о проценте обработанных данных, директория и имя файла.

Проверка данных по каждой строке таблицы включает:

·контроль наличия банка с заданным названием в базе данных;

·контроль наличия портфеля с заданным префиксом в базе данных;

·контроль соответствия найденного портфеля найденному банку;

·контроль наличия в базе данных должника по ключевым реквизитам "Номер договора", "Номер счета", "ID в системе клиента" с учетом ФИО должника, банка и портфеля;

·контроль единственности найденного долга;

·контроль соответствия заданного ФИО должника и ФИО для найденного долга;

·контроль соответствия заданного банка и банка для найденного долга;

·контроль соответствия заданного префикса портфеля и портфеля для найденного долга.

Реквизиты каждой строки автоматически обрезаются до максимально допустимой длины:

·64 символа — для банка и префикса портфеля;

·322 символа — для ФИО должника;

·128 символов — для номера договора;

·32 символа — для номера счета и ID в системе клиента.

По окончании поиска долгов в таблице на основе найденных в системе данных автоматически формируются следующие дополнительные столбцы:

·ID долга — идентификатор долга;

·ID персоны — идентификатор должника;

·ФИО — фамилия, имя, отчество должника;

·Банк долга — название банка;

·Портфель долга;

·№ договора;

·Л/С — лицевой счет должника;

·ID в системе клиента;

·Остаток к взысканию — остаток суммы долга на текущую дату;

·Статус долга;

·Причина закрытия;

·Комментарий;

·Начальный долг;

·Сумма подтвержденных платежей по долгу.

Ячейки таблицы, по которым зафиксированы ошибки, окрашиваются.

Если отсутствуют ошибки, будет выдано сообщение: "ID долга однозначно определены для всех <количество записей> записей". При наличии хотя бы одной ошибки будет выдано сообщение: "Однозначно определить долг не удалось для <количество записей с ошибками> записей из <общее количество записей>".

Кнопки Обработать, Выгрузить файлы Excel, Выгрузить файлы Csv и Отменить становятся доступными, восстанавливается возможность ручного редактирования ячеек таблицы.

Повторный поиск долгов выполняется после ручного редактирования данных в таблице. Если изменения отсутствуют, повторная проверка не выполняется, выводится сообщение: "Данные в ячейках таблицы не изменялись, повторный поиск долгов отменен". При наличии измененных данных повторный поиск долгов производится по описанному выше алгоритму.

Результаты поиска можно выгрузить в итоговые файлы.
