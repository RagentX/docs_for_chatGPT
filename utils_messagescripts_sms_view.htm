SMS-сообщение

Сценарии SMS используются в следующих случаях:

·формирование SMS

·из модуля Карточка должника ( Формирование SMS, формирование SMS поручителю, залогодателю);

·из модуля Центр обзвона должников (Создание заявки на отправку SMS должнику)

·массовая отправка SMS-сообщений из модуля "Исходящее информирование" (см. Формирование и отправка SMS).

Вкладка "SMS-сообщения"

Описание вкладки "SMS-сообщения"

·список сценариев SMS-сообщений, содержащий следующие поля:

·ID — идентификатор сценария, формируется автоматически;

·Номер — номер сценария;

·Ед. рассылка — признак того, что у оператора есть возможность создать заявку на отправку SMS-сообщения из модуля "Карточка должника";

·Транслит — признак транслитерации сообщения латинскими буквами;

·Название — название сценария;

·Получатель — получатель сообщения;

·Текст — текст сценария разговора;

·кнопки для добавления, редактирования, удаления сценариев.

Форма добавления/редактирования сообщения

Описание формы добавления/редактирования сообщения

·Название — название сценария. Поле обязательно для заполнения;

·Номер — номер сценария, который может использоваться для привязки сценария к узлу дерева контактов (см. Добавление результата контактов);

·Получатель — получатель сообщения;

·Транслит — установленный флаг означает, что будет выполнена транслитерация сообщения латинскими буквами;

·Учитывать в ограничениях — если флаг включен, то сообщение, отправленное по данному шаблону, будет учтено в атрибуте "Количество контактов, учитываемых в ограничении". Данное поле видно в том случае, если константа CommunicationLimit.ContactTypeList содержит значение 7;

·Текст — текст сценария. Поле может содержать как статичный текст, так и переменные, вместо которых при выполнении сценария будут подставлены конкретные значения (см. Подстановка переменных в текст сообщения);

Примечание. Лимит символов для одного SMS-сообщения в латинской раскладке составляет 160 латинских символов или 70 символов кириллицы (русских). Использование заглавных букв в тексте SMS-сообщения ведет к увеличению реального количества символов. При превышении лимита SMS-сообщение будет автоматически разбиваться на части.

·Единичная рассылка — установленный флаг означает, что оператор сможет создать заявку на отправку SMS-сообщения с указанными параметрами непосредственно из модуля "Карточка должника".

Действия со сценариями SMS

Создание сценария SMS

Для того чтобы создать сценарий SMS, выполните следующее:

1.В форме Сценарии сообщений для должников на вкладке SMS-сообщение нажмите кнопку .

2.В открывшейся форме добавления сообщения заполните нужные поля.

3.Нажмите кнопку ОК. Сценарий будет добавлен в список.

4.После создания сценария SMS-сообщения рекомендуется настроить доступ к нему пользователей. Для этого в модуле Роли и доступы выберите роль, а затем  на вкладке SMS-сообщение добавьте созданный сценарий с уровнем доступа "Формирование" данной роли. См. Управление доступом к шаблонам SMS-сообщений.

Редактирование сценария

Для того чтобы отредактировать сценарий SMS, выполните следующее:

1.В форме Сценарии сообщений для должников на вкладке SMS-сообщение выберите нужную запись и нажмите кнопку или выполните двойной щелчок левой клавишей мыши по выбранной записи.

2.В форме добавления/редактирования сообщения измените необходимые атрибуты.

3.Нажмите кнопку ОК, чтобы сохранить изменения.

Удаление сценария

Для того чтобы удалить сценарий SMS, выполните следующее:

1.В форме Сценарии сообщений для должников на вкладке SMS-сообщение выберите нужную запись.

2.Нажмите кнопку .

3.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить удаление.
