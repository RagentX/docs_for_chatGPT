Обработка записей с некритическими ошибками

В процессе загрузки данных осуществляется обработка записей с некритическими ошибками:

·если константа LoadStatusBulkInsert имеет значение "Истина", будет выдан запрос: "Внимание! Имеются записи (количество записей), для которых ФИО в записи не совпадает с ФИО должника. Загружать данные по записям с несовпадающим ФИО?". При положительном ответе (кнопка Yes) для всех записей с несовпадающим ФИО производится загрузка данных, при отрицательном ответе (кнопка No) все записи с несовпадающим ФИО исключаются из загрузки;

·если константа LoadStatusBulkInsert имеет значение "Ложь", то по каждой записи с несовпадающим ФИО будет выдан запрос: "Запись №' <номер записи>, должник = <ФИО должника>, банк = <название банка>, портфель = <название портфеля>, договор = <номер договора>, л/с = <номер счета>, ID клиента = <ID в системе клиента>. В базе найден должник = <ФИО должника>, ID долга = <ID долга>, статус будет загружен для него. Подтверждаете загрузку?". При положительном ответе (кнопка Yes) для соответствующей записи производится загрузка данных, при отрицательном ответе (кнопка No) нестандартная запись исключается из загрузки. Исключенные из загрузки записи не учитываются при расчете количества успешно загруженных записей.
