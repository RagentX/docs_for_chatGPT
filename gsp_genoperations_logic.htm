Логика активности кнопок и мультифильтров в модуле ГСП

Кнопки модуля Группы свободного поиска:

Описание операций и условий их доступности

1. "Выгрузить в Excel" — доступна в том случае, если в таблице отображены данные, и есть доступ 328, и дано право в РМ "Группы должников" "Выгрузка в Excel".

2. "Зарегистрировать контакт" — доступна, если на вкладке есть ID должника и ID долга.

3. "Редактировать должника"  —  доступна, если  на вкладке есть ID должника.

4. "Конструктор писем" — доступна в том случае, если в РМ "Группы должников" было предоставлено право "Конструктор писем".

5. "Входящий звонок" — доступна в том случае, если в РМ "Группы должников" было предоставлено право "Входящий звонок".

6. "Работа с оператором" — доступна, если  активен один из подпунктов:

·"Автораспределение" — видно, если активен один из подпунктов:

·"По региону" — видно, если активен один из подпунктов:

o"Выполнить" (Автораспределение по региону. Выполнение)

o"Настройка..." (Автораспределение по региону. Настройка)

·"По продукту и просрочке" — видно, если активен один из подпунктов:

o"Выполнить" (Автораспределение по продукту и просрочке. Выполнение)

o"Выполнить на основе файла Excel" (Автораспределение по продукту и просрочке. Из файла Excel)

o"Настройка..." (Автораспределение по продукту и просрочке. Настройка)

·"По карте" - видно, если активен один из подпунктов:

o"По многоугольникам" — видно, если активен один из подпунктов:

§"Выполнить" (Автораспределение по карте. По многоугольникам. Выполнение)

§"Настройка..." (Автораспределение по карте. По многоугольникам. Настройка)

o"По расстоянию" — видно, если активен один из подпунктов:

§"Выполнить" (Автораспределение по карте. По расстоянию. Выполнение)

§"Настройка..." ( Автораспределение по карте. По расстоянию. Настройка)

Данные подпункты активны в том случае, если на вкладке есть ID долга и в РМ "Группы должников" были предоставлены соответствующие права (в скобках).

·"Назначить оператора" — виден, если  активен один из подпунктов:

·"ОДВ" — видно, если на вкладке есть ID долга, есть доступ WORK_TASK_SET, и в РМ "Группы должников" было предоставлено право "Работа с оператором. Назначить оператора ОДВ".

·"ОСВ" — видно, если на вкладке есть ID иска/приказа, есть доступ LAW_ACT_CHANGE_OPERATOR, и в РМ "Группы должников" было предоставлено право "Работа с оператором. Назначить оператора ОСВ".

·"ОИП" — видно, если на вкладке есть ID ИП, есть доступ LAW_EXEC_CHANGE_OPERATOR, и в РМ "Группы должников" было предоставлено право "Работа с оператором. Назначить оператора ОИП".

·"Освободить оператора" — виден, если  активен один из подпунктов:

·"ОДВ" — видно, если на вкладке есть ID долга, есть доступ WORK_TASK_SET, и в РМ "Группы должников" было предоставлено право "Работа с оператором. Освободить оператора ОДВ".

·"ОСВ" — видно, если на вкладке есть ID иска/приказа, есть доступ LAW_ACT_CHANGE_OPERATOR, и в РМ "Группы должников" было предоставлено право "Работа с оператором. Освободить оператора ОСВ".

·"ОИП" — видно, если на вкладке есть ID ИП, есть доступ LAW_EXEC_CHANGE_OPERATOR, и в РМ "Группы должников" было предоставлено право "Работа с оператором. Освободить оператора ОИП".

7. "Работа с долгом" — доступна всегда. Составляющие:

·"Только выделенные" — видно всегда, при клике снимается/ставится флаг по варианту работы со списком.

·"Вернуть в работу" — видно, если в РМ "Группы должников" было предоставлено право "Работа с долгом. Вернуть в работу", есть доступ DEBT_RETURN_TO_WORK (26), и на вкладке есть ID долга.

·"Вернуть Клиенту" — видно, если в РМ "Группы должников" было предоставлено право "Работа с долгом. Вернуть клиенту", есть доступ DEBT_BACK_TO_BANK (69), и на вкладке есть ID долга.

·"Перевод в проблемные" — видно, если в РМ "Группы должников" было предоставлено право "Работа с долгом. Перевод в проблемные", есть доступ DEBT_RETURN_TO_WORK (26), и на вкладке есть ID долга.

·"Закрыть" — видно, если в РМ "Группы должников" было предоставлено право "Работа с долгом. Закрыть", есть доступ DEBT_CLOSE (20), и на вкладке есть ID долга.

·"Пользовательский статус" — видно, если в РМ "Группы должников" было предоставлено право "Работа с долгом. Пользовательский статус", есть доступ № 410 DEBT_CARD_CUSTOM_STATUS, в словаре № 6 есть статусы с кодом более 100, и на вкладке есть ID долга.

·"Аутсорсинг" — видно, если в РМ "Группы должников" было предоставлено право "Работа с долгом. Аутсорсинг", и на вкладке есть ID долга.

·"Выезды" — виден, если  активен один из подпунктов:

·"Установить необходимость выезда" — видно,  если на вкладке есть ID адреса и ID долга, и в РМ "Группы должников" было предоставлено право "Выезды. Установить необходимость выезда".

·"Снять необходимость выезда" — видно,  если на вкладке есть ID адреса и ID долга, и в РМ "Группы должников" было предоставлено право "Выезды. Снять необходимость выезда".

·"Подготовить выезд" — видно,  если на вкладке есть ID адреса и ID долга, и в РМ "Группы должников" было предоставлено право "Выезды. Подготовить выезд".

8. "Перевод на стадию" — доступна, если на вкладке есть ID должника или ID долга, и есть доступ № 32 DEBT_LIST_GROUP_UPDATE.

9. Дополнительные операции – видны в том случае, если есть доступ на одну из операций:

·"Операции по долгу" — доступна, если на вкладке есть ID долга, в РМ "Группы должников" было предоставлено право "Дополнительные операции по долгу", и в РМ "Роли и доступы"  предоставлен доступ хотя бы на одну произвольную функцию по долгу.

·"Операции по должнику" — доступна, если на вкладке есть ID должника, в РМ "Группы должников" было предоставлено право "Дополнительные операции по должнику", и в РМ "Роли и доступы"  предоставлен доступ хотя бы на одну произвольную функцию по должнику.

·"Операции по имуществу" — доступна, если на вкладке есть ID имущества, в РМ "Группы должников" было предоставлено право "Дополнительные операции по имуществу", и в РМ "Роли и доступы"  предоставлен доступ хотя бы на одну произвольную функцию по имуществу.

·"Процедуры" — доступна, если в РМ "Группы должников" было предоставлено право "Дополнительные операции: процедуры", и в РМ "Роли и доступы"  предоставлен доступ хотя бы на одну процедуру.

10. "Произвольные отчеты" — доступна, если  на вкладке есть ID долга, в РМ "Группы должников" было предоставлено право "Произвольные отчеты", в РМ "Роли и доступы"  предоставлен доступ хотя бы на формирование одного  произвольного отчета.

11. "Работа с картами" — доступна в том случае, если на вкладке есть ID должника, и в РМ "Группы должников" было предоставлено право "Работа с картами".

12. "Документы по долгу" — видна, если на вкладке есть ID долга, и есть доступ на одну из операций:

·"Выгрузка документов и писем по долгу" —  активна, если  в РМ "Группы должников" было предоставлено право "Выгрузка документов и писем по долгу".

·"Печать документов и писем по долгу" — "Группы должников" было предоставлено право "Печать документов и писем по долгу".

13. "Судебная работа" — видна, если на вкладке есть ID долга и ID должника, и в РМ "Группы должников" было предоставлено право "Судебная работа".

·Создать суд. дело (предв) — видна, если константа LawActUseIndefiniteAct равна "Истина", и доступ № 58 LAW_ACT_CREATE равен TRUE.

·Создать судебные приказы — доступна, если доступ № 58 LAW_ACT_CREATE равен TRUE.

·Создать судебные иски — доступна, если доступ № 58 LAW_ACT_CREATE равен TRUE.

·Создать банкротство — доступна, если доступ № 58 LAW_ACT_CREATE равен TRUE.

·Создать правопреемство — видна, если доступ № 58 LAW_ACT_CREATE равен TRUE, и константа LawActUseSuccession равна "Истина".

·Исполнительное производство — доступно, если доступ № 167 LAW_EXEC_CREATE равен TRUE, и LAW_EXEC_CREATE_FROM_DEBT равен TRUE.

14. "Утилиты загрузки" — видна, если есть доступ хотя бы на одну из утилит:

·"Загрузка документов" — видна, если у пользователя доступ DEBT_LIST_LOAD_DOC имеет значение TRUE, и в РМ "Группы должников" предоставлено право "Массовая загрузка документов".

·Блокировка телефонов — видна, если у пользователя доступ № 639 DEBT_LIST_BLOCK_PHONE имеет значение TRUE, и в РМ "Группы должников" было предоставлено право "Массовая блокировка телефонов".

·Блокировка адресов — видна, если у пользователя доступ № 640 DEBT_LIST_BLOCK_ADDRESS имеет значение TRUE, и в РМ "Группы должников" было предоставлено право "Массовая блокировка адресов".

15. "Установить цвет" — доступна, если на вкладке есть ID долга, и в РМ "Группы должников" было предоставлено право "Установка цвета".

 

Для того чтобы такие поля, как ID адреса и ID долга, были определены в запросе, необходимо, чтобы их название соответствовало перечисленным.

Так же определение полей в запросе участвует в отображении мультифильтров.

Если поле, соответствующее мультифильтру отсутствует, то мультифильтр не отобразится даже при наличии соответствующего доступа из модуля Группы должников.

Как это работает:

1.В  структуре запроса присутствуют поля, которые соответствуют системному названию.

2.Если по первому столбцу ничего не найдено, то ищем в описании полей (Пользовательское имя).
