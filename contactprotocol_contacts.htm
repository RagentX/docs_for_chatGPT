Просмотр данных о контактах с должниками

Show/Hide Hidden Text

На вкладке Контакты отображаются данные о контактах с должниками, зарегистрированных через дерево контактов или загруженных в привязке к нему с помощью загрузки истории работы.

При открытии формы списки на вкладках отображаются пустыми. Нажмите кнопку Найти, чтобы вывести в список все записи с учетом значений в фильтрах, заданных по умолчанию, или сначала настройте условия фильтра для отбора записей (см. Мультифильтры, Сложный фильтр).

Ограничения на вывод данных в списках

Используются следующие ограничения на отображение данных пользователю:

·Отображение контактов по долгам, закрепленным только за пользователем и его подчиненными (при их наличии) или отображение по всем долгам — регулируется положением пользователя в Структуре предприятия. См. Ограничения доступа к данным на основании структуры предприятия.

·Отображение контактов по долгам, обладающим определенными атрибутами (долги в определенных стадиях, принадлежащие определенным банкам и др.) для пользователя настраивается в модуле Настройка списка должников. См. Применение ограничений доступа к долгам.

Примечание.

·Если в роли пользователя доступ № 637 CONTACT_BLOCK_PHONE_VIEW  имеет значение TRUE, то в списках контактов и комментариев отображаются данные о контактах по заблокированным телефонам.

·Если в роли пользователя доступ № 638 CONTACT_BLOCK_ADDRESS_VIEW имеет значение TRUE, то в списках контактов и комментариев отображаются данные о контактах по заблокированным адресам.

       Описание полей списка

Информация о контактах с должниками отображается в следующих полях списка:

·ID долга;

·ID должника;

·Должник — ФИО должника или наименование организации;

·Обещ. сумма — обещанная сумма оплаты долга (отображается также и для неверифицированных обещаний);

·Тип должника — см. словарь № 54;

·Модуль регистрации — указывается одно из значений:

·Регистрация контакта — для контактов, зарегистрированных при помощи модуля "Регистрация контакта";

·Android — для контактов, зарегистрированных при помощи мобильной версии приложения.

·Название продукта — указывается название продукта, по которому возник долг;

·Банк — краткое название банка;

·Портфель — портфель, в который входит долг;

·Дата — фактическая дата регистрации контакта;

·Оператор — оператор, зарегистрировавший контакт;

·Отдел — подразделение, котором работает оператор (см. Структура предприятия);

·Тип — тип контакта (см. словарь № 3);

·Конт. — признак того, что контакт установлен;

·Результат — результат контакта (см. Настройка дерева контактов);

·Обещ. дата — обещанная дата (отображается также и для неверифицированных обещаний);

·Направление — номер телефона, по которому был установлен контакт;

·Тип направления — тип телефона;

·Код качества телефона; — см. словарь № 114;

·Комментарий — дополнительная информация о контакте;

·Валюта — валюта долга (см. Список валют);

·Остаток — остаток суммы долга на текущую дату, выводится в валюте долга, указанной в поле Валюта;

·Статус долга — см. словарь № 6;

·Договор — номер договора с должником из системы/базы данных клиента;

·Лицевой счет — лицевой счет должника из системы/базы данных клиента;

·ID Клиента;

·Часовой пояс — часовой пояс должника;

·Важность;

·Порядочность;

·Флаг Поручитель — включен, если контакт был зарегистрирован из каротчки поручителя;

·Флаг Залогодатель — включен, если контакт был зарегистрирован из карточки залогодателя;

·Флаг Запись — отмечает наличие записи разговора, при наличии соответствующих полномочий пользователь может прослушать запись разговора;

·ID Разговора — идентификатор данного контакта в Системе;

·Балл № 1 — см. словарь № 101;

·Балл № 2 — см. словарь № 102;

·Балл № 3 — см. словарь № 244;

·Балл № 4 — см. словарь № 245;

·Дата регистр. конт. — если у пользователя включен доступ CONTACT_REG_DATE_ALL, то отображается дата, которая была назначена для контакта при его регистрации; если доступ отсутствует или выключен, то указывается фактическая дата регистрации контакта;

·Гос. пошлина — сумма государственной пошлины;

·Остаток основного долга — сумма остатка основного долга, выводится в валюте долга, указанной в поле Валюта;

·Причина звонка должника —  см. словарь № 118;

·Филиал — если организация, использующая систему, имеет филиальную структуру, в данном поле указывается филиал, который должен работать с этим долгом. Значение выбирается из словаря № 61;

·Регион — регион, в котором выдан кредит, выбирается из словаря № 39;

·Реестр КА — исходящий портфель долга;

·Длительность — длительность разговора;

·Call-лист — название колл-листа автодайлера, в рамках которого был зарегистрирован контакт; функционал включается по согласованию с технической поддержкой (Техническая константа RegContact.CallList.Save).

·ID намерения, Дата намерения, Сумма намерения — данные о намерении; функционал включается по согласованию с технической поддержкой (Техническая константа ConfirmIntent.Use).
