Проверка данных

Для проверки данных в таблице нажмите кнопку Проверить в форме Утилита изменения статуса долга. Во время поиска долгов кнопки Проверить, Сохранить в БД и Отменить, а также возможность редактирования ячеек таблицы становятся недоступны. В процессе поиска долгов в нижней строке формы будет отображаться тип операции (поиск долгов), текущие данные о проценте обработанных данных, директория и имя файла.

В процессе проверки данных в нижней строке формы будет отображаться тип операции (проверка данных), текущие данные о проценте выполнения работы, директория и имя файла.

Проверка данных включает:

·контроль наличия обязательных реквизитов должника (банка, статуса);

·контроль наличия хотя бы одного реквизита из группы (ФИО, номера договора, номера счета или ID в системе клиента);

·контроль корректности значения статуса (статус долга должен совпадать с одним из допустимых значений: 3, 4, 5, 6, 8, 9, 10, 11, 12 или > 100);

·контроль наличия заданного кода причины в соответствующем словаре;

·контроль наличия банка с заданным названием в базе данных;

·контроль наличия портфеля с заданным названием в базе данных;

·контроль соответствия найденного портфеля найденному банку;

·контроль наличия в базе данных должника по ключевым реквизитам "Номер договора", "Номер счета", "ID в системе клиента" с учетом ФИО должника, банка и портфеля;

·контроль соответствия заданного ФИО и ФИО для найденного долга;

·контроль соответствия заданного банка и банка для найденного долга;

·контроль соответствия заданного портфеля и портфеля для найденного долга.

Текстовые реквизиты каждой строки автоматически обрезаются до максимально допустимой длины:

·64 символа — для банка и портфеля;

·322 символа — для ФИО;

·128 символов — для номера договора;

·32 символа — для номера счета и ID в системе клиента;

·512 символов — для комментария.

Некритической ошибкой является несоответствие заданного ФИО и ФИО для найденного долга. Все остальные ошибки считаются критическими. Ячейки таблицы, по которым зафиксированы ошибки, окрашиваются.

Если ошибки отсутствуют, по окончании проверки данных будет выдано сообщение "Данные полностью соответствуют формату загрузки".

Если все записи содержат критические ошибки, будет выдано сообщение: "Загрузка невозможна, ни одна запись не соответствует формату загрузки". Если имеются и корректные, и ошибочные записи, будет выдано сообщение: "Можно загрузить записей <количество корректных записей>, номер первой ошибочной записи <номер записи>".

Кнопки Проверить, Сохранить в БД и Отменить становятся доступными, восстанавливается возможность ручного редактирования ячеек таблицы.

Повторная проверка данных производится после их ручного редактирования. При повторной проверке проверяются только данные, измененные пользователем. Если изменения отсутствуют, будет выдано сообщение: "Данные в ячейках таблицы не изменялись, повторная проверка отменена". При наличии данных, измененных вручную, производится их повторная проверка так же, как описано выше.
