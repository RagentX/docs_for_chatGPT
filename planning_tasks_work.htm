Рабочая вкладка "Задачи"

Show/Hide Hidden Text

Вкладка, из которой осуществляется работа с функционалом, включает следующие элементы:

·Фильтр Все записи по должнику/Записи только по текущему долгу/Записи только по текущему делу;

·Флаг Показывать завершенные задачи — см. Работа флага "Показывать завершенные задачи";

·Список задач;

·Описание выбранной задачи;

·Результат выбранной задачи;

·Кнопки смены статуса задачи;

·Кнопка просмотра лога по текущей задаче.

На вкладке доступна ручная постановка задачи, изменение статуса и атрибутов задачи, а также просмотр задачи двойному клику на соответствующей записи.

Доступна из следующих модулей:

·Карточка должника;

·Карточка судебной работы;

·Карточка исполнительного производства;

·Вкладка "Имущество", подчиненная вкладка Задачи — для отображения задач по имуществу.

Примечание. Для задач, поставленных по делам судебной работы или ИП, в поле ФИО отображается ФИО ответчика/истца по делу: должника, поручителя или связанного лица.

Особенности отображения задач разных стадий взыскания

Отображение задач любых стадий взыскания для каждого из указанных модулей может быть настроено вендором (для этого обратитесь в техподдержку).См. Поддержка константы Task.TypeTaskInList.Mode.

Обычно задачи отображаются следующим образом:

·В карточке должника на вкладке "Задачи" отображаются только задачи досудебной стадии взыскания.

·В карточке судебной работы на вкладке "Задачи" отображаются задачи:

·судебной стадии взыскания;

·досудебной стадии взыскания.

·В карточке исполнительного производства отображаются задачи:

·исполнительной стадии взыскания;

·судебной стадии взыскания;

·досудебной стадии взыскания.

Особенности отображения задач по имуществу

Отображение задач по имуществу для каждого из следующих модулей может быть настроено вендором (для этого обратитесь в техподдержку) (См. Поддержка константы Task.TypeTaskInList.Mode):

·Карточка должника;

·Карточка судебной работы;

·Карточка исполнительного производства;

Если для модуля настроено отображение задач по имуществу, то на вкладке "Имущество" есть подчиненная вкладка "Задачи", на которой отображаются только задачи по имуществу.

Фильтрация записей по умолчанию

Фильтрация записей по умолчанию настраивается константой DebtTask.TaskListFilter.Default для следующих модулей Системы:

·Карточка должника:

·Если в константе DebtTask.TaskListFilter.Default есть значение = «9:1», то значением фильтрации записей по умолчанию будет «Все записи по должнику».

·Если в константе DebtTask.TaskListFilter.Default есть значение = «9:2», то значением фильтрации записей по умолчанию будет «Записи только по текущему долгу».

·Карточка судебной работы:

·Если в константе DebtTask.TaskListFilter.Default есть значение = «24:1», то значением фильтрации записей по умолчанию будет «Все записи по должнику».

·Если в константе DebtTask.TaskListFilter.Default есть значение = «24:2», то значением фильтрации записей по умолчанию будет «Записи только по текущему делу».

·Если в константе DebtTask.TaskListFilter.Default есть значение = «24:3», то значением фильтрации записей по умолчанию будет «Записи только по текущему долгу».

·Карточка исполнительного производства:

·Если в константе DebtTask.TaskListFilter.Default есть значение = «55:1», то значением фильтрации записей по умолчанию будет «Все записи по должнику».

·Если в константе DebtTask.TaskListFilter.Default есть значение = «55:2», то значением фильтрации записей по умолчанию будет «Записи только по текущему делу».

·Если в константе DebtTask.TaskListFilter.Default есть значение = «55:3», то значением фильтрации записей по умолчанию будет «Записи только по текущему долгу».

       Список полей на вкладке "Задачи"

·ID Задачи — уникальный идентификатор задачи;

·ID должника;

·ID долга;

·ID дела — идентификатор дела, для задач, созданным по делам судебной работы или ИП;

·ФИО — фамилия, имя, отчество персоны;

Примечание. Для задач, поставленных по делам судебной работы или ИП, в поле ФИО отображается ФИО ответчика/истца по делу (должника или поручителя).

·Номер договора;

·Портфель;

·Наименование задачи — поле, заполненное при настройке задачи;

·Тип задачи — поле, заполненное при настройке задачи, см. словарь № 229

·Подтип задачи — поле, заполненное при настройке задачи, см. словарь № 230

·Статус — текущий статус задачи;

·Причина перехода в текущий статус — только для тех статусов задачи, где это необходимо;

·Приоритет — поле, заполненное при настройке задачи

·Дата постановки — дата постановки задачи;

·Дата факт. выполнения — дата выполнения задачи;

·Крайний срок — срок выполнения задачи;

·Постановщик — оператор, поставивший задачу (поле заполнено только для задач, поставленных вручную);

·Исполнитель — оператор, назначенный исполнителем задачи;

·Выполнивший задачу — оператор, выполнивший задачу;

·Источник — название модуля, из которого была поставлена задача, только для задач, поставленных в автоматическом режиме;

·Тип — тип объекта, по которому создана задача, поле может не отображаться в модулях, если в них настроено отображение только задач по долгу.
