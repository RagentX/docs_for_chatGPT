Добавление долга

Информация по долгам может вводиться в систему:

·путем загрузки данных из файлов установленного формата (см. Загрузка данных из внешних источников);

·автоматизированно с помощью ETL и ночных процедур;

·вручную в карточке должника.

Данные о долгах находятся в карточке должника на вкладке Долги. Для того чтобы добавить информацию о долге вручную, выполните следующие действия:

1.Нажмите кнопку , относящуюся к списку долгов. Откроется форма "Информация о долге".

2.В форме введите необходимые данные о долге.

3.Нажмите кнопку ОК, чтобы сохранить данные.

Примечание. При добавлении долга осуществляется назначение его статуса по следующему правилу:

·если debt.debt_sum <= 0, то долгу присваивается статус 6 "Завершено";

·если debt.debt_sum > 0, то долгу присваивается статус 1 "Не распределен".

Примечание. Для отображения дополнительных данных по долгу может быть настроена вкладка "Дополнительно".

Форма добавления долга

Описание формы добавления долга

Форма добавление долга содержит следующие поля:

·Портфель — портфель, в который входит долг. Поле обязательно для заполнения. При этом автоматически будут заполнены поля Название банка, Телефон банка, Дата передачи, Дата окончания, Коэффициент;

·Балл № 1 — значение выбирается из словаря № 101, см. Скоринговые баллы;

·Балл № 2 — значение выбирается из словаря № 102, см. Скоринговые баллы;

·№ — идентификационный номер долга, присваивается автоматически при сохранении информации о долге (в дальнейшем фигурирует как ID долга);

·Название продукта — указывается название продукта, по которому возник долг; поле обязательно для заполнения;

·Тип продукта — см. словарь № 11;

·Лицевой счет — лицевой счет должника из системы/базы данных клиента;

·Договор — номер договора с должником из системы/базы данных клиента;

·ID Клиента — идентификатор должника из системы/базы данных клиента (при наличии);

Примечание. Атрибуты Лицевой счет, Договор, ID клиента могут использоваться в качестве идентификаторов при загрузке платежей и остатков (см. Загрузка платежей).

·Дата выдачи — дата выдачи кредита;

·Город — город, в котором зарегистрирован договор;

·Регион — регион, в котором выдан кредит, выбирается из словаря № 39;

·Филиал — если организация, использующая систему, имеет филиальную структуру, в данном поле указывается филиал, который должен работать с этим долгом. Значение выбирается из словаря № 61;

·Дата выхода на просрочку — дата, с которой долг считается просроченным. Поле обязательно для заполнения;

·Валюта — валюта долга (см. Список валют);

·Размер кредита — сумма кредита без учета процентов, комиссий и т.п., выводится в валюте долга, указанной в поле Валюта;

·Проц. ставка — процентная ставка по кредиту;

·Ост. осн. долга — сумма остатка основного долга, выводится в валюте долга, указанной в поле Валюта;

·Причина возникновения долга — значение выбирается из словаря № 13;

·Остаток — остаток суммы долга на текущую дату. Поле обязательно для заполнения;

Примечание. При загрузке/добавлении долга сумма в поле Остаток обычно равна сумме в поле Начальный долг (если специально не настроено иначе). При регистрации платежей сумма остатка долга уменьшается, а сумма начального долга не изменяется. Это позволяет формировать отчеты по эффективности сборов.

·Переплата — сумма переплаты по долгу;

Пример. Если сумма долга составляла 9887,12 руб., а должник внес платеж на 10000 руб., то остаток долга станет равным 0, а значение 12,88 руб. появится в поле Переплата.

·Дата расчета — дата, на которую рассчитано значение в поле Остаток;

·Дата акт. — дата актуальности. Банковский атрибут, который показывает, до какой даты является актуальным остаток долга, рассчитанный на определенную дату;

Пример. Например, остатки, рассчитанные банком на 01.06.2011, могут быть актуальны до 20.06. Данная информация может быть использована организацией для работы с должником.

·Вознаграждение % — поле, которое организация может использовать для хранения информации о сумме своего вознаграждения с данного долга. Возможность просмотра данного поля регулируется доступом AGENCY_RATE_VIEW, редактирование — доступом AGENCY_RATE_EDIT. Значение может вводиться вручную или загружаться с помощью загрузки данных (формат загрузки данных должен содержать поле Комиссия из таблицы долгов);

·Сумма платежей в руб. — сумма подтвержденных платежей по долгу. Поле не редактируется;

·Начальный долг — начальная сумма долга в выбранной валюте на момент передачи его в организацию;

·Мин. сумма платежа — минимальная сумма платежа, банковский атрибут, информационное поле;

·Крайний срок — дата крайнего срока погашения долга по кредиту, информационное поле;

·Штрафные санкции — описание штрафных санкций, информационное поле;

·Составляющие долга:

·Основной долг — сумма основного долга. Поле обязательно для заполнения;

·Проср. осн. долг — сумма просроченного основного долга;

·Проценты — сумма процентов по кредиту;

·Проср. проценты — сумма просроченных процентов по кредиту;

·Штрафы — сумма штрафов по кредиту;

·Пени — сумма пени;

·Комиссия — сумма комиссии;

·Проср. комиссия — сумма просроченной комиссии по кредиту;

·Гос. пошлина — сумма государственной пошлины;

·Акция — информация об акциях. Используется для информирования коллекторов о том, что по данному долгу есть возможность предложить должнику какие-либо льготные условия (например, частичное списание долга при условии оплаты до определенной даты). Информационное поле;

·Сумма на счете — остаток на банковском счете должника;

·Аннуитетный платеж — сумма ежемесячного платежа по кредиту, если размер ежемесячного платежа остаётся постоянным на всём периоде кредитования;

·Балл № 3 — значение выбирается из словаря № 244, см. Скоринговые баллы;

·Балл № 4 — значение выбирается из словаря № 245, см. Скоринговые баллы;

·Комментарий — дополнительная информация;

·Отчет коллектора — комментарий по работе с долгом. Текст комментария можно ввести вручную или нажать кнопку и в открывшейся форме выбрать комментарий в списке (см. словарь № 108);

·Комментарий по реструктуризации — дополнительная информация по реструктуризации долга.
