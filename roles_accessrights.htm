Управление основными доступами системы

Show/Hide Hidden Text

Для каждой роли должен быть определен набор доступов к объектам и функциям модулей системы. Такие доступы мы будем называть основными доступами системы. При назначении пользователю той или иной роли он сможет выполнять только те действия в системе, которые включены в список доступов назначенной ему роли.

Основные доступы могут иметь один из следующих типов:

·Логический — значение такого доступа может быть TRUE или FALSE;

·Число — значение такого доступа может быть целым числом;

·Строка — значение такого доступа может быть строкой текста.

См. также — Список основных доступов.

Если в списке доступов или в форме выбра доступов (форма "Выберите доступ", см. далее) вам нужно найти записи, удовлетворяющие определенным параметрам, то см. Поиск в списке доступов.

Логика работы доступов

·Если доступ добавлен в роль, то он работает согласно своему значению.

·Если доступ не добавлен в роль, то разные типы доступов работают по-разному:

·Логический — доступ работает так, как если бы он был добавлен со значением FALSE;

·Числовой — доступ работает так, как если бы он был добавлен с нулевым значением;

·Строковый — доступ работает так, как если бы он был добавлен с пустой строкой.

Форма выбора доступов

Поддержана мультиязычность

Форма включает следующие поля:

·ID — номер доступа (по умолчанию поле скрыто);

·Название — наименование доступа;

·Рабочее место — модуль, в котором работает доступ;

·Тип — тип доступа;

·Описание — описание доступа;

·Комментарий — дополнительная информация.

·Значение по умолчанию для доступов числового типа;

·Значение по умолчанию для доступов строкового типа.

Если поддерживается мультиязычность, в форме присутствует также поле:

·Описание на альтернативном языке.

Добавление основных доступов системы

1.В форме Роли и доступы выберите нужную роль. При этом на вкладке Доступы будут отображаться доступы, уже назначенные данной роли.

2.На вкладке Доступы нажмите кнопку . Откроется форма со списком доступов, которые еще не назначены данной роли.

3.В форме выберите записи, содержащие нужные доступы. Для выбора нескольких записей используйте клавиши SHIFT и/или CTRL.

4.Нажмите кнопку ОК. Выбранные доступы будут добавлены в список доступов, назначенных роли.

Редактирование основного доступа системы

Система позволяет запретить доступ к объекту или функции модуля системы, не удаляя его из списка доступов для роли. Для этого выполните следующие действия:

1.В форме Роли и доступы выберите нужную роль.

2.На вкладке Доступы выберите нужный доступ в списке.

3.Нажмите кнопку или выполните клик (двойной щелчок левой клавишей мыши) на выбранном доступе. Откроется форма редактирования доступа.

Поддержка мультиязычности

4.В форме измените значения следующих атрибутов:

·Значение — атрибут показывает, действует (TRUE) или не действует (FALSE) доступ для роли, для числовых или строковых доступов вводятся соответствующие значения;

·Комментарий — дополнительная информация о доступе. Комментарий относится к конкретной связке роль — доступ. Включая тот же доступ в другую роль, можно вводить различные комментарии.

Поля, недоступные для изменения:

·Название;

·Описание.

При поддержке мультиязычности есть также поле, недоступное для редактирования:

·Описание на альтернативном языке.

5.Нажмите кнопку ОК, чтобы сохранить изменения.

Удаление основных доступов системы

Примечание. Чтобы избежать удаления доступа, вы можете изменить значение логического доступа на FALSE в режиме редактирования основных доступов.

Для того чтобы удалить доступы у роли, выполните следующие действия:

1.В форме Роли и доступы выберите нужную роль.

2.На вкладке Доступы выберите нужные доступы в списке. Для выбора нескольких записей используйте клавиши SHIFT и/или CTRL.

3.Нажмите кнопку .

4.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить удаление. Доступы будут удалены из списка.
