Email-сообщение

Сценарии сообщений электронной почты используются в следующих случаях:

·формирование сообщений электронной почты из модуля "Карточка должника" (см. Формирование сообщения электронной почты);

·массовая отправка сообщений электронной почты из модуля "Исходящее информирование" (см. Формирование и отправка сообщений электронной почты).

Вкладка "Email-сообщение"

Описание вкладки "Email-сообщение"

·список сценариев сообщений электронной почты, содержащий следующие поля:

·ID — идентификатор сценария, формируется автоматически;

·Номер — номер сценария;

·Ед. рассылка — признак того, что у оператора есть возможность создать заявку на отправку сообщения электронной почты из модуля "Карточка должника";

·Название — название сценария;

·Текст — текст сценария;

·кнопки для добавления, редактирования, удаления сценариев.

Форма добавления/редактирования сценариев сообщений электронной почты

Если включен функционал HTML-форматирования сообщений Email, то сообщения могут иметь либо текстовый, либо HTML-формат (согласно системному словарю № 303).

Примечание. Включение функционала осуществляется вендором. Самостоятельное включение функционала приведет к ошибкам. Для включения поддержки HTML-форматирования Email-сообщений обратитесь в техническую поддержку.

Для включения функционала следует включить техническую константу Email.HTMLFormat.Use и выполнения соответствующего скрипта (патч 3223 ms, 3224 oracle, см. ТЗ к задаче 20372 Организатора). Кроме того нужно завести словарь № 303 и наполнить его значениями.

Описание формы для добавления/редактирования сообщения электронной почты

·Название — название сценария. Поле обязательно для заполнения;

·Номер — номер сценария, который может использоваться для привязки сценария к узлу дерева контактов (см. Добавление результата контактов);

·Текст — текст сценария. Поле может содержать как статичный текст, так и переменные, вместо которых при выполнении сценария будут подставлены конкретные значения (см. Подстановка переменных в текст сообщения);

·Единичная рассылка — установленный флаг означает, что оператор сможет создать заявку на отправку сообщения электронной почты с указанными параметрами непосредственно из модуля "Карточка должника".

·Отправлять администратору — если флаг включен, то копия сообщения направляется по адресу, заданному константой EmailAdminDepartment;

·Учитывать в ограничениях — если флаг включен, то сообщение, отправленное по данному шаблону, будет учтено в атрибуте "Количество контактов, учитываемых в ограничении". Данное поле видно в том случае, если константа CommunicationLimit.ContactTypeList содержит значение 8.

Действия со сценариями сообщений электронной почты

Создание сценария сообщения электронной почты

Для того чтобы создать сценарий сообщения электронной почты, выполните следующее:

1.В форме Сценарии сообщений для должников на вкладке Email-сообщения нажмите кнопку .

2.В открывшейся форме заполните нужные поля.

3.Нажмите кнопку ОК. Сценарий будет добавлен в список.

4.После создания сценария Email-сообщения рекомендуется настроить доступ к нему пользователей. Для этого в модуле Роли и доступы выберите роль, а затем  на вкладке Email-сообщения добавьте созданный сценарий с уровнем доступа "Формирование" данной роли. См. Управление доступом к шаблонам Email-сообщений.

Редактирование сценария сообщения электронной почты

Для того чтобы отредактировать сценарий сообщения электронной почты, выполните следующее:

1.В форме Сценарии сообщений для должников на вкладке Email-сообщения выберите нужную запись и нажмите кнопку или выполните двойной щелчок левой клавишей мыши по выбранной записи.

2.В форме добавления/редактирования сообщения измените необходимые атрибуты.

3.Нажмите кнопку ОК, чтобы сохранить изменения.

Удаление сценария сообщения электронной почты

Для того чтобы удалить сценарий сообщения электронной почты, выполните следующее:

1.В форме Сценарии сообщений для должников на вкладке Email-сообщения выберите нужную запись.

2.Нажмите кнопку .

3.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить удаление.
