Просмотр данных о платежах

Доступ для просмотра всех платежей в форме Платежи имеют только те пользователи, в роли которых доступ PAYMENTS_SHOW_ALL установлен в значение TRUE. Просматривать платежи закрытых долгов могут только те пользователи, в роли которых доступ PAYMENTS_SHOW_CLOSE установлен в значение TRUE.

Используются следующие ограничения на отображение долгов пользователю:

·Отображение долгов, закрепленных только за пользователем и его подчиненными (при их наличии) или отображение всех долгов — регулируется положением пользователя в Структуре предприятия. См. Ограничения доступа к данным на основании структуры предприятия.

·Ограничение доступа к платежам с помощью модуля Настройка списка должников регулируется вендором. Для применения данного ограничения обратитесь в техничекую поддержку.

При открытии формы списки на вкладках отображаются пустыми. Нажмите кнопку Найти, чтобы вывести в список все записи с учетом значений в фильтрах, заданных по умолчанию, или сначала настройте условия фильтра для отбора записей (см. Мультифильтры, Сложный фильтр).

Информация о платежах отображается в следующих колонках:

·Дата внесения — дата и время внесения платежа; выводится количество записей в выборке;

·Сумма платежа — сумма, поступившая в счет погашения задолженности;

·Валюта — валюта платежа (см. Список валют);

·Сумма в рублях — для иностранных валют вычисляется по курсу на дату регистрации платежа в банке; выводится сумма значений по выборке;

·Комиссия в рублях; выводится сумма значений по выборке;

·Статус платежа — статус может принимать следующие значения (см. словарь № 20):

·Не подтвержден — формируется автоматически при регистрации контакта с результатом "Должник утверждает, что уже оплатил". Также данный статус может быть сформирован вручную при регистрации платежа оператором;

·Подтвержден — устанавливается при выполнении операции подтверждение платежа. Также подтвержденный платеж может быть сформирован вручную при регистрации платежа оператором или автоматически при загрузке платежей;

·Отменен — устанавливается при выполнении операции отмены платежей.

·Дата регистрации;

·Коллектор (факт.) — оператор, который был ответственным (закреплен) за долг на дату платежа без учета времени;

·Коллектор (заявка) — если коллектор считает, что платеж должен быть закреплен за ним, то он может сделать заявку на этот платеж в модуле "Карточка должника" (отредактировать поле Коллектор). Далее сотрудник, который отвечает за распределение платежей по коллекторам, может принять или отклонить заявку коллектора;

·Дата учета информации — дата, которая будет учитываться в отчетах;

·Дата подтверждения платежа;

·Банковские реквизиты;

·Расчетный счет;

·Квитанция — признак наличия доказательства платежа (например, скан или ксерокопия платежного поручения или квитанция);

·% вознаграждения — величина процента вознаграждения по долгу;

·Размер вознаграждения —  сумма вознаграждения за подтвержденный платеж (зависит от процента вознаграждения по долгу, отображается при наличии доступа PAYMENTS_VIEW_FEE;

·ФИО плательщика;

·Номер договора к оплате;

·Номер квитанции — номер документа, доказывающего факт платежа (например, скан или ксерокопия платежного поручения или квитанции);

·Назначение платежа —  значение в соответствии со словарем № 130;

·Торговая точка — место выдачи кредита (адрес торговой точки).
