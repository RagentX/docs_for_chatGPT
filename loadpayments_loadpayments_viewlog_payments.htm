Данные в логе о загрузке платежей

Данные в файле лога о загрузке платежей формируются в следующей последовательности:

1.Данные о подтверждении ранее не подтвержденных платежей.

2.Данные о вводе новых платежей.

3.Данные об обновлении обещаний для новых и вновь подтвержденных платежей.

4.Данные об отмене намеченных выездов.

5.Данные о работе по правилу (для каждого правила, по которому имеется хотя бы одна запись с пустым остатком).

При подтверждении платежа в соответствующей записи таблицы БД debt_calc корректируются поля is_confirmed, confirm_dt, report_date. Перед подтверждением ранее не подтвержденных платежей в файле лога формируется сообщение: "SaveCalc. Подтверждение неподтвержденных платежей, количество корректируемых записей в debt_calc <количество записей>". Если константа LoadCalcBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveCalc. Дополнительные данные о подтверждении платежей:";

·сообщения по каждому вновь подтвержденному платежу вида "запись №<номер записи>, id debt_calc = <ID платежа>".

При вводе нового платежа в новой записи таблицы БД debt_calc формируются поля parent_id, sum, dt, is_confirmed, is_cancel, calc_date, r_currency_id, int_sum, report_date, dsc, confirm_dt. Перед вводом новых платежей в файле лога формируется сообщение: "SaveCalc. Ввод новых платежей, количество новых записей в debt_calc <количество записей>". Если константа LoadCalcBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveCalc. Дополнительные данные о новых платежах:";

·сообщения по каждому новому платежу вида "запись №<номер записи>, id debt_calc = <ID платежа>".

При обновлении обещаний для новых и вновь подтвержденных платежей в таблице БД debt_promise корректируется поле cover_sum, в таблице debt_calc корректируется поле r_promise_id, а в таблице debt корректируются поля state, prev_state, status. После обновления обещаний в файле лога формируется сообщение: "SaveCalc. Обновление обещаний для новых и вновь подтвержденных платежей (<количество обещаний>)". Если константа LoadCalcBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveCalc. Дополнительные данные об обновленных обещаниях:";

·сообщения по каждому обновленному обещанию вида "запись №<номер записи>, id debt_calc = <ID платежа>, id debt_promise = <ID обещания>".

При отмене намеченных выездов в таблице БД debt_promise корректируется поле cover_sum, в таблице visit_address корректируются поля check_dt, prepare_dt, r_user_id, r_admin_id, purpose, visit_dt, status, а в таблице address корректируется поле visit_state. Перед отменой намеченных выездов в файле лога формируется сообщение: "SaveCalc. Отмена намеченных выездов (<количество намеченных выездов>)". Если константа LoadCalcBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveCalc. Дополнительные данные об отменяемых выездах:";

·сообщения по каждому отменяемому выезду вида "запись №<номер записи>, id debt_calc = <ID платежа>, id address = <ID адреса>, id visit_address = <ID визита>".

При работе по правилу в таблице БД debt корректируются поля r_calc_id, status, dsc, debt_sum, debt_sum_extra, debt_exp_dt, а также соответствующая правилу выборка из полей basic_sum, exp_basic_sum, percent_sum, exp_percent_sum, penalty_sum, peni_sum, commission_sum, exp_commission_sum, due_sum. В таблице work_task корректируются поля r_user_id = null, prior = null, call_time = null, r_contact_id, r_admin_id, reason, finish_work_date. После работы по правилу в файле лога формируется сообщение: "SaveCalc. Работа по правилу <номер правила>, количество корректируемых записей <количество записей>". Если константа LoadCalcBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveCalc. Дополнительные данные о работе по правилу номер:";

·сообщения по каждой скорректированной записи вида "запись №<номер записи>, id debt = <ID долга>, id address = <ID адреса>, id work_task = <id work_task>".
