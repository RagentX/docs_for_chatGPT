Формирование SMS

Show/Hide Hidden Text

Сформировать SMS на контактный телефон должника можно для всех долгов, кроме тех, которые находятся в статусах 6 "Завершено", 7 "Аннулировано", 8 "Отозван клиентом". Возможность отправлять SMS из модуля "Карточка должника" определяется доступом DEBT_CARD_SMS. Количество SMS, которое оператор может зарегистрировать в день, определяется константой CollectorMaxDaySms.

       Учет ограничений коммуникации

Проверка наличия схемы коммуникации

1.Проверяется значение константы CommunicationLimit.ContactTypeList:

·Если константа содержит код 7 (см. словарь № 3), то для коммуникации типа "SMS" проверка наличия схемы коммуникации продолжается;

·Если данный код в константе отсутствует, то проверка не осуществляется, и коммуникация разрешается.

2.Проверяется в шаблоне сообщения флаг Учитывать в ограничениях (для звонков и выездов — см. Форма настройки исхода контакта в модуле Настройка дерева контактов; для SMS — см. SMS-сообщение; для email — см. Email-сообщение в модуле Сценарии сообщений):

·Если флаг включен, то проверка продолжается;

·Если флаг выключен, то проверка завершается, и коммуникация разрешается.

3.Проверяются Персональные ограничения коммуникации, настроенные для данного долга в Карточка должника.

·Если персональные ограничения настроены, то для данного типа контакта проверяется наличие Схемы коммуникации и соответствие даты совершения коммуникации ее периоду. Если ограничения настроены корректно, то осуществляется проверка лимита коммуникации по данной схеме (см. далее).

·Если персональные ограничения не настроены, или настроены некорректно или период не включает текущую дату, то проверка движется дальше.

4.Проверяются общие ограничения, налагаемые по формуле, указанной в значении константы, соответствующей данному типу коммуникации.

·Если в формуле указан существующий идентификатор Схемы коммуникации и дата совершения коммуникации входит в ее период, то осуществляется проверка лимита коммуникации по данной схеме (см. далее).

·Если общие ограничения не настроены, настроены некорректно или не включают текущую дату, коммуникация разрешается.

       Список констант для общих ограничений коммуникации

Для каждого типа коммуникации соответствующая константа содержит идентификатор формулы. по которой рассчитывается ограничение коммуникации в зависимости от параметров долга, участника договора, его роли и др. Результатом работы формулы является число — идентификатор той схемы коммуникации, которая должна быть использована в данном случае.

·Для звонка (набор номера) — CommunicationLimit.Call.Formula;

·Для SMS — CommunicationLimit.SMS.Formula;

·Для Email — CommunicationLimit.Email.Formula;

·Для выезда — CommunicationLimit.Visit.Formula;

·Для автоинформатора — CommunicationLimit.AutoInfo.Formula.

Проверка лимита коммуникации по схеме коммуникации

Процедура проверки лимита коммуникации осуществляет проверку возможности совершения коммуникации согласно текущей статистике для конкретного участника договора.

Если коммуникация разрешена, то она осуществляется. Если коммуникация превышает разрешенный лимит, то пользователю показывается соответствующее сообщение, и коммуникация не осуществляется.

Примечание. Для контактов типа SMS и Email отправка сообщения со свободным текстом не подвергается ограничениям.

       Возможность осуществления коммуникации при превышении лимита

Существует ряд доступов, дающих пользователю возможность осуществить коммуникацию сверх лимита.

Если в роли пользователя для данного типа коммуникации такой доступ включен, то при превышении лимита ему выдается соответствующее сообщение:

Формирование сообщения превысит допустимый лимит коммуникации. Вы действительно хотите <название действия для данного типа коммуникации>?

Выберите:

·Yes — если вы хотите совершить коммуникацию сверх лимита;

·No — если вы не хотите совершить коммуникацию сверх лимита.

Доступы на превышение лимитов коммуникации разных типов

·Доступ № 633 COMMUNICATION_SMS_EXCEED_LIMIT — дает право на формирование SMS – сообщений при превышении лимита согласно схеме коммуникации;

·Доступ № 634 COMMUNICATION_EMAIL_EXCEED_LIMIT — дает право на формирование EMAIL – сообщений при превышении лимита согласно схеме коммуникации;

·Доступ № 635 COMMUNICATION_CALL_EXCEED_LIMIT — дает право на совершение исходящего вызова при превышении лимита коммуникации;

·Доступ № 636 COMMUNICATION_VISIT_EXCEED_LIMIT — дает право на подготовку выезда при превышении лимита коммуникации.

 

Для того чтобы сформировать SMS-сообщение на контактный телефон должника, выполните следующие действия:

1.На вкладке Общая перейдите на вложенную вкладку Телефоны, выберите нужный телефон и нажмите кнопку . Откроется форма редактирования SMS-сообщения.

2.В открывшейся форме сформируйте текст сообщения одним из следующих способов:

·на вкладке Текст введите произвольный текст сообщения в поле Текст сообщения. Возможность отправки SMS-сообщений с произвольным текстом определяется доступом DEBT_CARD_SMS_TEXT;

·на вкладке Сценарий в поле SMS-сообщение выберите шаблон SMS-сообщения. Текст SMS-сообщения, сформированный по выбранному шаблону, будет отображаться в поле Текст сообщения. Если выбран шаблон, который содержит переменные, относящиеся к акции, то в форме будет активно поле Акция. В нем можно выбрать одну из активных акций, чтобы предложить ее клиенту. При отправке SMS-сообщения для долга (если он участвует в акции) будет установлен флаг Акция предложена и автоматически заполнено поле Источник информации об акции.

Примечание: шаблоны SMS-сообщений формируются в модуле "Сценарии сообщений" (см. Создание нового сценария SMS-сообщения). Доступ для пользователя к созданным шаблонам настраивается в модуле Роли и доступы на вкладке Управление доступом к шаблонам SMS-сообщений. В списке отображаются сообщения, доступ к которым был настроен.

Примечание: На вкладке отображаются только те сценарии, для которых установлен флаг Ед. рассылка.

Примечание. Если константа SmsOriginatorSelect имеет значение "Истина", то доступна возможность выбора отправителя при формировании SMS-сообщения. При этом по умолчанию в поле "Отправитель" вводится термин словаря № 96 "Отправитель SMS", указанный в значении константы SmsDefaultOriginator.

Примечание. Если значение константы DebtCardSmsBankRestriction равно "Истина", то будут доступны только шаблоны, настроенные для банка, которому принадлежит долг, в модуле Настройка ограничений на отправку SMS.

       Ввод параметров в сценарий сообщений

Ввод параметров в сценарии SMS осуществляется по кнопке . Кнопка видна и доступна только для сценариев, содержащих параметры.

Примечание. Добавление параметров в сценарий сообщения реализуется вендором. Обратитесь в техническую поддержку.

В настоящей версии Системы добавлен параметр Дата расчета.

Для того чтобы ввести параметр Дата расчета при формировании SMS-сообщения:

1.Откройте карточку должника.

2.На вкладке Общая перейдите на вложенную вкладку Телефоны, выберите нужный телефон и нажмите кнопку . Откроется форма редактирования SMS-сообщения.

3.Выберите сценарий сообщения, содержащий параметры. В нижней части формы появится кнопка .

4.Нажмите кнопку для ввода параметров. Откроется окно выбора параметров.

5.Для выбора параметра начните вводить его название в поле Выберите параметр или нажмите на для выбора параметра из списка доступных.

6.Когда параметр будет выбран, в форме Ввод параметров появится поле ввода для него.

Примечание. Поле ввода для параметров различного типа будет различаться.

Учет статистики по SMS

Статистика по смс должна обновляться (триггером) сразу после вставки сообщения в таблицу debt_sms. В статистике будут учитываться сообщения в следующих статусах: 

·Передано 

·Доставлено

·Заявка 

·В процессе отправки 

·Не известно

Если сообщение будет переведено в статус, отличный от описанных выше, то оно будет исключено из статистики, и наоборот, если сообщение перейдет в указанные статусы, то оно будет учитываться в статистике. 

В качестве даты сообщении для учета в статистике берется фактическая дата отправки (debt_sms.send_date), если эта дата не заполняется, то учитывается дата начала отправки (debt_sms.start_date).

В случае отложенной отправки сообщения, предполагаемые к отправке через некоторое время (в статусе "Готово к отправке") будут учитываться в статистике, соответствующей дню отправки. И такие сообщения будут учитываться при попытке поставить в очередь еще одно сообщение. 

 Обновление статистки по смс, которые изменили статус будет выполняться фоновым заданием.
