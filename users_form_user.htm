Описание формы "Пользователь"

Форма Пользователь включает следующие элементы:

·Вкладка Основные (вид вкладки, описание полей вкладки);

·Вкладка Дополнительно (вид вкладки, описание полей вкладки);

·Кнопка ОК для сохранения внесенных изменений.

Вкладка "Основные"

Вкладка Основные включает следующие элементы:

·Область данных "ФИО";

·Область данных "Вход в систему";

·Область данных "Настройки телефонии";

·Область данных "Контакты".

Область данных "ФИО"

Область данных ФИО включает следующие элементы:

·Фамилия — фамилия пользователя (обязательное для заполнения);

·Имя — имя пользователя;

·Отчество — отчество пользователя;

·Флаг Блокирован — установка флага в этом поле позволяет временно заблокировать доступ пользователя в систему. В случае если пользователь неправильно введет пароль для входа в систему три раза подряд, он будет заблокирован автоматически;

·Флаг Авт. перезакреп. — если этот флаг установлен у пользователя, то при регистрации контакта, у которого в настройках исхода выставлен флаг “перераспределение на оператора" по долгу, закрепленному за этим пользователем, долг будет перезакреплен за оператором, зарегистрировавшим контакт;

Пример. У пользователя Иванова установлен данный флаг, а у Петрова нет. Тогда:

·Если Петров зарегистрирует контакт с флагом “перераспределение на оператора" по долгу, закрепленному за Ивановым, то долг автоматически будет перезакреплен за Петровым.

·Если же Иванов зарегистрирует контакт с флагом “перераспределение на оператора" по долгу, закрепленному за Петровым, то долг останется закрепленным за Петровым.

·Флаг Виртуальный — признак, который можно использовать для отбора пользователей. Информационное поле;

·Флаг Запрет закрепления — если флаг включен, для данного пользователя запрещено закрепление долгов в ручных операциях.

·Флаг Очистка настроек — если он включен, то при сохранении пользователя происходит однократное удаление пользовательских настроек;

·поле для добавления фотографии. Для его заполнения нажмите кнопку и выберите файл с изображением в стандартном диалоговом окне Windows;

Примечание. Размер загружаемого файла не должен превышать значение, указанное в константе DocAttachMaxSize (в Кб).

·кнопка для работы с Дополнительными атрибутами пользователя, если в Системе настроен этот функционал.

Область данных "Вход в систему"

Область данных Вход в систему содержит следующие элементы:

·Логин — логин пользователя, который будет использоваться для входа в систему (обязательное для заполнения);

·Пароль — пароль пользователя (сохраненный пароль пользователя в поле не отображается. При вводе нового пароля символы заменяются на "*");

·Роль — роль из списка ролей пользователей, в соответствии с которой пользователю будут предоставлены права для работы в системе;

Примечание. Если значение константы _SecurityUseStrictRoleAccess равно "Истина", то запрещается вносить любые изменения в роль текущего пользователя.

Примечание. Разрешение на смену роли пользователя определяется доступом № 533 USERS_CHANGE_ROLE.

·График — график работы пользователя, выбирается из списка графиков рабочего времени;

·Филиал — филиал организации, использующей систему, в котором работает пользователь. Значение выбирается из словаря № 61;

·Тип — стадия взыскания долгов, на которой работает пользователь (выбирается из списка);

·Дата увольнения;

·Квалификация — см. словарь № 237 "Квалификация пользователя", поле может отсутствовать на форме, его добавление выполняется по согласованию с технической поддержкой;

Примечание для поддержки. Поле Квалификация включается технической константой Users.SkillEdit.Visible.

·Должность — должность пользователя;

·Имя Win — запись из каталога Active Directory (см. Привязка записи о пользователе к каталогу Active Directory);

·Табельный номер;

·План сбора — план сбора, назначенный пользователю;

·Внешний ID;

·Дата выхода в штат — необходимость заполнения данного поля определяется константой UserWorkStartDateMandatory;

Область данных "Настройки телефонии"

Область данных Настройки телефонии доступна при использовании интеграции Системы с телефонией.

·Имя агента — регистрационные данные агента;

·Пароль агента — регистрационные данные агента;

·ID Агента — идентификатор агента;

·Тип сервера — тип сервера телефонии.

Подробнее — см. Регистрация агента Genesys.

Область данных "Контакты"

Область данных Контакты включает следующие элементы:

·Мобильный — номер мобильного телефона пользователя;

·Рабочий — номер рабочего телефона пользователя;

·Внутренний — внутренний телефонный номер пользователя;

Примечание. Для того чтобы пользователь мог использовать функции телефонии, необходимо указать для него номер внутреннего телефона.

·Email — адрес электронной почты пользователя;

·Индекс — индекс места работы пользователя;

·Рабочий адрес — адрес места работы пользователя.

Вкладка "Дополнительно"

Вкладка Дополнительно содержит следующие элементы:

·Область данных "Android";

·Область данных "Комментарий";

·Область данных "Интерфейс";

·Область данных "Ссылки";

Область данных Android

 Примечание. Блок Android отображается, только если включена коммерческая константа @AndroidContact.

·Общее кол-во неудачных попыток — общее количество неверных попыток ввода пароля;

·Кол-во неудачных попыток подряд — количество неверных попыток ввода пароля подряд;

·Кнопка Сброс — кнопка для сброса количества неверных попыток ввода пароля подряд;

Примечание. Кнопка видна только пользователя, в роли которых есть доступ USERS_RESET_MOBILE_BLOCK.

·Дата последней неудачной попытки — дата последней неудачной попытки авторизации в системе;

·Дата последней успешной попытки — дата последней успешной попытки авторизации.

·Список Устройства:

Примечание. Список Устройства виден только, если включена константа Android.DeviceUserLink.Use.

В списке Устройства отображается список устройств, привязанных к учетной записи пользователя (см.Привязка мобильного устройства к учетной записи пользователя).

oТаблица со списком устройств включает поля:

§MAC-адрес — MAC - адрес устройства;

§Дата — дата последнего изменения;

oКнопки операций со списком устройств:

Примечание. Данные операции должны быть доступны, только если у текущего пользователя есть доступ № 611 USERS_DEVICE_USER_LINK.

§Добавить — добавление устройства, привязанного к учетной записи пользователя;

§Изменить — редактирование устройства, привязанного к учетной записи пользователя;

§Удалить — удаление устройства, привязанного к учетной записи пользователя;

Область данных "Комментарий"

Область данных Комментарий содержит текстовое поле для ввода дополнительной информации.

Область данных "Интерфейс"

Область данных Интерфейс включает следующие элементы:

·Кнопка — позволяет задать цвет карточек должника, закрепленных за данным пользователем.

Примечание. Установка цвета доступна при наличии у пользователя доступа USERS_SET_DEBT_CARD_COLOR.

Чтобы установить нужный цвет, нажмите кнопку и выберите его в палитре.

Чтобы вернуться к цвету, установленному по умолчанию, нажмите и выберите пункт Сбросить цвет.

·Язык — выберите язык пользователя

Область данных "Ссылки"

Область данных Ссылки отображается на форме, если константа Users.AttachLinks.Use имеет значение "Истина".

Область данных Ссылки включает следующие элементы:

·Таблица Ссылки, в которой отображается список ссылок, прикрепленных к данному пользователю;

·Кнопки для добавления, редактирования и удаления ссылок.
