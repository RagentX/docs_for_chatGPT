Загрузка истории работы с должником

Функция загрузки истории работы используется, как правило, при аутсорсинге или для операторов, работающих в удаленных филиалах организации и не имеющих доступа к системе, установленной в головной организации. В качестве результатов работы им передается файл MS Excel определенного формата для его загрузки в локальную систему. В результате загрузки данных в локальной системе появится информация о совершенных звонках, выездах и т. д.

Доступ к функции имеют только те пользователи, в роли которых доступ DEBT_LIST_CONTACT_HISTORY_LOAD установлен в значение TRUE.

Порядок действий

Для того чтобы загрузить историю работы, выполните следующие действия:

1.В форме Список должников в контекстном меню (вызывается правой кнопкой мыши) выберите пункт Загрузка истории работы.

2.В открывшемся диалоговом окне Windows выберите файл MS Excel в нужном формате и нажмите кнопку Open (Открыть).

Идентификация загруженных данных выполняется по следующему алгоритму:

1)Основным идентификатором является ID долга. При его заполнении в строке остальные идентификаторы (ID должника, Номер договора, Лицевой счет, ID Клиента) не учитываются.

2)При отсутствии данных в поле ID долга основным идентификатором является ID должника. Если у должника есть хотя бы один активный долг, контакт будет регистрироваться на первый загруженный активный долг данного должника. Если у должника все долги закрытые, то контакт будет регистрироваться на последний загруженный закрытый долг данного должника.

3)При отсутствии данных в столбцах ID должника и ID долга идентификаторами являются Номер договора, Лицевой счет, ID Клиента либо их сочетание, в зависимости от того, какие из этих полей заполнены.

4)При загрузке записи с типом исхода 1 (звонок) или 3 (входящий звонок), в случае если поле № 11 (ID телефона/ID Адреса) заполнено и значение в нем корректно (телефон с таким ID существует и принадлежит указанному должнику), загружается контакт на указанный ID телефона.

5)При загрузке записи с типом 2 (выезд):

·если поле № 11 (ID телефона/ID Адреса) заполнено и значение указано корректно (адрес с таким ID существует и принадлежит указанному должнику), загружается контакт на указанный ID адреса;

·если поле № 11 (ID телефона/ID адреса) не заполнено или телефон/адрес с таким ID не принадлежит указанному должнику, контакт загружается без привязки к телефону/адресу (в зависимости от типа контакта).

6)При загрузке записи с типом 7 (SMS) поле ID телефона обязательно к заполнению.

В случае если тип исхода не определен или не соответствует диапазону 1-5 или 7, данный контакт будет загружен с типом "Звонок", Результат "История работы" (независимо от кода исхода).

В случае если код исхода не определен или код исхода в файле не соответствует ни одному из кодов в РМ "Настройка дерева контактов" для указанного типа исхода, данный контакт будет загружен с результатом "История работы" для данного типа.

При загрузке записей с типом 7 (SMS), если не указан ID Сценария или указан некорректно (сценария с указанным номером для SMS-сообщений нет), SMS-сообщение загружается без привязки к сценарию.

Строка файла может быть не загружена, если:

·не идентифицирован долг (Тип 0);

·дата в файле больше текущей даты (Тип 2);

·длина строки (Направление + Комментарий) больше 2048 символов (Тип 3);

·у записи с типом 7 (SMS) не указан или некорректно указан ID Телефона.

Если при загрузке обнаружена хотя бы одна запись, которая не может быть загружена, будет предложено сохранить файл с ошибками. В файле будут отображаться все данные (для поля "Комментарий" — первые 950 символов), в последнем столбце будет содержаться причина ошибки.

В результате загрузки данных в локальной системе будут сформированы контакты на дату, которая указана в файле (столбец № 3), с типом из столбца № 9 и результатом в соответствии с кодом исхода (столбец № 10), значение в поле "Комментарий" — (Направление (столбец № 4) + Комментарий (Столбец № 5)), ), "Оператор" — Оператор (Столбец № 13) (если поле пустое, то оператор, выполняющий загрузку истории работы).

В случае если для контакта одновременно заполнены поля "Дата обещания" (Столбец № 14) и "Сумма платежа" (Столбец № 15), для указанного долга в систему загружается обещание с комментарием "Импортировано при загрузке истории работы", и у контакта будет ссылка на загруженное обещание.

В случае если контакт не загружается в систему, то и обещание тоже не будет загружено в систему.

При загрузке записей с типом исхода 1 или 3, в случае если ID телефона в загружаемом файле корректен (корректность телефона описана выше):

·если код исхода (Столбец № 10) в загружаемом файле соответствует значению константы DialerAutoinfoSuccessResult или константы DialerAutoinfoUncompleteResult и статус этого телефона "Не звонили ни разу" или "Результата не было", то статус этого телефона меняется на "Автоинформатор";

·если статус телефона "Не звонили ни разу" или "Результата не было, код исхода (Столбец № 10) в загружаемом файле не соответствует значениям констант DialerAutoinfoSuccessResult и DialerAutoinfoUncompleteResult и у данного исхода включен флаг Считать результат успешным, то статус телефона меняется на "Результат";

·если статус телефона "Не звонили ни разу", код исхода (Столбец № 10) в загружаемом файле не соответствует значениям констант DialerAutoinfoSuccessResult и DialerAutoinfoUncompleteResult, и у данного исхода не включен флаг Считать результат успешным, то статус телефона меняется на "Результата не было"

При этом если в файле дата контакта больше даты последнего звонка по этому телефону в системе, то телефону устанавливается дата последнего звонка из файла.

При загрузке записей с типом исхода 2, в случае если ID адрес в загружаемом файле корректен (корректность адреса описана в пункте Идентификация):

·если статус адреса "Не выезжали ни разу" или "Результата не было" и у данного исхода включен флаг Считать результат успешным, то статус адреса меняется на "Результат";

·если статус адреса "Не выезжали ни разу" и у данного исхода не включен флаг Считать результат успешным, то статус адреса меняется на "Результата не было".

При загрузке записей с типом 1 или 3, в случае если текст в поле "Комментарий" превышает 2048 символов, произойдет разбиение контакта на несколько по 2048 символов в комментарии в каждом. В результате будут загружено несколько контактов (в зависимости от количества символов в поле "Комментарий") с одинаковыми параметрами. Дата этих исходов будет отличаться на 1 секунду (для корректной сортировки). При этом если в файле для данного контакта есть обещание, на него будет ссылаться только один зарегистрированный контакт.

В карточке должника телефоны в статусе "Автоинформатор" окрашиваются в светло-оранжевый цвет.

В результате загрузки записей с типом исхода 7 будут сформированы SMS-сообщения со статусом "Передано" для телефона из поля "ID телефона" (Столбец № 11) на дату, которая указана в файле (столбец № 3), с текстом сообщения из поля "Комментарий" (Столбец № 5), Оператор — из поля  "Оператор" (Столбец № 13) (если поле пустое, то оператор, выполняющий загрузку истории работы), с номером сценария из поля "ID Сценария" (Сценарий № 12).

Формат файла

Исходный файл MS Excel должен иметь следующий формат:

1)ID должника;

2)ID долга;

3)Дата — обязательное поле;

4)Направление;

5)Комментарий — количество символов в комментарии для записей с типом исхода 1,2,3,4,5 не должно превышать 8000 символов; для записей с типом исхода 7 не должно превышать 512 символов;

Примечание: если количество символов в поле Комментарий превышает 8000 символов, строки слева будет обрезана до 8000 символов.

6)Номер договора;

7)Лицевой счет;

8)ID Клиента;

9) Тип исхода (1 — звонок, 2 — выезд, 3 — входящий звонок, 4 — визит в офис, 5 — Специальное, 7 — SMS);

10) Код исхода (настраивается в модуле "Настройка дерева контактов");

11) ID Телефона/ID Адреса;

12) ID Сценария;

13) ID Оператора;

14) Дата обещания;

15) Сумма обещания.

Первая строка файла должна содержать названия столбцов ("шапку"), которые могут быть любыми. Файл должен содержать 5 или более столбцов, в противном случае загрузка данных выполнена не будет.

Поля "Номер договора", "Лицевой счет", "ID клиента" могут отсутствовать в файле. Они необходимы для идентификации при отсутствии данных в столбце "ID должника" или "ID долга".
