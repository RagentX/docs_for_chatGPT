Завершение работы с долгом

Завершить работу с долгом можно в случае, если:

·информация о долге была заведена ошибочно;

·задолженность погашена;

·долг отозван клиентом по причине завершения работы с реестром, невозможности взыскания и т. п.

Для этого выполните следующие действия:

1.В карточке должника на вкладке Долги выберите нужную запись в списке долгов.

2.Нажмите кнопку и в раскрывшемся списке выберите:

·К возврату клиенту — для перевода долга в статус "К возврату клиенту";

·Отозван клиентом — для перевода долга в статус "Отозван клиентом";

·Завершить — для перевода долга в статус "Завершено";

3.Откроется диалоговое окно для подтверждения завершения работы с выбранным долгом. Нажмите Yes.

4.Откроется форма для указания причины завершения работы с выбранным долгом. Перечень возможных причин формируется (см. Списки причин для перевода долгов в определенные статусы):

·при переводе долга в статус 10 "К возврату клиенту" — в словаре № 27;

·при переводе долга в статус 8 "Отозван клиентом" — в словаре № 28;

·при переводе долга в статус 6 "Завершено" — в словаре № 21.

5.В форме заполните следующие поля:

·Причина — выберите причину аннулирования записей;

·Комментарий — поле обязательно для заполнения.

6.Нажмите кнопку ОК. Долг получит соответствующий статус и будет доступен в списке должников на вкладке Закрытые. Ответственный оператор будет освобожден, в поле "Ответственный" будет установлено значение "Не установлен."

Примечание. Есть возможность установить предыдущего оператора для закрытого долга.

Есть возможность настроить Систему так, чтобы при переводе долга в статусы (любая комбинация указанных статусов):

·6 — "Завершено";

·7 — "Аннулировано";

·8 — "Отозвано клиентом";

·10 — "К возврату клиенту"

автоматического освобождения оператора не происходило. Для этого обратитесь в техническую поддержку.

Для того чтобы при переходе долга в один из этих статусов долг не откреплялся от оператора, исключите код этого статуса из значения технической константы WorkTask.ClearOperator.DebtStatusList.
