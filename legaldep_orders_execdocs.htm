Создание исполнительного документа

Show/Hide Hidden Text

Создание исполнительных документов поддержано в Системе только при значении константы UseLawExecDoc равном "Истина".

Создание исполнительных документов доступно пользователям, в роли которых доступ LAW_EXEC_DOC_CREATE имеет значение TRUE.

Создание исполнительного документа по делу судебной работы аналогично переводу на исполнительное производство, поскольку при этом создается дело ИП (в статусе "Не создано"), но имеет ряд существенных отличий:

·Отсутствует проверка на наличие ранее созданных дел ИП или исполнительных документов;

·Текущее дело судебной работы не изменяет свой статус;

·Созданные исполнительные документы доступны на вкладке Исполнительные документы, и не доступны на вкладке Исполнительное производство.

Создание исполнительных документов осуществляется одинаково для дел следующих типов:

·Приказы — на вкладке Приказы;

·Правопреемственные дела — на вкладке Правопреемство;

·Иски — на вкладке Иски;

Порядок действий

Для создания исполнительных документов выполните следующие действия:

1.В форме Юридический департамент активируйте нужную вкладку и выберите одно или более дел. Для выбора нескольких записей используйте клавиши SHIFT и/или CTRL.

2.В контекстном меню (вызывается правой кнопкой мыши) выберите пункт Создать исполн. документ.

3.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить перевод.

4.Для каждго из выбранных дел судебной работы будут выполнены проверки статуса дела и статуса долга, связанного с ним:

       Список статусов судебных дел, для которых разрешен перевод на ИП

Для перевода на исполнительное производство судебное дело должно находиться в одном из следующих статусов:

Приказы (словарь № 18)

Коды статусов приказа, для которых разрешен перевод на исполнительное производство, указываются в константе LawAct.Order.TransferToExec.Allow.OrderStatusList. По умолчанию константа заполнена следующими значениями (не рекомендуется самостоятельно изменять значение этой константы без консультации с технической поддержкой):

·5 — "Приказ получен должником";

·7 — "Приказ получен".

Правопреемство (словарь № 25)

·39 — "Апелляция удовлетворена";

·40 — "Кассация удовлетворена";

·42 — "Правопреемство установлено";

·47 — "Частная жалоба удовлетворена".

Иски (словарь № 25)

Для перевода на исполнительное производство доступны иски, находящиеся в статусах, которые перечислены в значении константы LawAct.Act.TransferToExec.Allow.ActStatusList. Если значение константы пусто, то доступен перевод из любого статуса.

Исполнительная надпись (словарь № 25)

·1007 — "Надпись получена".

1)Если среди выбранных дел есть хотя бы одно не в разрешенном статусе, будет показано сообщение:

«Некоторые дела не будут переведены на исполнительное производство по причинам:

- Перевод осуществляется только после получения исполнительной надписи

Продолжить формирование исполнительного производства?»

2)Если среди выбранных дел, есть хотя бы одно, связанный долг по которому находится в одном из статусов, перечисленных в значении константы LawAct.TransferToExec.Ban.DebtStatusList, будет показано сообщение:

«Некоторые дела не будут переведены на исполнительное производство по причинам:

- Связанный с делом долг имеет статус, для которого запрещен перевод на ИП

Продолжить формирование исполнительного производства?»

3)Если выполняются сразу оба условия, перечисленные выше, то будет показано сообщение:

«Некоторые дела не будут переведены на исполнительное производство по причинам:

- Перевод осуществляется только после получения исполнительной надписи

- Связанный с делом долг имеет статус, для которого запрещен перевод на ИП

Продолжить формирование исполнительного производства?»

Перевод на исполнительное производство выполняется только для дел, которые находятся в разрешенных статусах, и для которых статус связанного долга не попадает под ограничение константы LawAct.TransferToExec.Ban.DebtStatusList.

5.По выделенным записям создаются исполнительные производства:

·им присваивается статус "Не создано";

·поле Дата возбуждения ИП не заполняется;

·статус дела, по которому создается исполнительный документ, не изменяется.

6.Действие протоколируется.

7.Осуществляется формирование настроенных запланированных действий.

8.По итогам успешного завершения действия на экран выводится сообщение "Исполнительные документы созданы (количество - <количество>)".

9.Созданные исполнительные документы доступны на вкладке Исполнительные документы.
