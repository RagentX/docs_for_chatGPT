Перевод должника между стадиями

Перевод должника между стадиями доступен пользователям, в роли которых доступ № 145 DEBT_CARD_STAGE имеет значение TRUE. Иначе кнопка Перевести на следующую стадию на форме Карточка должника отсутствует.

В карточке должника можно выполнить перевод должника:

·из стадии Soft в стадию Hard;

·из стадии Hard в стадию Legal;

·В стадии Legal создать судебное дело для того должника, у которого не имеется судебных дел.

Примечание. Действие доступно только тем пользователям, в роли которых доступ DEBT_CARD_STAGE имеет значение TRUE.

Перевод должника на стадию Hard

Для должника, находящегося на стадии Soft, доступен перевод на стадию Hard.

1.В карточке должника нажмите кнопку Перевести на следующую стадию.

2.В открывшемся диалоговом окне подтвердите перевод должника на следующую стадию.

3.Откроется форма для указания причины перевода. Перечень возможных причин перевода долга на стадию HARD содержится в словаре № 70.

4.В поле Причина выберите причину перевода и нажмите кнопку ОК. Перевод будет выполнен. При смене стадии может осуществляться перевод долгов в соответствующий портфель.Если включена константа DepartChangeClearOperator, то для долга будет очищена связь с оператором. Если эта константа выключена, оператор по долгу не изменится.

Перевод должника на стадию Legal

Для должника, находящегося на стадии Hard, доступен перевод на стадию Legal.

1.В карточке должника нажмите кнопку .

2.В открывшемся диалоговом окне подтвердите перевод должника на следующую стадию.

3.Откроется форма для указания причины перевода. Перечень возможных причин перевода долга на стадию LEGAL содержится в словаре № 71.

4.В поле Причина выберите причину перевода и нажмите кнопку ОК. Перевод будет выполнен. При смене стадии может осуществляться перевод долгов в соответствующий портфель. При смене стадии может осуществляться перевод долгов в соответствующий портфель.Если включена константа DepartChangeClearOperator, то для долга будет очищена связь с оператором. Если эта константа выключена, оператор по долгу не изменится.

Создание судебного дела на стадии Legal

Создание дела осуществляется для должников, находящихся на стадии Legal, у которых еще не создано судебное дело.

Тип создаваемого дела зависит от значения константы LawActUseIndefiniteAct.

LawActUseIndefiniteAct = "Истина"

При нажатии на кнопку осуществляется создание неопределенного судебного дела (не приказ и не иск).

LawActUseIndefiniteAct = "Ложь"

1.В карточке должника нажмите кнопку .

2.В открывшемся окне выберите тип судебного дела, которое нужно создать.

3.Нажмите OK. Дело будет создано. Если при выключенной константе DepartChangeClearOperator включена константа LawActGetOperatorFromPrevStage, то оператор долга станет оператором созданного дела.
