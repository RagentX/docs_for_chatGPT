Регистрация обещания и ограничения на дату и сумму

Show/Hide Hidden Text

В настройках результата контакта можно предусмотреть регистрацию обещания. Тогда оператор будет иметь возможность зарегистрировать обещание в процессе регистрации контакта.

Если контстанта PromRepeatedControlAll имеет значение "Истина", то при наличии активного обещания по этому долгу (или по должнику), срок ожидания по которому не истек, регистрировать новое обещание по тому же долгу (должнику) не разрешается.

Если константа PromRepeatedControlAll имеет значение "Ложь", но константа PromRepeatedControl имеет значение "Истина", то для операторв типа "Soft" при регистрации контакта типа "Звонок", "Выезд" или "Входящий звонок" также запрещено регистрировать обещание при наличии действующего обещания (как по долгу, так и по должнику).

При использовании константы RegContactDebtSumFormula сумма просроченной задолженности вычисляется динамически как результат формулы, идентификатор которой указан в значении константы.

Регистрация обещания при регистрации контакта подчиняется следующим ограничениям в зависимости от типа сотрудника (оператора).

Ограничения на дату обещания

Когда должник обещает осуществить платеж, он может указать дату не позднее, чем заранее заданное число дней от текущей даты. Это число дней называется сроком обещания и определяется для каждой стадии с использованием констант:

Для оператора типа Hard:

·если константа RegContactPromMaxDayHardFormula > 0, то срок рассчитывается по формуле, указанной в значении константы;

·если константа RegContactPromMaxDayHardFormula = 0, то:

·если есть доступ № 517 CONTACT_REG_MAX_PROMISE_DATE и его значение больше 0, то срок равен этому значению;

·иначе срок определяется константой PromMaxDayHard.

Для оператора другого типа, кроме Hard (например, Soft):

·если константа RegContactPromMaxDaySoftFormula > 0, то срок рассчитывается по формуле, указанной в значении константы;

·если константа RegContactPromMaxDaySoftFormula = 0, то:

·если есть доступ № 517 CONTACT_REG_MAX_PROMISE_DATE и его значение больше 0, то срок равен этому значению;

·иначе срок определяется константой PromMaxDaySoft.

При регистрации контакта в панели "Обещание" в поле "Дата" доступны для регистрации только те даты, которые входят в срок обещания.

Ограничение на обещанную сумму

Ограничения на минимальный размер обещанной суммы

Примечание. Использование функционала акций. Если константа UseDebtDiscount имеет значение "Истина", то для долгов, включенных в активные акции, проверка на минимальную сумму обещания не производится.

Для поддержки. Функционал использования акций включается коммерческой константой @Discounts.

Ограничение на минимальную сумму обещания задается как процент размера обещания относительно остатка к погашению.

Когда должник обещает осуществить платеж, сумма обещанного платежа сравнивается с заранее заданным процентом от остатка к погашению.

Примечание. Если флаг Обещание на уровне должника включен, остаток к погашению считается как сумма остатков всех долгов должника. Если флаг выключен, учитывается только текущий долг.

 Примечание. Функционал привязки обещания к должнику включается при помощи константы @DebtUsePromiseOnPerson.

Если сумма равна или превышает заданную, обещание регистрируется без проблем, если же сумма меньше, то дальнейшая логика зависит от значения доступа № 515 REG_CONTACT_INSUFFICIENT_PROM_SUM в роли оператора:

·Если доступ имеет значение FALSE, то обещание не регистрируется;

·Если доступ имеет значение TRUE, то обещание будет зарегистрировано со статусом "Не верифицировано" и для него потребуется подтверждение руководителя. Неверифицированные обещания не считаются полноценными и не отображаются в списочных РМ.

Для оператора типа Hard:

·если константа RegContactPromMinSumRateHardFormula > 0, то процент минимального обещания рассчитывается по формуле, указанной в значении константы;

·если константа RegContactPromMinSumRateHardFormula = 0, то:

·если есть доступ № 523 CONTACT_REG_MIN_PROMISE_RATE и его значение больше 0, то процент равен этому значению;

·иначе процент определяется константой PromMinSumRateHard.

Для оператора другого типа, кроме Hard (например, Soft):

·если константа RegContactPromMinSumRateSoftFormula > 0, то  процент минимального обещания рассчитывается по формуле, указанной в значении константы;

·если константа RegContactPromMinSumRateSoftFormula = 0, то:

·если есть доступ № 523 CONTACT_REG_MIN_PROMISE_RATE и его значение больше 0, то процент равен этому значению;

·иначе процент определяется константой PromMinSumRateSoft.

При регистрации контакта в панели Обещание в полях % от остатка и Сумма до начала ввода суммы обещания указаны значения минимального процента и соответствующей ему минимальной суммы обещания.

 

Ограничения на максимальный размер обещанной суммы

Если в роли пользователя доступ № 396 REG_CONTACT_MAX_PROM_SUM_CONTROL имеет значение TRUE, то у таких пользователей работает ограничение на максимальную сумму обещания.

Если обещанная сумма превышает сумму остатка на количество процентов, указанных в константе MaxPromSumPercent, то такое обещание не будет зарегистрировано. При попытке регистрации такого обещания пользователю отображается соответствующее сообщение.

Как настроить сквозное обещание (для каждой роли назначить свой максимальный срок и минимальную сумму)

Ограничения на срок и сумму обещания могут быть назначены собственные для каждой роли, независимо от типа пользователя.

Для этого выполните следующие действия:

1.Настройте константы:

·RegContactPromMaxDayHardFormula = 0;

·RegContactPromMaxDaySoftFormula = 0;

·RegContactPromMinSumRateHardFormula = 0;

·RegContactPromMinSumRateSoftFormula = 0.

2.Настройте значение доступа № 517 CONTACT_REG_MAX_PROMISE_DATE для каждой роли, для этого укажите максимальное количество дней. Например:

1)значение доступа 1, тогда при регистрации контакта 29.06.2015 — максимальный срок обещания — 30.06.2015;

2)значение доступа 5, тогда при регистрации контакта 29.06.2015 — максимальный срок обещания — 04.07.2015.

3.Настройте значение доступа № 523 CONTACT_REG_MIN_PROMISE_RATE для каждой роли, указав минимальный разрешенный процент размера обещания относительно остатка к погашению.

Например, если значение доступа 5, тогда при регистрации контакта минимальная сумма будет составлять 5% от остатка к погашению. В зависимости от значения доступа № 515 REG_CONTACT_INSUFFICIENT_PROM_SUM обещания на меньшую сумму будут либо запрещены, либо требовать подтверждения руководителя.
