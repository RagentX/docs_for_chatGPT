Групповое изменение атрибута

Show/Hide Hidden Text

Операция группового изменения атрибута доступна в форме Юридический департамент на вкладках Приказы, Иски, Судебные дела(предв.), Исполнительное производство для следующих атрибутов:

·Портфель;

Примечание. Право на групповое изменение портфеля определяется наличием в роли пользователя доступа:

oВкладка Приказы — доступ № 446 LAW_ACT_ORDER_SET_PORTFOLIO;

oВкладка Правопреемство — доступ № 565 LAW_ACT_SUCCESSION_SET_PORTFOLIO;

oВкладка Иски — доступ № 451 LAW_ACT_LAWSUIT_SET_PORTFOLIO;

oВкладка Банкротство — доступ № 586 LAW_ACT_BANKRUPT_SET_PORTFOLIO;

oВкладка Судебные дела (предв.) — № 504 LAW_ACT_INDEFINITE_SET_PORTFOLIO;

oВкладки Исполнит. документы, Исполнительное производство — доступ № 455 LAW_EXEC_SET_PORTFOLIO.

Примечание. Массовое изменение портфеля для банкротства имеет ряд особенностей. Подробное описание см. Массовое изменение портфеля в банкротстве.

·Ответственный — изменение ответственного происходит по другому алгоритму. См. далее Порядок действий для группового изменения ответственного.

Примечание. Право на групповое изменение ответственного определяется наличием в роли пользователя доступа:

oВкладка Приказы — доступ № 445 LAW_ACT_ORDER_SET_USER;

oВкладка Правопреемство — доступ № 564 LAW_ACT_SUCCESSION_SET_USER;

oВкладка Иски — доступ № 450 LAW_ACT_LAWSUIT_SET_USER;

oВкладка Банкротство — доступ № 585 LAW_ACT_BANKRUPT_SET_USER;

oВкладка Судебные дела (предв.) — № 503 LAW_ACT_INDEFINITE_SET_USER;

oВкладки Исполнит. документы, Исполнительное производство — доступ № 454 LAW_EXEC_SET_USER.

Примечание. Если в роли текущего пользователя доступ № 231 LAW_ACT_LIST_FULL имеет значение TRUE, то в списке доступных операторов отображаются все незаблокированные пользователей Системы, а также текущий оператор (даже если он заблокирован). Если доступ отсутствует или имеет значение FALSE, то список операторов отображается с учетом Ограничения доступа к данным на основании структуры предприятия.

·Примечание.

Примечание. Право на групповое изменение значения примечания определяется наличием в роли пользователя доступа:

·Вкладка Приказы — доступ № 499 LAW_ACT_ORDER_SET_DICT_VALUE;

·Вкладка Правопреемство — доступ № 567 LAW_ACT_SUCCESSION_SET_DICT_VALUE;

·Вкладка Иски — доступ № 500 LAW_ACT_LAWSUIT_SET_DICT_VALUE;

·Вкладка Банкротство — доступ № 588 LAW_ACT_BANKRUPT_SET_DICT_VALUE;

·Вкладка Судебные дела (предв.) — доступ № 666 LAW_ACT_INDEFINITE_SET_DICT_VALUE;

·Вкладка Исполнительное производство — доступ № 501 LAW_EXEC_SET_DICT_VALUE.

На остальных вкладках модуля групповое изменение примечания не разрешено.

 

Порядок действий для группового изменения атрибута

Для того чтобы изменить значение атрибута, выполните следующие действия:

1.В нужной вкладке формы при помощи фильтров сформируйте выборку записей, в которых требуется изменить значение атрибута.

2.В контекстном меню (вызывается правой кнопкой мыши) выберите пункт Установить значение (столбца) -> <Название атрибута>. Откроется окно для ввода нового значения атрибута.

3.Введите новое значение атрибута и нажмите кнопку ОК. Это значение атрибута будет установлено во всех записях, входящих в выборку.

Массовое изменение атрибутов протоколируется.

Порядок действий для группового изменения ответственного

Для того чтобы изменить значение атрибута, выполните следующие действия:

1.В нужной вкладке формы при помощи фильтров сформируйте выборку записей, в которых требуется изменить значение атрибута.

2.В контекстном меню (вызывается правой кнопкой мыши) выберите пункт Установить значение (столбца) -> Ответственный. Откроется форма Выбор оператора.

3.Выберите новое значение ответственного и нажмите кнопку ОК. Это значение атрибута будет установлено во всех записях, входящих в выборку.

Массовое изменение атрибутов протоколируется.

Примечание. Если константа Law.Operator.Change.Notification.Use имеет значение "Истина", то операторам, на которых распределены долги, и операторам, с которых были сняты долги, будут направлены соответствующие уведомления по внутренней почте системы.

Отображение данных в форме "Выбор оператора"

По умолчанию в форме скрыты дополнительные колонки. Добавить в таблицу их можно, включив флаги в списке выводимых полей. Также вы можете изменить порядок следования колонок в таблице. Полученный вид таблицы выбора оператора будет сохранен при следующем входе в модуль. Для каждого из указанных модулей будет сохранен свой вид таблицы. Изменения сохраняются вплоть до сброса пользовательских настроек.

Полный список колонок формы Выбор оператора:

·ФИО — ФИО оператора;

·Роль — роль оператора;

·Отдел — отдел оператора в модуле Структура предприятия;

·Количество в Списке должников — количество долгов, закрепленных за оператором;

·Кол-во судебных дел (предв.) — учитываются только неопределенные судебные дела, находящиеся в статусе 4;

Примечание. Колонка Кол-во судебных дел (предв.) отображается только в случае, когда константа LawActUseIndefiniteAct = "Истина".

·Кол-во приказов — учитываются только приказы, находящиеся в статусах 9, 10, 11;

·Кол-во исков — учитываются только иски, находящиеся в статусах 13, 14, 15, 18;

·Кол-во правопреемств — учитываются только правопреемственные дела, находящиеся в статусах 13, 14, 15;

Примечание. Колонка Кол-во правопреемств отображается только в случае, когда константа LawActUseSuccession = "Истина".

·Кол-во банкротств — учитываются только банкротства, находящиеся в статусах 6, 13, 14, 15;

Примечание. Колонка Кол-во банкротств отображается только в случае, когда константа LawActUseBankrupt = "Истина".

·Кол-во ИП — учитываются только дела исполнительного производства, находящиеся в статусах 3, 4, 5, 6, 10;

·Всего — в колонке отображается суммарное количество активных дел и долгов, закрепленных за оператором. Причем, если функционал неопределенных судебных дел, правопреемства или банкротства отключен (соответствующая константа равна "Ложь"), то дела данного функционала, закрепленные за оператором, не учитываются в суммарном количестве дел;

·Запрет закрепления — колонка отображается для досудебной стадии при значении доступа № 543 DEBT_BAN_SET_FLAG_IGNORE = TRUE.
