Планирование (Задачи)

Обновлено до 50 релиза 03.11.2020.

Функционал работает при включенной коммерческой константе @UseDebtTask

Основной целью создания единой системы планирования является:

·Учет заданий, запланированных на проведение в будущем, в едином интерфейсе.

·Просмотр истории работы (определенного оператора или с конкретным долгом) в рамках единого окна.

Возможности функционала

·Ручная постановка задач (кнопка "Поставить задачу");

·Автоматическая постановка задачи по результату контакта;

·Автоматическая постановка задач по расписанию;

·Автоматическая постановка задач в качестве настройки запланированных действий;

·Изменение статуса задачи в соответствии с жизненным циклом;

·Изменение атрибутов задачи;

·Просмотр списка задач по долгу/делу;

·Просмотр списка задач по сотруднику;

·Просмотр истории работы с долгом, включающий выполненные задачи, зарегистрированные контакты, отправленные SMS, звонки и др.

Объекты для задач

Задачи могут создаваться для объектов следующих типов:

·Долг.

·Дело судебной работы.

·Дело исполнительного производства.

·Имущество.

·Должник.

Отображение задач по каждому объекту для каждого из модулей может быть настроено вендором (для этого обратитесь в техподдержку))См. Поддержка константы Task.TypeTaskInList.Mode):

Словари, необходимые для работы функционала

Для корректной работы функционала "Планирование (Задачи)" следует наполнить терминами следующие словари:

·Словарь № 229 "Тип задачи".

·Словарь № 230 "Подтип задачи".

·Словарь № 231 "Причина отклонения задачи".

·Словарь № 232 "Причина снятия задачи".

·Словарь № 233 "Причина возврата в работу задачи".

·Словарь № 234 "Причина невыполнения задачи".
