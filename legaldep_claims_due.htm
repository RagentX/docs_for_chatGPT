Госпошлина

Функционала госпошлины для исковна вкладке Иски аналогичен функционалу госпошлины для приказоd на вкладке Приказы, но дополнительно можно настроить учет требваний неимущественного характера (см. далее).

Функционал госпошлины используется в Системе, если значение константы LawActUseRequestDueSum равно "Истина".

В контекстном меню (вызывается правой кнопкой мыши) доступен пункт:

·Госпошлина с подпунктами:

·Рассчитать ГП;

·Запросить ГП;

·Отметить оплату ГП.

Эти пункты доступны на вкладках модуля:

·Приказы;

·Иски.

Рассчитать ГП

Расчет госпошлины из вкладки Приказы доступен только пользователям, в роли которых доступ № 444 LAW_ACT_ORDER_SET_DUE имеет значение TRUE.

Расчет госпошлины из вкладки Иски доступен только пользователям, в роли которых доступ № 449 LAW_ACT_LAWSUIT_SET_DUE имеет значение TRUE.

Функционал, доступный по кнопке "Рассчитать ГП" осуществляет для выделенных приказов или исков действия, аналогичные вычислению расчетной пошлины в карточке судебной работы в приказном или исполнительном производстве соответственно.

Примечание. Автоматический расчет госпошлины осуществляется только после заполнения информации о суде.

Описание алгоритма автоматического расчета госпошлины — см. Алгоритм расчета госпошлины.

Примечание. Расчет госпошлины для дел с нулевым размером требования не осуществляется.

Запросить ГП

Запрос госпошлины из вкладки Приказы доступен только пользователям, в роли которых доступ № 507 LAW_ACT_ORDER_REQUEST_DUE имеет значение TRUE.

Запрос госпошлины из вкладки Иски доступны только пользователям, в роли которых доступ № 508 LAW_ACT_LAWSUIT_REQUEST_DUE имеет значение TRUE.

Действие осуществляется аналогично описанному в разделе Запрос оплаты госпошлины для карточки судебной работы в приказном или исковом производстве, только по нескольким делам. Действие может осуществляться только над приказами или исками, находящимися в статусах "Не оформлен" / "Не подготовлен" или "Запрос оплаты госпошлины". Предварительно у выбранных дел должна быть рассчитана госпошлина.

При окончании операции "Запросить госпошлину" выводится сообщение вида:

"Операция выполнена. Выделено дел: X. Для Y дел выполнен запрос оплаты госпошлины.",

где:

·X — количество дел, для которых была вызвана операция "Запросить госпошлину";

·Y — количество дел из выделенных, для которых операция не была выполнена или не выполнялась вообще.

Отметить оплату ГП

Отметка оплаты госпошлины из вкладки Приказы доступна только пользователям, в роли которых доступ № 507 LAW_ACT_ORDER_REQUEST_DUE имеет значение TRUE.

Отметка оплаты госпошлины из вкладки Иски доступна только пользователям, в роли которых доступ № 508 LAW_ACT_LAWSUIT_REQUEST_DUE имеет значение TRUE.

Примечание. Действие  не отображается, если константа LawAct.DueSumPaid.Use имеет значение "Ложь", а константа LawActUseRequestDueSum имеет значение "Истина".

Действие осуществляется аналогично описанному в разделе Госпошлина оплачена для карточки судебной работы в приказном производстве, только по нескольким делам.

Учет требований неимущественного характера

Только в исковом производстве и если значение константы LawActIskDueSumNoPropertyCharacter больше нуля, при расчете ГП осуществляется следующий шаг:

Появляется окно с предложением учесть требования неимущественного характера. В зависимости от ответа:

·Если вы выберите Yes, то к сумме расчетной госпошлины будет прибавлено значение константы LawActIskDueSumNoPropertyCharacter.

·Если вы выберите No, то к сумме расчетной госпошлины ничего прибавляться не будет.
