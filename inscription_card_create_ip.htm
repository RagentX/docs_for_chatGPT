Создать ИП

Show/Hide Hidden Text

Создание исполнительного производства по делам судебной работы доступно только пользователям, в роли которых доступ № 603 LAW_ACT_TRANSFER_TO_EXEC имеет значение TRUE.

В карточке исполнительной надписи кнопка Создать ИП доступна только если дело находится в статусе, для котрого разрешен перевод на ИП: 1007 "Надпись получена" (см. словарь № 25).

Порядок действий

Для перевода исполнительной надписи на исполнительное производство выполните следующее:

1.В карточке судебной работы в исполнительной надписи нажмите кнопку Исполнительная стадия и в открывшемся списке выберите пункт Создать ИП.

2.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить перевод.

3.Если в Системе настроена константа LawAct.TransferToExec.Ban.DebtStatusList, то осуществляется проверка статуса долга, связанного с данным делом. Если долг находится в одном из статусов, перечисленных в значении константы, то создание ИП не разрешено. Выполнение действия будет прервано, будет показано сообщение:

Связанный с делом долг имеет статус, для которого запрещен перевод на ИП.

4.Осуществляется проверка доступа 357 и наличия ранее созданного ИП по данному делу.

       Подробнее (кликните чтобы развернуть или свернуть список)

Проверка доступа 357 и наличия ранее созданного ИП по данному делу:

·Если доступ № 357 LAW_EXEC_MULTIPLE включен:

oЕсли ИП по данному долгу не существует, то создается новое ИП, связанное с данным делом.

oЕсли ИП по данному долгу существует, то пользователю выдается сообщение. Если пользователь выбирает продолжить формирование, то формируется новое ИП, связанное с данным делом.

·Если доступ № 357 LAW_EXEC_MULTIPLE выключен:

oЕсли константа LawExec.Create.CheckActiveStatus.Use включена, то:

§Если существует ИП в активном статусе для долга, то новое ИП создано не будет.

§Если не существует ИП в активном статусе для долга, то создается новое ИП, связанное с данным делом.

oЕсли константа LawExec.Create.CheckActiveStatus.Use выключена, то:

§Если по долгу нет ни одного ИП в любом статусе, то создается новое ИП, связанное с данным делом.

§Если по долгу есть ИП в любом статусе, то пользователю выводится сообщение, а новое ИП не создается.

Информация для поддержки

Доступы и константы

Полная схема проверок включает в себя проверки:

·Доступа № 357 LAW_EXEC_MULTIPLE;

·Обычной константы LawExec.Create.CheckActiveStatus.Use;

·Технической константы LawExec.Create.Return.Use.

Изначально для обычных клиентов они настроены:

·Доступ № 357 LAW_EXEC_MULTIPLE — по-разному;

·Константа LawExec.Create.CheckActiveStatus.Use = Истина;

·Константа LawExec.Create.Return.Use = Ложь.

Для Почта банка:

·Доступ № 357 LAW_EXEC_MULTIPLE = FALSE;

·Константа LawExec.Create.CheckActiveStatus.Use = Ложь;

·Константа LawExec.Create.Return.Use = Истина.

 Внимание! Важно! Пользовательский текст справки написан именно для этих двух случаев — для всех и для Летобанка (Почта банка).

Порядок проверок при создании ИП

·Если доступ № 357 LAW_EXEC_MULTIPLE включен:

oЕсли ИП по данному долгу не существует, то создается новое ИП, связанное с данным делом.

oЕсли ИП по данному долгу существует, то пользователю выдается сообщение. Если пользователь выбирает продолжить формирование, то формируется новое ИП, связанное с данным делом.

·Если доступ № 357 LAW_EXEC_MULTIPLE выключен:

oЕсли константа LawExec.Create.CheckActiveStatus.Use включена, то:

§Если существует ИП в активном статусе для долга, то новое ИП создано не будет.

§Если не существует ИП в активном статусе для долга, то создается новое ИП, связанное с данным делом.

oЕсли константа LawExec.Create.CheckActiveStatus.Use выключена, то:

§Если по долгу нет ни одного ИП в любом статусе, то создается новое ИП, связанное с данным делом.

§Если по долгу есть ИП в любом статусе, то пользователю выводится сообщение.

oЕсли константа LawExec.Create.Return.Use включена, то:

§Если по долгу существует ИП в активном статусе, то новое дело не создается;

§Если по долгу существует ИП не в активном статусе, то оно возобновляется, и для него устанавливается связь с данным судебным делом.

oЕсли константа LawExec.Create.Return.Use выключена, то новое ИП не создается.
