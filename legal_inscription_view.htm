Просмотр данных об исполнительной надписи

Данные о правопреемстве отображаются в форме Юридический департамент на вкладке Исполнительная надпись.

На вкладке отображаются исполнительные надписи, находящиеся в активных статусах (см. описание активных и архивных статсуов исполнительной надписи в описании ЖЦ надписи).

Для отображения архивных записей включите флаг Показывать архивные записи.

Вкладка содержит следующие поля:

·ID должника — идентификатор должника, с которым связано дело о правопреемстве;

·ID долга — идентификатор долга, с которым связано дело о правопреемстве;

·        ID дела — уникальный идентификатор (порядковый номер) дела в системе;

·        ФИО — фамилия, имя, отчество ответчика по делу (должника или поручителя);

·Дело поступило — дата поступления дела;

·Тип лица — тип ответчика (должника или поручителя), см. словарь № 54;

·        Размер требования — сумма долга, выводится в валюте долга, указанной в поле Валюта; выводится сумма значений по выборке;

·        Банк;

·        Портфель;

·Реестр КА;

·        Ответственный — оператор, ответственный за работу с делом;

·        Статус — статус судебного дела, см. словарь № 25;

·Номер договора — номер кредитного договора;

·        Лицевой счет — номер счета долга;

·Статус долга — см. Описание системных статусов долга;

·ID Клиента — дополнительный идентификатор долга в системе;

·Валюта — валюта долга (см. Список валют);

·Гос. пошлина — сумма госпошлины, выводится в основной валюте Системы; выводится сумма значений по выборке;

·Дата подачи — дата подачи исполнительной надписи;

·ФИО Нотариуса;

·        Комментарий;

·Дата проставления надписи;

·        Цвет — выделение цветом для удобства работы;

·Название продукта — указывается название продукта, по которому возник долг;

·Тип продукта — см. словарь № 11;

·        Остаток — сумма остатка, выводится в валюте долга, указанной в поле Валюта; выводится сумма значений по выборке;

·Дата рождения — дата рождения ответчика (должника или поручителя);

·Серия — серия паспорта ответчика-должника (если ответчик — поручитель, поле остается пустым);

·Номер — номер паспорта ответчика-должника (если ответчик — поручитель, поле остается пустым);

·        Филиал — филиал организации, к которому относится долг (см. словарь № 61);

·        Регион — регион, в котором находится филиал организации (см. словарь № 39);

·Номер исполнительной надписи;

·Сумма для взыскания по исполнительной надписи.

Примечание. Область, на которой отображается информация о датах, способах и результатах контактов с должником, информация о переводе долга на аутсорсинг и возврата из аутсорсинга, доступна для просмотра на форме, если константа DebtListContactHistory имеет значение "Истина". При этом в списке не отображаются контакты тех пользователей, чьи идентификаторы перечислены в значении константы ContactHistory.ContactLog.IgnoreUsersList.
