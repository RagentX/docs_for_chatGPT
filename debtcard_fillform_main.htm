Основные сведения о должнике

К основным сведениям о должнике относятся ФИО должника, его номер в системе, тип должника и ответственный за работу с должником. Для того чтобы указать основные сведения о должнике, в карточке должника используются следующие поля:

·№ — номер должника в системе. Данное поле заполняется автоматически при сохранении созданной карточки должника и недоступно для редактирования;

·Фамилия, Имя, Отчество / Компания — фамилия, имя, отчество должника (для физических лиц) или название компании (для юридических лиц). Поле Фамилия/Компания обязательно для заполнения;

·Тип — тип должника (см. словарь № 54);

·Ответственный — пользователь, ответственный за работу с долгом. Долги закрепляются вручную руководителями в РМ "Постановка задач" (см. Назначение ответственного оператор по долгам) или автоматически по факту регистраций контактов с должником;

·Краткая информация — краткая информация о должнике и его долгах. Данное поле заполняется автоматически и недоступно для редактирования.

Блок краткой информации можно настроить или оставить по-умолчанию.

·Размер вознаграждения — сумма вознаграждения от всей суммы долга должника (текущая сумма задолженности * % вознаграждения). Данное поле заполняется автоматически и недоступно для редактирования.

Информационный блок по умолчанию

В случае если информационный блок не настроен, область Краткая информация содержит следующие данные:

·Если требуется верификация контактных данных, то  первая запись в области краткой информации — это текст "ТРЕБУЕТСЯ ВЕРИФИКАЦИЯ ТЕЛЕФОНОВ", набранный красным цветом;

·Всего долгов;

·Прикреплено ссылок;

·Документов;

·Текущий долг: Сумма

·Портфель;

·Телефон;

·Начало работы;

·Окончание работы;

·Договор;

·Лицевой счет.

Настройка информационного блока для карточки должника

Настройка информационного блока с помощью формулы

Этот способ дает возможность:

·Создавать разные информационные блоки для разных долгов, разных пользователей и т.д.

·Создать информационный блок путем конкатенации более одного сценария сообщения типа "Специальное".

Для того чтобы настроить информационный блок с использованием формулы:

1.В модуле Произвольные формулы создайте формулу, имеющую тип Шаблон "Краткой информации" (например, такую), возвращающей в качестве результата своей работы список id сценариев сообщений типа "Специальное" через запятую.

2.Присвойте номер созданной формулы константе DebtCardMainInfoFormula. (для карточки должника) или CallCenter.MainInfo.Formula (для модуля Центр обзвона должников).

Примечание. Если в формуле текст информационного блока зависит от активного долга, то информационный блок "Краткая информация" определяется заново при смене активного долга на вкладке "Долги".

Настройка информационного блока без использования формулы

Этот способ применяется в случаях, когда нет необходимости в использовании формулы для настройки информационного блока, а также если:

·константа DebtCardMainInfoFormula не настроена;

·константа DebtCardMainInfoFormula настроена некорректно или ссылается на несуществующую формулу;

·формула, на которую ссылается константа, существует, но не имеет тип Шаблон "Краткой информации";

·формула ссылается на несуществующие шаблоны сообщений и т.д.

Для того чтобы настроить информационный блок для карточки должника без использования формулы:

1.Войдите в модуль Утилиты а Сценарии сообщений.

2.Активируйте вкладку Специальное.

3.Создайте сценарий обычным способом (добавление/редактирование информационного блока).

4.Запомните ID сценария и впишите его в значение константы DebtMainInfoFormat после символов "(1:".

Примечание. Если значение константы DebtMainInfoFormat пусто или после символов "(1:" указано число, которое не является ID сценария специального сообщения, то в области краткой информации отображаются данные по-умолчанию.

Примечание. Если в значении константы DebtMainInfoFormat скобка № 2 и далее отсутствуют, то в соответствующей юридической карточке краткая информация не отображается.

Примечание. Для модуля Центр обзвона должников. Если в значении константы DebtMainInfoFormat скобка № 6 отсутствует или константа пуста, а также константа CallCenter.MainInfo.Formula равна 0 или ссылается на несуществующую или некорректную формулу, то область краткой информации на форме Центр обзвона должников отсутствует.

Примечание. Для карточки должника и карточек судебной работы можно настроить как различные сценарии сообщений, так и одинаковые.
