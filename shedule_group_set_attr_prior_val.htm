Установка приоритетного значения атрибута (код 34)

См. далее — Логика работы мероприятия.

Порядок действий

1.Выберите тип мероприятия — Установка приоритетного значения атрибута (код 34). Откроется форма "Мероприятие" для  данного типа мероприятий.

2.Выберите группу объектов (объект "Должники"), для обработки которых предназначено мероприятие.

3.Настройте параметры мероприятия:

·Атрибут — открывается список используемых в Системе скоринговых баллов. Выберите одно значение, поле обязательно для заполнения;

·Приоритет — открывается форма, аналогичная форме мультифильтра для соответствующего скорингового балла. Выберите значения в нужном порядке. Поле обязательно для заполнения;

·Исключение — открывается форма, аналогичная форме мультифильтра для соответствующего скорингового балла. Выберите значения исключений. Поле не обязательно для заполнения;

·Флаг Исключить закрытые долги — для исключения из мероприятия закрытых долгов выбранных должников. По умолчанию включен.

Примечание. Словари для хранения значений скоринговых баллов (имеют названия, соответствующие настройке данных атрибутов):

·Балл № 1 — словарь 101;

·Балл № 2 — словарь 102;

·Балл № 3 — словарь 244;

·Балл № 4 — словарь 245.

4.Настройте График запуска мероприятия.

5.Заполните поля:

·Последовательность — при необходимости введите числовое значение последовательности. В случае одновременного запуска двух и более мероприятий первым будет выполняться задание с наименьшим значением в поле.

·Флаг "Вкл." — если флаг находится в положении Включено, мероприятие запускается, если флаг находится в положении Выключено, мероприятие не запускается. Использование флага дает возможность отменить запуск мероприятия, не удаляя его.

·Флаг "Проверять формирование групп". При включенном флаге мероприятие выполняется только для тех групп, которые были сформированы или переформированы в соответствии со следующими условиями:

·Если значение константы Scheduler.ReformGroupCheck.Mode равно 2, то группа считается переформированной, если дата формирования больше или равна текущей дате.

·Если значение константы Scheduler.ReformGroupCheck.Mode равно 1, то группа считается переформированной, если дата формирования больше текущей даты и времени минус 1 сутки (текущая логика).

·Если значение константы Scheduler.ReformGroupCheck.Mode не равно 1 или 2, то используется логика режима 1.

Внимание! Если мероприятие в модуле Расписание не предполагает использование автогруппы, флаг Проверять формирование группы должен находиться в положении Выключено.

·Комментарий — дополнительная информация.

6.Нажмите OK.

Логика мероприятия (job_schedule_group_event_3x)

По всем долгам должников группы из входного параметра выполняется изменение значения указанного атрибута по следующей логике:

·Среди всех долгов должника определяется наиболее приоритетное значение атрибута в соответствии с приоритетом из входного параметра Приоритет. Значения располагаются в порядке увеличения приоритета. Если значения нет в списке приоритетов, то оно не учитывается;

·Для всех остальных долгов должника устанавливается данное значение. Исключение составляются те долги, у которых текущее значение атрибута перечислено во входном параметре Исключения (для них значение не меняется).

Если включен чекбокс Исключить закрытые долги, то из логики определения приоритетного значения атрибута исключаются все долги в статусе 6, 7, 8. Также эти долги не должны менять значение атрибута.
