Функционал Планирование (Задачи). Информация для поддержки.

Здесь изложено подробное описание особенностей настройки и работы технической константы Task.TypeTaskInList.Mode.

Подробное описание настройки константы Task.TypeTaskInList.Mode

В константе Task.TypeTaskInList.Mode описываются типы отображаемых задач в модулях. Значения хранятся в круглых скобках и указываются через запятую. Каждая пара скобок отвечает за отдельный модуль. Возможны следующие типы задач (в БД тип задачи определяется по полю debt_task.obj_id): 

·8 — задачи по долгу.

·46 — задачи по судебной работе.

·47 — задачи по ИП.

·56 — задачи по имуществу.

·30 — задачи по должнику.

В настоящее время первая пара скобок отвечает за модуль law_act_card, вторая пара скобок за модуль law_exec_card, а их объединение за модуль law_act_list. Для сохранения работоспособности текущего функционала по видимости задач, в перечне значений константы Task.TypeTaskInList.Mode дополнительно добавляем справа новые пары скобок, в которых продублируем значения текущих двух скобок, изменяя коды модулей согласно следующей таблице соответствий.

Также необходимо дополнительные добавить 6 пар скобок, которые будут регулировать отображение задач в следующих модулях: "Карточка должника", "Юридический департамент", "Список должников", "Постановка задач", "Обработка должников", "Группы свободного поиска".

Значения в скобках для модулей "Карточка должника", "Список должников", "Постановка задач", "Обработка должников" указываются по аналогии с остальными модулями. 

 Примеры:

·(8:8,46,47) – в списке должников видны задачи по долгу, судебным делам и ИП;

·(8:8,46,47б56) – в списке должников видны задачи по долгу, судебным делам, ИП и имуществу;

·(13:8) – в постановке задач видны только задачи по долгу;

·(13:8,46) – в постановке задач видны задачи по долгу и судебным делам;

·(13:8,46,47) – в постановке задач видны задачи по долгу, судебным делам и ИП;

·(14:8,46,47) – в обработке должников видны задачи по долгу, судебным делам и ИП;

·(132:8) – в ГСП видны только задачи по долгу;

·(132:8,46) – в ГСП видны задачи по долгу и судебным делам;

·(132:8,46,47) – в ГСП видны задачи по долгу, судебным делам и ИП.

Работа константы Task.TypeTaskInList.Mode в модулях Системы

Добавление полей на вкладку "Задачи"

На вкладку "Задачи" добавлено две колонки:

· ID дела;

·Тип.

Колонка "ID дела" добавлена на вкладку "Задачи" в следующих модулях:

·Карточка должника;

·Список должников;

·Постановка задач;

·Обработка должников;

·Группы свободного поиска.

Колонка "Тип" видна всегда на вкладке "Задачи" в следующих модулях:

·Юридический департамент;

·Карточка судебной работы;

·Карточка исполнительного производства;

·Группы свободного поиска.

В следующих модулях на вкладке "Задачи" видимость колонки "Тип" определяется настроенной в константе Task.TypeTaskInList.Mode видимостью задач по судебным делам и/или ИП:

·Карточка должника;

·Список должников;

·Постановка задач;

·Обработка должников.

В колонке "Тип" для задач, созданных по долгу, отображается значение "Долг". Для задач, созданных по правопреемству, отображается значение "Правопреемство".

Отображение задач в модуле "Группы свободного поиска"

В модуле "Группы свободного поиска" добавлена вкладка "Задачи", которая расположена последней. Вкладка аналогична вкладке "Задачи" в модуле "Юридический департамент". Видимость типов задач определяется значением константы Task.TypeTaskInList.Mode (список типов указан в паре скобок с кодом модуля 132).

Видимость вкладки зависит от следующих условий:

·Если константа @UseDebtTask имеет значение "Ложь", то функционал "Планирование" не поддерживается, и вкладка не отображается.

·Если константа @UseDebtTask имеет значение "Истина", то видимость вкладки определяется наличием в роли пользователя доступа № 662 GROUP_VIEWER_DEBT_TASK_ VIEW:

·Если доступ имеет значение TRUE, то вкладка видна;

·Если доступ отсутствует или имеет значение FALSE, то вкладка не отображается.
