Общее описание

Конфигурационный файл

Сервис имеет конфигурационный файл, в котором настраиваются в том числе следующие параметры:

·Параметры соединения с БД;

·Параметры авторизации, если это необходимо для доступа к xml-фиду по ссылке;

·Путь к файлу xml;

·Название файла xml.

Представление БД для имущества, подлежащего экспонированию

Создано представление БД, содержащее список имущества, подлежащего экспонированию.

Для данного списка выбираются записи об имуществе по следующим условиям:

·значение параметра имущества "Процессы. Экспонирование" равно "В процессе";

·значение параметра имущества "Утвержденная стоимость" не равно NULL;

·по имуществу поставлена задача по процессу "Экспонирование", подтип "Публикация предложения", статус задачи не равен "Готова к проверке";

·по имуществу поставлена задача по процессу "Экспонирование", подтип "Стратегия экспонирования", статус задачи равен "Готова к проверке", результат по задаче равен "Онлайн".

Автоматический поиск имущества, подлежащего экспонированию

Проверка наличия новых записей, подходящих для публикации информации о них на площадках по недвижимости, осуществляется каждый день автоматически.

·Если значение параметра имущества "Экспонирование. ЦИАН" равно "Истина", то имущество попадает в файл в формате XML для ресурса ЦИАН.

·Если значение параметра имущества "Экспонирование. ЯндексНедвижимость" равно "Истина", то имущество попадает в файл в формате XML для ресурса "Яндекс.Недвижимость".

·Если значение параметра имущества "Экспонирование. Авито" равно "Истина", то имущество попадает в файл в формате XML для ресурса "Авито".

Сформированный файл XML выкладывается в сетевую папку, указанную в конфигурационном файле данного сервиса недвижимости.  Далее каждый день происходит повторный выбор из списка имущества и загрузка заново сформированного XML файла, с тем же именем и расширением, в эту же папку.  В обновленном файле должны содержаться все записи об имуществе, которые на момент формирования списка имеют задачу по реализации имущества.  При этом внешний идентификационный номер каждой записи имущества не изменяется.

Сервис автоматически проверяет наличие новых записей об имуществе при обновлении даты изменения файла и создает объявления по ним.
