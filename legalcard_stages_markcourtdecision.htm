Отметка о решении суда

Действия по кнопке Решение суда доступны для приказов в статусах:

·Подан;

·Возобновление;

·Отмена приказа (если LawAct.OrderCancel.IsFinal имеет значение "Истина").

В зависимости от значения константы LawAct.Order.SendAndReceive.Person.Use положительное решение суда будет разным:

·Если константа имеет значение "Истина", это пункт Приказ отправлен должнику. В результате дело переводится в статус "Приказ отправлен должнику". Далее следует выполнить этап Получение приказа.

·Если константа имеет значение "Ложь", это пункт Получение приказа, а в результате дело переводится в статус "Приказ получен".

Получение приказа

Данный пункт доступен только при значении константы LawAct.Order.SendAndReceive.Person.Use, равном "Ложь".

Для отметки получения приказа выполните следующие действия:

1.В карточке судебной работы на вкладке Приказное производство нажмите кнопку Решение суда, затем в выпадающем списке выберите Получение приказа.

2.Откроется форма Приказ получен. Заполните следующие поля:

·Дата получения — введите дату. Поле обязательно для заполнения;

·Комментарий — дополнительная информация.

3.Нажмите OK.

4.Выполните прикрепление документа.

5.Далее осуществляется формирование настроенных запланированных действий.

6.Дело будет переведено в статус "Приказ получен", перевод — запротоколирован.

После перевода дела в статус "Приказ получен" в карточке будут доступны следующие кнопки ЖЦ:

·Возражения.

Приказ отправлен должнику

Данный пункт доступен только при значении константы LawAct.Order.SendAndReceive.Person.Use, равном "Истина".

Для установки отметки о решении суда "Приказ отправлен должнику" выполните следующие действия:

1.В карточке судебной работы на вкладке Приказное производство в разделе Судебный приказ заполните поля:

·Дата вынесения решения;

·Способ доставки — способ доставки судебного приказа должнику, см. словарь № 17.

2.Нажмите кнопку Решение суда и в раскрывшемся списке выберите Приказ отправлен должнику.

3.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить выполнение операции. Если какие-либо обязательные поля не заполнены, будет выдано информационное сообщение. Нажмите в нем кнопку ОК, после чего произойдет возврат в карточку судебной работы для внесения данных.

4.Выполните прикрепление документа.

5.Далее осуществляется формирование настроенных запланированных действий.

6.После этого приказ будет переведен в статус "Приказ отправлен должнику".

Отказ в выдаче приказа

Для установки отметки о решении суда "Отказ в выдаче приказа" выполните следующие действия:

1.В карточке судебной работы на вкладке Приказное производство заполните поле Комментарий.

2.Нажмите кнопку Решение суда и в раскрывшемся списке выберите Отказ в выдаче приказа.

3.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить выполнение операции. Если поле Комментарий не заполнено, будет выдано информационное сообщение. Нажмите в нем кнопку ОК, после чего произойдет возврат в карточку судебной работы для внесения данных.

4.Выполните прикрепление документа.

5.Далее осуществляется формирование настроенных запланированных действий.

6.После этого приказ будет переведен в статус "Отказ в выдаче приказа", для него будет доступна операция подготовки заявления в суд.

Примечание. Если константа LawAct.Decision.Components.Compare имеет значение "Истина", то при выполнении действия Отказ в выдаче приказа дополнительно сравниваются текущие значения составляющих на момент подачи в суд со значениями составляющих по решению суда (составляющие для приказа перечислены в константе LawActOrderDebtComponentList), а также значение размера пошлины с утвержденным размером пошлины.

Если хотя бы одна пара значений не совпадает, пользователю отображается окно с сообщением:

Поданные и присужденные суммы не совпадают. Продолжить?

Нажмите ОК чтобы продолжить или Отмена чтобы прервать действие.

Примечание для поддержки

Если был выбран Отказ в выдаче приказа, то производится дополнительная проверка обязательности заполнения следующих полей карточки приказного производства, согласно значению технической константы LawAct.RejectResult.Fields.Mandatory:

·Утвержденная пошлина — если в скобке с единицей есть значение 741;

·Сумма по решению суда — если в скобке с единицей есть значение 740;

·Комментарий — если в скобке с единицей есть значение 433.

Возврат приказа

Данный пункт доступен, если константа LawAct.OrderReturn.Use имеет значение "Истина".

Для того чтобы осуществить возврат приказа, выполните следующие действия:

1.В карточке судебной работы на вкладке Приказное производство нажмите кнопку Решение суда, затем в выпадающем списке выберите Возврат приказа.

2.Откроется форма Возврат приказа. Заполните следующие поля:

·Дата возврата — укажите дату, поле обязательно для заполнения;

·Причина — выберите причину из списка (доступны термины словаря № 320 "Причина перевода приказа в текущий статус", имеющие в качестве родительского термин "Возврат приказа");

·Комментарий — дополнительная информация.

3.Нажмите OK.

4.Выполните прикрепление документа.

5.Далее осуществляется формирование настроенных запланированных действий.

6.Дело будет переведено в статус "Возврат приказа", перевод — запротоколирован.

После перевода дела в статус "Возврат приказа" в карточке будут доступны следующие кнопки ЖЦ:

·Подготовить;

·Запрос госпошлины – Запрос доплаты ГП.
