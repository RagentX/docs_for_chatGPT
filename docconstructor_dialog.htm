Диалоговое окно

Иногда возникает необходимость вывести в письме информацию, заданную непосредственно в момент формирования, чтоб не редактировать каждый раз шаблон перед формированием, можно настроить вывод диалогового окна, в котором пользователь задает нужные значения для формирования писем. Рассмотрим данную настройку на примере ситуации, когда в письмо требуется вывести ФИО и должность подписанта.

Примечание. На текущий момент в системе подобную задачу можно решить следующими способами:

1.Переменные в шаблоне письма связанные с константами вида "RepDebtLetter…". Например, sign_fio - ФИО лица, подписывающего письмо (константа RepDebtLetterSignFio), sign_pos - Должность лица, подписывающего письмо (константа RepDebtLetterSignPos). Удобно использовать, когда на всех шаблонах одно и тоже лицо выступает в качестве подписанта.

2.Переменные вида "u_...", например u_fio и u_position, ФИО и должность оператора за которым закреплен долг.

3.Использовать собственный SQL запрос.

4.Использовать диалоговое окно. Этот способ и описывается ниже.

Порядок действий

1.В контекстном меню по правой кнопке на вкладке "Page" выбираем пункт "Новая форма".

2.Создается пустая форма "DialogPage", на которой вы можете создать нужную вам форму диалогового окна, с полями для ввода, кнопками и всем необходимым. В левой части выделяем объект "DialogPage"  и на вкладке "Свойства" в поле "Caption" задаем текст "Входные параметры", который будет отображаться в названии диалогового окна.

3.Добавляем на форму:

·Label - подпись для поля

·Edit - поле для ввода текста

·BitBtn  - кнопка с картинкой (Button просто кнопка с текстом)

4.Label.

Выделяем объект Label, в левой части на вкладке "Свойства" в поле "Caption" задаем текст "ФИО подписанта", который будет отображаться в качестве подписи поля, и задаем размеры объекта или мышкой или в полях Height, Left, Top, Width.

Повторяем процедуру для второго объекта, для которого вводим текст "Должность подписанта".

5.Edit.

Выделяем объект Edit, в левой части на вкладке "Свойства" в поле "Name" задаем "edFIO". При необходимости в поле "Text" можно задать текст по умолчанию, который будет предварительно задан в поле. Также задаем размеры объекта или мышкой или в полях Height, Left, Top, Width.

Повторяем процедуру для второго объекта, для которого в поле "Name" задаем "edPosition".

6.BitBtn.

Выделяем объект BitBtn, в левой части на вкладке "Свойства" в поле "Caption" задаем текст "OK", который будет отображаться на кнопке, в поле "Kind" выбираем тип кнопки "bkOK". При необходимости задаем размеры объекта или мышкой или в полях Height, Left, Top, Width.

Повторяем процедуру для второго объекта, для которого в поле "Caption" задаем текст "Cancel", который будет отображаться на кнопке, в поле "Kind" выбираем тип кнопки "bkCancel".

7.Выделяем объект "MasterData1", переходим на вкладку "События" и делаем двойной клик в поле "OnBeforePrint", в результате чего автоматически попадаем на вкладку "Код" с автоматическим созданием процедуры. В блоке процедуры MasterData1OnBeforePrint помещаем следующий код:

UserName := edFIO.Text;

UserPosition := edPosition.Text;

Перед процедурой MasterData1OnBeforePrint добавляем переменные:

var UserName, UserPosition : String;

В результате должен получиться следующий код (!при условии что у вас ранее в этой процедуре не был задан другой код)

8.Далее переходим на вкладку "Page" и размещаем в нужном месте переменные [UserName] и [UserPosition].
