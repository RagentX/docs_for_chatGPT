Загрузка данных о статусах долгов в базу данных

Для загрузки данных в БД выполните следующие действия:

1.В форме Утилита изменения статуса долга нажмите кнопку Сохранить в БД. После этого кнопки Проверить, Сохранить в БД и Отменить, а также возможность редактирования вручную ячеек таблицы будут недоступны. В процессе загрузки данных в нижней строке формы будет отображаться тип операции (загрузка данных), текущие данные о проценте выполнения работы, директория и имя файла.

2.При наличии изменений в таблице перед загрузкой автоматически проводится повторная проверка данных в таблице согласно разделу Проверка данных. По результатам проверки:

·если в таблице отсутствуют записи, не содержащие критических ошибок, будет выдано сообщение: "Загрузка данных отменена из-за отсутствия записей, которые могут быть загружены в БД";

·если в таблице имеются ошибочные записи, будет выдан запрос на формирование файла ошибочных записей. Если требуется сформировать файл, нажмите Yes. В этом случае будет автоматически сформирован файл в формате MS Excel, содержащий ошибочные записи. В процессе формирования файла с ошибочными записями в нижней строке формы будет отображаться тип операции (формирование файла ошибочных записей) и текущие данные о проценте выполнения работы. По окончании формирования файла будет выдано информационное сообщение с указанием директории и имени файла с ошибочными записями. Нажмите кнопку ОК, чтобы закрыть его;

·если в таблице имеются и корректные, и ошибочные записи, перед загрузкой будет выдан запрос: "Внимание: имеются записи (<количество ошибочных записей> из <общее количество записей>), которые не могут быть загружены в БД. Сформировать файл ошибочных записей?". При положительном ответе (кнопка Yes) будет сформирован файл MS Excel с ошибочными записями. По окончании формирования файла будет выдано сообщение: "Ошибочные записи сохранены в файле <директория и имя файла>".

3.При наличии записей с некритическими ошибками (ФИО должника не соответствует ФИО для найденного долга) в процессе загрузки данных осуществляется диалог с пользователем, который принимает решение о загрузке или отказе от загрузки для нестандартных записей (см. Обработка записей с некритическими ошибками).

4.В ходе загрузки данных выполняется корректировка следующей информации в БД:

·о долгах (кредитах) в таблице debt;

·о задачах на обзвон для операторов в таблице work_task;

·о статусе долга в таблице ms_hard_status.

5.По окончании загрузки данных будет выдано сообщение о количестве успешно загруженных записей: "Успешно загружено <количество записей> записей". Нажмите кнопку ОК, чтобы закрыть сообщение. Кнопка Открыть файл вновь станет доступной, а кнопки Проверить, Сохранить в БД и Отменить — недоступны.

По окончании загрузки в БД записей, не содержащих ошибок, можно вернуться к ошибочным записям. После удаления из файла первого столбца (причины ошибок) он может быть использован в качестве исходного файла MS Excel для утилиты изменения статуса долга.
