Загрузка данных в базу данных

Для загрузки данных в БД выполните следующие действия:

1.В форме Загрузка данных нажмите кнопку Сохранить в БД.

Перед загрузкой автоматически проводится проверка данных в таблице согласно разделу Проверка данных. При выполнении проверки данных в нижней строке формы будет отображаться тип операции (проверка данных), текущие данные о проценте выполнения работы, директория и имя файла.

При наличии ошибки первого рода на экран выводится одно из сообщений:

·"Ошибка в формате. Повторное описание столбца <наименование>";

·"Ошибка в формате. Отсутствует описание столбца <наименование>".

Загрузка данных в БД не производится, состояние формы не изменяется.

При наличии ошибок второго или третьего рода (в отсутствии ошибок первого рода) будет выдано сообщение "Внимание: имеются записи (<число> из <общее число>), которые не могут быть загружены в БД. Сформировать файл ошибочных записей с их удалением из таблицы?". При положительном ответе (кнопка Yes) все ошибочные записи будут удалены из таблицы. В файле ошибочных записей будут сформованы исходные тексты записей и причины ошибок в текстовой форме. Будет выдано сообщение "Ошибочные записи сохранены в файле <директория и имя файла>".

Примечание. Для каждого ошибочного реквизита записи причина ошибки представляется в виде отдельной текстовой строки. Для реквизитов адреса и телефона название реквизита дополняется типом адреса / телефона и номером столбца исходного текста записи.

Примечание. По окончании загрузки в БД записей, не содержащих ошибок, пользователь имеет возможность вернуться к ошибочным записям. Файл ошибочных записей соответствует тому же формату, что исходный файл, и после удаления первого столбца (причины ошибок) может быть обработан заново.

2.Для загрузки в БД оставшихся в таблице записей повторно нажмите кнопку Сохранить в БД.

3.Если булевская константа LoadDataUseDataq имеет значение "Истина", а загружаемая информация содержит адреса, то откроется форма чистки адресов с помощью сервиса DaData. Форма содержит статистическую информацию об адресах и переключатели для выбора варианта работы:

·Не производить — обращение к системе DaData не производится;

·Производить только для адресов с ошибками;

·Производить для всех адресов.

4.В форме установите переключатель для нужного варианта и нажмите кнопку ОК.

Примечание. Если выбраны варианты с использованием системы DaData, то следует учитывать, что обращение производится отдельно по каждому адресу. Размер оплаты пропорционален количеству адресов.

5.Если пользователь имеет право на сохранение текста адресов без ссылки на справочные данные, отсутствуют ошибки первого и второго рода (ячейки желтого цвета), но имеются ошибки третьего рода (ячейки оранжевого или красного цвета), то будет выдано сообщение "Внимание: имеются записи (<число записей> из <общее число записей>), которые могут быть загружены только в текстовом виде. Подтверждаете загрузку?". Для подтверждения загрузки, а также если ошибки отсутствуют, нажмите кнопку Yes.

6.Если в формате отсутствует название портфеля и образующие колонки портфеля заданы частично, будет выдано предупреждение "Не все образующие колонки портфеля заданы в формате". Если название портфеля и полный набор образующих колонок отсутствуют, а также при наличии названия портфеля и хотя бы одной строки таблицы без ошибок и с пустым названием портфеля откроется форма выбора кредитора (банка) и портфеля (реестра). Поле Портфель обязательно для заполнения. После заполнения нажмите кнопку OK.

Примечание. Особенности загрузки данных из файлов с типом формата "Дозагрузка поручителей" описаны в разделе Особенности загрузки файлов с типом формата "Дозагрузка поручителей".

7.После этого начнется загрузка данных из таблицы в БД. В процессе загрузки данных в нижней строке формы отображается тип операции (загрузка данных), текущие данные о проценте выполнения работы, директория и имя файла.

8.В процессе загрузки данных может потребоваться обработка нестандартных записей.

Примечание. Если при загрузке не указан тип персоны, по умолчанию загружается person.typ=1 (Физ. лицо).

9.В случае невозможности загрузки в БД какой-либо записи будет выдано сообщение "Ошибка при загрузке записи <№ номер записи>. Не удается загрузить данные о паспорте (персоне, телефонах, адресах, долге, поручителе долга, расчетных лицах юр. лица, контактных лицах юр. лица, произвольных атрибутах, судебной работе, исполнительном производстве)". Соответствующая запись не будет учитываться при расчете количества успешно загруженных записей.

10.По окончании загрузки данных в БД будет выдано сообщение о количестве успешно загруженных записей. Нажмите кнопку ОК, чтобы закрыть его.
