Предварительная настройка констант

Перед началом загрузки данных о платежах и остатках к погашению необходимо в административном модуле Значения констант настроить константы, влияющие на эту процедуру. В случае отсутствия констант в БД при входе в форму Загрузка платежей будут выданы соответствующие предупреждения: "Отсутствует константа <имя константы>".

Название константы

Описание настройки

LoadCalcBulkInsert

Загрузка платежей и остатков. Включение режима массового ввода данных.

LoadCalcClearPlanVisit

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы в БД по умолчанию понимается как значение "Ложь".

Значение "Истина" означает автоматическую отмену намеченных выездов по адресам для подтвержденных платежей

LoadCalcConfirmDoublePayment

Константа может принимать значение "Истина" или "Ложь".

Если значение константы равно "Истина", то по окончании загрузки исходного файла платежей в БД осуществляется поиск и подтверждение (в диалоге с оператором) неподтвержденных платежей, для которых в БД есть хотя бы один подтвержденный платеж по тому же долгу, с той же датой, суммой и типом валюты.

LoadCalcDebtSumUpdateMode

Константа определяет способ выбора альтернативных долгов для загрузки остатка:

0 — все альтернативные долги;

1 — все активные альтернативные долги (не завершенные, не отозванные, не аннулированные), если активных нет, все закрытые альтернативные долги.

LoadCalcSelectDebtMode

Загрузка платежей. Критерий выбора долга при наличии нескольких долгов с одинаковым идентификатором:

0 — долг с меньшим ID;

1 — долг с меньшим ID среди активных долгов.

LoadCalcUsePortfolioStatusForReturn

Константа может принимать значение "Истина" или "Ложь". Признак дополнительной проверки статуса портфеля перед возвратом в работу долга, если загружается остаток более 0 и долг имеет статус "Завершено".

Если константа имеет значение "Ложь", то долг вернется в статус "Возврат в работу" без дополнительных условий

Если константа имеет значение "Истина", то долг вернется в статус "Возврат в работу" при условии,  что статус портфеля не равен 3 ("Закрыт")

PromCoverRate

Константа PromCoverRate определяет процент покрытия обещания для его выполнения и может принимать целое значение в диапазоне от 0 до 100. При отсутствии константы в БД по умолчанию используется значение 80.

В случае некорректного значения константы PromCoverRate (значение не входит в диапазон от 0 до 100) при входе в форму Загрузка платежей будет выдано предупреждение: "Некорректное значение константы PromCoverRate = <значение константы>, по умолчанию присвоено значение 80"

PromGracePeriod

Константа определяет количество дней после даты обещания, когда обещание считается выполненным либо частично выполненным, и может принимать целое значение от 0 и выше. При отсутствии константы в БД по умолчанию используется значение 2

LoadCalcRemainderInOtherValue

Константа может принимать значение "Истина" или  "Ложь". Отсутствие константы в БД по умолчанию понимается как "Ложь". Значение "Истина" предусматривает включение режима загрузки остатков в валюте, отличной от валюты долга, со сменой валюты долга.

LoadCalcDefaultPaymentCurrencyFromDebt

Константа может принимать значение "Истина" или  "Ложь". Отсутствие константы в БД по умолчанию понимается как "Ложь". Значение "Истина" предусматривает взятие типа валюты из долга в отсутствии данных о валюте в строке входного файла

LoadCalcDebtDscUpdateMode

Режим изменения комментария долга: 0 - изменяется для всех случаев, 1 - изменяется только, если долг переходит в статус "Завершено", 2 - изменяется только, если долг переходит в статус "Проблемные", 3 - изменяется только, если долг переходит в статус "Проблемные" или "Завершено", 4 - не изменяется для всех случаев
