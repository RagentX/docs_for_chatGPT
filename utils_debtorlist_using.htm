Применение ограничений доступа к долгам

Настроенные в модуле Настройка списка должников ограничения доступа к долгам по оператору или по роли работают следующим образом:

·По каждому параметру производится отдельная проверка.

·Сначала проверяется ограничение по оператору:

·Если ограничение по оператору настроено, то применяется оно, а ограничение по роли не проверяется.

·Если ограничение по оператору не настроено, то проверяется ограничение по роли. Если ограничение по роли настроено, то оно применяется.

См далее:

·Отображение долгов в списочных модулях;

·Ограничение на просмотр долгов в карточке должника;

·Ограничения в мультифильтрах;

·Ограничения при формировании произвольных отчетов;

Отображение долгов в списочных модулях

В списках долгов для пользователя отображаются только доступные ему долги, в соответствии с Настройками ограничений доступа оператора к долгам.

Модули, в которых ограничения работают всегда

В следующих списочных модулях ограничения, установленные в модуле Настройка списка должников, работают всегда:

·Список должников;

·Протокол контактов;

·Постановка задач;

·Обработка должников;

·Исходящее информирование;

·Группы свободного поиска.

Модули, в которых применение ограничений может быть настроено

В следующих модулях ограничения, установленные в модуле Настройка списка должников, работают в зависимости от настроек системы:

·Клиенты и портфели — в зависимости от значения константы BankPortfolio.OperatorScope.Use. При включенной константе ограничения накладываются, если для пользователя в полях Кредиторы и КА указаны списки разрешенных банков. Если хотя бы один из указанных списков пуст, то ограничения не накладываются.

·Входящий звонок — в зависимости от значения константы InnerCallUseDebtListOperatorScope.

·Платежи — применение ограничений, установленных в модуле Настройка списка должников, настраивается поддержкой.

 Примечание для поддержки.

Если техническая константа PaymentsUseOperatorScope включена, то в модуле Платежи в списке отображаются только те платежи, к которым у текущего пользователя настроен доступ в модуле Настройка списка должников.

Если техническая константа PaymentsUseOperatorScope выключена, ограничения не накладываются.

·Юридический департамент — в зависимости от значения константы LawActList.OperatorScope.Use.

Ограничение на просмотр долгов в карточке должника

Ограничения, настроенные для пользователя в модуле Настройка списка должников, накладываются на открытие карточки должника, только если карточка открывается вручную (т.е. без автоматического поднятия в рамках работы автодайлера) и включена константа DebtCard.OperatorScope.Use.

На вкладке Долги отображаются только доступные пользователю долги, в соответствии с Настройками ограничений доступа оператора к долгам.

Ограничения в мультифильтрах

Если в модуле работает ограничение на доступ к долгам, то отображение значений в мультифильтрах также подвергается ограничениям.

Описанные ниже ограничения работают в следующих модулях:

·Список должников;

·Протокол контактов;

·Постановка задач;

·Исходящее информирование;

·Платежи — только если настроено применение ограничений (см. выше);

·Юридический департамент — только если константа LawActList.OperatorScope.Use имеет значение "Истина";

·Группы свободного поиска.

Ограничения касаются следующих мультифильтров (см. Применение мультифильтров в списочных модулях):

·Портфели:

·Если константа OperatorScope.Portfolio.Use имеет значение "Истина", то отображение списка доступных портфелей в мультифильтре Портфели определяется для пользователя Настройками ограничений доступа оператора к долгам в поле Портфели. В списке доступных портфелей отображаются только входящие портфели из этого списка;

·Если константа OperatorScope.Portfolio.Use имеет значение "Ложь", то отображение списка доступных портфелей в мультифильтре Портфели определяется для пользователя Настройками ограничений доступа оператора к долгам в поле Кредиторы. В списке доступных портфелей отображаются только входящие портфели банков из этого списка;

·Реестр КА — отображение списка доступных исходящих портфелей в мультифильтре Реестр КА определяется для пользователя Настройками ограничений доступа оператора к долгам в поле КА. В списке отображаются только исходящие портфели коллекторских агентств из этого списка;

·Регион — в списке отображаются только разрешенные регионы;

·Филиал — в списке отображаются только разрешенные филиалы;

Примечание. Если включен функционал использования иерархической структуры филиалов(то есть, включена коммерческая константа @UseHierarchyFilial), то в списке отображаются все филиалы. Однако долги не разрешенных филиалов в списке долгов отображены не будут. Включение или выключение функционала иерархической структуры филиалов осуществляется вендором.

·Балл № 1 — в списке отображаются только разрешенные значения Балла № 1;

·Балл № 2 — в списке отображаются только разрешенные значения Балла № 2;

·Балл № 3 — в списке отображаются только разрешенные значения Балла № 3;

·Балл № 4 — в списке отображаются только разрешенные значения Балла № 4.

Ограничения при формировании произвольных отчетов

Ограничения, заданные при Настройке ограничений доступа оператора к долгам, распространяются на параметры произвольных отчетов следующих типов:

·Портфель. Если входной параметр произвольного отчета имеет тип "Портфель", то при формировании отчета откроется список, содержащий перечень всех входящих портфелей из модуля "Клиенты и портфели". В списке необходимо выбрать один или несколько портфелей так же, как при работе с мультифильтрами.

Примечание. Если в модуле Настройка списка должников для текущего пользователя в поле Кредиторы указан список разрешенных банков, то  в мультифильтре Портфели для данного пользователя доступны только портфели банков из этого списка.

·Банк. Если входной параметр произвольного отчета имеет тип "Банк", то при формировании отчета откроется список, содержащий перечень всех клиентов из модуля "Клиенты и портфели". В списке необходимо выбрать одного или нескольких клиентов так же, как при работе с мультифильтрами.

Примечание. Если в модуле Настройка списка должников для текущего пользователя в полях Кредиторы и/или КА указаны списки разрешенных банков, то  в мультифильтре Выберите клиентов для данного пользователя доступны только банки из этих списков.

·Исходящий портфель. Если входной параметр произвольного отчета имеет тип "Исходящий портфель", то при формировании отчета откроется список, содержащий перечень всех исходящих портфелей из модуля "Клиенты и портфели". В списке необходимо выбрать один или несколько портфелей так же, как при работе с мультифильтрами.

Примечание. Если в модуле Настройка списка должников для текущего пользователя в поле КА указан список разрешенных банков, то  в мультифильтре Выберите Реестр КА для данного пользователя доступны только исходящие портфели коллекторский агентств из этого списка.
