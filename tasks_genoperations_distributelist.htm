Распределение долгов по списку операторов

Функция распределения по списку операторов выполняется для всех долгов, кроме тех, которые находятся в статусах 6 "Завершено", 7 "Аннулировано" и 8 "Отозван клиентом". Такие долги распределяться не будут. При распределении по списку ограничения, заданные в модуле "Структура предприятия", не учитываются.

Право распределять долги по списку имеют только те пользователи, в роли которых доступ WORK_TASK_SET_LIST имеет значение TRUE.

Для выполнения операции на компьютере оператора должно быть установлено приложение Microsoft Excel. Предварительно необходимо подготовить файл определенного формата.

В первой строке должны содержаться заголовки столбцов ("шапка"), названия в ней могут быть любыми. Файл должен содержать от двух до пяти столбцов. Если в файле менее двух столбцов, он не пройдет даже первичный этап загрузки данных из файла. Столбцы должны содержать следующие данные:

·ID долга;

·Логин оператора — значение должно совпадать с логином пользователя в системе. Если данное поле в загружаемом файле пустое, долги будут сняты с текущего оператора и переведены в статус 1 "Не распределен";

·Номер договора;

·Лицевой счет;

·ID Клиента.

Столбцы Номер договора, Лицевой счет, ID Клиента могут отсутствовать в файле. Они необходимы для идентификации долга при отсутствии данных в столбце ID долга.

Для того чтобы распределить долги по списку операторов, выполните следующие действия:

1.В контекстном меню (вызывается правой кнопкой мыши) выберите команду Распределение по списку. Откроется стандартное диалоговое окно Windows для выбора файла.

2.Выберите файл, содержащий данные о распределении, и нажмите кнопку Open (Открыть). Будет выполнена попытка идентификации и распределения данных. Идентификация данных выполняется в следующем порядке:

a.Основным идентификатором является ID долга. При его заполнении в строке остальные идентификаторы (Номер договора, Лицевой счет, ID Клиента) не учитываются.

b.При отсутствии данных в поле ID долга идентификатором являются Номер договора, Лицевой счет, ID Клиента либо их сочетание, в зависимости от того, какие из этих полей заполнены. В случае нахождения в системе более одного долга с идентификатором, для распределения используется первый активный долг. Все идентифицированные долги, даже если они уже закреплены за другими операторами, будут распределены в соответствии со списком.

В результате выполнения операции:

·будет произведено перераспределение долгов на операторов из загружаемого файла с учетом ограничений, описанных выше;

·операторам, на которых распределены долги по списку, будут направлены соответствующие уведомления по внутренней почте системы;

·операторам, с которых снимаются долги при распределении по списку, будут направлены соответствующие уведомления по внутренней почте системы;

·если долг уже закреплен за оператором, указанным в списке для данного долга, то перераспределение не производится и уведомление не формируется.

Строка файла может быть не загружена, если:

·не идентифицирован долг (Тип 0);

·в системе не найден пользователь с указанным логином (Тип 2);

·долг находится в одном из статусов: 6 "Завершено", 7 "Аннулировано" или 8 "Отозван клиентом" (Тип 3).

В случае нахождения в файле хотя бы одной записи, которая не может быть загружена, будет предложено сохранить файл с ошибками. В файле будут отображаться все данные из файла, в последнем столбце — причина ошибки.

Учет запрета закрепления

Если в файле загрузки найдены операторы, у которых установлен флаг "Запрет закрепления", то пользователю выводится сообщение: "Для X указанных операторов запрещено закрепление" с вариантами ответа "Пропустить ошибочные записи",  "Продолжить, несмотря на запрет" и "Отмена". Кнопка "Продолжить, несмотря на запрет" видна только тем пользователям, в роли которых включен доступ DEBT _BAN_SET_FLAG_IGNORE.

·Если пользователь выбирает "Отмена", то окно закрывается.

·Если пользователь выбирает "Пропустить ошибочные записи", то появляется уведомление о кол-ве записей, которые не будут загружены (включить в него учет записей, где у пользователя установлен флаг "Запрет закрепления"), и предложение создать файл с описанием ошибок. В файл с ошибочными записями для таких случаев в поле "Описание ошибки" записывать "Запрет закрепления". После этого распределение должно пройти по всем доступным записям (распознан логин и не установлен флаг у оператора).

·Если пользователь выбирает "Продолжить, несмотря на запрет", то пользователи с флагом "Запрет закрепления" распределяются в стандартном режиме. Также появляется уведомление о кол-ве записей, которые не будут загружены, и предложение создать файл с описанием ошибок. В файл с ошибочными записями для таких случаев в поле "Описание ошибки" будет записываться "Запрет закрепления".
