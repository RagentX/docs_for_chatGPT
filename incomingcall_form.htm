Форма "Входящий звонок"

Форма Входящий звонок содержит следующие элементы:

·раздел Поиск, предназначенный для задания параметров и осуществления поиска должника;

·раздел Должники, содержащий найденные долги и объекты судебной работы для них (при наличии), а также действия с записями;

·раздел Телефоны, содержащий список контактных телефонов выбранного должника и действия с ними;

·раздел Работа с должником — информационная область, включающая данные об операторе, ответственном за работу с должником, и рекомендации к действию.

·Кнопки:

· Визит в офис — для Регистрации посещения офиса должником.

· Зарегистрировать контакт — для Регистрации входящего звонка должника.

При регистрации входящего звонка можно выполнить поиск должника, задавая условия поиска в разделе Поиск. Записи, удовлетворяющие заданным условиям, будут отображаться в разделе Должники.

При осуществлении поиска по должникам будут выводиться записи о должниках; при осуществлении поиска по поручителям будут выводиться записи о поручителях.

В списке можно выбрать нужную запись. В зависимости от типа ответственного  лица:

·Если в списке выбран должник, в области Телефоны будут отображаться телефоны должника;

·Если в списке был выбран поручитель, то  области Телефоны будут отображаться телефоны поручителя.

При поступлении входящего звонка открывается Карточка входящего звонка на том ответственном лице с номера которого поступает входящий звонок.

Если регистрируется входящий звонок по текущему долгу (т. е. долгу, выбранному в данный момент в каком-либо списке, например, в форме Список должников), в списке должников отображается только запись о выбранном долге, а в списке телефонов — телефоны выбранного должника.

Примечание. Поле раздела Поиск, в котором располагается курсор при открытии модуля, определяется значением константы  InnerCall.Open.CursorOnField:

·1 — ФИО/Название;

·0 или любое отличное от 1 — ID долга.

Раздел "Поиск"

Параметры поиска должника

Список полей для поиска должника в разделе Поиск:

·ID долга;

·ФИО — фамилия, имя и отчество ответственного лица: должника или поручителя;

·Договор — номер договора

·Лицевой счет — лицевой счет должника;

·Номер телефона — номер телефона ответственного лица: должника или поручителя;

·Адрес — адрес ответственного лица: должника или поручителя;

·Серия, Номер — паспортные данные ответственного лица: должника или поручителя;

·ID Клиента;

·Дата рождения — дата рождения ответственного лица: должника или поручителя;

Флаги:

·флаги По должникам — для выбора тип ответственного лица Должник (по умолчанию включен);

·флаг По поручителям для выбора типа ответственного лица Поручитель (по умолчанию выключен);

·флаг По связанным лицам — для поиска связанных лиц, удовлетворяющих остальным критериям поиска (по умолчанию выключен);

·флаг Показывать закрытые долги — для отображения в результатах поиска также и закрытых долгов (по умолчанию выключен).

Примечание. По согласованию с технической поддержкой доступно настроить так, чтобы флаги Показывать закрытые долги и По поручителям были по умолчнию включены на форме при открытии модуля. Техническая константа InnerCall.CheckboxDefaultTrue.Use.

Действия для поиска

· Очистить — для очистки введенных значений в полях для поиска;

· Найти — для осуществления Поиска должника.

Раздел "Должники"

Список найденных долгов

Список подчиняется Ограничениям на вывод объектов в список долгов.

Список найденных ответственных лиц: должников и/или поручителей в области Должники включает следующие поля:

·ID должника;

·ID долга;

·ID поручителя — заполняется, если поиск осуществлялся по поручителям;

·ФИО — фамилия, имя и отчество ответственного лица: должника или поручителя;

·Название — название продукта;

·Портфель — портфель долга;

·Банк — банк, к которому относится портфель;

·Телефон банка;

·Реестр КА — исходящий портфель, в состав которых входит долг;

·Лицевой счет — лицевой счет должника;

·Д. р. — дата рождения ответственного лица: должника или поручителя;

·Серия, Номер — паспортные данные ответственного лица: должника или поручителя;

·Остаток в руб. — остаток долга;

·Статус — статус долга;

·Дата выдачи кредита;

·Цвет;

·Филиал;

·Тип ответственного лица — если поиск осуществлялся по должникам, то выводится "Должник", а если поиск осуществлялся по поручителям, то выводится роль поручителя (см. словарь № 115);

·Кодовое слово.

Примечание для поддержки. Колонка Кодовое слово видна только если column_spec.user_name для записи, соответствующей полю debt.stock (column_spec.id=528), непустое.

  ­ Для наименования колонки используется термин 528 по словарю 279 "Атрибут объекта системы (column_spec)"

  ­ Если такой термин отсутствует, то наименование колонки берется из поля column_spec.user_name для записи, соответствующей полю debt.stock (column_spec.id=528).

Действия с записями списка долгов

В контекстном меню списка (вызывается правой кнопкой мыши) доступно Добавление долга в группу.

Кнопки для работы со списком:

· — Переход к карточке должника;

· — Просмотр истории комментариев к контактам с должником;

· — Назначение ответственного оператора;

· — Освобождение ответственного оператора;

· — Зарегистрировать обращение должника и Зарегистрировать обращение третьих лиц.

Примечание. Кнопка видна только тем пользователям, в роли которых доступ № 535 INNER_CALL_APPEAL_REG имеет значение TRUE.

Если пользователь имеет полномочия для Отображения в списке объектов судебной работы, то у него также есть возможность осуществить Переход к юридической карточке.

Раздел "Телефоны"

Раздел содержит список контактных телефонов выбранного должника.

Доступны все операции по Ведению списка телефонов должника.

Раздел "Работа с должником"

Раздел включает следующие данные:

·Ответственный — ФИО оператора, закрепленного за данным долгом должника;

·Отдел — подразделение, к которому относится оператор в модуле Структура предприятия;

·Тип — тип оператора;

·Моб. тел. — мобильный телефон оператора;

·Внутр. тел. — внутренний телефон оператора;

·Статус — статус агента телефонии;

·По графику — см. Графики рабочего времени;

·Рекомендация к действию.

Примечание.

·Для долгов на аутсорсинге или цессии (статус 14 "Аутсорсинг" или 17 "Цессия") в поле "Ответственный" выводится наименование контрагента, а в поле "Рекомендация к действию" выводится информация о контрагенте и реестре КА;

·При выборе записи об объекте судебной работы (см. Отображение в списке объектов судебной работы) в поле "Ответственный" выводится информация "Не установлен."
