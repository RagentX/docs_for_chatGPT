Вкладка "Процессы"

Вкладка "Процессы" включает следующие вкладки:

·Вкладка "Параметры процессов", включающая следующие подчиненные вкладки:

·Сопровождение.

·Экспонирование.

·Монетизация.

·Инвентаризация.

·Реализация.

·Списание.

·Выбытие с баланса.

·Оспаривание выбытия.

·Выселение.

·Оценка.

·Оценка по аренде.

·Аукцион.

·Иное.

·Вкладка "Аукцион".

Подчиненная вкладка "Сопровождение"

Подчиненная вкладка "Сопровождение" включает следующие элементы:

·Область данных "Параметры", включающая следующие элементы:

·Список параметров данного имущества, относящихся к процессу "Сопровождение" (см. словарь № 361);

·Кнопки:

§ Добавить — Добавление параметра имущества (доступна пользователям, в роли которых доступ № 760 PROPERTY_CARD_MAINTENANCE_ATTR_ADD имеет значение "TRUE");

§ Редактировать — Редактирование параметра имущества (доступна пользователям, в роли которых доступ № 761 PROPERTY_CARD_MAINTENANCE_ATTR_EDIT имеет значение "TRUE");

§ Удалить — Удаление параметра имущества (доступна пользователям, в роли которых доступ № 762 PROPERTY_CARD_MAINTENANCE_ATTR_DELETE имеет значение "TRUE").

§Договоры (расходы) — см. Договоры.

·Область данных "Задачи" — см. описание функционала "Планирование (Задачи)".

·Кнопка История изменений по задаче — см. Просмотр истории изменений по задаче.

Примечание. В списке задач дополнительно отображается поле "ID группы объектов".

Примечание. В области данных "Задачи" на вкладке "Сопровождение" доступны действия только с задачами, относящимися к процессу "Сопровождение" (см. словарь № 361).

Подчиненная вкладка "Оценка"

Подчиненная вкладка "Оценка" включает следующие элементы:

·Таблица со списком оценок со следующими полями:

·Процесс (property_estimate.r_task_id)

·Сумма оценки (property_estimate.estimate_sum)

·Номер оценки (property_estimate.estimate_num)

·Оценщик (property_estimate.estimator)

·Дата оценки (property_estimate.dt)

·Дата действия отчета об оценке (property_estimate.valid_estimate_dt)

·Дата составления отчета (property_estimate.create_dt)

·НДС (property_estimate.is_VAT. Если значение в поле 1, то необходимо выводить в интерфейсе "Да", если значение 0, то в интерфейсе необходимо выводить "Нет")

·Категория оценки (property_estimate.estimate_typ, вывод терминов из словаря "Категория оценки")

·ИНН оценщика (property_estimate.estimator_inn)

·Назначение оценки (property_estimate.assigning)

·Ссылка на отчет (property_estimate.doc_link)

·Цель оценки (property_estimate.estimate_goal, вывод терминов из словаря "Цель оценки")

·Номер рецензии на отчет об оценке (property_estimate.estimate_review_num )

·Дата рецензии на отчет об оценке (property_estimate.estimate_review_dt)

·Сумма, определенная рецензией (property_estimate.estimate_review_sum)

·Рецензент (property_estimate.reviewer, вывод терминов из словаря "Рецензент")

·Вывод о корректности рыночной стоимости (property_estimate.conclusion, вывод терминов из словаря "Вывод о корректности рыночной стоимости")

·Ликвидационная стоимость (property_estimate.liquid_sum)

·НДС по ликвид стоимости (property_estimate.is_liquid_VAT. Если значение в поле 1, то необходимо выводить в интерфейсе "Да", если значение 0, то в интерфейсе необходимо выводить "Нет")

·Стоимость аренды (руб./кв.м. в год) (property_estimate.rent_sum)

·НДС по стоим. Аренды (property_estimate.is_rent_VAT. Если значение в поле 1, то необходимо выводить в интерфейсе "Да", если значение 0, то в интерфейсе необходимо выводить "Нет").

·Кнопки

· Добавить — Добавление оценки (доступна пользователям, в роли которых доступ № 763 PROPERTY_CARD_EVALUATION_ATTR_ADD имеет значение "TRUE");

· Редактировать — Редактирование оценки (доступна пользователям, в роли которых доступ № 764 PROPERTY_CARD_EVALUATION_ATTR_EDIT имеет значение "TRUE");

· Удалить — Удаление оценки (доступна пользователям, в роли которых доступ № 765 PROPERTY_CARD_EVALUATION_ATTR_DELETE имеет значение "TRUE").

·История изменения — для отображения истории изменения оценки.

·Область данных "Задачи" — см. описание функционала "Планирование (Задачи)".

·Кнопка История изменений по задаче — см. Просмотр истории изменений по задаче.

Примечание. В списке задач дополнительно отображается поле "ID группы объектов".

Примечание. В области данных "Задачи" на вкладке "Оценка" доступны действия только с задачами, относящимися к процессу "Оценка" (см. словарь № 361).

Оценка по аренде

Подчиненная вкладка "Оценка по аренде" включает следующие элементы:

·Таблица со списком оценок со следующими полями:

·Id оценки  - значение атрибута rent_estimate.id.

·Процесс – id задачи, значение атрибута rent_estimate.r_task_id.

·Номер оценки – значение атрибута rent_estimate.estimate_num.

·Оценщик - значение атрибута rent_estimate.estimator.

·Дата оценки - значение атрибута rent_estimate.dt.

·Дата действия отчета об оценке - значение атрибута rent_estimate.valid_estimate_dt.

·Дата составления отчета - значение атрибута rent_estimate.create_dt.

·Стоимость аренды - значение атрибута rent_estimate.rent_sum.

·Стоимость за весь – признак полной стоимости, нужно отображать текст термина из словаря № 385 «Признак полной стоимости» по коду термина в значении rent_estimate.is_full_sum.

·Номер помещения - значение атрибута rent_estimate.building_num.

·ИНН оценщика - значение атрибута rent_estimate.estimator_inn.

·Ссылка на отчет - значение атрибута rent_estimate.doc_link. При нажатии на ссылку в данном поле должен открываться документ по сетевому пути.

·Область данных "Задачи" — см. описание функционала "Планирование (Задачи)".

·Кнопка История изменений по задаче — см. Просмотр истории изменений по задаче.

Примечание. В списке задач дополнительно отображается поле "ID группы объектов".

Примечание. В области данных "Задачи" на вкладке "Оценка по аренде" доступны действия только с задачами, относящимися к процессу "Оценка" (см. словарь № 361).

Вкладка "Аукцион"

Вкладка "Аукцион" включает следующие элементы:

·Область данных «Информация о лотах» с полями:

·Id лота – значение атрибута lot.id.

·Номер протокола – значение атрибута lot.protocol_num.

·Дата протокола – значение атрибута lot.protocol_dt.

·Номер аукциона – значение атрибута lot.auction_num.

·Дата начала подачи заявок на аукцион – значение атрибута lot.req_start_dt

·Дата окончания подачи заявок на аукцион – значение атрибута

·lot. req_end_dt.

·Дата проведения аукциона – значение атрибута lot.auction_dt.

·Дата подведения итогов – значение атрибута lot.summary_dt.

·Номер лота – значение атрибута lot.lot_num.

·Комментарий – значение атрибута lot.dsc.

·Ссылка на протокол – значение атрибута lot.doc_link. При нажатии на ссылку в данном поле должно происходить открытие документа.

·Начальная цена лота - значение атрибута lot.start_price.

·Область данных «Информация об участниках лотов»

Область данных «Информация об участниках лотов» является подчиненной по отношению к области «Информация о лотах». В ней отображаются данные об участниках выбранного лота.

Область данных «Информация об участниках лотов» включает следующие поля:

·id участника лота – значение атрибута lot_participant.id.

·id лота – значение атрибута lot_participant.r_lot_id.

·Номер лота - значение атрибута lot.lot_num по lot_participant.r_lot_id.

·Номер заявки - значение атрибута lot_participant.ticket_num.

·Дата заявки - значение атрибута lot_participant.ticket_dt.

·Наименование участника - значение атрибута lot_participant.ticket_num.

·ИНН участника - значение атрибута lot_participant.inn.

·Статус заявки – значение из словаря «Статус заявки участника лота» по коду термина в значении атрибута lot_participant.status.

·Комментарий - значение атрибута lot_participant.dsc.

·Телефон участника – значение атрибута lot_participant.phone.

·Представитель участника - значение атрибута lot_participant.assignee.

·Адрес электронной почты - значение атрибута lot_participant.email.

·Максимальная ставка - значение атрибута lot_participant.max_rate.

·Количество ставок - значение атрибута lot_participant.rate_cnt.
