Системная логика определения статуса обещания

Для всех клиентов, кроме Летобанка

По стандартной системной логике статус обещания определяется следующим образом:

·Если обещание зарегистрировано с нарушением ограничения на минимальную сумму обещания (debt_promise.is_unchecked=1), то возвращается статус 0 (Не верифицировано).

·Иначе:

·Если выполняются условия:

oСумма покрытия (debt_promise.cover_sum) > 0;

oСумма покрытия (debt_promise.cover_sum) >= (значение в константе PromCoverRate/100) * Сумма обещания (debt_promise.prom_sum),

то возвращается статус 1 (Выполнено).

·Иначе, если выполняются условия:

oСумма покрытия (debt_promise.cover_sum) > 0;

oСумма покрытия (debt_promise.cover_sum) < (значение в константе PromCoverRate/100) * Сумма обещания (debt_promise.prom_sum),

возвращается статус 2 (Част. выполнено).

·Иначе, если Обещанная дата платежа (debt_promise.prom_date) + количество дней, указанное в константе PromGracePeriod < текущей даты, то возвращается статус 3 (Не выполнено).

·Иначе если Обещанная дата платежа (debt_promise.prom_date) + количество дней, указанное в константе PromGracePeriod >= текущей даты И текущая дата > Обещанная дата платежа (debt_promise.prom_date) - (минус) количество дней, указанное в константе PromExpireDays, то возвращается статус  4 (Истекает).

·Иначе если Обещанная дата платежа (debt_promise.prom_date) - (минус) количество дней, указанное в константе PromExpireDays >= текущей даты, то возвращается статус  5 (Ожидание).

Для Летобанка

Добавлены следующие статусы обещаний:

·10001 "Выполнено с погашением просрочки".

·10002 "Выполнено без погашения просрочки".

·10003 "Частично выполнено с погашением просрочки".

·10004 "Частично выполнено без погашения просрочки".

Логика определения статуса обещания доработана следующим образом:

1.Если обещание зарегистрировано с нарушением ограничения на минимальную сумму обещания (debt_promise.is_unchecked=1), то возвращается статус 0 (Не верифицировано).

2.Иначе, если Сумма покрытия (debt_promise.cover_sum) <= 0, то:

·если Обещанная дата платежа (debt_promise.prom_date) + количество дней, указанное в константе PromGracePeriod < текущей даты, то возвращается статус 3 (Не выполнено);

·если Обещанная дата платежа (debt_promise.prom_date) + количество дней, указанное в константе PromGracePeriod >= текущей даты И текущая дата > Обещанная дата платежа (debt_promise.prom_date) - (минус) количество дней, указанное в константе PromExpireDays, то возвращается статус  4 (Истекает);

·если Обещанная дата платежа (debt_promise.prom_date) - (минус) количество дней, указанное в константе PromExpireDays >= текущей даты, то возвращается статус  5 (Ожидание).

3.Иначе:

a)определяется Сумма просрочки на момент регистрации обещания debt_promise.id=i_promise_id:

·по debt_balance_log ищется запись, которая имеет наибольшее значение debt_balance_log.dt, но при этом debt_balance_log.dt < debt_promise.dt:

·если запись найдена, то Сумма просрочки=debt_balance_log.debt_sum;

·иначе Сумма просрочки=debt.debt_sum для долга, к которому привязано обещание (debt.id=debt_promise.parent_id).

b)и затем возвращается статус обещания (debt_promise.status) по следующей логике:

·10001 (Выполнено с погашением просрочки), если выполняются условия:

·Сумма покрытия (debt_promise.cover_sum) >= Сумма просрочки;

·Сумма покрытия (debt_promise.cover_sum) >= Сумма обещания (debt_promise.prom_sum).

·10002 (Выполнено без погашения просрочки), если выполняются условия:

·Сумма покрытия (debt_promise.cover_sum) < Сумма просрочки;

·Сумма покрытия (debt_promise.cover_sum) >= Сумма обещания (debt_promise.prom_sum).

·10003 (Частично выполнено с погашением просрочки), если выполняются условия:

·Сумма покрытия (debt_promise.cover_sum) >= Сумма просрочки;

·Сумма покрытия (debt_promise.cover_sum) < Сумма обещания (debt_promise.prom_sum).

·10004 (Частично выполнено без погашения просрочки), если выполняются условия:

·Сумма покрытия (debt_promise.cover_sum) < Сумма просрочки;

·Сумма покрытия (debt_promise.cover_sum) < Сумма обещания (debt_promise.prom_sum).
