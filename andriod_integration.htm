·Описание: Андроид. Устанавливает тип дерева для регистрации контактов при использовании локальной авторизации. Указывается тип контакта, двоеточие, тип дерева и затем полученный текст заключается в скобки (Пример: (1:1)(2:1) – для регистрации контактов с типом "Звонок" должно отображаться дерево с типом 1, для регистрации контактов с типом "Выезд" должно отображаться дерево с типом 1);

·Тип данных: Строка;

·Тип константы: Некоммерческая, обычная;

·Первоначальное значение: "(1:1)(2:1)(3:1)".

Константа: Android.LocalAuth.Use

·Описание: Андроид. Использовать возможность входа в приложение с локальной авторизацией;

·Тип данных: Булевская переменная;

·Тип константы: Некоммерческая, обычная;

·Первоначальное значение: Истина.

Если константа включена

Пользователь может включить флаг "Локальная авторизация" произвольно. При этом работа приложения будет осуществляться в локальном режиме, независимо от наличия подключения к интернету.

Если флаг "Локальная авторизация" выключен, приложение будет работать в режиме online.

Если константа отключена

Если константа отключена, то у пользователя нет возможности использовать произвольную локальную авторизацию при входе в приложение. Флаг "Локальная авторизация" не видим на форме авторизации.

При этом авторизация происходит в режиме online всегда, когда устройство подключено к интернету. Если на устройстве отсутствует подключение к интернету, автоматически осуществляется локальная авторизация.

В процессе работы в приложении в случаях отключения интернета или проблем с мобильной связью приложение автоматически переходит в локальный режим. При появлении интернета или сети приложение автоматически переходит в режим online.

Вкладка "Android" в модуле "Протокол контактов"

 

Описание вкладки "Android"

На вкладке Android отображаются только выезды, зарегистрированные с помощью мобильной версии Системы (то есть те, у которых модулем регистрации является "Android").

На вкладке может осуществляться Контроль регистрации координат по адресу выезда.

Доступна возможность Просмотра маршрутов операторов мобильных устройств (в меню кнопки ).

Проверка GPS при запуске мобильного приложения

Поддержана константа Android.CheckGPS.Mode.

·Если значение константы равно 1, то:

·Перед запуском приложения осуществляется проверка, включен ли на устройстве GPS. Если GPS выключен, то пользователю отображается окно с соответствующим сообщением и кнопкой ОK. После нажатия на кнопку ОК вход в приложение не осуществляется. Текст сообщения:

½Определение местоположения на устройстве отключено. Для работы приложения необходимо включить определение местоположения в настройках устройства. Приложение будет закрыто.

·Раз в n-минут (значение числа n определяется разработчиком, исходя из его опыта) пока приложение запущено, осуществляется проверка статуса работы GPS. Если в результате проверки обнаружено, что GPS отключили, то пользователю будет показано на экране окно с сообщением-предупреждением (см. пункт выше), и после нажатия кнопки ОK пользователем работа с приложением завершится, при этом если в момент проверки происходил какой-то процесс в приложении (синхронизация, передача контактов на сервер, добавление документа и т.д.), то он будет корректно завершен, и только затем будет осуществлен выход из приложения.

·Координаты определяются только по gps. Метод определения координат по GSM не используется. Координаты определяются только по GPS во всех методах и местах кода, где определяются координаты (регистрация контакта, трекинг и т.д.)

·Если значение константы не равно 1, или константа не найдена, то работа в приложении ведется в прежнем режиме.
