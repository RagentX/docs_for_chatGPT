Предварительная настройка констант

Перед началом загрузки данных необходимо в модуле "Значения констант" настроить константы, влияющие на эту процедуру.

Название константы

Описание настройки

LoadDataAutoRegion

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы в БД по умолчанию понимается как значение "Ложь".

Если константа имеет значение "Истина", то будет автоматически формироваться колонка "Регион" при загрузке данных  в случае отсутствия в файле с данными MS Excel столбца "Регион".

LoadDataAutoText

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы в БД по умолчанию понимается как значение "Ложь".

Значение "Истина" предусматривает загрузку в БД без диалога с пользователем всех записей, содержащих адреса с альтернативными вариантами одноименных городов и/или улиц базовых городов — Москвы, Санкт-Петербурга, Екатеринбурга. При этом в БД формируются только названия городов и улиц без ссылок на справочники KLADR и ADDRESS_ZONE (улицы базовых городов).

При отсутствии в БД константы LoadDataAutoText или значении "Ложь" по каждому альтернативному варианту города (улицы) в процессе загрузки данных осуществляется диалог с пользователем. Для городов (улиц), по которым в результате диалога пользователь выбрал один из альтернативных вариантов, в БД формируются не только названия, но и ссылки на справочники (KLADR для городов, ADDRESS_ZONE для улиц).

LoadDataCSVChangePortfolio

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы в БД по умолчанию понимается как значение "Ложь".

Если константа имеет значение "Истина", то при обновлении долга в CSV будет изменена ссылка на портфель в таблице долгов.

LoadDataChangeDepart

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы в БД по умолчанию понимается как значение "Ложь".

Если константа имеет значение "Истина", то при загрузке данных для похожей персоны, не имеющей активных долгов, в случае обновления персоны и добавления долга значение атрибута "стадия" будет изменено на soft. Похожими считаются физические лица, у которых совпадают фамилии, имена, отчества, а также (если константы LoadDataEqualBirthDate и LoadDataEqualPassport имеют значение "Истина") даты рождения, серии и номера паспортов. Кроме того, похожими считаются юридические лица, у которых совпадают фамилии/названия компании юридического лица, а также ИНН (если константа LoadDataEqualInn имеет значение "Истина").

LoadDataCheckDublPersonFile

Не используется

LoadDataCheckStreet

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы в БД по умолчанию понимается как значение "Ложь".

Если константа имеет значение "Истина", то при загрузке данных будет выполняться проверка названий улиц на наличие и соответствие KLADR. Если в КЛАДР такой улицы не существует, то адрес записывается в текстовом формате. Возможные проблемы при включении: некоторые адреса будут загружены в общем текстовом виде, если улица не будет определена в КЛАДР.

При отсутствии в БД константы LoadDataCheckStreet или значении "Ложь" проверка выполняться не будет.

LoadDataDebugMode

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы в БД по умолчанию понимается как значение "Ложь".

Если константа имеет значение "Истина", то при загрузке данных будет включено автоматическое формирование текстового файла данных о загрузке.

При отсутствии в БД константы LoadDataDebugMode или значении "Ложь" указанный файл не формируется.

LoadDataEqualBirthDate

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы в БД по умолчанию понимается как "Ложь".

Если константа имеет значение "Истина", то при определении совпадения персоны с персонами из БД для юридического лица будет использоваться дата рождения

LoadDataEqualInn

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы LoadDataEqualInn в БД по умолчанию понимается как "Ложь".

Если константа имеет значение "Истина", то при определении совпадения персоны с персонами из БД для юридического лица будет использоваться ИНН

LoadDataEqualPassport

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы в БД по умолчанию понимается как "Ложь".

Если константа имеет значение "Истина", то при определении совпадения персоны с персонами из БД для физического лица будет использоваться серия и номер паспорта

LoadDataOblast

Код области для загрузки данных. Строковая константа LoadDataOblast задается двухсимвольным цифровым значением в диапазоне 01 — 89 (соответствует коду региона, например, 77 — Москва, 50 — Московская область) или пустым значением. Любые другие значения константы считаются некорректными. Отсутствие константы LoadDataOblast в БД, пустое или некорректное значение соответствует мягкому варианту проверки адресов, а цифровое значение в диапазоне от 01 до 89 — жесткому варианту проверки.

При поиске населенных пунктов в KLADR будет рассматриваться только заданная область. Населенные пункты других областей рассматриваться не будут. Например, если указан код области 50 (Московская обл.), то проверка в КЛАДР будет производиться только для адресов Московской области, адреса других регионов могут быть сохранены в текстовом формате

LoadDataOblast2

Дополнительный код региона для загрузки данных, если уже указано значение для константы LoadDataOblast. Например, LoadDataOblast = 77, LoadDataOblast2 = 50 обеспечивает отбраковку всех адресов, не относящихся к Москве и Московской области.

Константа LoadDataOblast2 также задается двухсимвольным цифровым значением в диапазоне 01 — 89 или пустым значением. Любые другие значения константы считаются некорректными. Некорректным считается также значение, совпадающее с текущим значением константы LoadDataOblast. Контроль корректности значения константы LoadDataOblast2 и работа с ней в мягком варианте проверки адресов не производится. Для перехода от жесткого варианта проверки адресов к мягкому достаточно присвоить пустое значение константе LoadDataOblast

LoadDataSaveEqualAddressMode

Числовая константа может принимать значения от 1 до 4. Отсутствие константы в БД по умолчанию понимается как значение 2.

Значение 1 предусматривает безусловное сохранение совпадающих адресов, адреса добавляются в БД как новые, без сравнения с имеющимися в БД адресами.

При значении 2 совпадающие адреса не добавляются в БД, сравнение адресов производится без учета типа. Если загружаемый адрес регистрации совпадает с фактическим адресом из БД, то тип "фактический" для адреса в БД автоматически заменяется типом "регистрация".

При значении 3 совпадающие адреса не добавляются в БД, сравнение адресов производится с учетом типа.

При значении 4 совпадающие адреса не добавляются, сравнение адресов выполняется без учета типа, тип адреса меняется на более приоритетный (меньший по коду термина в словаре типов адресов)

LoadDataSaveEqualPhoneMode

Числовая константа может принимать значения от 1 до 3. Отсутствие константы в БД по умолчанию понимается как значение 2.

Значение 1 предусматривает безусловное сохранение совпадающих телефонных номеров, телефоны добавляются в БД как новые, без сравнения с имеющимися в БД адресами.

При значении 2 совпадающие телефоны не добавляются в БД, сравнение телефонов производится без учета типа.

При значении 3 совпадающие телефоны не добавляются в БД, сравнение телефонов производится с учетом типа

LoadDataUseDataq

Константа может принимать значение "Истина" или "Ложь". Отсутствие константы в БД по умолчанию понимается как "Ложь".

Если константа имеет значение "Истина", то при загрузке данных будет выполняться проверка загружаемых контактных данных через сервис DaData по автоматической очистке и стандартизации почтовых адресов и приведению их к КЛАДР — официальному справочнику адресов ФНС России.

LoadData.Address.LoadWithTransform

Загрузка данных. Дозагружать адреса (Область, город и др.) с преобразованием по системной логике.

При необходимости исключить добавление системных сокращений «"д.", "корп.", "кв."» в полном адресе,

измените константу Address.FullAddr.AddAbbr. Если указан код КЛАДР, то формирование полного адреса

и его актуализация будет выполняться на основе КЛАДРа.

Address.FullAddr.AddAbbr

Адреса. Формирование полного адреса с добавлением системных сокращений ("д.", "корп.", "кв.")

LoadData.Phone.DictValue1.Default

Значение словарного атрибута телефона 1 (phone .dict_value_1), которое необходимо установить при загрузке новых телефонов с помощью модуля "Загрузка данных". Указывает код термина по словарю 258.

LoadData.SaveEqualPhone.Priority

При LoadDataSaveEqualPhoneMode = 2 приоритет типов для загрузки совпадающих телефонов определяется в первую очередь типами, указанными в строке LoadDataSaveEqualPhonePriority. Если строка пуста, то приоритет типов определяется только их кодами в словаре типов телефона.

LoadDataDebtNameDefault

Загрузка данных. Определяет название продукта по умолчанию при загрузке долга.

LoadDataPersonTypDefault

Загрузка данных. Определяет тип персоны по умолчанию при загрузке долга. Указывается код словаря 54.
