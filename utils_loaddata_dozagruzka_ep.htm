Особенности загрузки файлов с типом формата "ИП (Обновление)"

Особенности загрузки файлов с типом формата "ИП (Обновление)"

Доработка логики загрузки

1.В дополнение к существующим, поддержана загрузка следующих полей из таблицы "Исполнительное производство":

·Дата приостановления ИП (finish_date);

·Дата отказа в возбуждении ИП (reject_date);

·Дата вынесения (court_date);

·Тип исполнительного документа (executive_typ по словарю № 124);

·Дата вступления в силу (entry_force_dt);

·Название ФССП (law_court.name).

2.Логика загрузки доработана следующим образом:

·Если есть колонка "id долга", то вместо "ФИО/название компании" берется ФИО должника (person.f+i+o).

·Если есть колонка "id дела", то вместо "ФИО/название компании" берем ФИО ответчика через таблицу law_act_person_link:

·Если person_role = 1, то берем ФИО должника (person.f+i+o);

·Если person_role = 2, то берем ФИО поручителя (debt_guarantor.fio).

·Если ничего нет, то используется колонка "ФИО/название компании".

3.Исключена возможность указывать поле "ссылка на debt".

4.Поддержано прикрепление обновляемой записи "Исполнительного производства" к "Судебному делу", если дело связано с тем же самым долгом:

·Если в файле загрузки указано значение поля "ссылка на law_act" и по нему найдена запись в law_act принадлежащая тому же самому долгу (law_act.r_debt_id = law_exec.r_debt_id), то в обновляемой записи law_exec заполняется поле law_exec.r_act_id.

·Если в файле загрузки указано значение поля "ссылка на law_act" и по нему НЕ найдена запись в law_act принадлежащая тому же самому долгу (law_act.r_debt_id = law_exec.r_debt_id), то значение law_exec.r_act_id не обновляется. При проверке поле отмечается желтым цветом.

·Если поле "ссылка на law_act" не указано или не заполнено, дополнительных действий не требуется.

Доработка логики загрузки названия ФССП

Логика загрузки названия ФССП доработана следующим образом:

По полю "Название "ФССП" (law_court.name) проверяется наличие записи с такими данными в law_court.name по условию (law_court.name = %Название ФССП% AND law_court.typ = 2) в шаблоне:

·Если будет найдена одна запись, то в law_exec.r_court_id загружаемого ИП записывается law_court.id найденной записи.

·Если будет найдено несколько записей, то в law_exec.r_court_id загружаемого ИП записывается law_court.id с максимальным id.

·Если не будет найдено ни одной записи, то создается новая запись в law_court. Указывается law_court.name = %Название ФССП (по загружаемому документу)%; law_court.typ=2. В law_exec.r_court_id загружаемого ИП записать law_court.id созданной записи.

Поддержка константы LoadData.LawExecUpdate.Mode

Константа LoadData.LawExecUpdate.Mode поддержана следующим образом:

1.Если константа LoadData.LawExecUpdate.Mode = 1, то проверяется наличие ИП по объекту связи:

·Если в шаблоне, по которому производится загрузка данных, указана колонка "ID долга", то поиск ИП осуществляется по условию: law_exec.r_debt_id = debt.id и law_exec.r_act_id is null:

·Если найдено хотя бы одно ИП, то строка не загружается и отправляется в файл с незагруженными записями.

·Если ИП не найдено, то создается ИП с указанием следующих идентификаторов:

§law_exec.r_act_id = NULL;

§law_exec.r_debt_id = debt.id.

·Если в шаблоне, по которому производится загрузка данных, указана колонка "ID судебного дела", то поиск ИП осуществляется по условию: law_exec.r_act_id = law_act.id.

·Если найдено хотя бы одно ИП, то строка не загружается и отправляется в файл с незагруженными записями.

·Если ИП не найдено, то создается ИП с указанием следующих идентификаторов:

§law_exec.r_act_id = law_act.id;

§law_exec.r_debt_id = law_act.r_debt_id.

·Если в шаблоне, по которому производится загрузка данных, не указаны колонки "ID долга" и "ID судебного дела", остается старая логика загрузки.

2.Если константа LoadData.LawExecUpdate.Mode = 2, то проверяется наличие активного ИП (law_exec.state NOT IN(3, 4, 5, 6, 10, 13)) по объекту связи:

·Если в шаблоне, по которому производится загрузка данных, указана колонка "ID долга", то поиск ИП осуществляется по условию: law_exec.r_debt_id = debt.id и law_exec.r_act_id is null:

·Если найдено хотя бы одно ИП, то строка не загружается и отправляется в файл с незагруженными записями.

·Если не найдено ни одного ИП, то создается ИП с указанием следующих идентификаторов:

§law_exec.r_act_id = NULL;

§law_exec.r_debt_id = debt.id.

·Если в шаблоне, по которому производится загрузка данных, указана колонка "ID судебного дела", то поиск ИП осуществляется по условию: law_exec.r_act_id = law_act.id:

§Если найдено хотя бы одно ИП, то строка не загружается и отправляется в файл с незагруженными записями.

§Если ИП не найдено, то создается ИП с указанием следующих идентификаторов:

§law_exec.r_act_id = law_act.id;

§law_exec.r_debt_id = law_act.r_debt_id.

·Если в шаблоне, по которому производится загрузка данных, не указаны колонки "ID долга" и "ID судебного дела", то остается старая логика загрузки.

3.Если константа LoadData.LawExecUpdate.Mode = 3, то выполняется создание ИП без учета наличия ИП в БД:

·Если в шаблоне, по которому производится загрузка данных, указана колонка "ID судебного дела", то создается ИП с указанием следующих идентификаторов:

·law_exec.r_act_id = law_act.id;

·law_exec.r_debt_id = law_act.r_debt_id.

·Если в шаблоне, по которому производится загрузка данных, указана колонка "ID долга", то создается ИП с указанием следующих идентификаторов:

·law_exec.r_act_id = NULL;

·law_exec.r_debt_id = debt.id.

·Если в шаблоне, по которому производится загрузка данных, не указаны колонки "ID долга" и "ID судебного дела", остается старая логика загрузки.
