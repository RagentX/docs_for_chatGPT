Глоссарий

Все определения применяются только для системы Контакт и могут отличаться от общепринятых формулировок.

Термин 

Определение

Атрибут

Значение свойства сущности, отличающее один однотипный объект от другого

Банк

Внешняя компания, передающая портфели должников с целью проведения работ по сбору дебиторской задолженности

Банковская просрочка

Различают два значения банковской просрочки:

·Банковская просрочка (дней) — количество дней просрочки (поле dpd таблицы debt);

·Банковская просрочка (от даты) — значение рассчитывается как "Дата подписания реестра (portfolio.sign_date) – Дата выхода на просрочку (debt.start_date)"

Валюта долга

Валюта, в которой выражается сумма долга по договору

Валютный долг

Долг, для которого валюта долга не равна Основной валюте Системы

Вознаграждение %

Вознаграждение % — поле, которое организация может использовать для хранения информации о сумме своего вознаграждения с данного долга;

Действия

Активность пользователя системы по отношению к объектам системы

Дерево контактов

Системный объект, представляющий собой многоуровневую классификацию контактов по результату. Для каждого типа контакта в системе предусматривается свое дерево

Дней просрочки

Дней просрочки — если значение поля flag_preventive таблицы debt = 1 (Признак "Профилактика"), то в поле отображается 0, иначе значение рассчитывается как "Текущая дата – Дата выхода долга на просрочку (debt.start_date)". См. также Фактическая просрочка;

Долг

Обязанность физического или юридического лица выплатить некоторую сумму денежных средств, подтвержденная договором, заключенным этим лицом с банком. Договор регламентирует сроки и правила выплат

Должник

Физическое или юридическое лицо, имеющее долг перед одним или несколькими банками, дело которого было передано компании по работе с дебиторской задолженностью

Доступы

Возможность сотрудников компании производить определенные действия над объектами системы либо получать некоторую информацию из системы, связанную с наличием данного доступа

Коллектор

Специалист компании по работе с ранней или поздней задолженностью. В должностные обязанности коллектора входит осуществление контактов с должником с целью возврата долга перед банком.

Константа

Системный параметр, позволяющий настраивать механизм работы того или иного функционала системы "Контакт". Значение константы может изменяться администратором системы

Контакт

Удачная или неудачная попытка прямого общения сотрудников компании с должником. Разделяется на типы в зависимости от способа (телефон, личная встреча) и инициатора (сотрудник или сам должник)

Название продукта

Название продукта — указывается название продукта, по которому возник долг;

Направление контакта

Телефон или адрес, по которым был осуществлен контакт

Обещание

Зарегистрированная в системе информация от должника о том, что до определенной даты в банк будет внесена определенная сумма денежных средств

Объект

Любая сущность, используемая в процессе описания деятельности компании

Объект группы

Объект для группировки в модуле Группы должников. На досудебной стадии взыскания группируются долги по идентификатору долга, для судебной стадии — судебные дела по идентификаторам судебного дела, для исполнительной стадии — дела исполнительного производства по идентификаторам ИП.

Основная валюта Системы

Валюта, в которую конвертируются остальные валюты (см. Управление списком валют). В списке валют имеет первый номер. См. также Валютный долг.

Оператор

Коллектор, имеющий доступ к системе "Контакт"

Платеж

Зарегистрированная в системе информация о том, что должник внес определенную сумму денежных средств в банк

Покрытие обещания

Сумма платежей, отождествленных системой с обещанием должника произвести платежи. Одно обещание может быть ассоциировано с несколькими платежами, при этом один платеж может быть привязан только к одному обещанию

Пользователь

См. оператор

Портфель (реестр)

Группа сведений о должниках, передаваемая банком, для которой установлены единые сроки проведения работ по возврату задолженности, а также единые правила расчета агентского вознаграждения

Просрочка

Момент времени, когда должник не произвел оплату необходимой суммы в срок, предусмотренный договором с банком, чем нарушил свои обязательства в соответствии с данным договором

Рабочее место (модуль)

Группа экранных форм, представляющая собой единую сущность, как часть системы, обеспечивающая выполнение определенного набора базовых операций. В системе открытие рабочего места производится из главного меню или других рабочих мест

Роль

Набор доступов, предоставляемый тем или иным операторам системы, в зависимости от их должностных обязанностей

Словарь

Объект системы, содержащий множество текстовых значений (терминов), которое применяется в качестве множества значений атрибута некоторого объекта. Такой атрибут называется словарным, и он не может принимать значения, не описанные в соответствующем словаре. Словари делятся на две группы — системные и пользовательские. Последние доступны для изменения администратором системы.

Ссылка (связь)

Атрибут, отражающий иерархию отношений между объектами

Стадия взыскания

Основных стадий взыскания три: досудебная, судебная и исполнительная. Такие списочные РМ, как Список должников, Постановка задач, Обработка должников, Исходящее информирование работают с долгами на досудебной стадии. Модуль Юридический департамент работает с делами судебной и исполнительной стадий. Список стадий взыскания — см. словарь № 132.

Стадия должника

Основные стадии должника: soft, hard, legal, exec — см. словарь № 226.

Стадия портфеля

Стадии портфеля: к стадиям должника, описанным в словаре № 226 добавлена стадия bankrupt, соответствующая банкротству.

Статус

Специфический атрибут, отражающий состояние объекта. Для одного объекта возможно присвоение нескольких статусных атрибутов одновременно. Например, статусы долга см. в словаре № 6.

Супервизор

Специалист компании по работе с ранней или поздней задолженностью. В должностные обязанности супервизора входит управление группой коллекторов, в том числе формирование потоков работ, определение приоритетов и контроль результатов деятельности.

Тип пользователя, оператора

Тип пользователя соответствует стадии взыскания долгов, на которой работает пользователь, например soft, hard, legal, exec и т.д.

Тип продукта

Тип продукта — см. словарь № 11;

Фактическая просрочка

Фактическая просрочка — если значение поля flag_preventive таблицы debt = 1 (Признак "Профилактика"), то в колонке отображается 0; иначе значение рассчитывается как "Текущая дата – Дата выхода долга на просрочку (debt.start_date)". См. также Дней просрочки;

Формула (произвольная формула)

Объект, представляющий собой функцию, принимающую значения в зависимости от различных параметров и атрибутов системных объектов. Применяется для формирования условий выполнения некоторых процессов, а также для вычисления новых значений при автоматизированном обновлении данных

Форма

Средство отображения и ввода информации, используемые операторами системы

ID должника

Уникальный идентификатор (порядковый номер) должника в Системе

ID долга

Уникальный идентификатор (порядковый номер) долга в Системе

ID Клиента

Дополнительный идентификатор долга из системы/базы данных клиента (при наличии);

Soft

Досудебная стадия работы с должником — стадия ранней задолженности. Основной метод работы — удаленное общение с должником по телефону

Hard

Досудебная стадия работы с должником — стадия поздней задолженности. Основной метод работы — встреча с должником

Legal

Судебная стадия работы с должником — приказное, исковое производства

Exec

Исполнительная стадия работы с должником — исполнительное производство

KLADR

Внешняя база данных адресов. Позволяет организовать хранение и сегментацию адресов на уровне областей, районов и населенных пунктов
