Настройка константы CommunicationLimit.ContactTypeList

Для включения функционала настройте константу CommunicationLimit.ContactTypeList. Константа имеет тип "строка", и в ней должны быть перечислены через запятую типы контактов, для которых будет осуществляться подсчет статистики. Типы контактов указываются в виде их кодов по словарю № 3. При этом поддержаны следующие типы контактов:

·1 — Звонок;

·2 — Выезд;

·7 — СМС;

·8 — Email;

·10 — Автоинформирование.

Таким образом, значение данной константы может иметь вид, например, "1" или "1,7" или "1,2,7,8" и так далее.

Функционал выключен, если значение константы CommunicationLimit.ContactTypeList равно нулю, пусто или константа отсутствует. Кроме того, не обрабатываются перечисленные в ней значения, если они не поддерживаются.

После того, как вы включили константу CommunicationLimit.ContactTypeList, вы знаете, для каких именно типов контактов у вас поддерживаются ограничения.

Настройка учета ограничений на рабочих местах

Включите флаги "Учитывать в ограничениях" в следующих местах:

·в модуле Настройка дерева контактов;

·в модуле Сценарии сообщений;

·в форме Выгрузки данных в автодайлер в модулях:

·Группы должников;

·Список должников;

·Группы свободного поиска.

Настройка констант для кастомизации процесса

Константа CommunicationLimit.SameCall.Period

Возможна ситуация, когда во время разговора должник сообщает, что в данную минуту он по какой-то причине не может продолжать разговор, и просит оператора перезвонить позже для продолжения разговора. Оператор перезванивает через указанное время, разговор продолжается. Очевидно, что два данных телефонных звонка логично было бы рассматривать как одну коммуникацию. Для этого используется константа CommunicationLimit.SameCall.Period.

Укажите в значении константы CommunicationLimit.SameCall.Period количество минут, в течение которых контакты по данному участнику договора будут считаться одной коммуникацией в том случае, если они были зарегистрированы одним оператором.

Константы CommunicationLimit.SMS.SuccessStatus и CommunicationLimit.Email.SuccessStatus

Если SMS или Email сообщение было отправлено, но по какой-либо причине не дошло до адресата, то не имеет смысла считать это событие совершённой коммуникацией и учитывать в статистике. Для этих случаев используются константы CommunicationLimit.SMS.SuccessStatus и CommunicationLimit.Email.SuccessStatus, которые включают в себя списки статусов сообщения, соответствующих успешной коммуникации.

Укажите в значении константы CommunicationLimit.SMS.SuccessStatus список успешных статусов SMS по словарю № 50 для того, чтобы в статистике учитывались SMS только в этих статусах.

Укажите в значении константы CommunicationLimit.Email.SuccessStatus список успешных статусов Email по словарю № 65 для того, чтобы в статистике учитывались Email только в этих статусах.

Константа CommunicationLimit.Autoinf.Code

В константе CommunicationLimit.Autoinf.Code перечисляются через запятую коды дерева контактов, используемые для автоинформирования. Такие узлы не используются для звонков. Обычно они имеют включенный флаг "Не показывать при ручной регистрации контакта". Эти узлы будут использоваться для подсчета статистики по автоинформированию и, соответственно, будут исключены из статистики по звонкам.

При этом получается, что при регистрации контакта "Звонок" для ручной регистрации контакта и для автоинформирования используются разные узлы дерева контактов. Таким образом, если данная константа включена, то в статистике отдельно учитываются звонки оператора и автоинформирование. Если данную константу не включать (присвоить ей значение 0), то автоинформирование и звонки оператора будут учитываться как один тип контактов.

Примечание. При подсчете статистики по автоинформированию будут учитываться только те узлы из перечисленных в значении константы, при настройке которых был включен флаг "Учитывать в ограничениях".

Настройка формул для ограничения коммуникации

В модуле Произвольные формулы настройте формулы типа "Ограничения коммуникации".

В самом простом случае вы можете создать одну формулу, состоящую из одного числа, и она будет использоваться для всех типов коммуникации. Кроме того, вы можете назначить отдельные схемы коммуникации в зависимости от типа контакта, создав несколько формул, состоящих из одного числа.

Для более сложных настроек вы можете создавать более сложные формулы, включающие роль участника договора, значения произвольных атрибутов долга, а также все доступные атрибуты.

Вариант использования новых функций в формуле:

если Роль участника договора = 2,

 то = Схема для поручителя,

 иначе = если Роль участника договора = 3,

         то = Схема для залогодателя,

         иначе = если (ID произвольного атрибута 58 для ID долга) = 0,

                 то = Схема коммуникации 1,

                 иначе = если (Значение произвольного атрибута (число) 55 для ID произвольного атрибута 58 для ID долга) = 5,

                         то = Схема коммуникации 2,

                         иначе = Схема коммуникации 3.

Таким образом можно настроить разные схемы коммуникации для поручителя, залогодателя и должника, а также настроить разные схемы при наличии определенных значений произвольных атрибутов (например, при заявлении об отказе от взаимодействия).

Настройка констант для назначения общих схем коммуникации

После того как формулы настроены, присвойте их номера (ID из списка формул в модуле "Произвольные формулы") следующим константам:

Константа CommunicationLimit.Call.Formula

Присвойте константе идентификатор формулы, возвращающей номер схемы коммуникации для типа контакта "Звонок".

Константа CommunicationLimit.SMS.Formula

Присвойте константе идентификатор формулы, возвращающей номер схемы коммуникации для типа контакта "СМС".

Константа CommunicationLimit.Email.Formula

Присвойте константе идентификатор формулы, возвращающей номер схемы коммуникации для типа контакта "Email".

Константа CommunicationLimit.Visit.Formula

Присвойте константе идентификатор формулы, возвращающей номер схемы коммуникации для типа контакта "Выезд".

Константа CommunicationLimit.AutoInfo.Formula

Присвойте константе идентификатор формулы, возвращающей номер схемы коммуникации для типа контакта "Автоинформатор".
