Формирование списка для обзвона и выбор кампании обзвона

Есть два способа формирования списка для обзвона:

·Стандартый — используется для типов пользователя, отличных от Soft или при выборе кампании обзвона Стандартная;

·Кампания обзвона — если выбрана кампания обзвона (см. Кампании обзвона), отличная от Стандартная, порядок должников для обзвона будет формироваться в соответствии с выбранной кампанией с учетом значения константы CallCenterPromFreq.

Примечание. Во всех запросах на получение списка на обзвон или получение следующей записи осуществляется контроль значения константы CallCenter.DebtStatus.NotUse.List: из списка исключаются долги, находящиеся в статусах, перечисленных в указанной константе.

Стандартный список должников для обзвона

Стандартный список формируется по следующей логике:

1.Долг закреплен за текущим пользователем.

2.Долг не в статусах 6,7,8,9,10.

3.Выставлен приоритет либо дата для звонка меньше или равна текущей дате.

Данный список применяется:

·Для операторов с типом, отличным от Soft;

·Для операторов типа Soft, выбравших кампанию обзвона Стандартная.

Выбор кампании обзвона

Возможность выбора кампании обзвона доступна только пользователям типа Soft.

Если константа CallCenterStandard имеет значение "Истина", то в список кампаний обзвона включена также кампания Стандартная.

Для пользователей, имеющих тип Soft при запуске в модуля Центр обзвона должников открывается форма выбора кампании обзвона

В форме содержится список всех активных кампаний (т.е. находящихся в статусе "Запущена"), в которых авторизованный пользователь принимает участие. Настройка кампаний обзвона, списков входящих в них должников и их участников выполняется в модуле Кампании обзвона.

Для того чтобы перейти к обзвону должников нужной кампании, выберите кампанию обзвона в списке и нажмите кнопку ОК. Откроется форма Центр обзвона должников. Информация о текущей кампании обзвона будет отображаться в строке в нижней части формы.

Если список на обзвон пуст, появляется соответствующее предупреждение, и модуль не открывается.

Учет константы CallCenterPromFreq

Внутренний список обещаний коллектора

Внутренний список обещаний формируется по следующей логике:

1.Долг закреплен за текущим оператором.

2.Долг не в статусах 6, 7, 8, 9, 10.

3.Сумма покрытия последнего обещания равна 0.

4.Обещанная дата платежа по последнему обещанию меньше текущей даты, но больше даты последнего успешного контакта по долгу с типом "Звонок" или "Входящий звонок".

5.Обещанная дата платежа по последнему обещанию + количество дней, указанных в константе PromRecallDelayDays больше текущей даты.

Логика учета константы CallCenterPromFreq

В списке для обзвона, составленном для выбранной кампании, отличной от Стандартная, будут появляться должники из внутреннего списка обещаний коллектора. Количество должников из кампании, после которых появится один должник из внутреннего списка, определяется значением константы CallCenterPromFreq.
