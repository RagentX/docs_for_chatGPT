Логика прикрепления вложений и ссылок к объектам Системы

Включение функционала прикрепления ссылок и файлов к объектам Системы

Если константа DocLink.ChooseObjectToAttach.Use имеет значение "Истина", то функционал прикрепления ссылок к объектам Системы включен. Дополнительно по согласованию с технической поддержкой может быть настроено:

·Типы ссылок, которые могут прикрепляться только к определенным объектам.

·Отображение двух дополнительных полей на форме прикрепления ссылки.

·Правило для автоматического именования ссылок.

Если константа DocAttach.ChooseObjectToAttach.Use имеет значение "Истина", то функционал прикрепления файлов к объектам Системы включен. Дополнительно по согласованию с технической поддержкой может быть настроено:

·Типы файлов, которые могут прикрепляться только к определенным объектам.

·Отображение двух дополнительных полей на форме прикрепления файла.

·Правило для автоматического именования файлов.

Список модулей, в которых документы и ссылки прикрепляются к объектам

Для списков ссылок и файлов вложений добавлены поля, отображающие объект, с которым связана ссылка или файл. Данные доработки действуют в следующих модулях Системы:

·Карточка должника — вкладка "Вложения", списки "Ссылки" и "Файлы", соответственно.

·Карточка судебной работы — вкладки "Ссылки" и "Вложения" соответственно.

·Карточка исполнительного производства — вкладки "Ссылки" и "Вложения" соответственно.

·Регистрация контакта — панели прикрепления ссылок и файлов, соответственно.

Объекты связи

Объекты, с которыми может быть связано вложение или ссылка

Объект связи для ссылки или файла вложения — это тот объект, к которому прикреплена ссылка или файл. Механизм прикрепления вложений к объектам работал и предыдущих версиях Системы. В настоящей версии Системы добавлена возможность явного отображения объектов связи.

Объектами связи могут быть следующие объекты:

·Долги — файл или ссылка привязаны к долгу.

·Должники — файл или ссылка привязаны к должнику.

·Судебная работа — файл или ссылка привязаны к делу судебной работы (соответствует стадии взыскания "Судебная работа").

·Исполнительное производство — файл или ссылка привязаны к делу исполнительного производства (соответствует стадии взыскания "Исполнительное производство").

Объекты связи и стадии взыскания

Существуют ограничения на список объектов, к которым доступно прикрепление файла или ссылки, в зависимости от стадии взыскания:

·Досудебная стадия — доступны объекты "Долг", "Должник".

·Судебная стадия — доступны объекты "Долг", "Должник", "Судебная работа".

·Исполнительное производство — доступны объекты "Долг", "Должник", Исполнительное производство.

Доступность объектов из разных модулей

Доступность связи ссылок и файлов с объектами в зависимости от модуля, в котором выполняется прикрепление:

·Карточка должника — доступны объекты досудебной стадии "Долг", "Должник".

·Карточка судебной работы — доступны объекты судебной стадии "Долг", "Должник", "Судебная работа".

·Карточка исполнительного производства — доступны объекты стадии "Исполнительное производство" "Долг", "Должник", Исполнительное производство.

·Регистрация контакта — доступны объекты всех стадий.

Ссылки на файлы

Выбор объекта связи при прикреплении ссылки

Если константа DocLink.ChooseObjectToAttach.Use имеет значение "Истина", то при прикреплении ссылки в форме "Ссылка на документ" дополнительно отображается поле "Прикрепить к:".

В выпадающем списке поля "Прикрепить к:" отображаются доступные объекты связи. Каждый объект обозначен названием объекта связи (см. Объект связи) и его идентификатором, разделенными пробелом.

Для каждого модуля из списка, указанного выше, доступны разные объекты.

Объекты для ссылок, доступные из модуля "Карточка должника"

·Долги — все долги данного должника.

Примечание. Если в роли пользователя, добавляющего документ, доступ DEBT_CARD_LINK_ADD_TO_CLOSED имеет значение "TRUE", то в списке отображаются все долги данного должника. Иначе, если доступ имеет значение "FALSE", то отображаются только активные долги, не находящиеся в закрытых статусах 6,7 или 8.

·Должники — сам должник.

Объекты для ссылок, доступные из модуля "Карточка судебной работы"

·Судебная работа — данное дело судебной работы.

·Долги — долг по судебной работе.

·Должники — должник по долгу.

Объекты для ссылок, доступные из модуля "Карточка исполнительного производства"

·Исполнительное производство — данное дело исполнительного производства.

·Долги — долг по судебной работе.

·Должники — должник по долгу.

Объекты для ссылок, доступные из модуля "Регистрация контакта"

·Долг, по которому регистрируется контакт.

·Должник.

·Активная судебная работа по долгу.

Примечание. Активными считаются следующие дела судебной работы:

·Приказы — в статусах, кроме 9,10,11 (см. словарь № 18 "Статус Судебного приказа").

·Иски, правопреемство, банкротство — в статусах кроме 13,14,15,18 (см. словарь № 25 "Статус Судебного дела").

·Судебные дела (предв.) — в статусах кроме 4 (см. словарь № 252 "Статус судебного дела (предв.)").

·Исполнительная надпись — в статусах, кроме 1007,1008,1009,1010 (см. словарь № 25 "Статус Судебного дела").

·Активные исполнительные производства по долгу.

Примечание. Активными считаются дела исполнительного производства, которые не находятся в закрытых статусах 3,4,5,6 или 10.

Список возможных типов ссылок

Для каждого из объектов связи (см. Объект связи) можно настроить разные списки типов ссылок. Тогда после выбора в поле "Прикрепить к:" объекта связи в поле "Тип документа" будут доступны только те термины словаря № 99 "Тип вложения для ссылки", которые разрешены для данного объекта. Данная настройка выполняется по согласованию с технической поддержкой. Это настраивается в технической константе DocLink.TypeRestriction.

Дополнительные поля

Дополнительные технические поля отображаются на форме "Ссылка на документ", если их отображение согласовано с технической поддержкой. Они отображаются так же, как и в списке ссылок:

·Техническое поле типа словарь 1 — дополнительное поле типа "Словарь". Данное поле отображается по согласованию с технической поддержкой, так же, как и словарь, который соответствует данному полю. Наименование поля также согласовывается с поддержкой.

·Техническое поле типа дата 1 — дополнительное поле типа "Дата". Данное поле отображается по согласованию с технической поддержкой. Наименование поля также согласовывается с поддержкой.

Правило именования ссылок

В предыдущей версии Системы на форме "Ссылка на документ" в поле "Название" пользователь вводил название ссылки при каждом добавлении ссылки. Поле было доступно для редактирования.

В настоящей версии Системы добавлена возможность задать правило именования ссылок. Данное правило настраивается по согласованию с технической поддержкой. Если в вашей версии Системы задано правило именования ссылок, то поле "Название" недоступно для редактирования, в нем выводится название, сгенерированное согласно настроенному правилу.

Условия для настройки шаблона именования ссылок

1.Вы можете задать один шаблон именования ссылок для всех ссылок, прикрепляемых к любому объекту связи. Это называется "шаблон по умолчанию". Шаблон по умолчанию является обязательным.

2.Вы можете задать отдельные правила для объектов связи. Тогда для данного объекта связи будет выбираться заданный специально для него шаблон. А для тех объектов, для которых отдельные шаблоны не заданы, будет выбираться шаблон по умолчанию.

3.Шаблон имени ссылки может включать следующие атрибуты прикрепляемой ссылки:

·Объект связи — доступны следующие объекты (см. Объект связи):

·Долги.

·Должники.

·Судебная работа.

·Исполнительное производство.

·Идентификатор объекта связи.

·Тип ссылки — см. словарь № 99.

·Техническое поле типа словарь 1.

·Техническое поле типа дата 1.

4.Шаблон имени ссылки обязательно должен включать хотя бы один символ, который не заменяется значением переменной.

5.Если настроено правило именования, то поле "Название" недоступно для редактирования.

6.Если при прикреплении ссылки оказывается, что ссылка с таким же точно именем уже есть, то к сгенерированному названию добавляется символ подчеркивания "_", а затем порядковый номер.

Шаблон именования ссылок настраивается в технической константе DocLink.NameTemplate.
