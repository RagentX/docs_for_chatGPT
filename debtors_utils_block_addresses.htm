Блокировка адресов

Show/Hide Hidden Text

Утилита блокировки адресов осуществляет блокировку адресов по списку, переданному в Excel-файле. Сначала осуществляется проверка возможности осуществления данной процедуры. Затем осуществляется блокировка в соответствии с записями из файла, прошедшими проверку. Далее для заблокированных адресов осуществляется перенос истории контактов.

Утилита доступна только тем пользователям, в роли которых доступ № 640 DEBT_LIST_BLOCK_ADDRESS имеет значение TRUE.

Формат файла, содержащего список для блокировки

Для выбора доступен файл только в формате Excel.

Первая строка файла должна содержать "шапку".

Проверка списка адресов осуществляется, начиная со второй строки.

Список полей файла для загрузки

·ID адреса;

·Причина блокировки (указывается код термина по словарю № 314 "Причина блокировки адреса");

·ID адреса для переноса истории.

Обязательность заполнения полей таблицы

В файле обязательно должна быть заполнена колонка ID адреса.

Поле Причина блокировки является обязательным для заполнения, если включена константа Address.Block.Reason.Mandatory.

Порядок действий

Для того чтобы осуществить блокировку адресов:

1.Войдите в модуль Список должников.

2.Нажмите на кнопку Утилиты загрузки. В открывшемся списке выберите пункт Блокировка адресов.

3.Откроется стандартное окно для выбора файла. Выберите файл, затем нажмите Открыть (Open).

4.Откроется окно предварительного просмотра. Нажмите кнопку Проверить (остальные кнопки в данный момент неактивны). Будет осуществлена Проверка возможности осуществления блокировки.

5.По окончании проверки в таблице будут заполнены поля "Возможность блокировки" и "Причина невозможности блокировки". Вы можете сохранить файл с ошибочными записями, для этого нажмите кнопку Excel.

6.Нажмите кнопку Загрузка в БД. Будет выполнена блокировка.

7.Если заполнено поле ID телефона для переноса истории, будет осуществлен Перенос истории контактов по заблокированному номеру.

8.По окончании процесса будет показано информационное сообщение. Нажмите ОК.

Проверка возможности осуществления блокировки

Если формат файла не совпадает с требуемым, то выводить пользователю сообщение:

"Файл не соответствует требуемому формату".

Если формат файла верный, то проводится проверка данных в файле:

1.Выполняется проверка наличия в таблице address адреса с addresse.id из столбца "ID адреса". Если адрес не найден, то блокировка не производится с причиной:

"Не найден указанный адрес"

2.Если адрес найден, то определяется его статус блокировки, если адрес уже заблокирован, то блокировка не производится с причиной:

"Адрес уже заблокирован"

3.Выполняется проверка наличия термина в словаре "Причина блокировки адреса", указанного в столбце "Причина блокировки". Если код не найден (или он пустой) и включена константа Address.Block.Reason.Mandatory, то блокировка не производится с причиной:

"Некорректная причина блокировки"

По окончании проверки в таблице заполняются поля:

·Возможность блокировки — значения "Да" или "Нет";

·Причина невозможности блокировки — для записей, имеющих значение "Нет" в предыдущем поле указывается причина.

Выполнение блокировки

Блокировка выполняется только для записей, имеющих в поле Возможность блокировки значение "Да".

Если в списке были найдены строки, которые не подлежат загрузке, то пользователю выводится сообщение:

"Сформировать файл ошибочных записей?":

В зависимости от выбора пользователя:

·Если пользователь выбирает "Нет", то осуществляется блокировка адресов, которые прошли проверку;

·Если пользователь выбирает "Да", то осуществляется блокировка адресов и формируется Excel-файл, содержащий информацию об адресах, которые не были заблокированы и причину невозможности блокировки.

Перенос истории контактов по заблокированному адресу

Если блокировка для адреса была выполнена и в поле "ID адреса для переноса истории" указан идентификатор адреса, который найден в системе, то выполняются следующие действия:

1.Обновляется привязка контактов по блокируемому адресу к новому адресу, который указан в поле "ID адреса для переноса истории".

2.Обновляется привязка писем на блокируемый адрес к новому адресу.

3.Обновляется привязка заявок на письма по блокируемому адресу к новому адресу.

4.Для нового адреса устанавливается статус блокируемого адреса.
