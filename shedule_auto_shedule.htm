Расписание автоматических процедур

Добавление автоматической процедуры в расписание

Для того чтобы добавить автоматическую процедуру в расписание, выполните следующее:

1.Войдите в модуль "Расписание".

2.Во вкладке "Расписание" нажмите кнопку Добавить.

3.Заполните форму с параметрами процедуры.

·Название – выберите название для процедуры (см. Словарь № 10 "Название процедуры").

·Клиент – выберите клиента из списка.

·Портфель – выберите портфель.

·Формула – выберите формулу (см. модуль Утилиты а Произвольные формулы).

·Период запуска – см. Словарь № 9 "Тип периода".

Примечание. Если значение константы SchedulerUseExtendedScheduleMode = TRUE, то для настройки доступны все типы периода. Если же значение константы SchedulerUseExtendedScheduleMode = FALSE, то тип периода "Однократно" не доступен для настройки.

·Время – введите время запуска мероприятия. В случае еженедельного или ежемесячного запуска процедуры поле содержит также данные о дне запуска.

·Флаг Не запускать включается, если вы хотите временно перестать запускать процедуру, но не хотите ее удалять.

4.Нажмите OK.

Изменение автоматической процедуры в расписании

Для того чтобы изменить автоматическую процедуру в расписании, выполните следующее:

1.Войдите в модуль "Расписание".

2.Активируйте вкладку "Расписание".

3.Выберите процедуру в списке и нажмите кнопку Изменить или кликните мышью (выполните двойной щелчок левой клавишей). Откроется окно с параметрами процедуры.

4.Измените значения параметров.

5.Нажмите OK.

Удаление автоматической процедуры из расписания

Для того чтобы удалить автоматическую процедуру из расписания, выполните следующее:

1.Войдите в модуль "Расписание".

2.Активируйте вкладку "Расписание".

3.Выберите процедуру в списке.

4.Нажмите кнопку Удалить.

5.Подтвердите удаление.

Запуск автоматической процедуры вручную

Есть возможность запустить процедуру вручную, непосредственно, вне расписания.

Примечание. Пользователь может осуществить ручной запуск только тех процедур, идентификаторы которых перечислены в значении доступа  № 541 SCHEDULE_MANUAL_START_PROC в его роли. Процедуры, идентификаторы которых не указаны в значении данного доступа, не доступны для ручного запуска пользователем.

Для ручного запуска процедуры:

1.Войдите в модуль "Расписание".

2.Активируйте вкладку "Расписание".

3.Выберите процедуру в списке.

4.Нажмите кнопку Запустить.

5.Процедура начнет выполняться. Если на ее выполнение потребуется значительное время, вам будет показан прогресс выполнения

Просмотр лога процедуры

Право на просмотр лога процедуры имеют пользователи, в роли которых доступ SCHEDULE_LOG_VIEW имеет значение TRUE. Если доступ отсутствует или выключен, кнопка История на форме отсутствует.

Для того чтобы просмотреть список последних запусков процедуры:

1.Войдите в модуль "Расписание".

2.Активируйте вкладку "Расписание".

3.Выберите процедуру в списке.

4.Нажмите кнопку История.

5.В открывшемся меню выберите пункт Лог запуска.

Просмотр истории изменений

Право на просмотр истории изменений процедуры имеют пользователи, в роли которых доступ SCHEDULE_LOG_VIEW имеет значение TRUE. Если доступ отсутствует или выключен, кнопка История на форме отсутствует.

Для того чтобы просмотреть историю изменений выбранной процедуры:

1.Войдите в модуль "Расписание".

2.Активируйте вкладку "Расписание".

3.Выберите процедуру в списке.

4.Нажмите кнопку История.

5.В открывшемся меню выберите пункт История изменений.

 

Форма История изменений содержит следующие элементы:

·Область данных Версии, содержащая список всех редакций данной группы;

·Область данных Изменения, содержащая подробности выбранной редакции.

Типы изменений:

·I (insert) — присваивается при первой редакции группы, то есть ее создании;

·U (update) — присваивается при изменениях атрибутов группы;

·D (delete) — присваивается при удалении группы (недоступно для просмотра в данной форме).

Для просмотра выбранного поля "Старое значение" или "Новое значение" в области данных Изменения щелчком мыши можно открыть область, содержащую значение этого поля, размеры которой доступны для изменения. Это используется в тех случаях, когда, например, значение поля имеет длину, превышающую ширина столбца.
