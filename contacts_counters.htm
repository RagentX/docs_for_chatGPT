Регистрация показаний счетчиков

При регистрации тех исходов контакта, для которых настроены счетчики (произвольные атрибуты для исхода контакта), появляется панель с указанными произвольными атрибутами.

Для каждого атрибута отображается значение по следующему принципу:

·Если по выбранному долгу не создан данный атрибут, то значение — "пусто";

·Если по выбранному долгу такой атрибут уже существует, то:

·Если флаг Создание нового включен, то отображается пустое значение.

·Если флаг выключен — отображается текущее значение данного атрибута.

Нижнее поле предназначено для ввода значения выбранного произвольного атрибута.

Для того чтобы ввести значения счетчиков при регистрации контакта, выполните следующие действия:

1.При необходимости выберите долг должника.

2.Выберите номер телефона должника и выполните двойной клик мышью на этой строке.

3.При продвижении по дереву контактов выберите узел, в котором настроены счетчики. Выполните двойной клик.

4.Откроется форма ввода данных по результату контакта. Введите нужные значения счетчиков. Для этого:

a)Выберите счетчик.

b)Введите значение в поле ввода значения, расположенное ниже списка счетчиков.

c)При необходимости выберите следующий счетчик, и так далее.

5.Введите остальные данные о результате контакта.

6.Нажмите ОК.

После этого в карточке должника на вкладке Произвольные атрибуты появятся значения произвольных атрибутов, введенные при регистрации контакта. Если для атрибута включен флаг Создание нового, то таких записей будет столько, сколько раз атрибут создавался.

При регистрации тех исходов контакта, для которых настроены произвольные атрибуты для исхода контакта, появляется панель для ввода/изменения данных атрибутов.

В настоящей версии Системы после ввода нового значения произвольного атрибута добавлен вызов EPS-метода для проверки введенного значения. 

·Если метод возвращает результат 0, это означает, что проверка пройдена успешно, в поле значения атрибута устанавливается введенное значение. 

·Если метод возвращает 1, это означает, что проверка не успешна. В этом случае значение атрибута не изменяется на введенное, а пользователю показывается соответствующее сообщение об ошибке.

Например, для клиента Квадра разрешается вводить новое значение счетчика только если оно не меньше предыдущего.
