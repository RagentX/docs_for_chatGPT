Алгоритм распознавания адреса

Show/Hide Hidden Text

Распознавание адресов осуществляется в фоновом режиме. Сервис выполняет запрос к БД для получения адреса и отправляет его в сервис Yandex для получения GPS-координат. Ответ сервиса обрабатывается в соответствии со списком правил. Полученные координаты записываются в таблицу ADDRESS_GEO.

Логика распознавания адреса

Проверка идет последовательно по правилам, согласно приоритету, взятому из параметра <Priority> (см. Блок-схему на Рисунке I‑16.). То есть, берем правило с минимальным значением в <Priority> и отбираем все найденные координаты с указанными kind и precision. Если в правиле не указаны KindFilter или PrecisionFilter, то берем все значения для данного параметра.

1.Если найдена всего 1 пара координат – то считаем адрес распознанным.

2.Если найдено несколько координат, то анализируем параметр MaxDistance.

·если расстояние между адресами меньше значения MaxDistance, то считаем адреса идентичными. А в качестве координаты для записи берем среднее арифметическое

·если расстояние между адресами больше значения MaxDistance, то считаем адрес не распознанным.

3.Если адресов не найдено, то переходим к следующему правилу, если такого нет, то считаем не распознанным.
