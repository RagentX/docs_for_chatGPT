Объединение должников

Коммерческая константа для включения функционала @UseUnionPerson.

Функционал объединения должников доступен из модулей:

·Постановка задач;

·Группы свободного поиска.

Функция объединения должников позволяет перенести долги от должников с похожими (одинаковыми) данными на одного должника и тем самым удалить дублирующиеся данные о должниках. Доступ к функции объединения должников имеют только те пользователи, в роли которых доступ № 255 WORK_TASK_UNION_PERSON имеет значение TRUE.

Критериями для объединения должников могут быть:

·фамилия, имя, отчество;

·дата рождения;

·паспортные данные;

·номер договора;

·лицевой счет;

·ID в системе клиента;

·ИНН.

Объединение с выбором критериев (переносятся все долги)

Возможен произвольный выбор нескольких критериев. В случае выбора какого-либо условия для объединения должников (переноса долгов) необходимо, чтобы выбранные в критерии атрибуты долга/должника не были пустыми. Если в качестве критерия выбран адрес, то должен быть выбран еще хотя бы один (любой) критерий.

Если одним из критериев объединения должников является адрес, то сравнение производится по адресу выбранного в критериях типа, имеющего максимальный id (т. е. последний загруженный).

Можно объединить должников физических лиц с должниками юридических лиц, если они удовлетворяют выбранным критериям.

Объединение должников (перенос долгов) производится на должника с наименьшим ID персоны.

Адреса, телефоны, адреса электронной почты и комментарии при этом переносятся вместе с долгами.

Объединение без выбора критериев (переносятся только выбранные долги)

Также возможно объединение выделенных должников без выбора критериев. При этом критерием объединения будут серия и номер паспорта.

В таком случае все выделенные долги будут перенесены должнику с наименьшим ID персоны.

Адреса, телефоны, адреса электронной почты и комментарии при этом дублируются.

Порядок действий для объединения должников

Для объединения должников выполните следующие действия:

1.При необходимости в нужной вкладке модуля сформируйте выборку долгов с помощью фильтров и/или выделите записи с помощью клавиш SHIFT и CTRL.

2.В контекстном меню (вызывается правой кнопкой мыши) выберите пункт Объединить должников. Откроется окно выбора критериев объединения.

3.В окне установите флаги для тех критериев, при совпадении которых должники будут считаться одинаковыми, и их можно будет объединить.

4.Нажмите кнопку ОК. Откроется окно, содержащий перечень найденных должников с указанием выбранных критериев. В окне доступна возможность группировки и сортировки данных. На примере ниже показана группировка найденных должников по дате рождения.

Примечание: если должники с совпадающими критериями не найдены, будет выдано соответствующее сообщение.

5.В списке при необходимости выделите тех должников, которых требуется объединить, с помощью клавиш SHIFT и CTRL.

6.Нажмите кнопку Продолжить и в раскрывающемся списке выберите вариант выполнения операции: По всей выборке или Только выделенные. Будет выполнено объединение должников и перенос данных согласно правилам ниже.

7.По окончании объединения будет выдано соответствующее сообщение. Нажмите кнопку ОК, чтобы завершить операцию.

При объединении должников производится следующий перенос данных:

·Долги;

·Телефоны (в случае, когда не все долги должника переносятся на должника с наименьшим id, телефоны дублируются);

·Адреса (в случае, когда не все долги должника переносятся на должника с наименьшим id, адреса дублируются);

·Контактная информация (для юридического лиц) (в случае, когда не все долги должника переносятся на должника с наименьшим id, информация дублируется);

·Расчетные счета (для юридического лиц) (в случае, когда не все долги должника переносятся на должника с наименьшим id, информация дублируется);

·Документы (переносятся только в случае, если у должника, с которого переносятся долги, нет больше долгов);

·Ссылки (переносятся только в случае, если у должника, с которого переносятся долги, нет больше долгов);

·Контакты;

·Произвольные атрибуты;

·Поручители;

·Имущество (переносятся только в случае, если у должника, с которого переносятся долги, нет больше долгов);

·Письма;

·SMS-сообщения;

·Email-сообщения;

·Заявки на письма;

·Заявки на Email;

·Приказы/иски;

·ИП.

При объединении должников производится обновление следующих атрибутов (только в случае, если указанный атрибут должника не заполнен (пустой); если он заполнен, обновление указанного атрибута не производится):

·Дата рождения;

·Место работы;

·Должность;

·Комментарий к персоне;

·Семейное положение;

·Пол;

·Адрес электронной почты;

·ИНН (для юридического лица);

·КПП (для юридического лица);

·ОКВЭД (для юридического лица);

·ОКПО (для юридического лица);

·Расчетный счет (для юридического лица);

·Банк (для юридического лица);

·Кор. Счет (для юридического лица);

·БИК (для юридического лица);

·ОГРН (для юридического лица);

·Ген. Директор (для юридического лица);

·Вебсайт (для юридического лица).

Дубли телефонов и адресов, на которых не зарегистрированы контакты, письма, SMS, заявки на письма, удаляются (с учетом типа).
