Этап 1. Построчная проверка данных

Построчная проверка данных включает:

·контроль корректности реквизитов даты и времени;

·контроль корректности числовых реквизитов;

·контроль корректности целочисленных реквизитов;

·контроль корректности словарных реквизитов;

·контроль корректности текстовых реквизитов (в том числе, см. далее — Алгоритм контроля имени и отчества персоны).

На данном этапе любая ошибка при проверке строки таблицы является критической для этой строки. Ошибочные ячейки окрашиваются желтым цветом.

Если по окончании первого этапа проверки ошибок не выявлено, автоматически выполняется переход ко второму этапу (массовой проверке данных). При наличии ошибки хотя бы в одной строке будет выдано сообщение: "Внимание: имеются записи (<число ошибочных записей> из <общее число записей>), которые не могут быть загружены в БД. Сформировать файл ошибочных записей с их удалением из таблицы?"

Чтобы подтвердить удаление, нажмите кнопку Yes. Все ошибочные записи будут удалены из таблицы. В файле ошибочных записей будут сформированы исходные тексты записей и причины ошибок в текстовой форме, а также будет выдано информационное сообщение "Ошибочные записи сохранены в файле <директория и имя файла>". После этого осуществляется автоматический переход ко второму этапу (массовой проверке данных).

Примечание. По окончании загрузки в БД записей, не содержащих ошибок, можно вернуться к ошибочным записям. Файл ошибочных записей соответствует тому же формату, что исходный файл, и после удаления первого столбца (причины ошибок) может быть обработан заново.

При нажатии кнопки No проверка данных закончится досрочно, массовая проверка данных будет отменена.

Алгоритм контроля имени и отчества персоны при загрузке

1.При наличии колонки имени (отчества) и колонки типа персоны.

В случае пустого значения имени (отчества) для физического лица поле имени (отчества) бракуется, а в логическом файле формируется сообщение об отсутствии в строке значения обязательного реквизита.

2.При наличии колонки имени (отчества) и отсутствии колонки типа персоны.

В случае пустого значения имени (отчества) и значении константы LoadDataPersonTypDefault = 1 поле имени (отчества) бракуется, а в логическом файле формируется сообщение об отсутствии в строке значения обязательного реквизита.

3.При отсутствии колонки имени (отчества) и наличии колонки типа персоны.

В случае наличия в исходном файле хотя бы одного физического лица на экран выводится сообщение об отсутствии в формате описания обязательного реквизита. Аналогичное сообщение формируется в логическом файле.

4.При отсутствии колонки имени (отчества) и колонки типа персоны.

В случае значения константы LoadDataPersonTypDefault = 1 на экран выводится сообщение об отсутствии в формате описания обязательного реквизита. Аналогичное сообщение формируется в логическом файле.
