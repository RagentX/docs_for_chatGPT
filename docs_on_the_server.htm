Программное API Delphi для управления документами

Созданы стандартные процедуры для работы с документами в Системе.

Уникальное имя файла

Для каждого файла, сохраняемого на сервере, генерируется уникальное внутренне имя. Оно состоит из уникального идентификатора файла и расширения. Расширение файла соответствует расширению из пользовательского названия файла. 

Уникальное имя файла хранится в БД.

Полное расположение файла документа

Если документ сохраняется на сервере, то он сохраняется по следующему расположению:

1.Значение константы DocAttach.SavePath;

2.Относительный путь к файлу на сервере, который зависит от идентификатора соответствующей документу записи в таблице БД описывающей сохраненные документы. (также хранится в БД);

3.Внутренне имя документа.

Сохранение документа

Выходным параметром процедуры является идентификатор новой записи в таблице БД, описывающей сохраненные документы.

Процедура осуществляет сохранение документа в зависимости от значения технической константы DocAttach.SaveMode.

Если значение константы равно 1, то документ сохраняется в БД. Тело файла сохраняется в соответствующую таблицу.

Если значение константы равно 2, то документ сохраняется на сервере. При этом осуществляются следующие действия:

1.Генерируется уникальное имя файла и сохраняется в БД.

2.Формируется относительный путь к файлу на сервере.

3.Параметры документа сохраняются в БД.

4.Документ сохраняется на сервер по соответствующему расположению.

·если при сохранении возникают ошибки, то транзакция в БД откатывается;

·если при сохранении ошибок, не возникает, то транзакция в БД сохраняется.

Для документов, полученных из внешней системы, сохранение осуществляется при получении документа из внешней системы. 

Редактирование документа

Процедура осуществляет редактирование документа в зависимости от значения константы DocAttach.SaveMode.

·Если значение константы равно 1, то измененные параметры файла, а также, при необходимости, и тело файла, обновляются в соответствующей таблице БД.

·Если значение константы равно 2, то:

·Если изменяются атрибуты файла, то их значения в таблице БД обновляется.

·Если изменяется файл, то для него генерируется новое уникальное внутреннее имя, а затем он сохраняется на сервер по соответствующему расположению, расположение не изменяется при редактировании документа. Затем старый файл удаляется.

·если при сохранении возникают ошибки, то транзакция в БД откатывается;

·если при сохранении ошибок, не возникает, то транзакция в БД сохраняется.

Если с момента последнего сохранения документа данная константа была изменена:

·Если режим хранения меняется с 1 на 2, то после успешного сохранения фала на сервер очищается его тело в БД (поле doc_attach.filebody);

·Если режим хранения меняется с 2 на 1, то после успешного сохранения файла в БД удаляется файл с сервера из директории (значения константы DocAttach.SavePath + doc_attach.rel_server_path    + doc_attach.file_server_name).

Запрещается редактировать документы, полученные из внешней системы. Для этого проверяется значение атрибута doc_attach.ext_id. Если он не равен NULL, то редактирование документа не разрешается.

Удаление документа

При удалении документа осуществляются следующие действия:

1.Запись о документе удаляется из таблицы.

2.Если значение константы DocAttach.SaveMode равно 1, то удаляется файл с сервера.

Получение документа

Действия зависят от значения атрибута doc_attach.save_mode:

·Если doc_attach.save_mode = 1 или (doc_attach.save_mode = null и doc_attach.filebody <> null), это означает, что файл хранится в БД. Тело файла получают из doc_attach.filebody;

·Если doc_attach.save_mode = 2 или (doc_attach.save_mode = null и doc_attach.filebody = null), это означает, что файл хранится на сервере. Путь к файлу определяется по соответствующему расположению следующим образом: 

½DocAttach.SavePath + doc_attach.rel_server_path    + doc_attach.file_server_name;

·Если doc_attach.save_mode = 3, это означает, что файл хранится во внешней системе. Тогда содержимое файла получают с помощью специальной процедуры, реализованной отдельно для каждого конкретного случая.

 Примечание. При получении документа значение константы DocAttach.SaveMode не проверяется.

Единая карточка документа

Создана единая карточка документа, которая вызывается из всех рабочих мест Системы, где это необходимо.

Карточка документа содержит следующие элементы:

·Название — название документа, обязательное поле, может быть заполнено автоматически через выбор файла по кнопке  "Добавить документ";

·Тип документа — см. словарь № 100. Если значение константы LawActExecUseCardDocAttachMandatoryType равно TRUE, то поле является обязательным для заполнения;

·Номер — номер документа, необязательное поле;

·Версия — два поля, в каждое из которых можно ввести целое число. Необязательно для заполнения. 

 Примечание. Если поля Версия не заполнены, то им присваиваются значения по умолчанию "1" и "0".

·Кнопка  Добавить документ для открытия окна выбора файла;

·Кнопка ОК для сохранения данных.

 Примечание: Поля Версия отображаются только при открытии формы из модулей:

·Юридический департамент;

·Карточка судебной работы;

·Карточка исполнительного производства.

В модуле Карточка должника

Карточка документа

Карточка документа содержит следующие элементы:

·Название — название документа, обязательное поле, может быть заполнено автоматически через выбор файла по кнопке  "Добавить документ";

·Тип документа — см. словарь № 100. Если значение константы DebtCardUseDocAttachMandatoryType равно TRUE, то поле является обязательным для заполнения;

·Номер — номер документа, необязательное поле;

·Кнопка  Добавить документ для открытия окна выбора файла;

·Кнопка ОК для сохранения данных.

Действия с документами в Системе

В Системе поддержаны следующие действия с документами:

Добавление документа

В Системе реализовано два варианта процесса добавления документа:

·Через вызов карточки документа. Данный вариант используется для следующих объектов Системы (см. словарь № 117):

·8 — Долги;

·30 — Должники;

·46 — Судебная работа;

·47 — Исполнительное производство;

·63 — Уголовное преследование;

·66 — Обращения.

·Через окно выбора файла (без карточки документа). Данный вариант используется для объектов Системы:

·49 — Данные о портфелях;

·52 — Запросы;

·53 — Принятые меры;

·54 — Жалобы;

·58 — Исполнительные листы на мировое соглашение;

·68 — Информация о банке.

Вызывается программное API для хранения документа.

Редактирование документа

Вызывается карточка документа. Вызывается программное API для редактирования документа.

Удаление документа

Вызывается программное API для удаления документа.

Открытие документа

Документ сохраняется во временную папку с уникальным временным названием. Далее оттуда документ открывается средствами операционной системы.

Выгрузка документа

Документ сохраняется в выбранную папку.

Печать документа

Документ сохраняется во временную папку с уникальным временным названием. Далее оттуда документ отправляется на печать средствами операционной системы. Затем документ удаляется из временной папки.

Действия с документами, доступные из разных РМ

Карточка должника

В карточке должника на вкладке Документы доступны следующие операции:

·Добавление документа;

·Редактирование документа;

·Удаление документа;

·Открытие документа;

·Выгрузка документа.

Карточка судебной работы

В карточке судебной работы действия с документами осуществляются на следующих вкладках:

·Вкладка Документы: 

·Добавление документа;

·Редактирование документа;

·Удаление документа (по данной операции выключается флаг активности данного документа, поэтому используется операция редактирования документа);

·Открытие документа;

·Печать документа;

·Вкладка Мировые соглашения:

·Добавление документа;

·Удаление документа (по данной операции выключается флаг активности данного документа, поэтому используется операция редактирования документа);

·Вкладка Протокол:

·Открытие документа — осуществляется для тех записей, где документ был прикреплен;

·Вкладка Принятые меры по ИП; 

·Добавление документа;

·Удаление документа (по данной операции выключается флаг активности данного документа, поэтому используется операция редактирования документа).

Карточка исполнительного производства

В карточке исполнительного производства действия с документами осуществляются на следующих вкладках:

·Вкладка Документы: 

·Добавление документа;

·Редактирование документа;

·Удаление документа (по данной операции выключается флаг активности данного документа, поэтому используется операция редактирования документа);

·Открытие документа;

·Вкладка Протокол:

·Открытие документа — осуществляется для тех записей, где документ был прикреплен;

·Вкладка Принятые меры:

·Добавление документа;

·Удаление документа (по данной операции выключается флаг активности данного документа, поэтому используется операция редактирования документа);

·Вкладка Жалобы:

·Добавление документа;

·Удаление документа (по данной операции выключается флаг активности данного документа, поэтому используется операция редактирования документа);

·Вкладка Запросы;

·Добавление документа;

·Удаление документа (по данной операции выключается флаг активности данного документа, поэтому используется операция редактирования документа).

Клиенты и портфели

Кнопка Документы окна Контрагенты и кнопка Документы окна Портфели:

·Добавление документа (осуществляется без открытия карточки документа);

·Удаление документа;

·Открытие документа;

·Выгрузка документа.

Обращения

Форма Обращение:

·Добавление документа;

·Редактирование документа;

·Удаление документа;

·Открытие документа;

·Выгрузка документа.

Карточка уголовного преследования

Вкладка Документы:

·Добавление документа;

·Редактирование документа;

·Удаление документа;

·Открытие документа.

Функционал прикрепления документа

Исполнение каждого этапа жизненного цикла юридического производства сопровождается прикреплением документа. При этом осуществляется добавление документа путем выбора файла, без открытия карточки документа.

Сокрытие функционала "Ссылки"

Если константа DocLink.Use выключена, то функционал Ссылки скрывается в следующих модулях:

·Модуль Карточка должника (При этом на вкладке Документы блок Файлы отображается на всю вкладку);

·Модуль Карточка судебной работы;

·Модуль Карточка исполнительного производства;

·Модуль Юридический департамент (Скрывается действие контекстного меню Добавить документ -> Добавить ссылку на файл);

·Модуль Карточка уголовного преследования.
