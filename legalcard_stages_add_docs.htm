Прикрепление документа к действию

Исполнение каждого действия судебной работы в юридическом производстве сопровождается прикреплением документа.

Пользователю предлагается форма прикрепления документа. В ней:

·Если требуется прикрепить новый документ:

·установите переключатель Новый документ;

·нажмите кнопку , выберите документ в стандартном диалоговом окне Windows и нажмите кнопку Open (Открыть);

·в поле Тип вложения выберите тип (см. словарь № 100 "Тип вложения для файла").

Прикрепленный документ будет доступен на вкладке Вложения. Можно прикрепить до 10 документов.

·Если требуется прикрепить существующий документ:

·установите переключатель Существующий документ;

·в раскрывающемся списке отображаются документы из списка на вкладке Вложения (для соответствующей стадии), выберите нужный документ.

·Если прикреплять документ не требуется, установите переключатель Не прикреплять.

Регулирование обязательности прикрепления документа к действию судебной работы

Обязательность прикрепления документа к действию юридического производства регулируется по согласованию с технической поддержкой технической константой LawMandatoryDocAttachActivityList.

В константе перечислены идентификаторы (ID) действий судебной работы, для которых прикрепление документов обязательно. Если выполняемое действие указано в значении константы, то переключатель Не прикреплять отсутствует, требуется обязательно прикрепить документ.

Регулирование обязательности прикрепления документа к запросу

1.В первую очередь проверяется значение константы DocAttachAvailability.ActivityList.

1.1Если константа DocAttachAvailability.ActivityList не включает значение 142 (идентификатор действия "Создание запроса"), то при массовом формировании запросов прикрепление документов не предусмотрено.

1.2Если константа DocAttachAvailability.ActivityList включает значение 142, то после заполнения формы «Запрос» и нажатия кнопки «ОК» будет открываться форма для прикрепления документов.

2.Далее выполняется проверка значения константы LawMandatoryDocAttachActivityList.

2.1Если константа LawMandatoryDocAttachActivityList не включает значение 142, значит прикрепление документа к запросу не является обязательным. В форме прикрепления документа отображается вариант «Не прикреплять».

2.2Если константа LawMandatoryDocAttachActivityList включает значение 142, то алгоритм движется дальше.

3.Проверяется значение константы LawExec.DocAttachMandatory.RequestList, в которой перечисляются типы запросов, для которых прикрепление документов обязательно.

3.1Если константа LawExec.DocAttachMandatory.RequestList пустая, то это означает, что прикрепление документа обязательно для всех типов запросов. В этом случае после заполнения формы «Запрос» и нажатия кнопки «ОК» будет открываться форма для прикрепления документов, в которой вариант «Не прикреплять» не отображается. При попытке закрыть окно отображается сообщение с текстом:

Необходимо обязательно прикрепить документ. Формирование запроса будет прервано, продолжить?

и вариантами «Да» и «Нет». При выборе ответа «Да» операция по созданию запроса прерывается. При выборе ответа «Нет» пользователь возвращается обратно на форму прикрепления документа.

3.2Если константа LawExec.DocAttachMandatory.RequestList не пустая, то:

·Если константа LawExec.DocAttachMandatory.RequestList включает данный тип запроса (соответствующий код по словарю 44), то прикрепление документа обязательно. Форма прикрепления документа и возможности действий на ней будут реализованы так же, как в пункте 3.1.

·Если константа LawExec.DocAttachMandatory.RequestList не включает данный тип запроса, то прикрепление документа не обязательно. Форма прикрепления документа и возможности действий на ней будут реализованы так же, как в пункте 2.1.

Примечание. Если выполнено массовое прикрепление документа, то это означает, что документ прикрепляется для каждого ИП, то есть файл многократно заносится в систему.
