Проверка данных для формата "Загрузка платежей"

Следующим этапом обработки данных о платежах и остатках к погашению, полученных при открытии файла MS Excel, является их проверка. Для того чтобы проверить данные, в форме Загрузка платежей нажмите кнопку Проверить. При выполнении проверки данных в нижней строке формы будет отображаться тип операции (проверка данных), текущие данные о проценте выполнения работы, директория и имя файла.

Проверка данных по каждой строке таблицы включает:

·контроль наличия обязательного реквизита должника (банка);

·контроль наличия хотя бы одного реквизита обязательной группы (ФИО, номера договора, номера счета или ID в системе клиента);

·контроль парности реквизитов платежа (даты внесения платежа и суммы платежа). Парные реквизиты должны одновременно иметься в наличии или отсутствовать;

·контроль парности реквизитов остатка (остатка погашения и даты расчета);

·контроль наличия хотя бы одной из пар реквизитов (платежа и/или остатка);

·контроль корректности заданной даты внесения платежа;

·контроль ограничений на дату внесения платежа (при ее корректности). Дата внесения платежа не должна превышать текущую дату;

·контроль корректности заданной суммы платежа;

·контроль ограничения на сумму платежа (при ее корректности). Сумма платежа должна иметь положительное значение;

·контроль корректности типа валюты;

·контроль корректности заданной даты учета информации;

·контроль корректности заданного остатка погашения;

·контроль корректности заданной даты расчета;

·контроль корректности заданной даты актуальности остатка;

·контроль ограничения на дату актуальности остатка (при ее корректности). Дата актуальности остатка должна превышать дату расчета;

·контроль корректности заданной комиссии;

·контроль корректности типа назначения платежа (с использованием словаря);

·контроль наличия банка с заданным названием в базе данных;

·контроль наличия портфеля с заданным названием в базе данных;

·контроль соответствия найденного портфеля найденному банку;

·контроль наличия в базе данных должника по ключевым реквизитам (номер договора, номер счета, ID в системе клиента) с учетом ФИО должника, банка и портфеля;

·контроль соответствия заданного ФИО и ФИО для найденного долга;

·контроль соответствия заданного банка и банка для найденного долга;

·контроль соответствия заданного портфеля и портфеля для найденного долга;

·контроль наличия в базе данных хотя бы одного неподтвержденного платежа для найденного должника с аналогичными данными по дате платежа, сумме платежа и валюте (в отсутствии аналогичных подтвержденных платежей);

·контроль наличия в базе данных хотя бы одного подтвержденного платежа для найденного должника с аналогичными данными по дате платежа, сумме платежа и валюте.

Текстовые реквизиты каждой строки автоматически обрезаются до максимально допустимой длины:

·64 символа – для банка и портфеля;

·322 символа – для ФИО;

·128 символов – для номера договора;

·32 символа – для номера счета и ID в системе клиента;

·512 символов – для комментария к платежу.

Некритическими ошибками при проверке строки таблицы является:

·несоответствие ФИО для найденного долга заданному ФИО;

·наличие в базе данных неподтвержденного платежа;

·наличие в базе данных подтвержденного платежа.

Все остальные ошибки являются критическими.

В процессе контроля корректности заданного типа валюты проверяется:

·наличие данных о типе валюты в словаре;

·наличие в строке даты внесения платежа (если тип валюты отличается от рубля);

·наличие в базе данных курса валюты на дату внесения платежа (если тип валюты отличается от рубля).

При отсутствии в строке данных о типе валюты автоматически фиксируется значение "Рубль".

При наличии в строке остатка погашения со значением меньше одной копейки он автоматически заменяется нулевым значением.

В зависимости от результатов проверки будет выдано соответствующее сообщение:

·при отсутствии критических ошибок — "Данные полностью соответствуют формату загрузки";

·если все записи содержат критические ошибки — "Загрузка невозможна, ни одна запись не соответствует формату загрузки";

·если имеются и корректные, и ошибочные записи — "Можно загрузить <записей количество корректных записей>, номер первой ошибочной <записи номер записи>".

По окончании проверки данных таблицы состояние окна формы автоматически изменяется следующим образом:

·ячейки таблицы, по которым зафиксированы ошибки, окрашиваются определенным цветом в зависимости от типа ошибки (см. Цветовое отображение ошибок);

·при отсутствии критических ошибок на экран выводится сообщение: "Данные полностью соответствуют формату загрузки";

·если все записи содержат критические ошибки, на экран выводится сообщение: "Загрузка невозможна, ни одна запись не соответствует формату загрузки";

·если имеются и корректные, и ошибочные записи, на экран выводится сообщение: "Можно загрузить записей количество корректных записей, номер первой ошибочной записи номер записи";

·кнопки "Проверить" и "Отменить" становятся доступными, восстанавливается возможность редактирования вручную ячеек таблицы;

·кнопка "Сохранить в БД" остается блокированной в случае пустой таблицы данных, а также, если все строки таблицы содержат критические ошибки;

·при наличии в таблице хотя бы одной записи, не содержащей критических ошибок, кнопка "Сохранить в БД" становится доступной.
