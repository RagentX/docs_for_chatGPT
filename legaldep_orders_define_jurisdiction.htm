Определение подсудности

Функционал определения подсудности работает на вкладках Приказы и Иски.

Функционал определения подсудности доступен только в том случае, если Система использует интеграцию с сервисом для определения подсудности и подведомственности CSBI, то есть при включенной коммерческой константе @CSBI.Use.

Данный функционал доступен только пользователям, в роли которых доступ № 721 LAW_ACT_LIST_ADD_ACT_TO_CSBI_PACKAGE имеет значение TRUE.

Определение подсудности выбранных дел состоит в добавлении их в пакет CSBI для определения подсудности, находящийся в состоянии "Формирование". Определение подсудности возможно только при наличии на вкладке Пакеты CSBI, на подчиненной вкладке Определение подсудности хотя бы одного пакета CSBI в состоянии "Формирование".

Порядок действий при определении подсудности

Для того чтобы создать заявки на определение подсудности, выполните следующее:

1.В модуле Юридический департамент активируйте вкладку Приказы или Иски.

2.При необходимости сформируйте выборку долгов с помощью фильтров и/или мультифильтра.

3.Для выбора нескольких записей используйте клавиши SHIFT и/или CTRL.

4.В контекстном меню (вызывается правой кнопкой мыши) выберите пункт Определение подсудности.

5.Откроется форма Добавление дел в пакет для выбора пакета, в который будут добавляться заявки. В поле Пакет выберите пакет из списка пакетов для определения подсудности, находящихся в состоянии "Формирование", затем нажмите ОК.

6.Будут проведены следующие проверки:

·Если дело уже есть хотя бы в одном пакете с типом "Определение подсудности", имеющим состояние "Формирование", "Сформирован", "Передан" И заявка по данному делу имеет статус "Ожидание отправки" или "В обработке", то такое дело не добавляется в выбранный пакет.

·Иначе, если для дела не получается найти адрес согласно настройкам выбора адреса для пакета, оно не добавляется в пакет. Рассматриваются только незаблокированные адреса. В случае если подходит несколько адресов, например, отобрано 2 адреса с типом "Регистрация", то выбирается последний добавленный (с наибольшим значением даты добавления адреса).

·Иначе дело добавляется в выбранный пакет, т.е. в выбранном пакете создается заявка на определение подсудности по данному делу. Заявка находится в статусе "Ожидание отправки".

7.По результатам выполнения действия пользователю выводится сообщение с указанием количества дел, заявки по которым были успешно созданы и количества дел, заявки по которым не были созданы.
