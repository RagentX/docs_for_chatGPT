Изменение запроса

Для того чтобы изменить данные о запросе, выполните следующие действия:

1.В карточке исполнительного производства на вкладке Запросы выберите нужную запись и нажмите кнопку .

2.В форме редактирования данных о запросе измените необходимые атрибуты.

3.Нажмите кнопку ОК, чтобы сохранить изменения.

Обработка трек-номера при редактировании запроса

Если в Системе включена интеграция с почтой России (Коммерческая константа @PostRussia.Use), то на форме дополнительно отображается поле Трек-номер. В этом случае для значения, введенного в данное поле, выполняется следующая обработка.

При нажатии кнопки ОК при редактировании запроса, если поле Трек-номер непустое, для него выполняются следующие проверки:

·если в поле "Трек-номер" введено значение, содержащее буквенные символы (т.е. не только цифры) или введенная строка меньше 14 символов, пользователю выводится сообщение с текстом:

"Трек-номер не соответствует внутрироссийскому формату почтового идентификатора. Продолжить?"

с вариантами "Да", "Нет".

·при нажатии кнопки "Да" выполняются следующие действия:

·если значение статуса текущего письма в таблице, хранящей письма и их статусы, не пусто, выводится сообщение с текстом:

"Текущий статус письма будет сброшен. Продолжить?"

с вариантами "Да", "Нет". При нажатии кнопки "Да" обновляется соответствующая запись в таблице, хранящей письма и их статусы. При нажатии кнопки "Нет" пользователь "возвращается" на форму "Запрос".

·иначе введенное значение сохраняется в новой записи таблицы, хранящей письма и их статусы.

·при нажатии кнопки "Нет" пользователь "возвращается" на форму "Запрос".

·Иначе, т.е. если в поле Трек-номер строка из 14 цифр, выполняются следующие действия:

·если значение статуса текущего письма в таблице, хранящей письма и их статусы, не пусто, выводится сообщение с текстом:

"Текущий статус письма будет сброшен. Продолжить?"

с вариантами "Да", "Нет". При нажатии кнопки "Да" обновляется соответствующая запись в таблице, хранящей письма и их статусы. При нажатии кнопки "Нет" пользователь "возвращается" на форму "Запрос".

·иначе введенное значение сохраняется в новой записи таблицы, хранящей письма и их статусы.

Действие протоколируется. Дополнительно сохраняется номер запроса.
