Состав и структура сообщений, формируемых в файле в процессе загрузки

Данные о загрузке в файле лога формируются в следующей последовательности:

·сообщение "SaveStatus. CompanyName <> Morgan&Stout";

·данные о корректировке полей state, prev_state, int_color, dsc в таблице debt при наличии записей со статусом 8 (отозван клиентом), отличающимся от статуса в debt;

·данные о корректировке полей state, prev_state, dsc в таблице debt при наличии записей со статусом 11 (возврат в работу), отличающимся от статуса в debt;

·данные о корректировке поля status в таблице debt при наличии записей со статусом, отличающимся от статуса в debt;

·данные о корректировке полей r_user_id, prior, call_time, r_contact_id, r_admin_id, reason, finish_work_date в таблице work_task при наличии записей со статусом 8 (отозван клиентом);

·данные о корректировке полей r_contact_id, r_admin_id, reason, call_time, finish_work_date в таблице work_task при наличии записей со статусом 11 (возврат в работу);

·данные о корректировке поля reason в таблице work_task при наличии записей со статусами 6 (завершено), 12 (без перспектив).

Перед корректировкой в debt для записей со статусом 8 в файле лога формируется сообщение: "SaveStatus. Корректировка полей state, prev_state, int_color, dsc для записей со статусом 8, количество корректируемых записей в debt количество записей". Если булевская константа LoadStatusBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveStatus. Дополнительные данные о корректировке:";

·сообщения по каждой записи со статусом 8 вида "запись №номер записи, id debt = ID долга".

Перед корректировкой в debt для записей со статусом 11 в файле лога формируется сообщение: "SaveStatus. Корректировка полей state, prev_state, dsc для записей со статусом 11, количество корректируемых записей в debt количество записей". Если булевская константа LoadStatusBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveStatus. Дополнительные данные о корректировке:";

·сообщения по каждой записи со статусом 11 вида "запись №номер записи, id debt = ID долга".

Перед корректировкой в debt поля status в файле лога формируется сообщение: "SaveStatus. Корректировка поля status, количество корректируемых записей в debt количество записей". Если булевская константа LoadStatusBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveStatus. Дополнительные данные о корректировке:";

·сообщения по каждой записи вида "запись №номер записи, id debt = ID долга, новый статус = новый статус".

Перед корректировкой в work_task для записей со статусом 8 в файле лога формируется сообщение: "SaveStatus. Корректировка полей r_user_id, prior, call_time, r_contact_id, r_admin_id, reason, finish_work_date для записей со статусом 8, количество корректируемых записей в work_task количество записей". Если булевская константа LoadStatusBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveStatus. Дополнительные данные о корректировке:";

·сообщения по каждой записи со статусом 8 вида "запись №номер записи, id work_task = ID work_task".

Перед корректировкой в work_task для записей со статусом 11 в файле лога формируется сообщение: "SaveStatus. Корректировка полей r_contact_id, r_admin_id, reason, call_time, finish_work_date для записей со статусом 11, количество корректируемых записей в work_task количество записей". Если булевская константа LoadStatusBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveStatus. Дополнительные данные о корректировке:";

·сообщения по каждой записи со статусом 11 вида "запись №номер записи, id work_task = ID work_task".

Перед корректировкой в work_task для записей со статусами 6, 12 в файле лога формируется сообщение: "SaveStatus. Корректировка поля reason для записей со статусами 6 и 12, количество корректируемых записей в work_task количество записей". Если булевская константа LoadStatusBulkInsert имеет значение "Ложь", то в файле лога дополнительно формируются:

·сообщение "SaveStatus. Дополнительные данные о корректировке:";

·сообщения по каждой записи со статусами 6, 12 вида "запись №номер записи, id work_task = ID work_task".
