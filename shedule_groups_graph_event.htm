Привязка мероприятия к событию

Привязка мероприятия к событию реализована для того, чтобы запуск мероприятия мог зависеть от завершения некоторых событий в Системе. Например, мероприятие по формированию групп или по распределению должников должно стартовать после того, как в будет закончен импорт данных в Систему. А если добавление данных в Систему не производилось, то не имеет смысла запуска мероприятие. В этих случаях рекомендуется использовать данный функционал.

Примечание. Привязка мероприятия к событию доступна только при значения константы SchedulerUseExtendedScheduleMode равном TRUE.

Для использования функционала следует выполнить следующие условия:

1.Создать словарь словаря № 260 "Типы событий для запуска мероприятий по расписанию" и заполнить его.

2.Создать таблицу, в которую указанные события будут записывать факт своего окончания.

3.Присвоить константе SchedulerUseExtendedScheduleMode значение TRUE.

4.Настроить привязку нужных мероприятий к событиям.

Примечание. Если данные действия вызывают у вас затруднения, обратитесь в техподдержку.Таблица называется Event_execution.

Описание области формы

Область Привязка к событию формы График содержит следующие элементы:

·Поле Событие — выберите событие из словаря № 260;

·Поле Время последнего возможного запуска — выберите время;

·Поле Период (ч) для события в пред. сутках — выберите количество часов;

·Кнопка Очистить время последнего запуска и период анализа события — для удобства редактирования.

Логика привязки мероприятия к событию

Мероприятие в расписании, привязанное к событию, может быть выполнено только после окончания этого события.

Попытки выполнить мероприятие будут начаты в момент времени, указанный в поле Время запуска, согласно расписанию выполнения мероприятия. Будет анализироваться специальная таблица, в которую должны записываться данные об окончании событий, описанных в словаре № 260.

При первой попытке будет осуществлен поиск события, время окончания которого приходится на текущие сутки или ранее на выбранное количество часов в поле Период (ч) для события в пред. сутках. Если такая запись будет найдена, то мероприятие будет выполняться.

Если при первой попытке запуска мероприятия запись о событии в таблице не будет найдена, то в дальнейшем будут осуществляться новые попытки найти эту запись, вплоть до времени указанного в поле Время последнего возможного запуска. Если до этого момента запись о совершении события не появится, то мероприятие не будет выполняться.

Если до истечения времени, указанного в поле Время последнего возможного запуска, запись об окончании выбранного события успеет появиться, то мероприятие будет выполнено.
