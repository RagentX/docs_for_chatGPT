Формирование лога загрузок данных о платежах и остатков погашения

Файл лога формируется автоматически и содержит данные о функционировании модуля загрузки платежей за текущие сутки. Имя файла: load_calc <сведения о дате>.log (например: load_calc 01042011.log).

Каждое сообщение в файле лога начинается со сведений о дате и времени формирования сообщения, например: 01.04.2011 12:29:48.

В файл лога заносятся следующие сообщения:

·при загрузке модуля формируются два сообщения: "FormShow. Версия-<номер версии>" и "FStart. Возврат в исходное состояние";

·перед открытием файла с данными формируется сообщение: "buOpenClick. Старт открытия файла". При успешном открытии файла формируется сообщение: "buOpenClick. Файл-<директория и имя файла> успешно открыт";

·в случае несоответствия количества информационных колонок в открытом файле, а также при наличии в исходном файле ошибочной ячейки, не позволяющей открыть файл, формируются соответствующие сообщения;

·в начале проверки данных формируется сообщение: "buCheckClick. Проверка данных". В зависимости от результата проверки формируется одно из сообщений:

·"Данные полностью соответствуют формату загрузки";

·"Загрузка невозможна, ни одна запись не соответствует формату загрузки";

·"Можно загрузить записей <количество корректных записей>, номер первой ошибочной записи <номер записи>";

·в процессе проверки данных формируются сообщения о создании, наполнении и корректировке временных таблиц;

·перед окраской фона ячеек таблицы данных формируется сообщение: "CheckData. Окраска ячеек в строках grid". По окончании окраски фона ячеек формируется сообщение: "CheckData. Результаты проверки данных: всего записей <общее количество записей>, из них ошибочных <количество ошибочных записей>";

·если константа LoadCalcBulkInsert имеет значение "Ложь", то для каждой проверяемой записи формируется сообщение: "CheckData. Старт проверки данных записи <номер записи>" и, при наличии ошибок, отдельное сообщение по каждой выявленной критической и некритической ошибке. Если для проверяемой записи найден id долга, формируется соответствующее сообщение;

·в начале загрузки данных в логическом файле формируется сообщение: "buSaveClick. Загрузка данных". При отсутствии изменений данных повторная проверка не осуществляется, формируется сообщение: "buSaveClick. Данные в ячейках таблицы не изменялись, повторная проверка отменена";

·при наличии в таблице данных ошибочных записей в случае решения пользователя сформировать файл ошибочных записей формируется сообщение: "buSaveClick. Имеются ошибочные записи (<количество ошибочных записей>). Старт формирования файла ошибок", а затем сообщение: "buSaveClick. Сформирован файл ошибок <директория и имя файла>";

·при наличии в таблице данных корректных дублированных записей (с одинаковыми ID долга) формируется сообщение: "buSaveClick. Имеются записи с одинаковыми id debt. Остатки по дублированным записям (<количество дублированных записей>) не учитываются при загрузке в БД". Если константа LoadCalcBulkInsert имеет значение "Ложь", то дополнительно формируются:

·сообщение "buSaveClick. Дополнительные данные о дублировании записей:";

·сообщения по каждой дублированной записи вида "запись №<номер записи>, id debt = <ID долга>";

·при наличии записей с несовпадающим ФИО и значении "Истина" константы LoadCalcBulkInsert формируется сообщение: "buSaveClick. Имеются записи (<количество записей>) с несовпадающим ФИО". В зависимости от решения пользователя формируется одно из сообщений:

·"buSaveClick. Пользователь отказался от загрузки данных по записям с несовпадающим ФИО";

·"buSaveClick. Пользователь согласен загружать данные по записям с несовпадающим ФИО";

·при наличии записей с несовпадающим ФИО и значении "Ложь" константы LoadCalcBulkInsert в зависимости от решений пользователя по каждой записи с несовпадающим ФИО формируется одно из сообщений:

·"buSaveClick. Сообщение "Запись №<номер записи>, должник = <ФИО должника>, банк = <название банка>, портфель = <название портфеля>, договор = <номер договора>, л/с = <номер счета>, ID клиента = <ID в системе клиента>. В базе найден должник = <ФИО должника>, ID долга = <ID долга>". Пользователь отказался от загрузки данных по записи с несовпадающим ФИО";

·"buSaveClick. Сообщение "Запись №<номер записи>, должник = <ФИО должника>, банк = <название банка>, портфель = <название портфеля>, договор = <номер договора>, л/с = <номер счета>, ID клиента = <ID в системе клиента>. В базе найден должник = <ФИО должника>, ID долга = <ID долга>". Пользователь согласен загружать данные по записи с несовпадающим ФИО";

·при наличии записей с платежами, подтвержденными ранее, и значении "Истина" константы LoadCalcBulkInsert формируется сообщение: "buSaveClick. Имеются записи (<количество записей>) с платежами, подтвержденными ранее". В зависимости от решения пользователя формируется одно из сообщений:

·"buSaveClick. Пользователь отказался от загрузки данных по подтвержденным платежам";

·"buSaveClick. Пользователь согласен загружать данные по подтвержденным платежам";

·при наличии записей с платежами, подтвержденными ранее, и значении "Ложь" константы LoadCalcBulkInsert в зависимости от решений пользователя по каждой записи с платежом, подтвержденным ранее, формируется одно из сообщений:

·"buSaveClick. Сообщение "Запись №<номер записи>, дата платежа = <дата платежа>, сумма платежа = <сумма платежа>. Такой подтвержденный платеж уже есть в debt_calc, id = <идентификатор в debt_calc>". Пользователь отказался от загрузки данных по записи в части подтвержденного платежа";

·"buSaveClick. Сообщение "Запись №<номер записи>, дата платежа = <дата платежа>, сумма платежа = <сумма платежа>. Такой подтвержденный платеж уже есть в debt_calc, id = <идентификатор в debt_calc>". Пользователь согласен загружать данные по записи в части подтвержденного платежа";

·в процессе загрузки формируются сообщения о корректировке временных таблиц;

·состав и структура сообщений, формируемых в процессе загрузки данных об остатках погашения, приведены в разделе Данные в логе о загрузке остатков погашения;

·состав и структура сообщений, формируемых в процессе загрузки данных о платежах, приведены в разделе Данные в логе о загрузке платежей.

·по окончании загрузки данных формируется сообщение: "buSaveClick. Загрузка окончена. Успешно загружено <количество записей> записей".
