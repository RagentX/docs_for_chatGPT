Возврат ИП на предыдущую стадию (в иск или приказ)

Show/Hide Hidden Text

В модуле Карточка исполнительного производства по кнопке осуществляется возврат дела на судебную стадию — дело преобразуется в иск или приказ, а исполнительное производство переводится в архив.

Меню кнопки состоит из пунктов:

·Преобразовать в иск;

·Преобразовать в приказ.

Преобразование в иск

Преобразование ИП в иск доступно только пользователям, в роли которых доступ № 498 LAW_ACT_LIST_EXEC_TO_ISK имеет значение TRUE.

Для преобразования ИП в иск выполните следующие действия:

1.Откройте карточку нужного исполнительного производства.

2.Нажмите кнопку , в открывшемся меню выберите пункт "Преобразовать в иск".

3.По данному делу и долгу будут осуществлены следующие проверки:

·Если ИП создано по иску:

·ИП переводится в архив (действие протоколируется);

·Связанный иск переводится в статус "Возобновлено" (действие протоколируется);

·Если ИП создано по приказу:

·ИП переводится в архив (действие протоколируется);

·Осуществляется перевод приказа в иск, связанный с данным ИП;

·Если ИП создано без связи с судебным делом:

·Появляется сообщение, что по данному ИП нет связанного судебного дела. Операция не возможна, действие не осуществляется.

Преобразование в приказ

Преобразование ИП в приказ доступно только пользователям, в роли которых доступ № 625 LAW_ACT_LIST_EXEC_TO_ORDER имеет значение TRUE.

Для преобразования ИП в приказ выполните следующие действия:

·Если ИП создано по иску:

·ИП переводится в архив (действие протоколируется);

·Осуществляется перевод иска в приказ, связанный с данным ИП;

·Если ИП создано по приказу:

·ИП переводится в архив (действие протоколируется);

·Связанный приказ переводится в статус "Возобновлено" (действие протоколируется);

·Если ИП создано без связи с судебным делом:

Информация для поддержки

В полном алгоритме возврата ИП на предыдущую стадию из карточки ИП используется техническая константа:

·LawExec.ReturnToPrevStage.WithCreateLawAct;

Изначально для всех клиентов эта константа выключена, а для Почта Банка включена.

Полный алгоритм проверок

       Для перевода в иск

·Если константа LawExec.ReturnToPrevStage.WithCreateLawAct имеет значение TRUE, то:

oЕсли ИП создано по иску:

§ИП переводится в архив (действие протоколируется);

§Связанный иск переводится в статус "Возобновлено" (действие протоколируется);

oЕсли ИП создано по приказу:

§Осуществляется перевод приказа в иск, связанный с данным ИП;

oЕсли ИП создано без связи с судебным делом:

§Если судебные дела по данному долгу есть, то:

oОткрывается форма выбора дела для преобразования в иск. Далее:

oЕсли вы хотите создать новый иск, выберите "Создать новый иск". Иск будет создан, если:

oЕсли вы хотите преобразовать в иск, связанный с данным ИП, существующее дело, то выберите дело из списка и нажмите "Преобразовать ИП в выбранное дело". Дело будет преобразовано в связанный иск;

§Если судебных дел по данному долгу нет, то:

oИП переводится в архив (действие протоколируется);

oСоздается новый иск, связанный с данным ИП;

·Если константа LawExec.ReturnToPrevStage.WithCreateLawAct имеет значение FALSE, то:

§ИП переводится в архив (действие протоколируется);
