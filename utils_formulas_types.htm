Типы формул и их особенности

Show/Hide Hidden Text

Формулы в Системе имеют тип. Существующие типы формул перечислены в словаре № 144.

В настоящей версии Системы поддерживаются следующие типы формул:

1.Сценарий разговора.

2.Расчет комиссии.

3.График платежей.

4.Регистрация контакта.

5.Дерево контактов.

6.Окраска карточки должника.

7.Шаблон "Краткой информации".

8.Ограничение коммуникации.

9.Расчет переменной для SMS сообщения (число).

Каждый тип формул имеет собственные входные параметры. Результат работы формулы так же типизирован, поскольку его значение используется в Системе в определенных контекстах.

Для каждого типа формул доступен свой набор атрибутов и функций. Операции доступны для всех типов формул.

Доступность функций для типов формул — см. Список функций.

Доступность атрибутов — см. далее.

Примечание При построении дерева формул могут быть использованы операнды различных типов. Соответственно, для каждого операнда доступны только те операции, атрибуты и функции, чей тип соответствует типу операнда.

Важно! Если в Системе осуществляется вызов некорректной формулы, которая:

·Отсутствует;

·Работает с ошибкой;

·Не завершена;

·Выдает результат не подходящего типа,

то вместо результата работы формулы подставляется пустое значение, а сообщение об ошибке не выдается.

Описание типа формул "Сценарий разговора"

Входные параметры

·ID долга;

·ID пользователя.

Результат

Результатом работы формулы типа Сценарий разговора может быть одно число или строка, содержащая несколько чисел, разделенных запятой. Эти числа соответствуют идентификаторам сценариев разговора — см. модуль Сценарии сообщений, вкладка Разговор.

Формула используется в модуле Настройка дерева контактов в форме настройки исхода контакта.

В поле Формула для автосценария задается номер формулы, в результате работы которой при регистрации контакта при переходе к данному узлу для оператора отображается один или более (один за другим по порядку) сценариев.

Описание типа формулы "Регистрация контакта"

Входные параметры

·ID долга;

·ID пользователя.

Результат

Формулы типа Регистрация контакта используются в модуле Регистрация контакта (см. Регистрация обещания и ограничения на дату и сумму).

Результатом работы формулы является число. Оно может обозначать:

·количество дней для ограничения даты обещания:

·RegContactPromMaxDayHardFormula;

·RegContactPromMaxDaySoftFormula;

·количество процентов для ограничения суммы обещания:

·RegContactPromMinSumRateHardFormula;

·RegContactPromMinSumRateSoftFormula.

Особенности передачи входных параметров

Если краткая информация будет выводиться в модуле Карточка должника, то:

·Параметр Кампания обзвона (i_campaign_id) всегда передаётся в формулу как -1 (i_campaign_id = -1).

Если краткая информация будет выводиться в модуле Центр обзвона должников, то:

·Если тип кампании "Стандартная", то в формулу передаётся параметр Кампания обзвона (i_campaign_id), равный нулю;

·Если тип кампании отличается от "Стандартная", то в формулу передаётся параметр Кампания обзвона, соответствующий campaign.id;

·Параметр Карточка поднята по автодайлеру (i_is_predictive) для центра обзвона должников всегда передаётся как 0.
