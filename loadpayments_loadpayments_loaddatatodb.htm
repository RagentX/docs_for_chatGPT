Загрузка данных о платежах и остатках погашения в базу данных для формата "Загрузка платежей"

После проверки данных и при необходимости их корректировки осуществляется загрузка данных о платежах должников и остатках погашения в базу данных. Для этого выполните следующие действия:

1.В форме Загрузка платежей нажмите кнопку Сохранить в БД. При наличии изменений в таблице перед загрузкой автоматически проводится повторная проверка данных в таблице. При выполнении проверки данных в нижней строке формы будет отображаться тип операции (проверка данных), текущие данные о проценте выполнения работы, директория и имя файла.

2.Если в таблице имеются ошибочные записи, будет выдан запрос на формирование файла ошибочных записей. Если требуется сформировать файл, нажмите Yes. В этом случае будет автоматически сформирован файл в формате MS Excel, содержащий ошибочные записи (см. Просмотр файла ошибочных записей). В процессе формирования файла с ошибочными записями в нижней строке формы будет отображаться тип операции (формирование файла ошибочных записей) и текущие данные о проценте выполнения работы. По окончании формирования файла будет выдано информационное сообщение с указанием директории и имени файла с ошибочными записями. Нажмите кнопку ОК, чтобы закрыть его.

3.В процессе загрузки данных в БД в нижней строке формы будет отображаться тип операции (загрузка данных), текущие данные о проценте выполнения работы, директория и имя файла. Автоматически осуществляются следующие действия:

·исключение из загрузки дублированных записей, относящихся к одному и тому же долгу;

·корректировка данных об остатке в таблице долгов для записей с ненулевым остатком;

Примечание. Если для долга в статусе 6 "Завершено" и с нулевым остатком загружено положительное значение остатка, то статус долга изменяется на 11 "Возврат в работу".

·корректировка даты расчета остатка в таблице долгов для записей с пустым остатком;

·установка статуса 11 в таблице долгов для записей с ненулевым остатком и корректировка соответствующих строк таблицы задач на обзвон;

·установка статуса 6 в таблице долгов для записей с нулевым остатком и корректировка соответствующих строк таблицы задач на обзвон;

Примечание Если в результате расчета по правилу расчета остатков в портфеле долга получается, что платеж полностью покрывает остаток долга, то статус долга изменяется на 6 "Завершено" с причиной "Автоматически при работе с правилом <ID правила расчета>".

·работа по правилам, корректировка данных в таблице долгов для записей с пустым остатком и корректировка соответствующих строк таблицы задач на обзвон;

·подтверждение неподтвержденных платежей в таблице платежей;

·ввод новых строк в таблицу платежей;

·корректировка данных об обещаниях в таблице обещаний для новых и вновь подтвержденных платежей;

·корректировка соответствующих обновленным обещаниям строк таблицы платежей и таблицы долгов;

·отмена намеченных выездов в таблице адресов для новых и вновь подтвержденных платежей;

·проверка, участвует ли долг в акциях. Если участвует, то для долга обновляется поле Сумма платежей по акции в рублях для каждой акции в зависимости от флага Единовременный платеж:

·если флаг установлен, то в поле попадает сумма максимального платежа за период с даты начала акции по дату, до которой необходимо внести платежи по акции (если не заполнена, то до текущей);

·если флаг не установлен, то в поле попадает сумма всех платежей за этот период.

Примечание. Остаток по акции рассчитывается как Сумма к оплате по акции (в рублях) — Сумма платежей по акции в рублях.

При нулевом остатке по акции:

·агентские долги (т.е. поле Тип договора в портфеле имеет значение 1) переводятся в статус 9 "Проблемный" с причиной "Долг погашен по акции";

·цессионные долги (т.е. поле Тип договора в портфеле имеет значение 2) автоматически закрываются, при этом в качестве причины закрытия указывается "Закрыт по акции", в комментарии — "Закрыт по акции <Название акции>".

Возможна ситуация, когда платеж покрывает и сумму к оплате по акции, и сумму задолженности. В этом случае долг будет закрыт один раз по акции, закрывать его повторно не требуется.

4.Если в таблице имеются ошибочные записи, перед загрузкой на экран выводится запрос: "Внимание: имеются записи (<количество ошибочных записей> из <общее количество записей>), которые не могут быть загружены в БД. Сформировать файл ошибочных записей? Да, Нет".

При положительном ответе пользователя осуществляется автоматическое формирование файла MS Excel ошибочных записей. В процессе формирования файла в нижней строке окна формы автоматически отображается тип работы (формирование файла ошибочных записей) и текущие данные о проценте выполнения работы. По окончании формирования файла на экран выводится сообщение: "Ошибочные записи сохранены в файле <директория и имя файла>".

5.При наличии записей с некритическими ошибками в процессе загрузки данных осуществляется диалог с пользователем, который принимает решение о загрузке или отказе от загрузки для нестандартных записей (см. Обработка записей с некритическими ошибками).

6.По окончании загрузки будет выдано информационное сообщение с указанием количества успешно загруженных записей. Исключенные из загрузки записи с несовпадающим ФИО не учитываются при расчете количества успешно загруженных записей. Исключенные из загрузки записи с платежами, подтвержденными ранее, не учитываются при расчете количества успешно загруженных записей при условии отсутствия остатков. Нажмите кнопку ОК, чтобы закрыть сообщение.

7.Если значение константы LoadCalcConfirmDoublePayment равно "Истина", то по окончании загрузки исходного файла платежей в БД осуществляется поиск и подтверждение (в диалоге с оператором) неподтвержденных платежей, для которых в БД есть хотя бы один подтвержденный платеж по тому же долгу, с той же датой, суммой и типом валюты.
