       Автоматический выбор формы дерева контактов в случае использования функционала независимых деревьев контакта

При запуске модуля Регистрация контакта осуществляется проверка значения константы RegContactTreeTypFormula. Если ее значение больше нуля, то в модуле Произвольные формулы выполняется формула, имеющая такой номер. Результатом работы формулы будет номер типа дерева контактов. В данном случае (на рисунке) автоматически открыто альтернативное дерево контактов.

Форма Регистрация контракта

Описание формы

Форма Регистрация контакта содержит следующие элементы:

·Сценарий разговора — описание сценария контакта, который меняется в зависимости от выбранного результата. Он определяется в результате работы формулы, номер которой указан в поле "Формула для автосценария" формы исхода контакта. Сценарий создается путем конкатенации сценариев сообщений, номера который перечислены в формуле. Кроме того, в поле может содержаться напоминание о необходимости указать причину задолженности (если при настройке исхода контакта был включен флаг "Ввод причины возникновения задолженности"). Если значение для узла не задано, то по умолчанию будет взято то значение, которое является результатом работы формулы с типом Сценарий разговора, номер которой указан в константе FormulaAutoMessage;

Примечание. Сценарий разговора отображается не для всех типов контактов.

 Примечание для поддержки 1. Конкатенация нескольких сценариев доступна только при значении константы _RegContactNewModeTalkPattern равном True. Если константа выключена, то можно использовать только одно сообщение.

       Примечание для поддержки 2. EPS-точка

Создана EPS-точка, кастомизирующая логику для определения сценария разговора в модуле "Регистрация контакта".

Определения ID сценария разговора по EPS-процедуре реализуется в зависимости от следующих атрибутов: 

·ID долга (debt_id);

·Текущий пользователь (r_user_id);

·Код выбранного узла дерева (result_code);

·        ID кампании обзвона (для звонка через ЦОД);

·Признак поднятия карточки по автодайлеру.

Если кастомизированная логика настроена, то используется получение сценария с помощью соответствующей клиентской процедуры. Если кастомизированная логика не настроена, или ее результат пустой, то используется системная логика.

 

·Дополнительные поля:

·ФИО контактного лица — только если при настройке исхода контакта включен флаг "Ввод ФИО конт. лица"

·Панель выбора автокомментария — только если при настройке исхода контакта в поле "Ввод автокомментария" выбрано более одного автокомментария;

·Поле для ввода комментария;

·Дерево возможных результатов контакта, которое различается в зависимости от типа контакта. Настройка вариантов дерева для разных типов контактов осуществляется в модуле Настройка дерева контактов;

·кнопка Причина для указания причины возникновения долга;

·кнопка ОК для сохранения результата контакта.

       Запрет на выход из формы без регистрации результата контакта

Если значение доступа REG_CONTACT_CLOSE равно FALSE и значение константы "RegContactCloseMode" равно "Истина", то пользователь не может выйти из модуля "Регистрация контакта" без регистрации результата контакта. При нажатии кнопки появляется предупреждение, и окно не закрывается. Таким образом, каждый контакт, осуществлённый оператором, будет зарегистрирован.
