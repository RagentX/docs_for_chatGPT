Формирование лога с данными о работе утилиты

Если константа LoadDataDebugMode имеет значение "Истина", то в текущей директории утилиты автоматически создается текстовый файл load_status <дата выполнения>.log (например, load_status 01042011.log) с данными о функционировании утилиты за текущие сутки.

Каждое сообщение в файле начинается со сведений о дате и времени формирования сообщения, например: 01.04.2011 12:29:48.

При загрузке утилиты load_status в файле формируются два сообщения: "FormShow. Версия-<номер версии>" и "FStart. Возврат в исходное состояние".

Перед открытием файла MS Excel в файле формируется сообщение: "buOpenClick. Старт открытия файла". При успешном открытии файла MS Excel в файле формируется сообщение: "buOpenClick. Файл-директория и имя файла успешно открыт".

В случае несоответствия количества заполненных столбцов в открытом файле MS Excel формату, а также при наличии в исходном файле MS Excel ошибочной ячейки, не позволяющей открыть файл, в файле формируются соответствующие сообщения.

В начале проверки данных в файле формируется сообщение: "buCheckClick. Проверка данных". В зависимости от результата проверки в логическом файле формируется одно из сообщений:

·"Данные полностью соответствуют формату загрузки";

·"Загрузка невозможна, ни одна запись не соответствует формату загрузки";

·"Можно загрузить записей <количество корректных записей>, номер первой ошибочной записи <номер записи>".

В процессе проверки данных в файле формируются сообщения о создании, наполнении и корректировке временных таблиц.

Перед окраской фона ячеек таблицы данных в файле формируется сообщение: "CheckData. Окраска ячеек в строках grid". По окончании окраски фона ячеек в логическом файле формируется сообщение: "CheckData. Результаты проверки данных: всего записей <общее количество записей>, из них ошибочных <количество ошибочных записей>.

Если константа LoadStatusBulkInsert имеет значение "Ложь", то для каждой проверяемой записи в логическом файле формируется сообщение: "CheckData. Старт проверки данных записи <номер записи>" и, при наличии ошибок, отдельное сообщение по каждой выявленной ошибке. Если для проверяемой записи найден id долга, в логическом файле формируется соответствующее сообщение.

В начале загрузки данных в логическом файле формируется сообщение: "buSaveClick. Загрузка данных". В отсутствии изменений данных пользователем повторная проверка не осуществляется, в логическом файле формируется сообщение: "buSaveClick. Данные в ячейках таблицы не изменялись, повторная проверка отменена". В отсутствии записей без критических ошибок в логическом файле формируется сообщение: "buSaveClick. Загрузка данных отменена из-за отсутствия записей, которые могут быть загружены в БД".

При наличии в таблице данных и корректных, и ошибочных записей в случае решения пользователя сформировать файл ошибочных записей в логическом файле формируется сообщение: "buSaveClick. Имеются ошибочные записи (<количество ошибочных записей>). Старт формирования файла ошибок", а затем сообщение: "buSaveClick. Сформирован файл ошибок директория и имя файла".

При наличии в таблице данных корректных дублированных записей (с одинаковыми ID долга) в логическом файле формируется сообщение: "buSaveClick. Имеются записи с одинаковыми id debt. Дублированные записи (<количество дублированных записей>) не учитываются при загрузке в БД". Если булевская константа LoadStatusBulkInsert имеет значение "Ложь", то в логическом файле дополнительно формируются:

·сообщение "buSaveClick. Дополнительные данные о дублировании записей:";

·сообщения по каждой дублированной записи вида "запись №<номер записи>, id debt = <ID долга>".

При наличии записей с некритическими ошибками и значении "Истина" булевской константы LoadStatusBulkInsert в логическом файле формируется сообщение: "buSaveClick. Имеются записи (<количество записей>) с несовпадающим ФИО". В зависимости от решения пользователя в логическом файле формируется одно из сообщений:

·"buSaveClick. Пользователь отказался от загрузки данных по записям с несовпадающим ФИО";

·"buSaveClick. Пользователь согласен загружать данные по записям с несовпадающим ФИО".

При наличии записей с некритическими ошибками и значении "Ложь" булевской константы LoadStatusBulkInsert в зависимости от решений пользователя по каждой записи с несовпадающим ФИО в логическом файле формируется одно из сообщений:

·"buSaveClick. Сообщение "Запись №' <номер записи>, должник = <ФИО должника>, банк = <название банка>, портфель = <название портфеля>, договор = <номер договора>, л/с = <номер счета>, ID клиента = <ID в системе клиента>. В базе найден должник = <ФИО должника>, ID долга = <ID долга>". Пользователь отказался от загрузки данных по записи с несовпадающим ФИО";

·"buSaveClick. Сообщение "Запись №' <номер записи>, должник = <ФИО должника>, банк = <название банка>, портфель = <название портфеля>, договор = <номер договора>, л/с = <номер счета>, ID клиента = <ID в системе клиента>. В базе найден должник = <ФИО должника>, ID долга = <ID долга". Пользователь согласен загружать данные по записи с несовпадающим ФИО".

Состав и структура сообщений, формируемых в файле в процессе загрузки, описаны в разделе Состав и структура сообщений, формируемых в логическом файле в процессе загрузки.

По окончании загрузки данных в логическом файле формируется сообщение: "buSaveClick. Загрузка окончена. Успешно загружено <количество записей> записей".
