Клиентские теги

Для отдельных клиентов специально насроенные теги отображаются как обычные в нужных сценариях сообщений.

1.Системный словарь № 323 "Пользовательские переменные в сценариях сообщений" содержит клиентские теги.

2.Таблица debt_msg_user_tag для хранения пользовательских тегов:

Поле

Тип данных

(MSSQL/Oracle)

Комментарий

id

int/number

 

code

int/number

Код термина в словаре "Пользовательские переменные в сценариях сообщений"

name

nvarchar(64)/

varchar2(64)

Название (выводится, если нет ссылки на словарь)

tag

nvarchar(64)/

varchar2(64)

Тег. Писать в %%

tag_type

int/number

Тип сценария: 1 - разговор, 2 – СМС, 3 – Email, 4 – Соц. Сети, 5 – Автоматический комментарий, 6 - Специальное

3.При добавлении/редактировании сценария на вкладках Разговор, SMS-сообщения, Email-сообщения, Соц. Сети, Автоматический комментарий, Специальное в окне добавления/редактирования сценария дополнительно выводить в выпадающем списке переменные по таблице debt_msg_user_tag:

·для вкладки Разговор дополнительно выводятся термины словаря "Пользовательские переменные в сценариях сообщений", коды которых соответствуют debt_msg_user_tag.code, для которых debt_msg_user_tag.tag_type=1 или debt_msg_user_tag.tag_type=NULL. Если такого термина в словаре нет, а запись в debt_msg_user_tag есть, то выводится debt_msg_user_tag.name

·для вкладки SMS-сообщения дополнительно выводятся термины словаря "Пользовательские переменные в сценариях сообщений", коды которых соответствуют debt_msg_user_tag.code, для которых debt_msg_user_tag.tag_type=2 или debt_msg_user_tag.tag_type=NULL. Если такого термина в словаре нет, а запись в debt_msg_user_tag есть, то выводится debt_msg_user_tag.name

·для вкладки Email-сообщения дополнительно выводятся термины словаря "Пользовательские переменные в сценариях сообщений", коды которых соответствуют debt_msg_user_tag.code, для которых debt_msg_user_tag.tag_type=3 или debt_msg_user_tag.tag_type=NULL. Если такого термина в словаре нет, а запись в debt_msg_user_tag есть, то выводится debt_msg_user_tag.name

·для вкладки Соц. Сети дополнительно выводятся термины словаря "Пользовательские переменные в сценариях сообщений", коды которых соответствуют debt_msg_user_tag.code, для которых debt_msg_user_tag.tag_type=4 или debt_msg_user_tag.tag_type=NULL. Если такого термина в словаре нет, а запись в debt_msg_user_tag есть, то выводится debt_msg_user_tag.name

·для вкладки Автоматический комментарий дополнительно выводятся термины словаря "Пользовательские переменные в сценариях сообщений", коды которых соответствуют debt_msg_user_tag.code, для которых debt_msg_user_tag.tag_type=5 или debt_msg_user_tag.tag_type=NULL. Если такого термина в словаре нет, а запись в debt_msg_user_tag есть, то выводится debt_msg_user_tag.name

·для вкладки Специальное дополнительно выводятся термины словаря "Пользовательские переменные в сценариях сообщений", коды которых соответствуют debt_msg_user_tag.code, для которых debt_msg_user_tag.tag_type=6 или debt_msg_user_tag.tag_type=NULL. Если такого термина в словаре нет, а запись в debt_msg_user_tag есть, то выводится debt_msg_user_tag.name

4.При добавлении такой переменной вставлять в сценарий соответствующее значение поля debt_msg_user_tag.tag.

 Примечание. Была доработка для САВД по заявке 24656, потом для КапМосБанк по заявке 29840.

Для САВД еще

EPS-точка

Требуется доработка процедуры UR_EPS_WRP$SINGLE_SMS_UPD_SAVD.

Требуется поддержать обработку нового тега:

·%utag.first_creditor% - произвольный атрибут долга с типом 25 по словарю 53.

Для КапМосБанк еще

ur_eps_wrp$single_sms_upd_capmos

Требуется доработка процедуры ur_eps_wrp$single_sms_upd_capmos. Необходимо поддержать в процедуре замену пользовательского тега:

·%utag.prediction_sum%

Требуется подставлять сумму, указанную в значении debt_detail.number_value_9
