Примеры входных параметров различных типов

В данном разделе рассмотрены примеры входных параметров различных типов:

·Дата.

·Число.

·Портфель.

·Оператор.

·Банк.

·Строка.

·Словарь.

·Список Excel.

·Исходящий портфель.

·Словарь ++.

·Чекбокс.

Дата

Если входной параметр произвольного отчета имеет тип "Дата", то при формировании отчета необходимо будет выбрать в календаре нужную дату и время.

 

Число

Если входной параметр произвольного отчета имеет тип "Число", то при формировании отчета необходимо ввести число в поле ввода.

Портфель

Если входной параметр произвольного отчета имеет тип "Портфель", то при формировании отчета откроется список, содержащий перечень всех входящих портфелей из модуля "Клиенты и портфели". В списке необходимо выбрать один или несколько портфелей так же, как при работе с мультифильтрами.

Примечание. Если в модуле Настройка списка должников для текущего пользователя в поле Кредиторы указан список разрешенных банков, то  в мультифильтре Портфели для данного пользователя доступны только портфели банков из этого списка.

Оператор

Если входной параметр произвольного отчета имеет тип "Оператор", то при формировании отчета откроется список, содержащий перечень всех операторов из модуля "Пользователи". В списке необходимо выбрать одного или нескольких операторов так же, как при работе с мультифильтрами.

Примечание. Если в роли текущего пользователя доступ № 642 CUSTOM_REPORTS_ALL_USERS_VIEW имеет значение TRUE, то в списке доступных операторов отображаются все незаблокированные пользователей Системы, а также текущий оператор (даже если он заблокирован). Если доступ отсутствует или имеет значение FALSE, то список операторов отображается с учетом Ограничения доступа к данным на основании структуры предприятия.

Банк

Если входной параметр произвольного отчета имеет тип "Банк", то при формировании отчета откроется список, содержащий перечень всех клиентов из модуля "Клиенты и портфели". В списке необходимо выбрать одного или нескольких клиентов так же, как при работе с мультифильтрами.

Примечание. Если в модуле Настройка списка должников для текущего пользователя в полях Кредиторы и/или КА указаны списки разрешенных банков, то  в мультифильтре Выберите клиентов для данного пользователя доступны только банки из этих списков.

Строка

Если входной параметр произвольного отчета имеет тип "Строка", то при формировании отчета необходимо ввести строку в поле ввода. Строку необходимо вводить в одинарных кавычках (см. рисунок). Также можно использовать подстановочные символы:

·"%" — означает "любая (в том числе пустая) строка символов";

·"_" — означает один любой символ.

Словарь

Если входной параметр произвольного отчета имеет тип "Словарь", то при формировании отчета откроется список, содержащий перечень всех словарных терминов из словаря, указанного настройках. Словарь представляет из себя мультифильтр и поддерживает множественный выбор.

Список Excel

Входной параметр типа "Список Excel" применяется в том случае, если необходимо сформировать отчет по определенным долгам. Идентификационные данные для определения этих долгов должны содержаться в файле MS Excel. Файл должен иметь следующий формат:

·таблица не содержит "шапки", данные начинаются с первой строки;

·содержать не более пяти столбцов. Если файл содержит более пяти столбцов, будет выдано информационное сообщение, и его обработка будет прекращена;

·содержать как минимум в одном столбце данные, по которым будет выполняться поиск долгов в БД. Рекомендуется следующий порядок столбцов:

·Номер договора;

·Лицевой счет;

·ID клиента;

·ID долга;

·ФИО.

Примечание: если столбцы в файле следуют в другом порядке, их можно переместить в нужное положение в форме Список долгов.

Примечание: если файл содержит пустые строки, в процессе поиска долгов они будут пропущены.

После открытия файла его содержимое появится в окне Список долгов.

При нажатии кнопки ОК выполняется валидация данных:

·если данные корректные, то идентификаторы долгов появятся в окне ввода входных параметров (подробнее о формировании отчетов см. Формирование произвольных отчетов);

·если в файл содержит некорректные данные, появится информационное сообщение. В этом случае закройте окно Список долгов, отредактируйте файл и откройте его заново.

Исходящий портфель

Если входной параметр произвольного отчета имеет тип "Исходящий портфель", то при формировании отчета откроется список, содержащий перечень всех исходящих портфелей из модуля "Клиенты и портфели". В списке необходимо выбрать один или несколько портфелей так же, как при работе с мультифильтрами.

Примечание. Если в модуле Настройка списка должников для текущего пользователя в поле КА указан список разрешенных банков, то  в мультифильтре Выберите Реестр КА для данного пользователя доступны только исходящие портфели коллекторских агентств из этого списка.

Словарь ++

Если входной параметр произвольного отчета имеет тип "Словарь ++", то при формировании отчета откроется список, содержащий перечень всех подразделений, настроенных в модуле Структура предприятия.

Чекбокс

Если входной параметр произвольного отчета имеет тип "Чекбокс", то при формировании отчета появится поле, содержащее флаг (чекбокс).

Параметр с типом "Чекбокс" в отчет, процедуру или функцию передается в виде 0/1, где 0 – отключенный чекбокс, а 1 – включенный;

Признак обязательности к заполнению не участвует при работе данного поля, так как по умолчанию всегда формируется отключенный чекбокс.
