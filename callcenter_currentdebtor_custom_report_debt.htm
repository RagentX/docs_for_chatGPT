Формирование произвольных отчетов по долгу

 

Для формирования произвольных отчетов по долгу на форме Центр обзвона должников расположена кнопка Отчеты, дающая доступ к заранее определенным произвольным отчетам.

Список отчетов, доступных для формирования указывается в значении доступа № 656 CALL_CENTER_REQUEST_REPORT_LIST, что дает возможность настроить для каждой роли свой список отчетов, доступных для формирования по кнопке Отчеты:

·Если доступ в роли пользователя отсутствует, то пользователь не имеет доступа к отчетам по долгу. Кнопка на форме Центр обзвона должников не отображается.

·Если значение доступа равно "ALL", то в списке отображаются все настроенные отчеты.

·Если в значении доступа перечислены номера отчетов, то в списке отображаются только перечисленные в доступе отчеты.

·Если номер в значении доступа указан неверно или в константе отсутствует ссылка на существующий отчет, то в списке отчетов такая строка будет отсутствовать.

Максимальное количество отчетов — 9.

Для настройки уже существующего в Системе произвольного отчета в качестве отчета по долгу, вам следует обратиться к сопровождению: зарегистрировать заявку в Help Desk, указав в ней ID и название произвольного отчета для активации. Для создания отчета также рекомендуется обратиться к сопровождению.

Важно! Чтобы отчет корректно выгружался из карточки, должны быть описаны все поля отчета. Также, чтобы отчет формировался по конкретному долгу, обязательно, чтобы был входной параметр на id долга с алиасом "id".

Отчеты задаются константами. Наименования отчетов передаются через таблицу strings.

Список технических констант, которые могут содержать ID произвольного отчета, отображаемого на вкладке Долги:

·DebtCardRequestReport1;

·DebtCardRequestReport2;

·DebtCardRequestReport3;

·DebtCardRequestReport4;

·DebtCardRequestReport5;

·DebtCardRequestReport6;

·DebtCardRequestReport7;

·DebtCardRequestReport8;

·DebtCardRequestReport9.

Если значения этих констант равно 0, то они игнорируются.
