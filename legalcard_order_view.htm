Вкладка "Приказное производство"

В карточке судебной работы на вкладке Приказное производство отображается общая информация о должнике, его телефонах и адресах, информация о судебном приказе. Вкладка содержит следующие элементы:

·раздел Общая информация, содержащий общие данные об ответчике по делу (должнике или поручителе);

·раздел Судебный приказ, содержащий данные о судебном приказе;

·кнопки для выполнения действий с приказом (см. Этапы работы в приказном производстве);

·раздел Телефоны, содержащий список телефонов ответчика по делу (должника или поручителя) и кнопки для работы с ним;

·раздел Адреса, содержащий список адресов ответчика по делу (должника или поручителя) и кнопки для работы с ним;

·поле Комментарий для ввода дополнительной информации и кнопку для просмотра истории комментариев по работе с должником;

·поле Указание руководителя — право вводить информацию в поле имеют только те пользователи, для которых доступ LAW_ACT_CARD_INSTRUCT установлен в значение TRUE.

Ссылка: Подробная информация о редактировании полей в карточках судебной работы и ИП.

Раздел "Общая информация"

Раздел Общая информация содержит следующие поля:

·ФИО должника — данное поле отображается только для тех дел, в которых ответчиком является поручитель или связанное лицо;

·Реестр КА — название исходящего портфеля для долга;

·Дата поступления дела;

·Крайняя дата подачи — поле доступно для редактирования (см. также Автоматический расчет крайней даты подачи);

·Укомплектованность — поле доступно для редактирования, см. словарь № 19;

·Статус — статус судебного приказа, см. словарь № 18;

·Номер договора;

·Валюта — валюта долга (см. Список валют);

·Тип продукта — см. словарь № 11;

·Примечание — см. словарь № 247;

Примечание. Право на редактирование примечания определяется доступом LAW_ACT_ORDER_SET_DICT_VALUE.

 Примечание для поддержки. Колонка в Юридическом департаменте на вкладках судебных дел и данное поле в карточке судебной работы видны только если в column_spec у колонки с id=884 заполнено поле user_name. Наименование колонки реализовано по стрингу law_act_list.act_dict_value_1.

·Составляющие — список составляющих долга в приказном производстве определяется константой LawActOrderDebtComponentList;

·переключатель По договору/На момент подачи в суд/По решению суда, показывающий, какие составляющие долга указаны в предыдущем поле.

Примечание. Составляющие на момент подачи в суд доступны только если константа LawAct.FiledInCourt.Components.Use имеет значение "Истина".

Примечание. Редактирование составляющих на момент подачи в суд доступно для дел в статусах, перечисленных в значении доступа № 717 LAW_ACT_ORDER_DELIVERY_COMPONENT_EDIT_STATUS_LIST (или доступно для всех статусов, если значение доступа "ALL").

Примечание. Для долгов по ЖКХ редактирование на момент подачи в суд доступно только для тех составляющих, которые перечислены в значении доступа № 847 LAW_ACT_CARD_DEBT_DETAIL_EDIT (или доступно для всех, если значение доступа "ALL").

Примечание. Для долгов по ЖКХ и только в том случае, если используется функционал создания дел судебной работы по периодам задолженности, в списке составляющих долга по решению суда дополнительно присутствует переключатель Взыскано/Отказано, по которому отдельно отображаются взысканные судом суммы и отказанные суммы (для приказов и исков)

Для поддержкки:

·При активном радиобаттоне «Взыскано» (по умолчанию активен) суммы выводятся из law_act_dd_info_sum.court_sum

·При активном радиобаттоне «Отказано» суммы выводятся из law_act_dd_info_sum.failure_sum.

Раздел "Судебный приказ"

Раздел Судебный приказ содержит следующие поля:

·Предмет приказа — см. словарь № 327;

·Размер требования — значение поля рассчитывается путем суммирования полей, перечисленных в  константе LawActOrderTotalSumFormula, выводится в валюте долга, указанной в поле Валюта (редактирование поля разрешается пользователю в соответствии с значением достуа № 710 LAW_ACT_FIELD_EDIT в его роли);

·        Остаток —  выводится в валюте долга, указанной в поле Валюта;

·Утвержденная пошлина — утвержденная сумма пошлины, выводится в основной валюте Системы (редактирование поля разрешается пользователю в соответствии с значением достуа № 710 LAW_ACT_FIELD_EDIT в его роли);

·Расчетная пошлина — если константа LawAct.Prepare.DueSum.Mandatory имеет значение "Истина", то поле обязательно для заполнения. Подробнее см. Расчет госпошлины), выводится в основной валюте Системы (редактирование поля разрешается пользователю в соответствии с значением достуа № 710 LAW_ACT_FIELD_EDIT в его роли);

·Дата оплаты пошлины;

·Дата подачи в Суд — дата подачи заявления в суд;

·Номер платеж. поручения;

·Период ожидания вынесения — целое число, в днях;

·Способ подачи — способ доставки заявления в суд, см. словарь № 16;

·Дата вынесения решения;

·Дата вступления в силу;

·Способ доставки — способ доставки судебного приказа должнику, см. словарь № 17;

·Получен Агентством — дата получения приказа Агентством;

·Номер приказа;

·Дата приказа;

·Результат по делу — см. словарь № 135;

·Доп. информация — при нажатии открывается окно, содержащее сценарий специального сообщения, настроенного в модуле Сценарии сообщений (вкладка Специальное) и имеющего идентификационный номер, указанный в значении константы DebtMainInfoFormat после символов "(2:". Настройка информационного блока аналогична настройке информационного блока для карточки должника.

Примечание. Если в значении константы DebtMainInfoFormat после символов "(2:" указано число, не являющееся идентификационным номером специального сценария, то поле Доп. информация отсутствует.

Раздел "Телефоны"

В разделе Телефоны отображаются телефоны ответчика по данному делу:

·Если ответчиком является должник, то отображаются телефоны должника. Если если включена константа CardAllPhoneShow, то отображается также  информация по телефонам поручителей и залогодателей.

·Если ответчиком является поручитель, то в списке отображаются только телефоны поручителя.

Раздел Телефоны содержит следующие элементы:

·Список телефонов с полями:

·Тип — тип телефона, см. словарь № 4;

·Номер — номер телефона;

·Контактное лицо;

·Кнопки для работы со списком телефонов.

Действия со списком телефонов — см. Ведение списка контактных телефонов должника.

 

Окраска телефонов в зависимости от статуса

Строки в списке телефонов выделены цветом фона в зависимости от статуса контактного телефона (см. Статусы телефонов):

·Не звонили ни разу — серый цвет.

·Результата не было — белый цвет.

·Неверный номер — красный цвет.

·Результат — зеленый цвет .

·Последний результат — голубой цвет.

·Автоинформатор — розовый цвет.

Если у пользователя есть доступ 127 DEBT_CARD_ALL_PHONE, то в карточке должника он может видеть заблокированные номера телефонов. Такие записи в списке телефонов выделены серым шрифтом на белом фоне.

Раздел "Адреса"

В разделе Адреса отображаются адреса ответчика по данному делу.

·Если ответчиком является должник, то отображаются адреса должника. Если включена константа CardAllAddressShow, отображается также информация по адресам поручителей и залогодателей.

·Если ответчиком является поручитель, то в списке отображаются только адреса поручителя.

Раздел Адреса содержит следующие элементы:

·Список адресов с полями:

·Тип — тип адреса, см. словарь № 5;

·Адрес — адрес должника;

·Комментарий — поле с дополнительной информацией о должнике и кнопка для просмотра истории комментариев;

·Кнопки для работы со списком адресов.

Действия со списком адресов — см. Ведение списка адресов должника.

 

Окраска адресов в зависимости от статуса

Строки в списке адресов выделены цветом фона в зависимости от статуса адреса. (см. Статусы адресов):

·Не выезжали ни разу — серый цвет;

·Результата не было — белый цвет;

·Неверный адрес — красный цвет.

·Результат — зеленый цвет;

·Последний результат — голубой цвет.

Если у пользователя есть доступ 280 DEBT_CARD_ALL_ADR, то в карточке должника он может видеть заблокированные адреса. Такие записи в списке адресов выделены серым шрифтом на белом фоне.
