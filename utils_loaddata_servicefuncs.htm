Сервисные функции

Show/Hide Hidden Text

В модуле "Загрузка данных" доступны следующие сервисные функции:

Функции доступны из контекстного меню модуля, которое вызывается щелчком правой кнопки мыши в области таблицы данных, или при нажатии соответствующих "горячих" клавиш.

Меню сервисных функций вызывается нажатием правой кнопки мыши и обеспечивает:

·подготовку текста активной ячейки таблицы к копированию;

·копирование подготовленного текста в активную ячейку;

·переход к следующей проблемной ячейке;

·проверку текущей строки;

·отмену последних изменений;

·поиск населенного пункта или улицы;

·корректировку в ячейках таблицы по результатам поиска данных об области, районе и городе;

·корректировку в активной ячейке таблицы по результатам поиска данных об улице;

·подготовку данных для автозамены;

·формирование файла EXCEL в универсальном формате;

·подготовку данных по регионам;

·поиск кодов городов;

·просмотр легенды.

Подготовка текста активной ячейки таблицы к копированию осуществляется нажатием левой кнопки мыши в соответствующей строке меню сервисных функций либо одновременным нажатием клавиш Ctrl и C (или Ctrl и Insert). Никаких изменений на экране не происходит.

Копирование подготовленного текста в активную ячейку осуществляется через меню либо одновременным нажатием клавиш Ctrl и V (или Shift и Insert). В результате в активной ячейке отображается ранее подготовленный текст.  Подготовленный текст может использоваться многократно для изменения двух или более ячеек таблицы.

Переход к следующей проблемной ячейке осуществляется через меню либо нажатием клавиши F3. В результате активизируется очередная проблемная ячейка таблицы (с желтой, оранжевой  или красной окраской). Если новая активная ячейка отсутствовала на экране, содержимое экрана автоматически изменяется, при этом активная ячейка располагается в левом верхнем углу поля таблицы. Если предшествующая проблемная ячейка была последней в файле, то активизируется начальная ячейка таблицы.

Примечание. Если пользователь имеет право на сохранение текста адресов без ссылки на справочные данные, то ячейки с оранжевой и красной окраской автоматически пропускаются, переход осуществляется к очередной желтой ячейке.

Проверка текущей (активной) строки осуществляется через меню либо одновременным нажатием клавиш Ctrl и R. В результате проверки данных цвет фона ячеек строки на экране может измениться. Сообщения об окончании проверки данных текущей строки на экран не выводятся.

Отмена последних изменений осуществляется через меню либо одновременным нажатием клавиш Ctrl и Z. В результате отменяются последнее изменение координат активной ячейки (при наличии изменения координат активной ячейки), последнее изменение текста активной или пассивной ячейки (при наличии изменения текста ячейки), последнее изменение названия города (при наличии изменения названия города) и последнее изменение названия улицы (при наличии изменения названия улицы). В результате отмены последних изменений цвет фона ячеек на экране может измениться. Сообщения об окончании отмены последних изменений на экран не выводятся.

Обращение к функции поиска населенного пункта или улицы осуществляется через меню либо нажатием клавиши F2 (предварительно целесообразно активизировать нужную ячейку таблицы). На экран выводится окно поиска населенных пунктов и улиц. Пользователь выбирает вариант поиска (населенный пункт или улица), формирует минимально необходимый для успешного поиска фрагмент названия населенного пункта или улицы и активизирует кнопку "Поиск". В результате фрагмент окаймляется слева и справа специальным символом  %, на экран выводится список населенных пунктов или улиц из справочника kladr или address_zone, названия которых содержат заданный пользователем фрагмент. Каждый элемент списка содержит текстовую информацию о типе и названии населенного пункта или улицы, коде в справочнике, области и районе размещения, почтовом индексе (для населенного пункта) или географическом направлении (для улицы).

В случае населенного пункта имеется дополнительная возможность, задать код kladr в окне поиска. Код kladr должен состоять из 13 цифр, ошибочные коды игнорируются. При наличии в окне поиска корректного кода kladr игнорируется фрагмент названия населенного пункта. После активизации кнопки "Поиск" на экран выводится список населенных пунктов, содержащий единственную строку или пустой список, если сформированный код kladr отсутствует в справочнике.

Если по результатам поиска пользователю удается найти нужный населенный пункт или улицу, ему достаточно активизировать соответствующую строку списка и нажать кнопку "Вставить". Окно поиска удаляется, а в ячейках таблицы осуществляются необходимые изменения.

В случае населенного пункта обеспечивается:

·изменение данных об области, районе и городе в соответствии с информацией из выбранной строки списка населенных пунктов;

·сброс оранжевого цвета в ячейке города;

·автоматическая проверка данных для ячейки улицы того же адреса;

·по результатам проверки, при необходимости – сброс или установка оранжевого цвета в ячейке улицы.

 

В случае улицы обеспечивается:

·изменение данных об улице в соответствии с информацией из выбранной строки списка улиц;

·автоматическая проверка измененных данных для ячейки улицы;

·по результатам проверки, при необходимости – сброс оранжевого цвета в ячейке улицы.

Обращение к функции подготовки данных для автозамены осуществляется через меню либо нажатием клавиши F4. На экран выводится окно данных для автозамены. Пользователь имеет возможность добавлять, изменять и удалять данные для автозамены.

По окончании корректировки данных для автозамены пользователь нажимает кнопку "ОК". Окно данных для автозамены удаляется, а в проблемных ячейках таблицы, содержащих тексты для замены,  формируются новые тексты. Автоматически производится повторная проверка данных в таблице.

Для формирования файла EXCEL в универсальном виде пользователь выбирает соответствующий пункт меню либо нажимает клавишу F5. В открывшемся стандартном диалоговом окне пользователь выбирает нужную директорию и имя формируемого файла. После нажатия кнопки "ОК" диалоговое окно удаляется и осуществляется формирование файла EXCEL. По окончании формирования файла на экран выводится сообщение: "Создание файла имя файла закончено".

Описание файлов универсального формата и работа с ними описана в разделе Работа с файлом в универсальном формате.

Обращение к функции подготовки данных по регионам осуществляется через меню либо нажатием клавиши F6. На экран выводится окно формирования данных по регионам. Пользователь имеет возможность добавлять, изменять и удалять коды и названия регионов.

По окончании корректировки данных по регионам пользователь нажимает кнопку "ОК". Окно формирования данных по регионам удаляется, а в соответствующем словаре базы данных фиксируются выбранные пользователем коды и номера регионов, которые при дальнейшей работе позволяют автоматически идентифицировать регион по коду города в справочнике kladr.

Обращение к функции поиска кодов городов осуществляется через меню либо нажатием клавиши F7. На экран выводится окно ввода данных о диапазоне адресов в БД. Пользователь задает начальное и конечное значение идентификатора в таблице адресов.

После нажатия кнопки "ОК" диалоговое окно удаляется, для строк таблицы адресов в заданном диапазоне при отсутствии ссылки на справочник kladr организуется автоматический поиск с учетом значений области, района и города в строке. При положительном результате поиска (в kladr найдена единственная строка) в строке таблицы адресов автоматически формируется ссылка на справочник kladr и текст полного адреса.

Просмотр легенды осуществляется через меню либо нажатием клавиши F11. На экран выводится окно легенды с информацией об окраске фона ячеек таблицы по результатам проверки данных:

      Некорректное значение стандартного реквизита;

      Некорректное значение элемента адреса;

      Посторонний город, район или область;

      Имеется несколько вариантов города или улицы.

Более подробное описание некоторых функций

       Поиск населенного пункта или улицы

Для того чтобы выполнить поиск населенного пункта или улицы, выполните следующие действия:

1.Активизируйте нужную ячейку таблицы с названием населенного пункта или улицы.

2.В контекстном меню выберите пункт Поиск города/улицы или нажмите клавишу F2.

3.В открывшейся форме поиска населенных пунктов и улиц:

a.Выберите вариант поиска, установив соответствующий переключатель.

b.В поле Фрагмент названия введите фрагмент названия населенного пункта или улицы.

c.В случае поиска населенного пункта в поле Код KLADR можно задать код KLADR. Он должен состоять из 13 цифр, ошибочные коды игнорируются. Если для поиска задан корректный код KLADR, фрагмент названия населенного пункта будет игнорироваться.

d.Нажмите кнопку Поиск. Если поиск выполняется по фрагменту, то будет выдан список населенных пунктов или улиц из справочника KLADR или ADDRESS_ZONE, названия которых содержат заданный пользователем фрагмент. Каждый элемент списка содержит текстовую информацию о типе и названии населенного пункта или улицы, коде в справочнике, области и районе размещения, почтовом индексе (для населенного пункта) или географическом направлении (для улицы).
Если поиск выполняется по коду KLADR, будет выдана строка с информацией о населенном пункте с указанным кодом или пустой список, если указанный код отсутствует в справочнике.

4.Если по результатам поиска удается найти нужный населенный пункт или улицу, выделите соответствующую строку списка и нажмите кнопку Вставить. В ячейку таблицы будут внесены необходимые изменения.

       Автозамена данных в таблице

Функция автозамены данных позволяет заменить в проблемных ячейках указанный текст на новый. Для автозамены данных выполните следующие действия:

1.В контекстном меню выберите пункт Данные для автозамены или нажмите клавишу F4.

2.В открывшейся форме Данные для автозамены нажмите кнопку Добавить.

3.В открывшейся форме в поле Заменить введите текст, который требуется заменить, а в поле На — новый текст.

4.Нажмите кнопку ОК. Данные для автозамены будут добавлены в список.

5.При необходимости повторите шаги 2, 3, чтобы добавить все необходимые данные.

Данные в форме Данные для автозамены можно изменять и удалять. Для изменения данных выделите нужную строку, нажмите кнопку Изменить, в открывшейся форме измените данные и нажмите кнопку ОК. Для удаления данных выделите нужную строку, нажмите кнопку Удалить и в открывшемся диалоговом окне подтвердите удаление, нажав кнопку Yes.

6.По окончании формирования списка данных для автозамены в форме Данные для автозамены нажмите кнопку ОК. В проблемных ячейках таблицы, содержащих тексты для замены, будут сформированы новые тексты. Автоматически будет выполнена повторная проверка данных в таблице.

       Регистрация данных по регионам

Функция регистрации данных по регионам позволяет сохранить в БД указанные пользователем код и номер региона, а также информацию о часовом поясе региона, которые в дальнейшем позволят автоматически идентифицировать регион по коду города в справочнике KLADR.

Для того чтобы зарегистрировать данные по регионам, выполните следующие действия:

1.В контекстном меню выберите пункт Данные по регионам или нажмите клавишу F6.

2.В открывшейся форме Данные по регионам нажмите кнопку Добавить.

3.В открывшейся форме заполните следующие поля:

·Фрагмент кода kladr — фрагмент кода KLADR, по которому будет выполняться идентификация;

·Название региона — см. словарь № 39;

·Часовой пояс — см. словарь № 51.

4.Нажмите кнопку ОК. Данные будут добавлены в список.

5.При необходимости повторите шаги 2, 3, чтобы добавить все необходимые данные.

Данные в форме Данные по регионам можно изменять и удалять. Для изменения данных выделите нужную строку, нажмите кнопку Изменить, в открывшейся форме измените данные и нажмите кнопку ОК. Для удаления данных выделите нужную строку, нажмите кнопку Удалить и в открывшемся диалоговом окне подтвердите удаление, нажав кнопку Yes.

6.По окончании формирования списка данных для автозамены в форме Данные по регионам нажмите кнопку ОК. Данные будут зафиксированы в соответствующем словаре системы.

       Поиск кодов городов

В результате поиска кодов городов для каждой строки заданного диапазона таблицы address при отсутствии кода kladr и наличии данных о городе осуществляется поиск строки справочника kladr населенных пунктов РФ и автоматическое формирование поля r_kladr_id в строке address при успешном результате поиска. Код r_kladr_id формируется на основе поля code таблицы kladr.

Для того чтобы выполнить поиск кодов городов, выполните следующие действия:

1.Выберите в контекстном меню пункт Поиск кодов городов или нажмите клавишу F7.

2.В открывшейся форме введите начальное и конечное значение идентификатора в таблице адресов (address).

3.Нажмите кнопку ОК. Для каждой строки таблицы адресов из заданного диапазона при отсутствии в ней кода KLADR и наличии данных об области, районе, городе выполняется автоматический поиск соответствующей строки справочника  KLADR. При положительном результате поиска (в KLADR найдена единственная строка) в строке таблицы адресов автоматически формируется ссылка на справочник KLADR (значение в поле r_kladr_id, которое ссылается на поле code таблицы kladr) и текст полного адреса.

4.По окончании поиска будет выдано сообщение и завершении поиска. Нажмите кнопку ОК, чтобы закрыть его.

       Проверка адресов с помощью сервиса DaData

Для того чтобы проверить адреса с помощь интегрированного сервиса DaData, выберите в контекстном меню пункт Проверка адресов в DaData или нажмите клавишу F9.

       Просмотр легенды (информация об окраске фона ячеек таблицы по результатам проверки)

Легенда содержит информацию об окраске ячеек таблицы по результатам проверки данных. Для ее просмотра выберите в контекстном меню пункт Легенда или нажмите клавишу F10.
