Перевод дел исполнительного производства в иски

В модуле Юридический департамент пункт контекстного меню (вызывается правой кнопкой мыши) Преобразовать в иск доступен только пользователям, в роли которых доступ № 498 LAW_ACT_LIST_EXEC_TO_ISK имеет значение TRUE.

Для преобразования приказа в судебный иск выполните следующие действия:

1.В форме Юридический департамент на вкладке Исполнительное производство выберите одно или более дел. Для выбора нескольких записей используйте клавиши SHIFT и/или CTRL.

2.В контекстном меню выберите пункт Преобразовать в иск.

Примечание: возможность создания нескольких судебных дел разных типов для одного долга доступна при наличии у пользователя доступа № 356 LAW_ACT_MULTIPLE. Возможность создания нескольких судебных дел одного типа — доступа № 626 LAW_ACT_MULTIPLE_DIFFERENT_TYPE

3.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить перевод.

4.Для каждого дела из списка будут осуществлены следующие проверки:

·Если ИП создано по иску:

oИП переводится в архив (действие протоколируется);

oСвязанный иск переводится в статус "Возобновлено" (действие протоколируется);

·Если ИП создано по приказу:

oИП переводится в архив (действие протоколируется);

oОсуществляется перевод приказа в иск, связанный с данным ИП;

·Если ИП создано без связи с судебным делом:

oЕсли по данному долгу есть судебные дела:

§Выдается предупреждение: "По некоторым ИП не будет выполнено, так как присутствует неоднозначная логика преобразования"; для таких дел действие не осуществляется;

oЕсли по данному делу нет судебных дел, то:

§Пользователю выдается сообщение "По некоторым ИП действие не будет выполнено, так как по ним нет судебного приказа или иска."; для таких дел перевод не осуществляется.

5.По окончании перевода будет выдано сообщение о результатах выполнения операции. Нажмите кнопку ОК, чтобы закрыть его. Судебные иски доступны на вкладке Иски.

Примечание. При переводе исполнительного производства в иск значение госпошлины обнуляется, если исполнительное производство было создано по приказу.

Информация для поддержки

В полном алгоритме массового возврата ИП на предыдущую стадию из карточки ИП используются две технические константы:

·LawExec.ReturnToPrevStage.WithCreateLawAct;

·LawExec.ReturnToPrevStage.UnconnectedAct.

Изначально для всех клиентов эти константы выключены, а для Почта Банка включены обе. Текст справки написан именно для этих двух случаев — для всех и для Летобанка.

 Внимание! Важно! Пользовательский текст справки написан именно для этих двух случаев — для всех и для Летобанка (Почта банка).

Полный алгоритм проверок

·Если ИП создано по иску:

oИП переводится в архив (действие протоколируется);

oСвязанный иск переводится в статус "Возобновлено" (действие протоколируется);

·Если ИП создано по приказу:

oИП переводится в архив (действие протоколируется);

oОсуществляется перевод приказа в иск, связанный с данным ИП;

·Если ИП создано без связи с судебным делом:

oЕсли по данному долгу есть судебные дела:

§Если таких дел больше одного:

oВыдается предупреждение: "По некоторым ИП не будет выполнено, так как присутствует неоднозначная логика преобразования"; для таких дел действие не осуществляется;

 Примечание. Для таких дел можно осуществить преобразование в иск в карточке исполнительного производства. Там можно выбрать судебное дело, в которое преобразовать текущее ИП. 

§Если такое дело только одно, то:

oЕсли константа LawExec.ReturnToPrevStage.UnconnectedAct включена, то:

§ИП переводится в архив (действие протоколируется);

§Если это иск, то он связывается с данным ИП и переводится в статус "Возобновлено"; если это приказ, то он преобразуется в иск, связанный с данным ИП;

oЕсли константа LawExec.ReturnToPrevStage.UnconnectedAct выключена, то:

§Выдается предупреждение: "По некоторым ИП не будет выполнено, так как присутствует неоднозначная логика преобразования"; для таких дел действие не осуществляется;

oЕсли по данному делу нет судебных дел, то:

§Если константа LawExec.ReturnToPrevStage.WithCreateLawAct имеет значение TRUE, то:

oИП переводится в архив (действие протоколируется);

oСоздается новый иск, связанный с данным ИП;

§Если константа LawExec.ReturnToPrevStage.WithCreateLawAct имеет значение FALSE, то:

oПользователю выдается сообщение "По некоторым ИП действие не будет выполнено, так как по ним нет судебного приказа или иска."; для таких дел перевод не осуществляется.
