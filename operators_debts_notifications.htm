Информирование операторов о назначении/снятии долгов/дел

В Системе осуществляется уведомление операторов о назначении им или снятии с них долгов или дел путем формирования сообщений электронной почты в модуле "Почта".

Снятие или назначение долгов осуществляется в следующих модулях:

·Постановка задач;

·Входящий звонок;

·Группы свободного поиска;

·Расписание.

Снятие или назначение дел осуществляется в модулях:

·Юридический департамент;

·Карточка судебной работы;

·Карточка исполнительного производства;

·Входящий звонок;

·Группы свободного поиска.

Уведомления формируются в зависимости от стадии взыскания. См. далее:

·Досудебная стадия взыскания:

·Информирование операторов о назначении долгов;

·Информирование операторов о снятии долгов.

·Судебная стадия взыскания:

·Информирование операторов о назначении дел судебной работы;

·Информирование операторов о снятии дел судебной работы.

·Исполнительная стадия взыскания:

·Информирование операторов о назначении дел исполнительного производства;

·Информирование операторов о снятии дел исполнительного производства.

Досудебная стадия взыскания

Информирование операторов о назначении долгов

Автоматическое уведомление оператора о назначении ему новых долгов осуществляется при назначении оператору долгов в следующих модулях Системы:

·Постановка задач (для действий по назначению оператора и по распределению долгов, но только если в роли оператора, которому назначают долги, доступ № 321 WORK_TASK_SEND_NOTIFICATION имеет значение TRUE);

·Входящий звонок (при назначении оператора, но только если в роли оператора, которому назначают долги, доступ № 321 WORK_TASK_SEND_NOTIFICATION имеет значение TRUE);

·Группы свободного поиска (для действий по назначению оператора ОДВ и по распределению долгов, но только если в роли оператора, которому назначают долги, доступ № 321 WORK_TASK_SEND_NOTIFICATION имеет значение TRUE);

·Расписание (условия на формирование уведомлений см. в описании соответствующих мероприятий 15 и 26).

Уведомление о назначении долгов формируется таким образом, что в одном сообщении для оператора указан список всех долгов, которые были назначены данному оператору в рамках одного действия.

Формат сообщения

Тема сообщения:

"Новые должники для работы".

В следующих случаях в начале сообщения указывается причина назначения долгов:

·Если выполнялось действие по назначению оператора (ОДВ) из модулей "Постановка задач", "Входящий звонок" или "Группы свободного поиска", и значение константы WorkTask.Operator.Change.Reason.Use равно "Истина".

·Если выполнялось мероприятие "Смена оператора" (код 15), "Распределение долгов" (код 26) в модуле "Расписание", и при этом одновременно значение константы WorkTask.Operator.Change.Reason.Use равно "Истина", и в значении константы Schedule.Operator.Change.Reason.Default указан номер существующего термина словаря № 326 "Причина назначения/снятия долга".

Текст сообщения включает список новых долгов, закрепленных за оператором.

Для каждого долга в списке указывается:

·ФИО должника;

·ID долга;

·Портфель;

·Предыдущий оператор — указывается только если предыдущий оператор у долга есть.

Информирование операторов о снятии долгов

Автоматическое уведомление оператора о снятии долгов осуществляется при снятии долгов с оператора, в том числе и если оно осуществлялось в рамках назначения этих долгов другому оператору, в следующих модулях Системы:

·Постановка задач (для действий по снятию/назначению оператора и по распределению долгов, но только если в роли оператора, с которого снимают долги, доступ № 321 WORK_TASK_SEND_NOTIFICATION имеет значение TRUE);

·Входящий звонок (для действий по снятию/назначению оператора для долга, но только если в роли оператора, с которого снимают долги, доступ № 321 WORK_TASK_SEND_NOTIFICATION имеет значение TRUE);

·Группы свободного поиска (для действий по снятию/назначению оператора ОДВ и по распределению долгов, но только если в роли оператора, с которого снимают долги, доступ № 321 WORK_TASK_SEND_NOTIFICATION имеет значение TRUE);

·Расписание (условия на формирование уведомлений см. в описании соответствующих мероприятий).

Уведомление о назначении долгов формируется таким образом, что в одном сообщении для оператора указан список всех долгов, которые были сняты с данного оператора в рамках одного действия.

Формат сообщения

Тема сообщения:

"Снятие должников".

В следующих случаях в начале сообщения указывается причина снятия долгов:

·Если выполнялось действие по снятию/назначению оператора (ОДВ) из модулей "Постановка задач", "Входящий звонок" или "Группы свободного поиска", и значение константы WorkTask.Operator.Change.Reason.Use равно "Истина".

·Если выполнялось мероприятие, предусматривающее снятие оператора, в модуле "Расписание", и при этом одновременно значение константы WorkTask.Operator.Change.Reason.Use равно "Истина", и в значении константы Schedule.Operator.Change.Reason.Default указан номер существующего термина словаря № 326 "Причина назначения/снятия долга".

Текст сообщения включает список новых долгов, закрепленных за оператором.

Для каждого долга в списке указывается:

·ФИО должника;

·ID долга;

·Портфель.

Судебная стадия взыскания

Информирование операторов о назначении дел судебной работы

Автоматическое уведомление оператора о назначении ему новых дел судебной работы осуществляется при назначении оператору дел в следующих модулях Системы (если константа Law.Operator.Change.Notification.Use имеет значение "Истина"):

·Юридический департамент (при назначении делам нового оператора);

·Карточка судебной работы (при назначении делу нового оператора);

·Входящий звонок (при назначении дел оператору);

·Группы свободного поиска (для действий по назначению оператора ОСВ).

Уведомление о назначении дел формируется таким образом, что в одном сообщении для оператора указан список всех дел, которые были назначены данному оператору в рамках одного действия.

Формат сообщения

Тема сообщения:

"Новые судебные дела для работы".

Текст сообщения включает список новых дел, закрепленных за оператором.

Для каждого долга в списке указывается:

·ФИО должника;

·ID дела (с типом дела);

·Предыдущий оператор — указывается только если предыдущий оператор у дела есть.

Информирование операторов о снятии дел судебной работы

Автоматическое уведомление оператора о снятии с него дел судебной работы осуществляется при снятии с него дел, в том числе и если оно осуществлялось в рамках назначения этих дел другому оператору, в следующих модулях Системы (если константа Law.Operator.Change.Notification.Use имеет значение "Истина"):

·Юридический департамент (при изменении оператора по делам);

·Карточка судебной работы (при изменении оператора по делу);

·Входящий звонок (при снятии/назначении дел оператору);

·Группы свободного поиска (для действий по снятию/назначению оператора ОСВ).

Уведомление о снятии дел формируется таким образом, что в одном сообщении для оператора указан список всех дел, которые были сняты с данного оператора в рамках одного действия.

Тема сообщения:

"Снятие судебных дел".

Текст сообщения включает список новых дел, закрепленных за оператором.

Для каждого долга в списке указывается:

·ФИО должника;

·ID дела (с типом дела).
