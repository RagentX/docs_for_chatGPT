Обжалование

Примечание. Функционал обжалования используется в приказном производстве, если константа LawAct.Appeal.Use включает значение 1.

По кнопке Обжалование доступны следующие типы жалоб:

·Апелляция;

·Кассация;

·Надзор.

Обжалование доступно для дел в статусах:

·Отказ в выдаче приказа;

·Апелляция отклонена;

·Кассация отклонена.

После выбора одного из типов жалобы статус судебного приказа изменяется в зависимости от настроек Системы.

Примечание. Статус "Надзор отклонен" является финальным, обжалование для него не доступно.

Использование полного либо сокращенного жизненного цикла жалобы

В Системе может быть настроено использование полного либо сокращенного жизненного цикла жалобы.

Тип жизненного цикла жалобы регулируется константой LawActUsePrepareAppeal:

·Истина — используется полный цикл;

·Ложь — используется сокращенный цикл.

Примечание. По вопросам изменения сокращенного цикла жалобы на полный или наоборот — обратитесь в техподдержку.

Полный жизненный цикл жалобы

1.Первоначальная подача жалобы (выбор типа жалобы). По кнопке Обжалование осуществляется выбор типа жалобы. Дело переводится в статус:

·Апелляция — "Не подготовлена апелляция";

·Кассация — "Не подготовлена кассация";

·Надзор — "Не подготовлен надзор".

Если константы LawActUseRequestDueSum и LawAct.Appeal.RequestDueSum.Use имеют значение "Истина", то госпошлина включена в жизненный цикл жалобы. Следующим этапом будет п. 2 — Запрос доплаты ГП.

Если одна из этих констант или обе имеют значение "Ложь", то госпошлина не включена в жизненный цикл жалобы. Следующим пунктом будет п. 4 — Подготовить.

2.Запрос доплаты ГП. По кнопке Запрос госпошлины -> Запрос доплаты ГП.

3.Госпошлина оплачена.

4.Подготовить.

5.Подать.

6.После этого переходите к отметке результатов обжалования.

Сокращенный жизненный цикл жалобы

1.Подача жалобы. По кнопке Обжалование осуществляется выбор типа жалобы. Дело переводится в статус:

·Апелляция — "Подана апелляция";

·Кассация — "Подана кассация";

·Надзор — "Подан надзор".

2.После этого переходите к отметке результатов обжалования.

Описание действий жизненного цикла жалобы

Подача жалобы (выбор типа жалобы)

Для установки отметки о подаче жалобы выполните следующие действия:

1.В карточке судебной работы на вкладке Приказное производство нажмите кнопку Обжалование и в раскрывшемся списке выберите тип жалобы.

2.В открывшемся диалоговом окне нажмите кнопку Yes, чтобы подтвердить выполнение операции.

3.Выполните прикрепление документа.

4.Далее осуществляется формирование настроенных запланированных действий.

5.Действие протоколируется.

Если константа LawActUseRequestDueSum имеет значение "Истина", то:

·если константа LawAct.Appeal.RequestDueSum.Use имеет значение "Истина", то становится доступной кнопка Запрос госпошлины.

·если константа LawAct.Appeal.RequestDueSum.Use имеет значение "Ложь", то становится доступной кнопка Подготовить.

Если константа LawActUseRequestDueSum имеет значение "Ложь", далее становится доступной кнопка Подготовить.

Запрос и оплата госпошлины для обжалования

Эти действия доступны только если обе константы LawActUseRequestDueSum и LawAct.Appeal.RequestDueSum.Use равны "Истина".

По этой кнопке доступен только подпункт Запрос доплаты госпошлины. Описание — см. Запрос доплаты госпошлины.

Окно запроса доплаты госпошлины имеет дополнительную кнопку Взять текущий размер ГП, которая позволяет в поле Введите размер доплаты ввести значение, указанное в поле Текущая госпошлина.

В результате статус дела изменится в зависимости от вида жалобы следующим образом:

·Апелляция — "Запрос ГП апелляции";

·Кассация — "Запрос ГП кассации";

·Надзор — "Запрос ГП надзора".

После запроса госпошлины выполните действие по кнопке Госпошлина оплачена (см. Госпошлина оплачена). Статус дела изменится:

·Апелляция — "ГП апелляции оплачена";

·Кассация — "ГП кассации оплачена";

·Надзор — "ГП надзора оплачена".

Кнопка Подготовить будет доступна только при статусе дела "ГП апелляции/кассации/надзора оплачена".

Подготовка обжалования

Если приказ находится в статусе "Не подготовлена апелляция/кассация/надзор" при значении константы LawActUseRequestDueSum равном "Ложь" или в статусе "ГП апелляции/кассации/надзора оплачена" при значении константы LawActUseRequestDueSum равном "Истина", то становится доступна кнопка Подготовить.

1.Нажмите кнопку Подготовить.

2.Появится окно подтверждения. Нажмите Да.

3.Выполните прикрепление документа.

4.Далее осуществляется формирование настроенных запланированных действий.

5.В результате подготовки приказ получит статус в зависимости от вида жалобы:

·Апелляция — "Подготовлена апелляция";

·Кассация — "Подготовлена кассация";

·Надзор — "Подготовлен надзор".

6.Действие протоколируется.

Подача обжалования

Для дел, находящихся в статусе "Подготовлена апелляция/кассация/надзор" становится доступна кнопка Подать.

Для подачи обжалования:

1.Нажмите кнопку Подать.

2.Появится окно подтверждения. Нажмите Да.

3.Выполните прикрепление документа.

4.Далее осуществляется формирование настроенных запланированных действий.

5.В результате подачи иск принимает статусы:

·Апелляция — "Подана апелляция";

·Кассация — "Подана кассация";

·Надзор — "Подан надзор".

6.Действие протоколируется.
