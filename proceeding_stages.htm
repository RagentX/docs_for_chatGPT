Этапы работы в исковом производстве

На странице Визуальная схема жизненного цикла искового производства наглядно отображаются этапы работы приказного производства.

Право на управление жизненным циклом иска имеют пользователи, в роли которых доступ № 459 LAW_ACT_LAWSUIT_FLOW_FULL имеет значение TRUE. Для остальных пользователей все кнопки жизненного цикла в карточке искового производства недоступны.

Работа в исковом производстве включает следующие этапы:

1.Госпошлина. Этот этап доступен только при значении константы LawActUseRequestDueSum равном "Истина".

2.Подготовка иска в суд — на данном этапе иск переводится в статус "Подготовлен" (см. словарь № 25).

3.Подача иска в суд — на данном этапе иск переводится в статус "Подан" (см. словарь № 25).

4.Принятие иска — на данном этапе в зависимости от принятого решения иск может быть переведен в один из следующих статусов:

·"Определение о принятии к производству";

·"Оставление иска без движения";

·"Отказ в принятии";

·"Возвращение заявления";

·"Определение об оставлении без рассмотрения".

5.Предварительное заседание — на данном этапе иск переводится в статус "Определение о рассмотрении по существу";

6.Судебное разбирательство — на данном этапе в зависимости от принятого решения иск может быть переведен в один из следующих статусов:

·"Удовлетворение иска";

·"Отказ в удовлетворении иска" (данный статус является конечным);

·"Определение о прекращении" (данный статус является конечным);

·"Определение об оставл. без рассмотрения";

·"Частичное удовлетворение иска".

7.Обжалование — на данном этапе в зависимости от вида жалобы иск переводится в статусы:

·Если используется полный  цикл жалобы значение технической константы LawActUsePrepareAppeal равно "Ложь":

·Апелляция — "Не подготовлена апелляция";

·Кассация — "Не подготовлена кассация";

·Надзор — "Не подготовлен надзор".

Затем осуществляется запрос и оплата госпошлины. После этого — подготовка жалобы. Следующим этапом будет подача жалобы (см. подробное описание этапа Обжалование).

·Если используется сокращенный цикл жалобы значение технической константы LawActUsePrepareAppeal равно "Истина", то вышеописанные этапы пропускаются, и дело переводится в статусы:

·Апелляция — "Подана апелляция";

·Кассация — "Подана кассация";

·Надзор — "Подан надзор".

Примечание. По вопросам изменения сокращенного цикла жалобы на полный или наоборот — обратитесь в техподдержку.

8.Отметка результата обжалования — на данном этапе в зависимости от принятого решения иск может быть переведен в один из следующих статусов:

·"Решение Суда обжаловано":

·Апелляция — "Апелляция удовлетворена";

·Кассация — "Кассация удовлетворена";

·Надзор — "Надзор удовлетворен".

·"Жалоба отклонена":

·Апелляция — "Апелляция отклонена";

·Кассация — "Кассация отклонена";

·Надзор — "Надзор отклонен".

9.На некоторых этапах доступно приостановление дела. В этом случае дело получает статус "Приостановлено". Дальнейшие действия с ним доступны только после отмены приостановления.

10.Работа с архивом включает следующие действия:

·Перевод в архив — переводит ИП в статус "Архив";

·Возобновление — переводит ИП в статус "Возобновлено".

На любом этапе работы после подачи иска в суд можно установить отметку о заключении мирового соглашения с должником. В этом случае иск перейдет в статус "Мировое соглашение", а при получении исполнительного листа на мировое соглашение — в статус "Получение ИЛ на МС". Работа с исполнительными листами — см. Мировые соглашения.

Переход по этапам работы в приказном производстве выполняется при нажатии соответствующих кнопок в карточке судебной работы на вкладке Исковое производство. Если для каких-либо кнопок настроено автоматическое формирование запланированных действий, то при этом будут автоматически сформированы и добавлены на вкладку Планирование соответствующие запланированные действия.

Примечание: автоматическое формирование запланированных действий зависит от значения константы LawActivitySetMode.

Примечание. При смене статуса иска пользователь имеет возможность прикреплять до 10 различных документов в режиме добавления новых документов. При этом есть возможность уже загруженные документы помечать, как ненужные к загрузке. При внесении записи в протокол указывается ссылка на последний загруженный документ (см. здесь).
